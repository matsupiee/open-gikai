## 概要

- サイト: https://www.town.ichinomiya.chiba.jp/info/gikai/2/
- 分類: PDF 形式で年度別公開（既存アダプターでは対応不可）

## PDF ファイル名パターン

PDF ファイル名は年度によって命名規則が異なる。

### パターン 1: 元号 + 月 + `kaigiroku`（平成30年〜令和7年）

```
H30.{月}kaigiroku.pdf     （平成30年）
R{年}.{月}kaigiroku.pdf    （令和元年〜令和7年）
```

例:
- `H30.12kaigiroku.pdf` → 平成30年12月
- `R6.9kaigiroku.pdf` → 令和6年9月
- `R7.3kaigiroku.pdf` → 令和7年3月

同月に複数の会議がある場合は `(2)` 等の連番が付与される:
- `R6.9kaigiroku.pdf` → 令和6年第2回定例会
- `R6.9kaigiroku(2).pdf` → 令和6年第3回定例会

### パターン 2: `YYYYMMDD`（一部の会議）

```
YYYYMMDD.pdf
```

例:
- `20250317.pdf` → 2025年3月17日
- `20240529.pdf` → 2024年5月29日
- `20260306.pdf` → 2026年3月6日

### パターン 3: `gikai` + 年号2桁 + 月日（平成25年〜29年頃）

```
gikai{YY}{MMDD}.pdf
```

例:
- `gikai251205.pdf` → 平成25年12月5日
- `gikai25918(1).pdf` → 平成25年9月18日
- `gikai25304.pdf` → 平成25年3月4日

### パターン 4: 全角括弧付き元号（一部古い年度）

```
（{年号}.{回}kaigiroku.pdf  ※ URL エンコードされた全角括弧
```

例:
- `%EF%BC%A825.1kaigiroku.pdf` → （25.1kaigiroku.pdf

## 会議の種類

- **定例会**: 年4回（第1回〜第4回）
- **臨時会**: 不定期（年0〜4回程度）

各年度ページのリンクテキストから会議種別を抽出する:
- `令和６年第４回定例会` → 定例会
- `平成30年第１回臨時会` → 臨時会

---

## スクレイピング戦略

### Step 1: 年度別一覧ページから PDF URL を収集

1. トップページ `/info/gikai/2/` から全年度ページへのリンクを取得（またはハードコードした URL マッピングを使用）
2. 各年度ページにアクセスし、`<a>` タグの `href` 属性から PDF URL を抽出
3. リンクテキストから会議名（例: `令和６年第４回定例会`）を抽出

**抽出対象:**
- PDF URL: `href` 属性が `.pdf` で終わるリンク
- 会議名: リンクのテキストノード

### Step 2: PDF のダウンロード

収集した PDF URL からファイルをダウンロードする。

### Step 3: PDF からテキスト抽出

PDF からテキストを抽出し、会議録データとして構造化する。

#### メタ情報の抽出

リンクテキストからメタ情報を抽出する:

```typescript
// 会議名パターン
const meetingPattern = /(?:令和|平成)([\d元]+)年第(\d+)回(定例会|臨時会)/;
// 例: "令和６年第４回定例会" → 年="６", 回="４", 種別="定例会"
```

#### PDF テキストからの発言抽出

PDF の内容はテキスト抽出ツール（pdf-parse 等）で処理する。町議会の会議録 PDF は一般的に以下の構造を持つ:

- 冒頭にメタ情報（開催日時、出席議員、議事日程等）
- 発言は「議長」「○○番（氏名）」等の形式で記録
- 具体的なパースパターンは PDF の実際の内容を確認して決定する

## ページネーション

なし。各年度ページに当該年度の全会議録リンクが1ページで表示される。

---

## 注意事項

- PDF ファイル名の命名規則が年度によって異なるため、URL パターンによる推測ではなくページからのリンク抽出で対応する
- 古い年度（平成25年頃）は PDF 格納ディレクトリが `/assets/files/yakubaannai2/` と異なる
- 一部の PDF ファイル名に全角括弧が含まれており、URL エンコードが必要
- 同月に複数の会議がある場合、ファイル名に `(2)` 等の連番が付与される

---

## 推奨アプローチ

1. **一覧ページからのリンク抽出を優先**: ファイル名パターンが不規則なため、各年度ページの HTML をパースして PDF URL を収集する
2. **年度ページ URL はハードコード**: ページ ID が不規則なため、トップページからのリンク抽出またはハードコードで対応
3. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` 等のライブラリで PDF からテキストを抽出
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 年度別ページ構成のため、最新年度ページのみを再取得することで差分更新が可能
