## PDF ファイル名パターン

### 令和以降（令和2年〜）

命名規則が統一されており、年号・月日・種別が固定フォーマット。

```
R{YY}{MM}{DD}kaigiroku.pdf
```

例：
- `R070304kaigiroku.pdf`（令和7年3月4日）
- `R060910kaigiroku.pdf`（令和6年9月10日）

各年度につき、4つの定例会（第1回〜第4回）が存在し、各定例会で初日・最終日の 2 ファイルが公開される（計8ファイル／年度）。

### 平成（平成25〜31年）

命名規則が不統一。年号・月日の後に初日／最終日を示すサフィックスが付くが、表記揺れが多い。

例：
- `H260306.shoniti.pdf`（初日）
- `H260314saishuubi.pdf`（最終日）
- `H250307syonichi.377KB.pdf`（ファイルサイズが名前に含まれる例）
- `saishuubiH251212.pdf`（サフィックスが先頭に来る例）

→ 平成分は、年度別一覧ページの HTML からリンクを直接抽出する方式が確実。

## スクレイピング戦略

### Step 1: 年度別一覧ページの取得

カテゴリトップ（`category_list.php?frmCd=8-0-0-0-0`）は frmId の全量を保持するため、以下のどちらかで frmId リストを取得する。

- **静的管理**: 上記対応表を初期値として使い、新年度追加時に定期クロールで更新
- **動的取得**: カテゴリトップを GET して `contents_detail.php?co=cat&frmId=(\d+)&frmCd=8-6-0-0-0` の正規表現で frmId を抽出

### Step 2: 各年度ページから PDF リンクを収集

各年度ページ（`contents_detail.php?co=cat&frmId={frmId}&frmCd=8-6-0-0-0`）を GET し、`<a href="...\.pdf">` のリンクを Cheerio 等で抽出する。

**注意:**
- リンクは相対パス（`./cmsfiles/...`）で記述されているため、ベース URL と結合して絶対 URL に変換する
- cmsfiles ディレクトリの `0000000` / `0000001` / `0000003` 等のセグメントは年度によって異なる → リンクから直接取得するのが確実

### Step 3: PDF のダウンロードとテキスト抽出

各 PDF ファイルをダウンロードし、`pdftotext` 等のツールでテキストを抽出する。

- PDF は `PDF-1.7` 形式、`FlateDecode` 圧縮（テキスト埋め込み済み）
- ファイルサイズは 200〜700KB 程度

## PDF の構造とパース

### メタ情報

冒頭に以下の形式で記載される：

```
令和７年第１回度会町議会定例会会議録

招集年月日

令和７年３月４日

招集場所

度会町議会議場

開議

令和７年３月４日（午前９時00分）
```

抽出項目：
- 年・回次・会議種別: `令和X年第N回度会町議会定例会会議録`
- 招集年月日: `令和X年X月X日`
- 開議時刻: `午前/午後HH時MM分`

### 発言の構造

発言者行のパターン：

```
○議長（若宮　淳也）
○町長（中村　忠彦）
○副町長（西岡　一義）
○X番（氏名）
```

- `○` で始まり、役職名または議員番号が括弧の前に来る
- 括弧内が氏名
- 発言内容は発言者行の後に続くテキストブロック

議事区切りのパターン：

```
◎開会の宣告
◎会議録署名議員の指名
◎議案の上程（議案第X号～議案第XX号）
```

- `◎` で始まる行は議事の区切り（発言者ではない）

### パース用正規表現（案）

```typescript
// 会議録タイトルの抽出
const titlePattern = /^((?:令和|平成)\d+年第\d+回度会町議会(?:定例会|臨時会))会議録/;

// 招集年月日の抽出
const datePattern = /招集年月日\s*((?:令和|平成)\d+年\d+月\d+日)/;

// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（若宮　淳也） → role="議長", name="若宮　淳也"
// 例: ○１番（山北　佳宏） → role="１番", name="山北　佳宏"

// 議事区切りの検出
const sectionPattern = /^◎(.+)/;
```

## 注意事項

- **平成分のファイル名は不統一**: 命名規則に複数のバリエーションがあり、ファイル名の推測による直接 URL 生成は困難。各年度ページの HTML からリンクを抽出する方式を採用する
- **PDF のパス階層の揺れ**: `cmsfiles/contents/` 以下の `0000000` / `0000001` / `0000003` セグメントが年度によって異なる。HTML から取得したリンクを使用すること
- **年度ページの frmId は非連番**: 将来の年度追加時は、カテゴリトップを定期クロールして新規 frmId を検出する
- **1年度あたり PDF は最大8件**: 第1〜4回定例会 × 初日・最終日。ただし臨時会が追加される場合もある

---

## 推奨アプローチ

1. **静的 frmId 管理 + 定期更新**: 対応表を初期値として実装し、カテゴリトップの定期クロールで新年度の frmId を自動検出する
2. **HTML からリンク抽出**: 年度ページの HTML をパースして PDF リンクを収集し、ファイル名パターンの揺れに対処する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: frmId を年度キーとして管理し、既取得 frmId 以降のみを処理する差分更新が可能
