# 八丈町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.hachijo.tokyo.jp/kakuka/gikai/kaigiroku.html
- 分類: 公式ウェブサイト上で会議録を PDF ファイルで公開（年度ごとに整理）
- 文字コード: UTF-8（HTML ページ）
- 特記: 会議録は PDF 形式のみで公開。検索システムなし。単一ページに全年度分を掲載。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（トップ） | `https://www.town.hachijo.tokyo.jp/kakuka/gikai/kaigiroku.html` |
| 会議録 PDF | `https://www.town.hachijo.tokyo.jp/kakuka/gikai/pdf/kaigiroku/{西暦年}/{ファイル名}.pdf` |

### PDF ファイル名の規則

- 定例会: `{西暦年}{定例会番号(2桁)}{号数(2桁)}.pdf`
  - 例: `20250101.pdf` → 2025年・第1回定例会・第1号
  - 例: `20250104.pdf` → 2025年・第1回定例会・第4号
  - 例: `20240201.pdf` → 2024年・第2回定例会・第1号
  - 例: `20240302.pdf` → 2024年・第3回定例会・第2号
- 臨時会: `{西暦年}{臨時会番号(2桁)}r.pdf`（末尾に `r` サフィックス）
  - 例: `20250701r.pdf` → 2025年・第1回臨時会（※ファイル名の番号体系は定例会と異なる場合あり）
  - 例: `20240101r.pdf` → 2024年・第1回臨時会
  - 例: `20230102r.pdf` → 2023年・第2回臨時会

※ 臨時会のファイル名は `{西暦年}{連番}r.pdf` の形式だが、連番部分は必ずしも臨時会の回数と一致しない場合がある。HTML ページのリンクから正確に取得する必要がある。

---

## データ範囲

- 平成26年（2014年）〜 現在（令和7年・2025年）
- 年4回の定例会（第一回〜第四回）+ 臨時会（年0〜3回程度）
- 各定例会は複数号（1号〜5号程度）で構成

---

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録一覧ページ `kaigiroku.html` から全 PDF リンクを抽出する。

- 単一ページに全年度分が掲載されている（ページネーションなし）
- `<a>` タグの `href` 属性から `pdf/kaigiroku/` を含むリンクを収集
- リンクテキストから号数と開催日を抽出

**収集方法:**

1. `kaigiroku.html` を取得し、Cheerio で全 `<a>` タグを走査
2. `href` が `pdf/kaigiroku/` を含むリンクを抽出
3. ベース URL `https://www.town.hachijo.tokyo.jp/kakuka/gikai/` と結合して絶対 URL を生成
4. リンクテキストおよび周辺の見出し要素から以下を抽出:
   - 年度（令和X年 / 平成X年）
   - 会議種別（第X回定例会 / 第X回臨時会）
   - 号数（第X号）
   - 開催日（RX.X.X / HX.X.X 形式）

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF は数十〜数百ページの大容量ファイル（定例会1ファイルで数百ページのものもある）
- `pdf-parse` や `pdf.js` 等のライブラリでテキスト抽出
- 1つの PDF に定例会全体（複数日分）の会議録が含まれる場合がある

### Step 3: 会議録のパース

#### メタ情報

PDF の表紙・冒頭から以下を抽出:

```
令和７年第一回定例会

八丈町議会会議録

令和７年　３月３日　開会
令和７年　３月28日　閉会
```

各号の冒頭:

```
令和７年第一回八丈町議会定例会会議録

議　事　日　程　（第１号）

　　　　　　　　　　令和７年３月３日（月曜日）午前９時開会
```

- 定例会名: `令和X年第X回八丈町議会定例会会議録`
- 号数: `議事日程（第X号）`
- 開催日時: `令和X年X月X日（X曜日）午前X時開会`

#### 出席議員

```
出席議員（１２名）
　１番　　真　田　幸　久　君　　　　２番　　淺　沼　隆　章　君
　３番　　奥　山　幸　子　君　　　　４番　　浅　沼　清　孝　君
```

- 議員番号（全角数字）+ 氏名（全角スペース区切り）+ 「君」
- 12名の町議会

#### 説明出席者

```
地方自治法第１２１条の規定により説明のため出席した者の職氏名
　町　　長　山　下　奉　也　君　　副 町 長　山　越　　　整　君
　公営企業管理者　奥　山　　　勉　君　　教 育 長　大　澤　道　明　君
```

- 役職名 + 氏名 + 「君」

#### 発言の構造

発言者パターン:

```
○議長（山本忠志君）
○総務課長（高野秀男君）
○１番（真田幸久君）
○町長（山下奉也君）
○会計課長補佐（大澤知史君）
```

- `○` で始まり、役職 or 番号 + 括弧内に氏名 + 「君」
- 議長: `○議長（氏名君）`
- 議員: `○{番号}番（氏名君）`
- 執行部: `○{役職名}（氏名君）`

#### 議事区切り

```
◎行政報告
◎会議録署名議員の指名
◎会期の決定
◎諸般の報告
◎施政方針
```

- `◎` で始まる見出しが議事日程の各項目を示す
- 罫線 `───...───` で区切られる

#### その他の表記

```
（「異議なし」の声あり）
（発言する者なし）
（午前　９時００分）
```

- 括弧内に場内の状況、時刻等を記載

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（山本忠志君） → role="議長", name="山本忠志"
// 例: ○１番（真田幸久君） → role="１番", name="真田幸久"

// 議事項目の検出
const agendaPattern = /^◎(.+)/;

// 開催日の抽出（各号冒頭）
const datePattern = /(?:令和|平成)[０-９]+年[０-９]+月[０-９]+日/;

// 号数の抽出
const issuePattern = /第([０-９]+)号/;

// 時刻の抽出
const timePattern = /（(午前|午後)　*[０-９]+時[０-９]*分?）/;
```

---

## 注意事項

- 会議録は PDF 形式のみで公開されているため、HTML パースではなく PDF テキスト抽出が必要
- PDF は定例会ごとに1ファイルにまとめられている場合がある（例: 第一回定例会の全4号が1つの PDF）
- 一方、号ごとに個別の PDF として公開されている場合もある（年度によって異なる）
- 全角数字・全角スペースが多用されている
- ページネーションなし（単一ページに全データ）
- 議員数は12名と小規模な議会

---

## 推奨アプローチ

1. **一覧ページから全 PDF リンクを一括取得**: ページネーションがないため、1回のリクエストで全 PDF の URL を収集可能
2. **PDF テキスト抽出ライブラリの活用**: `pdf-parse` 等を使い、PDF からテキストを抽出してパース
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの PDF リンク数を前回と比較し、新規追加分のみダウンロード・パースする
5. **ファイルサイズに注意**: 定例会全体を含む PDF は数百ページになることがあるため、メモリ使用量に留意
