# つがる市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.tsugaru.aomori.jp/soshiki/shigikai/kaigiroku/index.html
- 分類: 自治体 CMS による PDF 公開（年度別リンク集 → PDF ファイル）
- 文字コード: UTF-8
- 特記: 会議録はすべて PDF ファイルで提供。平成25年以降の会議録が公開されている。

検索システムは存在しない。年度別のリンク集ページから PDF ファイルへの直接リンクで構成されている。

## HTML 構造

### 会議録トップページ（index.html）

- `h1` 見出し「会議録検索」
- 説明文「平成25年以降の市議会の会議録が閲覧できます」
- `ul > li > a` で年度別ページへのリンクを列挙（14年度分）

### 年度別会議録一覧ページ

- 各会議録の PDF ファイルへのリンクが `ul > li > a` のリスト形式で列挙
- リンクテキストに会議名（例: 「令和7年第1回定例会本会議」）を含む
- ファイルサイズが括弧付きで表記（例: `(PDFファイル: 1.9MB)`）

### 会議種別

年度ごとに以下の会議種別が存在:

- **定例会本会議**: 年4回（3月、6月、9月、12月）
- **臨時会本会議**: 不定期
- **予算特別委員会**: 定例会に対応して開催
- **予算・決算特別委員会**: 9月定例会に対応して開催（決算審査を含む）

## PDF ファイル名パターン

ファイル名の命名規則は年度によって統一されていない。

### 令和6年以降（比較的統一）

```
R{和暦2桁}{月2桁}{会議種別}.pdf
```

例:
- `R0703teireikai.pdf` → 令和7年3月定例会本会議
- `R0701rinnjikai.pdf` → 令和7年1月臨時会
- `R0703yosanntokubetuiinnkai.pdf` → 令和7年3月予算特別委員会
- `R0609yosannkessanntokubetuiinnkai.pdf` → 令和6年9月予算・決算特別委員会

### 平成31年・令和元年（混在）

```
310213rinji.pdf          → 平成31年2月臨時会

r0106t.pdf               → 令和元年6月定例会
201909kaigirokuhonnkaigi.pdf → 令和元年9月本会議（西暦パターン）
```

## スクレイピング方針

### Step 1: 年度別ページ URL の収集

会議録トップページ `index.html` から年度別ページへのリンクを収集する。

- `a[href*="/soshiki/shigikai/kaigiroku/"]` で年度別ページのリンクを抽出
- `index.html` 自体へのリンクは除外

### Step 2: PDF リンクの収集

各年度別ページから PDF ファイルへのリンクを収集する。

- `a[href$=".pdf"]` で PDF リンクを抽出
- リンクテキストから会議名を取得（例: 「令和7年第1回定例会本会議」）
- PDF の URL はすべて `/material/files/group/27/` 配下

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

- PDF パーサー（`@opendataloader/pdf` 等）でテキスト化
- 発言者・発言内容の構造化は PDF 内部のレイアウトに依存

### Step 4: メタ情報の抽出

リンクテキストから以下を抽出:

- 年度（和暦 → 西暦変換）
- 会議種別（定例会 / 臨時会）
- 会議名（本会議 / 予算特別委員会 / 予算・決算特別委員会）
- 回次（第1回、第2回 など）

```typescript
// リンクテキストからのメタ情報抽出（案）
const metaPattern = /(?:令和|平成)(\d+)年第(\d+)回(定例会|臨時会)(本会議|予算特別委員会|予算・決算特別委員会)/;
// 例: "令和7年第1回定例会本会議" → year="7", round="1", type="定例会", name="本会議"
```

---

## 注意事項

- 会議録はすべて PDF のため、HTML パースではなく PDF テキスト抽出が必要
- PDF ファイルサイズは 300KB〜2MB 程度（ダウンロード時のレート制限に注意）
- ファイル名の命名規則が年度によって異なるため、ファイル名からのメタ情報抽出は不安定（リンクテキストを優先すべき）
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **2段階クロール**: トップページ → 年度別ページ → PDF リンク収集の2段階でリンクを網羅的に取得
2. **リンクテキスト活用**: ファイル名よりもリンクテキストからメタ情報を抽出する方が安定
3. **PDF テキスト抽出**: `@opendataloader/pdf` を使用して PDF からテキストを抽出
4. **差分更新**: 年度別ページ ID は昇順のため、新しいページ ID のみを対象にした差分更新が可能
5. **レート制限**: PDF ダウンロード時は特にファイルサイズが大きいため、適切な間隔を空ける
