## 会議種別

各年度ページ内で `<h2>` タグにより会議種別が区分される。

- **定例会**: 年4回（3月・6月・9月・12月）
- **臨時会**: 不定期（年2〜3回程度）

会議録は会議日ごとに分割され、「第N日」として個別の PDF で公開される。各定例会・臨時会には目次 PDF も付属する。

---

## PDF ファイルの命名規則

ファイル名は `{元号下2桁}{月2桁}{種別}{日数}.pdf` のパターンに従う。

### パターン

| 種別 | ファイル名パターン | 例 |
| --- | --- | --- |
| 定例会 目次 | `{YY}{MM}mokuzi.pdf` | `0712mokuzi.pdf`（令和7年12月定例会目次） |
| 定例会 第N日 | `{YY}{MM}teirei{N}.pdf` | `0712teirei1.pdf`（令和7年12月定例会 第1日） |
| 臨時会 目次 | `{YY}{MM}mokuzi.pdf` | `0710mokuzi.pdf`（令和7年10月臨時会目次） |
| 臨時会 第N日 | `{YY}{MM}rinzi.pdf` | `0710rinzi.pdf`（令和7年10月臨時会 第1日） |

### 注意

- `{YY}` は令和の年数2桁（令和7年 → `07`）
- `{MM}` は月2桁（12月 → `12`）
- 臨時会のローマ字表記に揺れがある: `rinzi` と `rinnzi` が混在（令和6年では `rinnzi`、令和7年では `rinzi`）
- 一部ファイル名に月のずれがある（例: 令和7年3月定例会の第4日が `0704teirei4.pdf`）

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録トップページ `/bunya/gikai_kaigiroku/` または記事一覧ページ `/bunya/gikai_kaigiroku/gikai/` から、年度別会議録ページへのリンクを収集する。

**収集方法:**

1. トップページの `div.units ul` 内の `<a>` タグから `/docs/{記事ID}/` 形式のリンクを抽出
2. 記事一覧ページ `/bunya/gikai_kaigiroku/gikai/` にはトップページに表示されない古い年度も含まれる（平成27〜28年）

```typescript
// リンク抽出パターン
const yearPagePattern = /href="(\/docs\/\d+\/)"/g;
```

### Step 2: 各年度ページから PDF リンクの抽出

各年度ページの `div.body` 内から PDF へのリンクを抽出する。

**HTML 構造:**

```html
<div class="body">
  <h2>令和７年 第４回（１２月）定例会</h2>
  <p><a class="iconFile iconPdf" href="./files/0712mokuzi.pdf">第４回定例会目次(86KB)</a></p>
  <p><a class="iconFile iconPdf" href="./files/0712teirei1.pdf">第４回定例会（第１日）(266KB)</a></p>
  ...
</div>
```

**収集方法:**

1. `div.body` 内の `a.iconPdf`（または `a[href$=".pdf"]`）を全て抽出
2. `href` は相対パス（`./files/xxx.pdf`）のため、ベース URL `/docs/{記事ID}/` と結合して絶対 URL を生成
3. 直前の `<h2>` タグから会議名（定例会/臨時会、回数、月）を取得し、PDF と紐付ける

```typescript
// PDF リンクの抽出
const pdfLinkSelector = 'div.body a[href$=".pdf"]';

// 会議名の抽出（h2 タグ）
const sessionPattern = /^(令和|平成)\S+年\s+第(\d+)回（(\d+)月）(定例会|臨時会)$/;
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 各 PDF を `http://www.town.mihama.wakayama.jp/docs/{記事ID}/files/{ファイル名}.pdf` からダウンロード
2. PDF からテキストを抽出（pdf-parse 等を使用）
3. 目次 PDF（`mokuzi.pdf`）はメタ情報として活用可能だが、本文データとしては不要

---

## 注意事項

- **HTTP のみ**: サイトが HTTPS に対応していないため、`http://` で接続する必要がある
- **レスポンス遅延**: 過去の調査でタイムアウトが発生している。接続タイムアウトを 30 秒以上に設定し、リトライ処理を実装する
- **ファイル名の揺れ**: 臨時会の PDF ファイル名が年度によって `rinzi` / `rinnzi` と統一されていない。`href` 属性から直接取得する方が安全
- **相対パス**: PDF リンクは `./files/xxx.pdf` の相対パスで記述されている。URL 結合時に注意が必要
- **テキスト検索システムなし**: 全文検索機能は提供されていないため、PDF を全件ダウンロードしてテキスト抽出する必要がある
- **新年度の追加**: 新しい年度は記事 ID が予測不可能なため、トップページのリンク一覧から動的に取得する

---

## 推奨アプローチ

1. **2段階クロール**: まずトップ・一覧ページから年度別ページの URL を収集し、次に各年度ページから PDF リンクを収集する
2. **目次 PDF のスキップ**: `mokuzi.pdf` は目次のみのため、本文データとしてはスキップ可能（メタ情報として保持するかは要検討）
3. **レート制限**: リクエスト間に 2〜3 秒の待機時間を設ける
4. **リトライ処理**: タイムアウト（30 秒）＋最大 3 回リトライで安定性を確保する
5. **差分更新**: 年度別ページの記事 ID と PDF リンク一覧をキャッシュし、前回取得済みの PDF はスキップする
6. **RSS/Atom の活用**: トップページに RSS (`index.rss`) / Atom (`index.atom`) フィードが提供されているため、新規記事の検出に活用可能
