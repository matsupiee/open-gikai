## HTML 構造

### 一覧ページ（トップ）

年度別ページへのリンクが `<ul class="level1col2">` 内の `<li class="page">` として列挙される。

```html
<ul class="level1col2 clearfix">
  <li class="page">
    <a href="https://www.town.nagawa.nagano.jp/gyoseijoho/gikai/teireikai-rinjikai/1/2426.html">令和7年定例会会議録</a>
  </li>
  ...
</ul>
```

- ページネーション: なし（全年度が 1 ページに表示）

### 年度別ページ

PDF リンクが `<p class="file-link-item">` として列挙される。

```html
<p class="file-link-item">
  <a class="pdf" href="//www.town.nagawa.nagano.jp/material/files/group/11/{ファイル名}.pdf">
    令和6年長和町議会第1回定例会会議録 (PDFファイル: 1.4MB)
  </a>
</p>
```

- 各定例会（年 4 回: 3月/6月/9月/12月）の PDF がリスト表示される
- リンクテキストから会議名・回次を抽出可能
- ファイルサイズがリンクテキスト内に記載

## PDF ファイル命名規則

ファイル名に統一的な命名規則はなく、年度によって異なるパターンが使われている。

| 年度 | パターン | 例 |
| --- | --- | --- |
| 令和5年, 令和4年, 令和3年 | `R{年}nen{月}gatuteireikaikaigiroku.pdf` | `R5nen3gatuteireikaikaigiroku.pdf` |
| 令和6年, 令和2年 | `reiwa{年}nennagawamatigikaidai{回}kaiteireikaikaigiroku.pdf` | `reiwa6nennagawamatigikaidai1kaiteireikaikaigiroku.pdf` |
| 令和7年 | 不規則（ファイルごとに異なる） | `reiwa7nen3gatsureireikaikaigiroku.pdf` |

ファイル名のパターンに依存せず、HTML から href を直接取得する方針が安全。

## PDF 内部構造

### 全体構成

1 つの PDF に定例会全日程分の会議録がまとめられている。構成は以下の通り:

1. **表紙**: 「令和X年X月定例会 長和町議会会議録」、開会・閉会日
2. **議会関係日程表**: カレンダー形式で本会議・委員会の日程一覧
3. **各号（日ごとの会議録）**: 「第1号（X月XX日）」のように号数と日付で区分
4. **議事日程**: 日程番号・議案番号・議題の一覧
5. **出席議員一覧**: 番号・氏名
6. **説明出席者一覧**: 役職・氏名
7. **議会事務局出席者**
8. **本文（発言録）**: 発言者ごとの逐語録

### 発言者パターン

発言者は `○` で始まり、役職（氏名）の形式で記載される。

```
○議長（森田公明君）
○事務局長（米沢　正君）
○６番（羽田公夫君）
○代表監査委員（丸山淳子君）
○土地開発公社理事長（高見沢高明君）
○町長（羽田健一郎君）
○教育長（藤田仁史君）
○企画財政課長（藤田健司君）
```

### 発言者パターンの特徴

- `○` で始まる行が発言者の切り替わり
- 役職名には「議長」「事務局長」「X番」「町長」「教育長」「XX課長」「代表監査委員」等がある
- 氏名は全角括弧 `（）` 内に記載、敬称は `君` が付く
- 議員番号は全角数字で表記（例: `６番`）

### 議事区切りパターン

```
◎日程第１　会議録署名議員の指名
◎日程第２　会期の決定
```

- `◎` で始まる行が議事日程の区切り
- 罫線 `─────` で議事の区切りが示される場合もある

### その他の記述パターン

```
（「異議なし」の声あり）
（全　員　挙　手）
（「なし」の声あり）
```

- 議場の状況は全角括弧 `（）` で囲まれる

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（森田公明君） → role="議長", name="森田公明"
// 例: ○６番（羽田公夫君） → role="６番", name="羽田公夫"

// 議事日程の区切り
const agendaPattern = /^◎日程第(\d+)/;

// 議場状況（発言ではない）
const situationPattern = /^（.+）$/;

// 開催日・開会時刻の抽出（各号冒頭）
const sessionHeaderPattern = /令和(\d+)年(\d+)月(\d+)日.+午(?:前|後)\s*(\d+)時(\d+)分開会/;
```

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

一覧ページ `index.html` から `<li class="page"> > a` のリンクを取得する。

- セレクタ: `ul.level1col2 li.page a`
- リンクテキストから年度情報を抽出

### Step 2: PDF URL の収集

各年度別ページから `<p class="file-link-item"> > a.pdf` のリンクを取得する。

- セレクタ: `p.file-link-item a.pdf`
- `href` はプロトコル省略形式（`//www.town.nagawa.nagano.jp/...`）のため、`https:` を補完する
- リンクテキストから会議名（「令和X年長和町議会第X回定例会会議録」）を抽出

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で全文テキスト化する。

### Step 4: テキストのパース

1. PDF テキストを号（日）ごとに分割（「第X号（X月XX日）」で区切り）
2. 各号から出席議員一覧・説明出席者一覧を抽出
3. 発言者パターン `○{役職}（{氏名}君）` で発言を分割
4. 各発言に対して発言者・本文を構造化

---

## 注意事項

- PDF ファイルのため、テキスト抽出精度に依存する（フォント埋め込み、スキャン画像等の影響）
- 全角スペースが多用されているため、テキスト抽出時にスペースの正規化が必要
- 1 定例会で 1 PDF のため、複数日分の会議録が 1 ファイルに含まれる
- ファイルサイズは 768KB〜3.6MB 程度
- 臨時会の会議録は掲載されていない（定例会のみ）
- ページ ID は連番ではなく不規則なため、一覧ページからの動的取得が必須

---

## 推奨アプローチ

1. **HTML からの PDF URL 取得を優先**: ファイル名パターンが不規則なため、必ず HTML ページから PDF リンクを取得する
2. **PDF テキスト抽出**: pdf-parse や pdfjs-dist 等のライブラリでテキスト化し、正規表現でパースする
3. **号単位での分割**: 1 PDF に複数日の会議録が含まれるため、「第X号」で分割してから各日の発言をパースする
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページの年度リンクと PDF リンクを比較し、新規追加分のみダウンロードする
