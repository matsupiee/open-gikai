# 広川町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.hirokawa.fukuoka.jp/soshiki/gikai_jimukyoku/1/1/1/1/index.html
- 分類: 町公式サイト内で年度別に PDF ファイルを公開（独立した検索システムなし）
- 文字コード: UTF-8
- 特記: Joruri CMS ベースの公式サイト。各会議録は個別ページとして存在し、PDF ファイルへのリンクを提供する。1回の会議（複数日にまたがる場合は日ごと）に対して PDF が1ファイル存在する

## 会議録個別ページの構造

各個別ページは以下を含む:

- 会議名（タイトル）
- 開催日（本文テキスト）
- PDF ダウンロードリンク（日程ごとに1ファイル）
- 問い合わせ先（議会事務局）

**PDF URL の形式:**

```
https://www.town.hirokawa.fukuoka.jp/material/files/group/37/{ファイル名}.pdf
```

### PDF ファイル名パターン

2通りのパターンが混在している:

| パターン | 例 | 説明 |
| --- | --- | --- |
| `R{年}{月}{日}.pdf` | `R060105.pdf` | 令和表記: R + 年(2桁) + 月(2桁) + 日(2桁) |
| `{年}{月}{日}.pdf` | `060606.pdf` | 数字6桁: 年(2桁) + 月(2桁) + 日(2桁) |

例:
- `R060105.pdf` → 令和6年1月5日
- `060606.pdf` → 令和6年（平成換算年）6月6日
- `060613.pdf` → 令和6年6月13日

## スクレイピング戦略

### Step 1: 年度別一覧ページから個別ページ URL を収集

各年度の `index.html` を取得し、会議録個別ページへのリンクを抽出する。

- リンクは `<a href="{数値ID}.html">` 形式の相対パス
- ページネーションはなし（1ページに全件掲載）
- 年度ディレクトリの URL は `5_{N}` 形式で推定可能

**年度ディレクトリのマッピング（確認済み）:**

| 年度 | ディレクトリ |
| --- | --- |
| 令和4年度 | `5_2` |
| 令和5年度 | `5_3` |
| 令和6年度 | `5_4` |
| 令和7年度 | `5_5` |
| 令和8年度 | `5_6` |

令和3年度以前については、トップページ `index.html` のリンクを確認して特定する。

### Step 2: 個別ページから PDF URL を抽出

各個別ページを取得し、PDF ファイルへのリンクを Cheerio で抽出する。

- PDF リンクは `href` に `/material/files/group/37/` を含む `<a>` タグ
- 1ページにつき複数 PDF がある場合（複数日程にまたがる定例会）も存在する

### Step 3: PDF のダウンロードと解析

- PDF を取得してテキストを抽出する
- テキスト形式の会議録 PDF であるため、PDFtoText 等でテキスト抽出可能

---

## 注意事項

- 個別ページの数値 ID に連番規則があるが、飛び番が存在するため全量取得は年度別一覧から辿る方式を推奨
- PDF ファイル名は日付ベースだが `R` プレフィックスの有無が混在するため、ファイル名からのメタ情報推定には注意が必要
- 年度ディレクトリ `5_{N}` は令和4年度が `5_2` であり、令和3年以前は別の体系の可能性がある
- トップの年度一覧ページ（`index.html`）は JavaScript で動的生成されているが、各年度の `index.html` は静的 HTML

---

## 推奨アプローチ

1. **トップページから年度リスト取得**: `index.html` から各年度ディレクトリの URL を動的に取得（将来の年度追加に自動対応）
2. **年度別一覧の静的クロール**: 各年度 `index.html` から Cheerio で個別ページ URL を抽出
3. **PDF URL 抽出**: 各個別ページから `/material/files/group/37/` を含むリンクを抽出
4. **差分更新**: 既取得の個別ページ ID を記録し、新規追加分のみ処理
5. **レート制限**: 町公式サイトへの負荷を抑えるため、リクエスト間に 1〜2 秒の待機を設ける
