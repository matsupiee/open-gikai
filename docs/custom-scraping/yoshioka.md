# 吉岡町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.yoshioka.lg.jp/gikai/kaigiroku/
- 分類: PDF 年別公開（検索機能なし）
- 文字コード: UTF-8
- 特記: 会議録は全て PDF ファイルで公開。HTML 上の会議録本文は存在しない

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録トップ（最新年） | `https://www.town.yoshioka.lg.jp/gikai/kaigiroku/` |
| 過去年度一覧 | `https://www.town.yoshioka.lg.jp/gikai/kaigiroku/bn_{西暦年}.html` |
| 会議録 PDF | `https://www.town.yoshioka.lg.jp/gikai/kaigi/pdf/{ファイル名}.pdf` |
| 会議録 PDF（一部） | `https://www.town.yoshioka.lg.jp/gikai/kaigi/docs/{URLエンコード済みファイル名}.pdf` |

### 過去年度一覧の URL 例

- 2024年: `bn_2024.html`
- 2023年: `bn_2023.html`
- ...
- 2009年（平成21年）: `bn_2009.html`

※ 最新年（2025年）はトップページ `/gikai/kaigiroku/` に直接掲載される。

---

## 公開範囲

- 2009年（平成21年）〜 現在（2025年）
- 年あたり定例会 4 回 + 臨時会数回（年により異なる）
- 各会議につき PDF 1 ファイル（会議全体が 1 つの PDF にまとまっている）

---

## 会議の種類

| 種別 | 説明 |
| --- | --- |
| 定例会 | 第1回〜第4回（3月・6月・9月・12月） |
| 臨時会 | 年により回数が異なる（0〜5回程度） |

---

## PDF ファイル名のパターン

ファイル名に統一的な命名規則はなく、年度やアップロード時期により異なる。

| パターン | 例 |
| --- | --- |
| ハッシュ値型 | `f4dab915d596211a651c4213ea72dfe2_1.pdf` |
| 命名規則型（旧） | `kaigiroku_21t04.pdf`（平成21年第4回定例会） |
| 命名規則型（中期） | `r05-01t.pdf`（令和5年第1回定例会） |
| 混合型 | `6_kaigiroku_r5_3t.pdf` |
| URL エンコード日本語型 | `令和７年第３回定例会.pdf`（docs ディレクトリ配下） |

※ ファイル名からメタ情報を機械的に抽出するのは困難。HTML ページ上の会議名テキストを優先してメタ情報を取得する。

---

## HTML ページの構造

### トップページ / 年別ページ共通

- 各会議は `<h3>` 見出し + `<ul><li>` リストで構成
- `<h3>` に会議名（例: `令和6年第4回定例会`）
- `<li>` 内に会期（例: `2024年12月02日(月曜日)〜12月12日(木曜日)`）と PDF リンク
- PDF リンクのテキストに `会議録(PDF:2.0 MB)` のようにファイルサイズが記載

---

## スクレイピング戦略

### Step 1: 年別ページの URL リスト構築

トップページ `/gikai/kaigiroku/` から過去年度リンク `bn_{year}.html` を収集する。

- トップページ自体が最新年のデータを含む
- `bn_2009.html` 〜 `bn_2024.html` の 16 ページ + トップページ = 計 17 ページ

### Step 2: 各年ページから会議情報と PDF URL を抽出

各年別ページを Cheerio でパースし、以下を抽出:

- **会議名**: `<h3>` テキスト（例: `令和6年第4回定例会`）
- **会期**: `<li>` 内の日付テキスト
- **PDF URL**: `<a>` タグの `href` 属性（`/gikai/kaigi/pdf/...` または `/gikai/kaigi/docs/...`）

### Step 3: PDF のダウンロード

抽出した PDF URL からファイルをダウンロードする。

### Step 4: PDF からテキスト抽出

PDF からテキストを抽出する。ツールは `pdf-parse` や `pdftotext` 等を使用。

### Step 5: テキストのパース

PDF から抽出したテキストを解析し、発言単位に分割する。

#### メタ情報の抽出

HTML ページの `<h3>` テキストから:

```
令和6年第4回定例会
```

- 年号: `令和6年` → 西暦変換
- 回次: `第4回`
- 種別: `定例会` / `臨時会`

```typescript
const meetingPattern = /^(令和|平成)(\d+)年第(\d+)回(定例会|臨時会)$/;
```

会期は HTML の日付テキストから:

```typescript
const datePattern = /(\d{4})年(\d{2})月(\d{2})日/;
```

#### 発言者パターン（PDF テキスト内、要実地確認）

PDF 内の発言者表記は実際のファイルを確認して決定する必要がある。一般的な町議会 PDF では以下のパターンが想定される:

```
○議長（山田太郎君）
○1番（佐藤花子君）
○町長（鈴木一郎君）
```

※ 実際の PDF を数ファイルダウンロードして発言者パターンを確定させること。

---

## 注意事項

- **検索機能なし**: サイトに会議録検索機能は存在しない。全量取得のみ
- **PDF のみ**: HTML 形式の会議録本文は公開されていない
- **ファイル名不統一**: PDF ファイル名に統一規則がないため、ファイル名からのメタ情報抽出は不可
- **1 会議 1 PDF**: 各会議につき 1 つの PDF（複数日程の会議でも 1 ファイルにまとまっている）
- **ファイルサイズ**: 定例会は 1.7〜3.5 MB、臨時会は 100〜900 KB 程度

---

## 推奨アプローチ

1. **HTML ページからメタ情報を取得**: 会議名・会期・PDF URL は HTML ページから抽出する（PDF ファイル名に依存しない）
2. **全量取得**: 17 ページ分の HTML をクロールし、全 PDF URL を収集してからダウンロード
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **PDF テキスト抽出の品質確認**: PDF の生成方法（スキャン画像 vs テキスト埋め込み）により抽出品質が異なるため、事前に数ファイル確認すること
5. **差分更新**: 年別ページの構造上、最新年のトップページのみを定期的にチェックし、新規 PDF があれば追加取得する
