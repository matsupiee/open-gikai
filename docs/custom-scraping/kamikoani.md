# 上小阿仁村議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.vill.kamikoani.akita.jp/menu/6
- 分類: 村公式サイトで年度別に PDF 形式の会議録を公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF ファイルで提供。検索機能なし。4 つの年度グループページに分かれて公開。

## PDF ファイル名の規則

### 定例会

年度グループにより命名規則が異なる。

**令和7年以降:**

```
R{和暦年}-{回}-{連番}.pdf
例: R7-8-1.pdf → 令和7年第8回定例会 第1分割
    R8-1-1.pdf → 令和8年第1回臨時会
```

**令和元年〜令和4年:**

- 数字連番パターン: `R{和暦年}-{回}-{連番}.pdf`（例: `R4-2-1.pdf`）
- アルファベットパターン: `R{和暦年}-{回}-{a-g}.pdf`（例: `R2-3-a.pdf`）
- 日付パターン（臨時会）: `{和暦年}.{月}.{日}.rinji.pdf`（例: `1.11.7.rinji.pdf`）

**平成31年:**

```
31-3-{a-f}.pdf（定例会）
31.3.28rinji.pdf（臨時会）
```

**平成26年〜平成30年:**

```
H{和暦年}-{月}-{a-h}.pdf（定例会）
{和暦年}.{月}.{日}.pdf（臨時会）
例: H30-12-a.pdf → 平成30年12月定例会 第1分割
    30.9.26.pdf → 平成30年9月26日臨時会
```

**平成21年〜平成25年:**

```
H{和暦年}-{月}-{a-h}.pdf（定例会）
例: H25-12-a.pdf
```

### ファイル名の注意点

- 命名規則が年度グループごとに統一されていない
- 同一年度内でも数字連番とアルファベットが混在する場合がある
- ディレクトリ構造も `/pdf/gikai/` と `/pdf/` と `/dload/gikai/` が混在

## 会議録の構造

### 会議種別

- **定例会**: 年 4〜6 回程度。複数日にわたり、複数の PDF に分割される。
- **臨時会**: 不定期。通常 1 日で完結し、1 つの PDF にまとめられる。

### 定例会の分割パターン

定例会は内容に応じて複数の PDF に分割されている:

1. 開会〜施政方針・行政報告
2. 一般質問（議員ごとに 1 ファイル）
3. 議案上程〜散会
4. 開議〜閉会

分割数は定例会によって異なる（4〜9 ファイル程度）。

### メタ情報

一覧ページのリンクテキストから以下を取得可能:

- 会議種別（定例会 / 臨時会）
- 回次（第 N 回）
- 開催日（月日）
- 内容区分（開会〜行政報告、一般質問（議員名）、議案上程 等）
- 一般質問の発言議員名

## スクレイピング戦略

### Step 1: 一覧ページから PDF リンクを収集

4 つの一覧ページ（`/info/663`, `/info/302`, `/info/288`, `/info/301`）をクロールし、全 PDF リンクを収集する。

**収集方法:**

1. 各一覧ページの HTML を取得
2. `<a>` タグから PDF へのリンク（`.pdf` で終わる href）を Cheerio で抽出
3. リンクテキストからメタ情報（会議種別、回次、日付、内容区分）をパース
4. ページネーションなし（各ページに全件表示）

**リンクテキストのパースパターン:**

```typescript
// 丸数字付きのテキストからメタ情報を抽出
// 例: "①開会～施政方針・行政報告"
// 例: "②一般質問（河村良満）"
// 例: "⑦開議～閉会"
const contentPattern = /^[①-⑳](.+)/;

// 一般質問の議員名を抽出
const questionPattern = /一般質問[（(](.+?)[）)]/;
```

### Step 2: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、テキストを抽出する。

**処理方法:**

1. PDF をダウンロード（レート制限を遵守）
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容を構造化

### Step 3: 会議録のパース

PDF から抽出したテキストを構造化データに変換する。

#### 発言者パターン（想定）

PDF の会議録テキストは一般的に以下のパターンを含む:

```
○議長（氏名）
○N番（氏名）
○村長（氏名）
○副村長（氏名）
```

※ 実際のパターンは PDF テキスト抽出後に確認・調整が必要。

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（氏名） → role="議長", name="氏名"

// 開催日の抽出（一覧ページのテキストから）
const datePattern = /(\d+)月(\d+)日/;
```

---

## 注意事項

- PDF ファイルのため、テキスト抽出の精度は PDF の作成方法に依存する
- スキャン画像ベースの PDF の場合は OCR が必要になる可能性がある
- ファイル名の命名規則が年度によって異なるため、一覧ページのリンクから URL を取得するのが最も確実
- 一部の PDF パスが `/div/gikai/dload/gikai/` になっているものがある（主に古い年度）
- PDF 内のテキストレイアウト（段組み等）によってはパース精度が低下する可能性がある

---

## 推奨アプローチ

1. **一覧ページ起点**: ファイル名規則が不統一なため、URL を推測するのではなく必ず一覧ページからリンクを収集する
2. **PDF テキスト抽出の事前検証**: 少数の PDF をサンプルとしてダウンロードし、テキスト抽出の品質を確認する（スキャン画像 vs テキスト埋め込み）
3. **メタ情報は一覧ページから取得**: 会議種別・回次・日付・発言議員名は PDF 内よりも一覧ページのリンクテキストから取得するほうが確実
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 最新の一覧ページ（`/info/663`）は更新されるため、定期的にチェックして新規 PDF のみを取得する
