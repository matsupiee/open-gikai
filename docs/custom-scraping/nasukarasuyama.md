## サイト構造

### 会議録一覧ページ (`index.html`)

- 年別のリンクが単一ページにリスト表示される
- ページネーションなし
- サイドバーにも同じ年別リンク一覧がある

### 年別会議録ページ (`page{ID}.html`)

- `<h2>` タグで会議種別（定例会・臨時会）ごとにセクション分け
- 各セクション内に `<ul><li>` で PDF リンクが列挙される
- 各 `<li>` に PDF の URL、リンクテキスト、ファイルサイズが含まれる

HTML 構造の例:

```html
<h2>第5回12月定例会</h2>
<ul>
  <li><a href="https://www.city.nasukarasuyama.lg.jp/data/doc/1771461228_doc_40_0.pdf" target="_blank" rel="noopener">第5回12月定例会 目次</a> [PDF形式／67.86KB]</li>
  <li><a href="https://www.city.nasukarasuyama.lg.jp/data/doc/1771461232_doc_40_0.pdf" target="_blank" rel="noopener">第5回12月定例会 第1日（11月28日）</a> [PDF形式／430.38KB]</li>
  ...
</ul>
```

### 会議種別

- 定例会: 3月、6月、9月、12月（年4回が基本）
- 臨時会: 不定期

リンクテキスト形式:

- `第{回}回{月}月定例会 目次`
- `第{回}回{月}月定例会 第{日}日（{月}月{日}日）`
- `第{回}回{月}月{日}日臨時会 目次`
- `第{回}回{月}月{日}日臨時会 第{日}日（{月}月{日}日）`

## PDF の構造

### メタ情報（1〜4ページ目）

各 PDF の冒頭に以下の情報が記載される:

1. **ヘッダー**: `令和７年第５回12月定例会　第１日` （全ページ共通）
2. **タイトル**: `令和７年第５回那須烏山市議会１２月定例会（第１日）`
3. **開催日**: `令和7年１１月２８日（金）`
4. **開会・散会時刻**: `開会　午前１０時００分` / `散会　午後　１時３７分`
5. **出席議員一覧**: 番号 + 氏名
6. **欠席議員**
7. **説明のため出席した者の職氏名**: 役職 + 氏名
8. **事務局職員出席者**
9. **議事日程**

### 発言の構造（本文部分）

発言者パターン:

```
○議長（中山五男）　　おはようございます。...
○市長（川俣純子）　　...
○１２番（渋井由放）　　...
○副市長（熊倉精介）　　...
○教育長（内藤雅伸）　　...
○総合政策課長（小原沢一幸）　　...
```

- `○` で始まり、役職 or 番号 + 括弧内に氏名
- 発言内容は同行の氏名閉じ括弧の後に続く
- 全角数字を使用

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（中山五男） → role="議長", name="中山五男"
// 例: ○１２番（渋井由放） → role="１２番", name="渋井由放"

// ヘッダーから会議情報を抽出
const headerPattern = /(.+?)年第(.+?)回(.+?月)定例会/;
// 例: 令和７年第５回12月定例会

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 開会時刻
const openingPattern = /開会\s+(.+)/;
```

## スクレイピング戦略

### Step 1: 年別ページ URL の収集

会議録一覧ページ (`index.html`) またはサイドバーから年別ページの URL を取得する。

- サイドバーの `<li class="iconPage">` から年別ページ URL を抽出
- 既知の ID 一覧（上記テーブル）をハードコードしても可

### Step 2: PDF URL の収集

各年別ページの `<article id="contents">` 内から PDF リンクを収集する。

**収集方法:**

1. 年別ページの HTML を取得
2. `<h2>` タグから会議種別（定例会名・臨時会名）を取得
3. 各 `<h2>` に続く `<ul>` 内の `<a>` タグから PDF URL とリンクテキストを取得
4. リンクテキストから会議名、日次、開催日を抽出

**リンクテキストのパース:**

```typescript
// 「第5回12月定例会 第1日（11月28日）」→ 会議名, 日次, 月, 日
const linkPattern = /^(.+?)\s+(目次|第(\d+)日（(\d+)月(\d+)日）)$/;
```

- 「目次」を含むリンクはスキップ可能（目次 PDF は発言内容を含まない）

### Step 3: PDF のダウンロードとテキスト抽出

1. 各 PDF をダウンロード
2. PDF からテキストを抽出（`pdf-parse` 等のライブラリを使用）
3. 抽出したテキストをパースして発言データを構築

### Step 4: 発言データのパース

1. ヘッダー部分（出席議員一覧等）と本文（発言部分）を分離
2. `○` で始まる行を発言者の区切りとして認識
3. 各発言ブロックから発言者（役職・氏名）と発言内容を抽出

---

## 注意事項

- PDF のテキスト抽出精度はライブラリに依存する。レイアウトが複雑な場合（2段組の出席議員一覧等）は抽出順序が乱れる可能性がある
- 全角数字と半角数字が混在する（ヘッダーでは半角、本文では全角が多い）
- 年別ページの ID は不規則なため、新しい年が追加された場合はインデックスページから動的に取得する必要がある
- PDF の URL に含まれる ID（例: `1771461228`）はタイムスタンプ風の数値で、連番ではない

---

## 推奨アプローチ

1. **インデックスページから動的取得**: 年別ページ URL をハードコードせず、`index.html` のサイドバーから取得することで新年度の追加に自動対応
2. **目次 PDF はスキップ**: 「目次」リンクの PDF は発言内容を含まないためダウンロード不要
3. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` 等を使用し、テキストレイヤーから発言を抽出
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 各年別ページの PDF リンク数を前回と比較し、新規追加分のみダウンロードする
