## データ公開形式

### 会議録の構造

会議録は **PDF ファイル** で公開されている。HTML 上にテキストとして会議録が掲載されることはない。

各定例会・臨時会の詳細ページには以下の情報が含まれる:

- 会議名（例: 令和7年 第1回定例会（3月））
- 開催日一覧
- PDF ファイルへのリンク（会議種別ごと）

### PDF ファイルの種別

1. **本会議** - 複数日分の会議録が 1 つの PDF にまとまっている
2. **予算決算常任委員会** - 補正予算審査、予算審査などに分かれる場合がある

### PDF ファイル名のパターン

```
# 本会議
20250306_kaigiroku_honkaigi.pdf
20221206_kaigiroku_honkaigi.pdf
R6_4_kaigiroku_honkaigi.pdf

# 予算決算常任委員会
R7_3_yosankaigiroku1.pdf
R7_3_yosankaigiroku2.pdf
20231206_R4_4teireikai_kaigiroku_yosannkessann.pdf
20241210_kaigiroku_yosankessan.pdf

# 臨時会
R7_rinzikai_1_kaigiroku.pdf
```

※ ファイル名の命名規則は統一されておらず、年度や会議によって異なる。

## 中継録画配信サイト（gikai.town.itakura.gunma.jp）

中継録画配信サイトは会議録テキストを含まず、以下の情報のみを提供:

- 議題・議案の一覧（見出しレベル）
- 一般質問の質問項目と議員名
- YouTube 動画へのリンク

会議録テキストの取得には使用できない。

## スクレイピング戦略

### Step 1: 年度別ページから会議一覧を収集

会議録一覧ページ（`d000070/d000030/index.html`）から各年度のリンクを取得し、年度別ページから定例会・臨時会の詳細ページ URL を収集する。

**収集方法:**

1. `d000070/d000030/index.html` から年度別ページのリンクを取得
2. 各年度ページ（`d000070/d000030/d000{NNN}/index.html`）から会議詳細ページのリンクを取得
3. リンクのパターン: `/cont/s034000/d034010/{ID}.html`

### Step 2: 詳細ページから PDF リンクを収集

各会議の詳細ページ（`/cont/s034000/d034010/{ID}.html`）にアクセスし、PDF ファイルへのリンクを抽出する。

- PDF リンクは相対パス `../d034040/{filename}.pdf` で記載されている
- 絶対パスに変換: `https://www.town.itakura.gunma.jp/cont/s034000/d034040/{filename}.pdf`

### Step 3: PDF のダウンロードとテキスト抽出

1. PDF ファイルをダウンロード
2. PDF からテキストを抽出（pdf-parse 等のライブラリを使用）
3. 抽出したテキストから発言者・発言内容を構造化

### Step 4: 会議録のパース

#### メタ情報

詳細ページの HTML から以下を抽出:

- 会議名（例: 令和7年 第1回定例会（3月））
- 開催日一覧
- 会議種別（本会議、予算決算常任委員会 等）

#### PDF テキストからの発言抽出

PDF テキストの構造は要調査。一般的な議会会議録 PDF では以下のパターンが想定される:

```
○議長（氏名）
  発言内容...

○N番（氏名）
  発言内容...
```

※ 実際の PDF 内部構造は PDF をダウンロードして確認する必要がある。

## 注意事項

- 会議録は PDF のみで公開されており、HTML テキストとしての会議録は存在しない
- PDF ファイルのサイズは数百 KB〜数 MB と幅がある（本会議は 3〜14 MB 程度）
- PDF ファイル名の命名規則が統一されていないため、ファイル名からのメタ情報抽出は困難
- 中継録画配信サイト（gikai.town.itakura.gunma.jp）は動画配信専用であり、会議録テキストの取得には使用しない
- PDF からのテキスト抽出精度は PDF の作成方法に依存する（スキャン画像 PDF の場合は OCR が必要）

---

## 推奨アプローチ

1. **2 段階クロール**: まず年度別→会議別で PDF リンクを収集し、次に PDF をダウンロード
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使用してテキストを抽出
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **PDF キャッシュ**: ダウンロード済み PDF はローカルにキャッシュし、再取得を回避
5. **差分更新**: 年度別ページの構造が安定しているため、最新年度のみ再クロールで差分更新が可能
