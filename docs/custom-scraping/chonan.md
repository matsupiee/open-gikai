## サイト構造

### 3 階層構成

1. **会議録一覧ページ**: 年度別リンクの一覧（平成23年〜令和7年）
2. **年度別ページ**: 各年度の定例会・臨時会ごとに PDF リンクを掲載
3. **PDF ファイル**: 会議録本文（テキスト埋め込み型 PDF）

### 年度別ページの構成

各年度ページでは、定例会・臨時会ごとにセクションが分かれ、以下の PDF がリンクされている:

- **目次**: 議事日程の概要
- **第○号（○月○日）**: 各日の会議録本文

例（令和6年第4回定例会）:
- 目次
- 第１号（12月3日）
- 第２号（12月6日）

### ページネーション

なし。各年度ページに当該年度の全会議録 PDF リンクが一括掲載される。

## PDF の構造

### メタ情報（1〜2ページ目）

PDF 冒頭に以下の情報が記載される:

1. **会議名**: `令和６年第４回長南町議会定例会`
2. **議事日程**: `議事日程（第1号）`
3. **開催日時**: `令和6年12月3日（火曜日）午前9時開会`
4. **議事日程の詳細**: 日程第1〜第N の一覧
5. **出席議員一覧**: 番号・氏名の一覧（例: `1番 太田久之`）
6. **欠席議員一覧**
7. **説明出席者一覧**: 町長、副町長、教育長、各課長の職名・氏名
8. **事務局職員一覧**: 事務局長、書記の職名・氏名

### 発言の構造（3ページ目以降）

#### 発言者パターン

```
○議長（松野唱平）
○町長（平野貞夫）
○議会運営委員長（森川剛典）
○長生郡市広域市町村圏組合議会議員（御園生　明）
○１番（太田久之）
```

- `○` で始まり、役職 or 番号 + 括弧内に氏名
- 発言内容は発言者行の後に続くテキスト（次の発言者行まで）

#### 登壇表記

```
〔町長　平野貞夫登壇〕
〔議会運営委員長　森川剛典登壇〕
```

- `〔` で始まり `登壇〕` で終わる
- 役職と氏名が全角スペース区切り

#### セクション見出し

```
◎開会の宣告
◎開議の宣告
◎議事日程の報告
◎会議録署名議員の指名
◎会期日程等の議会運営について
◎会期の決定
◎諸般の報告
◎行政報告
◎陳情第１号の上程、討論、採決
◎一般質問
```

- `◎` で始まるセクション見出し
- 議事進行の区切りを示す

#### 区切り線

- 罫線（`──────────`）でセクション間が区切られている

#### 時刻表記

```
（午前　９時００分）
```

- 括弧内に全角数字で時刻が記載される

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（松野唱平） → role="議長", name="松野唱平"
// 例: ○１番（太田久之） → role="１番", name="太田久之"

// 登壇の検出
const podiumPattern = /^〔(.+?)登壇〕/;
// 例: 〔町長　平野貞夫登壇〕

// セクション見出しの検出
const sectionPattern = /^◎(.+)/;
// 例: ◎開会の宣告

// 開催日の抽出（PDF冒頭メタ情報から）
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 会議名の抽出（PDF冒頭から）
const meetingPattern = /(?:令和|平成)\d+年第(\d+)回長南町議会(定例会|臨時会)/;
```

## スクレイピング戦略

### Step 1: 年度別ページから PDF URL を収集

会議録一覧ページ `/chousei/gikai/会議録/` から年度別ページへのリンクを取得し、各年度ページ内の PDF リンクを収集する。

**収集方法:**

1. 会議録一覧ページをフェッチし、年度別ページの URL を取得
2. 各年度ページをフェッチし、`/wp-content/uploads/` にマッチする PDF リンクを全て抽出
3. リンクテキスト（例: `令和６年第４回定例会第１号　12月3日`）から会議種別・回次・号数・日付を抽出

**リンクテキストのパースパターン:**

```typescript
// リンクテキストから情報を抽出
const linkPattern = /(?:令和|平成)\d+年第(\d+)回(定例会|臨時会)(目次|第(\d+)号)[\s　]+(\d+)月(\d+)日/;
```

### Step 2: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、テキストを抽出する。

- PDF はテキスト埋め込み型のため、`pdf-parse` や `pdfjs-dist` 等のライブラリでテキスト抽出が可能
- 目次 PDF はスキップ可能（本文 PDF に同等の情報が含まれる）

### Step 3: 会議録のパース

抽出されたテキストから、メタ情報と発言データを構造化する。

1. **メタ情報の抽出**: PDF 冒頭から会議名・開催日・出席者を取得
2. **発言の分割**: `○` で始まる行を発言者の区切りとして分割
3. **発言者情報の抽出**: 役職・氏名を正規表現で取得
4. **セクション情報の付与**: `◎` 見出しを議題情報として発言に紐付け

---

## 注意事項

- PDF のファイル名はハッシュ値のため、URL からは会議の内容を推測できない。リンクテキストの解析が必須
- 古い年度（平成23〜27年頃）の PDF は `/wp-content/uploads/2015/10/` 配下にまとめてアップロードされている（WordPress 移行時の一括アップロードと推測）
- 議員定数は9名（小規模町議会）
- 定例会は年4回（3月・6月・9月・12月）、臨時会は随時開催
- PDF のページ数は会議により異なる（数十ページ程度）

---

## 推奨アプローチ

1. **HTML → PDF URL 収集を分離**: まず全年度ページから PDF URL とメタ情報（リンクテキスト）を収集し、NDJSON として保存
2. **PDF ダウンロード → テキスト抽出を分離**: PDF のダウンロードとテキスト抽出を別ステップとし、中間ファイルとして保存
3. **テキスト → 構造化データ変換**: 抽出テキストを発言単位に分割し、最終的な NDJSON を生成
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 年度別ページの記事 ID が既知のため、新規追加された年度ページのみを対象にした差分更新が可能
