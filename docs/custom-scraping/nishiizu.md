# 西伊豆町議会 カスタムスクレイピング方針

## 概要

- サイト: http://www.town.nishiizu.shizuoka.jp/kakuka/gikai/gijiroku/index.html
- 分類: 独自 CMS による HTML 公開（専用の会議録検索システムは存在しない）
- 文字コード: UTF-8
- 特記: 議会事務局ページで会議録を PDF 形式で提供。会議名一覧から中間ページを経由して PDF へリンクされている

## PDF リンクの構造

### 中間ページの構成

各中間ページ（`78_{番号}.html`）には会議名・開催日程・主な内容と、PDF ファイルへのリンクが掲載されている。

- 臨時会: 通常 1 件の PDF（単一日）
- 定例会: 複数件の PDF（開催日数分）、各ファイルに「○日目」のテキストが付く

## スクレイピング戦略

### Step 1: 一覧ページから中間ページ URL を収集

一覧ページ `index.html` を取得し、`./78_{番号}.html` パターンの `<a>` タグを全て抽出する。

- ページネーションは存在しない（1 ページで完結）
- 各リンクの `href` を絶対 URL に変換する
- リンクテキストが会議名（例: 「令和7年第3回定例会」）として利用可能

### Step 2: 中間ページから PDF URL を収集

各中間ページを取得し、`/pdf/gikai/` を含む `<a>` タグを全て抽出する。

- 臨時会: 通常 1 件の PDF
- 定例会: 複数件の PDF（「会議録1日目」「会議録2日目」等のリンクテキスト）
- 中間ページ内に開催日情報（例: 「第一日（9月3日）」）が記載されている

### Step 3: PDF の取得

収集した PDF URL から各ファイルをダウンロードし、テキストを抽出する。

- PDF はテキスト埋め込み形式（OCR 不要の可能性が高い）
- ファイル名からの日付推測は補助的に使用する（信頼性が低いため）

---

## 注意事項

- `78_{番号}` の番号は連番ではなく欠番がある（例: 117→120→121 など）。そのため、番号の推測による URL 生成は行わず、必ず一覧ページからリンクを収集する
- PDF URL の `/pdf/gikai/{西暦年}/` のディレクトリ部分は西暦年（例: 2024, 2025, 2026）で管理されており、年をまたぐ定例会は開会年のディレクトリに格納される
- ファイル名パターンが会議ごとに微妙に異なるため（`_01ringi` あり・なし等）、ファイル名からの規則的な URL 生成は困難

---

## 推奨アプローチ

1. **一覧ページを起点とする**: `index.html` → 中間ページ → PDF の 2 ホップ構造を確実にたどる
2. **差分更新**: 既取得の `78_{番号}` を記録し、新規追加分のみ処理する（番号は増加方向に追加されるため）
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **PDF テキスト抽出**: PDF はスキャン画像ではなくテキスト埋め込みと想定されるが、実際のファイルで確認すること
