## 本会議一覧ページの構造

`/gikai/2022111000029/` が全会議録のインデックスページ。以下の構造を持つ：

- ページネーションなし（単一ページに全年度分を表示）
- テーブル形式で年度・会議種別ごとに整理

| 列 | 内容 |
| --- | --- |
| 年 | 令和・平成の年号 |

会議録のリンクは「議案上程」「一般質問（1）」「一般質問（2）」「質疑」「採決」等の区分でラベルされている。

## 会議録詳細ページの構造

### メタ情報

- `<h2>` タグに会議名（例: 「令和7年八幡浜市議会12月定例会会議録第1号」）
- 冒頭に「議事日程 第N号」「令和X年X月X日（曜日） 午前10時開議」
- 担当課: 議会事務局

### 発言者パターン

```
○議長（菊池　彰君）
○市長（大城一郎君）
○総務課長（河野光徳君）
〔市長　大城一郎君登壇〕
```

- `○` で始まり、役職 + `（` 氏名 + `君）` の形式
- 登壇表記は `〔役職　氏名君登壇〕` 形式
- 専用の HTML タグはなく、`<p>` タグのプレーンテキストとして記述される

### 本文構造

- 発言内容は `<p>` タグ複数で構成
- 段落内の論理区切りには全角スペース（`　`）を使用
- 見出し構造: `<h2>`（会議名） → `<h3>`（各議案セクション）

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（菊池　彰君） → role="議長", name="菊池　彰"

// 登壇の検出
const podiumPattern = /^〔(.+?)君登壇〕/;

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 会議録タイトルの解析
const titlePattern = /(?:令和|平成)(\d+)年(.+)会議録第(\d+)号/;
```

## 賛否一覧 PDF

本会議一覧テーブルの「賛否」列から PDF へ直接リンクされる。

### ファイル名パターン

| ファイル名例 | 対応する会期 |
| --- | --- |
| `r0803.pdf` | 令和8年3月定例会 |
| `r0712.pdf` | 令和7年12月定例会 |
| `r0709ri.pdf` | 令和7年9月臨時会 |
| `r0709-2.pdf` | 令和7年9月定例会（第2回） |
| `h3012.pdf` | 平成30年12月定例会 |

命名規則: `[r/h][年号2桁][月2桁][suffix].pdf`
- `r` = 令和、`h` = 平成
- 臨時会は `ri` や `rinnzi` がサフィックスにつく場合がある
- 古いファイルは長いパス（`2022111000029_docs_2014082700084_files_*.pdf`）を持つ

## スクレイピング戦略

### Step 1: 会議録 URL の収集

本会議一覧ページ `https://www.city.yawatahama.ehime.jp/gikai/2022111000029/` をスクレイピングし、会議録詳細ページの URL を全件抽出する。

1. テーブル内の「会議録」列から `/gikai/{コード}/` パターンの URL を抽出
2. 取得範囲: 令和（最新）〜 平成25年（2013年）
3. 過去の本会議ページ（平成17〜25年）も別途アクセスが必要

**注意**: ページネーションなし。1ページに全件掲載のため、単一リクエストで完結。

### Step 2: 会議録 HTML の取得とパース

各会議録詳細 URL にアクセスし、本文を抽出する。

1. `<h2>` から会議名・年度・号数を抽出
2. `<p>` タグのテキストを順次処理し、`○` で始まる行を発言者として識別
3. 発言者行に続くテキストを発言内容として紐付ける

### Step 3: 一般質問ページの個別取得

一覧ページの「一般質問（議員別）」リンクも個別の URL として提供されている場合がある。議員名と質問内容が1ページにまとまっているため、必要に応じて追加取得する。

## 注意事項

- 会議録は HTML 形式で全文公開されており、PDF からのテキスト抽出は不要
- 賛否一覧のみ PDF 形式（テキスト抽出が必要な場合は PDF パーサーを使用）
- 過去の本会議（平成17〜25年）は別ページに分離されており、リンクをたどる必要がある
- URL コード（`2022111000029` 等）は日付との規則的な対応がないため、必ず一覧ページから収集する
- 氏名に「君」付けの敬称が使われる（例: 「菊池　彰君」）
- リクエスト間に適切な待機時間（1〜2 秒）を設けること

## 推奨アプローチ

1. **本会議一覧ページを起点**: `/gikai/2022111000029/` の1ページから全 URL を収集できる
2. **HTML 全文取得**: 会議録は HTML で公開されているため、PDF 変換不要
3. **過去アーカイブの別途収集**: 平成17〜25年は別ページへのリンクをたどる
4. **差分更新**: 本会議一覧ページの新着行を定期チェックし、未取得 URL のみ処理
5. **レート制限**: 1〜2 秒のウェイトを設ける
