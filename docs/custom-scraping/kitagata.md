## 会議録の構成

各年度ページには複数の会議が掲載されている。会議の種別は以下の通り。

| 種別 | 開催時期の目安 | 1年あたりの回数 |
| --- | --- | --- |
| 定例会 | 3月・6月・9月・12月 | 4回 |
| 臨時会 | 随時 | 年1〜2回程度 |

各会議は複数号（最大4号程度）で構成され、号ごとに PDF ファイルが存在する。また会議録 PDF のほか「付議案件」「議案と議決結果」などの関連資料 PDF も掲載される。

---

## PDF ファイル名のパターン

会議録 PDF のファイル名は以下のルールで命名されている。

```
r{年}{月}{日}gijiroku.pdf
```

| パターン | 例 | 意味 |
| --- | --- | --- |
| `r{YY}{MM}{DD}gijiroku.pdf` | `r70901gijiroku.pdf` | 令和7年9月1日の会議録 |
| `r{YY}{MM}{DD}gijiroku.pdf` | `r60301gijiroku.pdf` | 令和6年3月1日の会議録 |
| `r{YY}{MM}{DD}gijiroku.pdf` | `r40606gijiroku.pdf` | 令和4年6月6日の会議録 |

- 年は和暦の年数（令和7年 → `7`、令和4年 → `4`）
- 月・日はゼロ埋め2桁
- 一部の古いファイルは大文字プレフィックス（`R040114_rinzi.pdf` など）の場合がある

## スクレイピング戦略

### Step 1: 年度別ページから PDF リンクを収集

上記の年度別ページ ID 一覧を使い、各年度ページを順に取得する。

- ページネーションはなし（全会議録が1ページに掲載）
- Cheerio で `a[href$=".pdf"]` を抽出し、`/material/files/group/12/` を含む href のみを対象とする
- 関連資料（付議案件・議決結果等）と会議録 PDF の区別は href のファイル名で判定する（`gijiroku` を含むものが会議録本文）
- リンクテキストから会議名・号数・開催日を取得する

**HTML 構造例（年度別ページ）:**

```html
<h2>第4回（9月）定例会</h2>
<ul>
  <li><a href="//www.town.kitagata.gifu.jp/material/files/group/12/r60901gijiroku.pdf">
    北方町定例会第1号 令和6年9月2日（PDFファイル：60.7KB）
  </a></li>
  ...
</ul>
```

### Step 2: PDF のダウンロードとテキスト抽出

PDF を取得し、`pdf-parse` 等のライブラリでテキストを抽出する。

- 会議録は PDF のみで提供されるため、HTML からテキストを直接得ることはできない
- PDF 内テキストの文字コードは UTF-8 が前提だが、古いファイルは異なる場合がある

### Step 3: 抽出テキストのパース

PDF から抽出したテキストを解析してメタ情報・発言内容を取得する。

#### メタ情報

リンクテキストおよび PDF 冒頭から以下を抽出する:

- 開催日: `令和X年X月X日` 形式
- 会議名: `北方町定例会第N号` または `北方町臨時会` 形式
- 号数: `第N号` から抽出

#### 発言の構造（PDF 抽出テキストより）

北方町の会議録は標準的な議事録形式。発言者パターンは次の通りと推定される:

```
○議長（氏名）
○○番（氏名）
○町長（氏名）
```

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（山田太郎） → role="議長", name="山田太郎"

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 会議名の抽出
const meetingPattern = /北方町(?:定例会|臨時会)第?(\d*)号?/;
```

## 注意事項

- **HTML テキストなし**: 会議録本文はすべて PDF で提供されており、HTML 上に発言テキストは存在しない
- **ファイル命名の揺れ**: 古い年度のファイルは大文字プレフィックス（`R04...`）や別命名パターン（`_rinzi` 等）が含まれる場合がある。href を正確に取得することが重要
- **関連資料の除外**: 付議案件・議決結果など会議録以外の PDF も同ページに掲載されるため、`gijiroku` を含むファイル名で会議録本文を識別する
- **ページ ID は非連番**: 年度別ページ ID は連番ではなくランダムに割り当てられているため、上記の対応表を使ってハードコードする

---

## 推奨アプローチ

1. **年度ページ ID をハードコード**: ページ ID は予測不可能なため、上記の対応表を静的に保持してクロールに使用する
2. **会議録 PDF のみを対象**: `gijiroku` を含む href を会議録本文として扱い、関連資料は取得対象外とする
3. **PDF テキスト抽出**: PDF ライブラリを用いて各 PDF からテキストを抽出し、発言単位でパースする
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 最新年度ページのみ定期的に再取得し、新規 PDF リンクが増えた場合に差分取得する
