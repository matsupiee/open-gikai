# 城里町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.shirosato.lg.jp/page/dir002730.html
- 分類: 独自会議録検索システムによるHTML公開
- 文字コード: UTF-8
- 特記: 令和4年度以降、独自の会議録検索システムを提供

---

## URL 構造

会議録ページの主要な URL パターンは以下の通り:

| ページ | URL パターン |
| --- | --- |
| 会議録トップページ | `https://www.town.shirosato.lg.jp/page/dir002730.html` |
| 会議録検索システム | `https://www.town.shirosato.lg.jp/` (令和4年度以降の会議録検索用) |

※ 詳細な会議録検索システムの URL 構造、検索パラメータについては、実際のシステムアクセスで確認が必要。

---

## 会議の種類

城里町議会における標準的な会議の種別:

| 会議種別 | 説明 |
| --- | --- |
| 本会議 | 本会議（定例会・臨時会） |
| 常任委員会 | 総務委員会、生活経済委員会、建設委員会等 |
| 特別委員会 | 必要に応じて設置 |
| 議会運営委員会 | 議会の運営に関する事項 |
| 全員協議会 | 全議員による協議 |

---

## データ提供形式

会議録は以下の形式で提供されている可能性がある:

- **HTML形式**: 検索システムから直接閲覧
- **PDF形式**: 個別の会議録ドキュメント

詳細は実際のシステムで確認が必要。

---

## 時間範囲

- **カバー開始時期**: 令和4年度（2022年度）
- **現在までのカバー**: 最新の会議録まで随時更新

---

## スクレイピング戦略（推奨アプローチ）

### Step 1: 会議録検索システムの実装詳細を確認

1. 会議録検索システムへのアクセス URL を特定
2. 検索パラメータ（年、月、会議種別等）を確認
3. 検索結果の HTML 構造、ページネーション形式を分析

### Step 2: 会議一覧の取得

検索条件を組み合わせて会議一覧ページを取得:

- 年ごと（令和4年〜現在）
- 会議種別ごと（本会議、委員会等）
- その他の検索パラメータがあれば活用

### Step 3: 会議録詳細ページのパース

各会議録ページから以下の情報を抽出:

#### メタ情報

- 開催日（年月日）
- 会議名（本会議 / 委員会名）
- 定例会 / 臨時会の区別
- 会期（開始日〜終了日、または単一日）

#### 会議録本文の構造

発言者パターン（想定される形式）:

```
○○○○○ 議長
○○○○○ 副議長
○○○○○ 議員
```

発言内容のパース:

- 発言者名の抽出
- 発言内容テキストの取得
- 登壇・質問パターンの検出（あれば）

### Step 4: 差分更新の実装

- 最後の取得日時を記録
- 新規追加の会議録のみを差分更新
- または、年ごと・月ごとの更新スケジュール

---

## 注意事項

- **会議録検索システムの仕様変更**: 令和4年度に新システムが導入されているため、それ以前のデータとの構造の違いに注意
- **令和3年度以前の会議録**: 旧システムまたは別途の公開方法を確認する必要がある可能性がある
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
- **システムへのアクセス可能性**: 実際のシステムへのアクセスが可能か、JavaScript による動的ロードが必要か事前確認が必須

---

## 推奨作業フロー

1. **事前調査**: 会議録検索システムに実際にアクセスし、以下を確認
   - 検索パラメータの形式
   - 検索結果ページの HTML 構造
   - 詳細ページの URL パターン
   - JavaScript による動的ロードの有無

2. **テスト用スクレイパー開発**: 1 年分（または 1 月分）のデータで試行

3. **本体実装**: 全時間範囲でのスクレイピング実装

4. **定期更新**: 新規会議録の追加に対応する差分更新スケジュール
