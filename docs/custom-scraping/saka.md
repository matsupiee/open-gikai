# 坂町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.saka.lg.jp/2017/05/30/29/
- 分類: 町公式サイト（WordPress）内での直接公開（外部の会議録検索システムは使用していない）
- 文字コード: UTF-8
- 特記: 会議録は PDF 形式で提供。WordPress の単一投稿ページ（post ID: 1195）に全年度の会議録 PDF リンクが集約されている。WordPress REST API（`/wp-json/wp/v2/posts/1195`）からも HTML コンテンツを取得可能。

## ページ構造

### 会議録一覧ページ

単一の WordPress 投稿ページに全年度の会議録が掲載されている。

#### 年度セクション（h2/h3 見出し）

近年分は `<h2>` 見出しで年度別に区切られる:

- `令和７年に開催された定例会・臨時会の会議録`
- `令和６年に開催された定例会・臨時会の会議録`
- `令和５年に開催された定例会・臨時会の会議録`
- `令和４年に開催された定例会・臨時会の会議録`
- `令和3年に開催された定例会・臨時会の会議録`
- `令和2年に開催された定例会・臨時会の会議録`

過去分は `過去の会議はこちら` という `<h2>` の下に、`<h3>` 見出しで年度別に区切られる:

- `【令和元年】`
- `【平成30年】`〜`【平成23年】`

#### リスト構造

各年度セクション内は `<ul>` リストで会議録が列挙される:

```html
<!-- 臨時会: 1つのPDFリンク -->
<li><a href="...pdf">令和７年坂町議会第１０回臨時会</a></li>

<!-- 定例会: 複数日にまたがる場合は日ごとにPDFリンク -->
<li>令和７年坂町議会第９回定例会（<a href="...pdf">１日目</a>/<a href="...pdf">２日目</a>）</li>

<!-- 3日間の場合 -->
<li>令和７年坂町議会第７回定例会（<a href="...pdf">１日目</a>/<a href="...pdf">２日目</a>/<a href="...pdf">３日目</a>）</li>
```

#### 会議の種類

- **定例会**: 年4回程度。複数日（1〜4日目）に分割された PDF で提供
- **臨時会**: 年数回。1つの PDF で提供

#### PDF 総数

216件（2025年3月時点）。平成23年（2011年）から令和7年（2025年）まで。

## スクレイピング戦略

### Step 1: 会議録一覧の取得

WordPress REST API を利用して、会議録一覧ページの HTML コンテンツを取得する。

```
GET https://www.town.saka.lg.jp/wp-json/wp/v2/posts/1195
```

レスポンスの `content.rendered` フィールドに全会議録の HTML が含まれる。

**メリット:**
- ナビゲーション等の不要な HTML を含まず、本文のみ取得可能
- JSON 形式のため、パースが容易
- ページネーションなし（単一ページに全件掲載）

### Step 2: PDF リンクの抽出

取得した HTML から全 PDF リンクを抽出する。

```typescript
// 全 PDF リンクの抽出
const pdfLinks = html.match(/href="([^"]*\.pdf[^"]*)"/g);
```

### Step 3: メタ情報の抽出

PDF リンクの前後のテキストから以下のメタ情報を抽出する:

- **年度**: `<h2>` または `<h3>` 見出しから（例: `令和７年`, `【平成30年】`）
- **会議種別**: リンクテキストまたは `<li>` テキストから（`定例会` / `臨時会`）
- **回数**: テキストから（例: `第９回`, `第１０回`）
- **日目**: リンクテキストから（例: `１日目`, `２日目`）

#### メタ情報抽出の正規表現（案）

```typescript
// 年度の抽出（h2）
const yearPatternH2 = /(令和|平成)(\d+|[０-９]+)年/;

// 年度の抽出（h3）
const yearPatternH3 = /【(令和|平成)(\d+|[０-９]+)年】/;

// 会議名の抽出（臨時会）
const rinji = /(令和|平成)\S+坂町議会第\S+回.*臨時会/;

// 会議名の抽出（定例会）
const teirei = /(令和|平成)\S+坂町議会第\S+回.*定例会/;

// 日目の抽出
const dayPattern = /([１２３４1234])日目/;
```

### Step 4: PDF のダウンロードとテキスト抽出

抽出した PDF URL からファイルをダウンロードし、PDF パーサーでテキストを抽出する。

## 注意事項

- 会議録ページは WordPress の単一投稿（post ID: 1195）であり、ページネーションは存在しない
- PDF ファイル名は日本語を含むものが多く、URL エンコードが必要（例: `令和７年坂町議会第１０回臨時会-1.pdf`）
- 同名ファイルの重複回避のため、WordPress が自動でサフィックスを付与している（例: `-1`, `-2`, `-10`）
- `１日目.pdf` のようにファイル名だけでは会議を特定できないため、必ず HTML 上の文脈（`<li>` テキストと年度見出し）からメタ情報を取得すること
- 会議録の作成にはおよそ2ヶ月程度を要する旨がページに記載されている
- リクエスト間には適切な待機時間（1〜2秒）を設けること
- PDF の格納ディレクトリが年代によって異なるため、href 属性の値をそのまま使用すること（パターンから URL を生成しない）

---

## 推奨アプローチ

1. **REST API を活用**: WordPress REST API（`/wp-json/wp/v2/posts/1195`）を使い、本文 HTML のみを取得する。通常のページ取得よりもパースが容易
2. **全量一括取得**: ページネーションがないため、1回のリクエストで全会議録の PDF リンクを取得可能
3. **PDF からテキスト抽出**: 会議録は PDF 形式のみで提供されるため、PDF パーサー（pdf-parse 等）でテキストを抽出する必要がある
4. **差分更新**: REST API レスポンスの `modified` フィールド（最終更新日時）を前回取得時と比較し、更新があった場合のみ再取得する。新規追加はページ先頭に追記されるため、既知の PDF URL リストと差分を取る
