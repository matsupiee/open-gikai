- 1 会議あたり 1〜4 件程度の PDF

## PDF 内部構造

PDF は Microsoft Word for Microsoft 365 で作成されており、以下の構成を持つ。

### メタ情報（冒頭ページ）

1. **告示**: `茂木町告示第XX号` + 招集文
2. **期日・場所**: `１．期　日　令和X年X月X日` / `２．場　所　茂木町議会議事堂`
3. **応招・不応招議員**: 議員番号 + 氏名の一覧（`X番　姓　名　君`）
4. **議事日程**: `議事日程（第N号）` + `令和X年X月X日（曜）午前X時XX分開会`
5. **出席議員**: 議員番号 + 氏名の一覧
6. **欠席議員**
7. **説明のため出席した人**: 役職 + 氏名（町長、副町長、教育長、各課長など）
8. **職務のため出席した者の職氏名**: 事務局長など

### 発言の構造

発言者パターン:

```
○議長（古田土志郎君）
○議会運営委員長（磯　稲藏君）
○６番（山口文明君）
○町長（古口達也君）
```

- `○` で始まり、役職名 + 括弧内に氏名 + `君`
- 役職例: `議長`, `議会運営委員長`, `町長`, `副町長`, `教育長`, `総務課長` など
- 議員は `○X番（氏名君）` 形式（番号 + 氏名）
- 発言内容は発言者行の後に続くテキスト（同じ行に続く場合もある）

登壇表記:

```
〔議会運営委員長　磯　稲藏君登壇〕
〔町長　古口達也君登壇〕
〔６番　山口文明君登壇〕
```

- `〔` で始まり `登壇〕` で終わる
- 役職または番号 + 氏名 + `君登壇`

セクション見出し:

```
◎開会の宣告
◎諸般の報告
◎開議の宣告
◎議事日程の報告
◎会議録署名議員の指名
◎会期の決定
◎町政に対する一般質問
```

- `◎` で始まるセクション区切り
- 区切り線（罫線）がセクション間に挿入される場合あり

時刻表記:

```
（午前　９時００分）
```

議場の動作:

```
〔「異議なし」と呼ぶ者あり〕
```

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（古田土志郎君） → role="議長", name="古田土志郎"
// 例: ○６番（山口文明君） → role="６番", name="山口文明"

// 登壇の検出
const podiumPattern = /^〔(.+?)君登壇〕/;

// セクション見出しの検出
const sectionPattern = /^◎(.+)/;

// 開催日の抽出（議事日程ページから）
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 時刻の抽出
const timePattern = /（(午前|午後)\s*(\d+)時(\d+)分）/;
```

## スクレイピング戦略

### Step 1: 会議一覧の取得

会議録一覧ページ `nextpage.php?cd=17800&syurui=1` から全会議へのリンクを収集する。

- ページネーションなし（全件が 1 ページに表示）
- `<a>` タグの `href` から `cd` パラメータを抽出
- リンクテキストから会議名（定例会/臨時会）と年月を抽出

**収集方法:**

1. 一覧ページを取得
2. `nextpage.php?cd={ID}&syurui=2` パターンのリンクを Cheerio で抽出
3. リンクテキストから会議種別・年月をパース

### Step 2: PDF リンクの収集

各会議ページ `nextpage.php?cd={会議ID}&syurui=2` から PDF ダウンロード URL を収集する。

- 各ページには 1〜4 件の PDF リンクが含まれる
- リンク先は `/motegi/download/{ファイルID}.pdf` 形式
- PDF ファイル名から会議録番号（第N号）または開催日を取得可能

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

- PDF は Word 由来のためテキストレイヤーが存在する（画像 PDF ではない）
- `pdf-parse` や `pdfjs-dist` 等のライブラリでテキスト抽出が可能
- フォント埋め込み・ストリーム圧縮があるため、適切なライブラリの選定が必要

### Step 4: テキストのパース

抽出したテキストから発言データを構造化する。

1. `◎` で始まるセクション見出しで会議の進行を区切る
2. `○` で始まる発言者行で発言を分割
3. 発言者行から役職・氏名を抽出
4. 冒頭ページからメタ情報（開催日、出席議員など）を抽出

---

## 注意事項

- 会議録はすべて PDF 形式で提供されるため、HTML パースではなく PDF テキスト抽出が必要
- 古い会議録（平成24年頃）はファイル名が `hXX.M.D.pdf` 形式で、新しいものは `令和X年X月　会議録第N号.pdf` 形式
- PDF のページ数は 1 ファイルあたり 40 ページ前後（会議録第1号の場合）
- 議員定数は 12 名（4番・9番は欠番）
- PDF 内のテキストは全角数字が多用されている（番号、日付など）

---

## 推奨アプローチ

1. **一括リスト取得**: 一覧ページから全会議 ID を一括取得（ページネーションなし）
2. **PDF ダウンロード優先**: HTML 上に会議録本文がないため、PDF のダウンロード → テキスト抽出の流れが必須
3. **テキスト抽出ライブラリ**: `pdf-parse` または `pdfjs-dist` を使用し、ページ順にテキストを結合
4. **発言者パターンの一貫性**: `○` で始まる発言者行のパターンは全会議で統一されており、正規表現で安定的に抽出可能
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
6. **差分更新**: 一覧ページを取得して新規の `cd` が増えているかを確認することで差分更新が可能
