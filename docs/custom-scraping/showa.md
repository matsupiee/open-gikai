- サイト: https://www.vill.showa.gunma.jp/kurashi/gyousei/assembly/kaigiroku.html

- 特記: 会議録本文は PDF ファイルで公開。Google Analytics（G-GPE661EWEV）使用

## サイト構造

### 会議録トップページ

- 年度ごとに `<h5>` で「令和X年」と見出しがある
- 各定例会・臨時会へのリンクが `<h6>` 内に `<a>` タグで列挙される
- 現在公開されている範囲: 令和5年〜令和7年（3年分）
- ページネーションなし（全件が1ページに表示）
- 検索フォームなし（サイト内検索はあるが会議録専用の検索機能はない）

### 各定例会ページ

- 各号（第1号、第2号、第3号...）ごとに「名簿」と「本文」の2つの PDF リンクがある
- PDF ファイル名の命名パターンは統一されていない:
  - 新しい会議: `{YYYYMMDD}_meibo.pdf` / `{YYYYMMDD}_honbun.pdf`（例: `20251203_honbun.pdf`）
  - やや古い会議: `{和暦年}-{回}-{号}-{種別}.pdf`（例: `6-3-1-2.pdf`）
  - 古い会議: `dai{号}gou{種別}.pdf`（例: `dai2gouhonbun.pdf`）

### 会議種別

| 種別 | 説明 |
| --- | --- |
| 定例会 | 年4〜5回開催（第1回〜第5回） |
| 臨時会 | 不定期開催 |

※ 委員会の会議録は公開されていない（本会議のみ）。

## PDF 本文の構造

### メタ情報

PDF 冒頭に以下の情報が記載される:

- 開会時刻: `午前１０時００分開会`
- 議事区分: `◎議長挨拶及び諸般の報告` のように `◎` で始まるセクション見出し

ただし、PDF 自体には明示的な開催日や会議名の記載がない。これらは各定例会ページの見出し（例: 「令和7年第4回定例会」）とファイル名から推定する必要がある。

### 発言者パターン

発言者は `○` で始まり、役職と括弧内の氏名で表記される:

```
○議長（永井一行君）
○村長（髙橋幸一郎君）
```

### 議事進行の表記

```
◎議長挨拶及び諸般の報告    （セクション見出し: ◎ で始まる）
◎開会の宣告
◎会期の決定について
◎日程第１　会議録署名議員の指名
◎日程第３　議案第４２号　...
〔村長　髙橋幸一郎君発言〕    （発言者切替の注記: 〔〕で囲まれる）
〔係長朗読〕                   （事務局による朗読）
〔「異議なし」の声あり〕       （議場の反応）
```

### パース用正規表現（案）

```typescript
// 発言者の抽出（○で始まる行）
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（永井一行君） → role="議長", name="永井一行君"
// 例: ○村長（髙橋幸一郎君） → role="村長", name="髙橋幸一郎君"

// セクション見出しの抽出（◎で始まる行）
const sectionPattern = /^◎(.+)/;
// 例: ◎日程第３　議案第４２号　... → section="日程第３　議案第４２号　..."

// 発言者切替注記の抽出（〔〕で囲まれる行）
const notePattern = /^〔(.+?)〕/;
// 例: 〔村長　髙橋幸一郎君発言〕 → note="村長　髙橋幸一郎君発言"

// 開会時刻の抽出
const openingTimePattern = /午[前後](\d+)時(\d+)分開会/;
```

## スクレイピング戦略

### Step 1: 会議一覧の収集

会議録トップページ `kaigiroku.html` から全定例会・臨時会ページへのリンクを収集する。

- `<h5>` タグで年度を、`<h6>` 内の `<a>` タグで各会議へのリンクを抽出
- 年度と会議名（例: 「第1回定例会」「第1回臨時会」）を紐付けて保持

**収集方法:**

1. `kaigiroku.html` を取得し Cheerio でパース
2. `h5` → `h6` の順に走査し、年度と会議名・URL の対応を作成

### Step 2: PDF リンクの収集

各定例会ページから「本文」PDF のリンクを収集する。

- 各号ごとに「名簿」と「本文」の2つの PDF がある
- 「本文（honbun）」のみを対象とする
- リンクテキストに「本文」を含むものを抽出

**収集方法:**

1. 各定例会ページを取得
2. `<a>` タグから PDF リンクを抽出
3. ファイル名またはリンクテキストに `honbun` または `本文` を含むものをフィルタ
4. 号数（第1号、第2号...）と紐付け

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

- PDF は Word から生成されたテキストベースの PDF（スキャン画像ではない）
- `pdf-parse` や `pdfjs-dist` 等のライブラリでテキスト抽出が可能

### Step 4: テキストのパース

抽出したテキストから発言者と発言内容を構造化する。

1. `○` で始まる行を発言者として検出
2. 発言者行から次の発言者行までを発言内容として抽出
3. `◎` で始まる行をセクション見出しとして抽出
4. `〔〕` で囲まれた行を注記として処理

---

## 注意事項

- PDF のテキスト抽出時、改行位置が元の文書と異なる場合がある（PDF のレイアウト依存）
- PDF ファイル名の命名規則が統一されていないため、名簿と本文の区別にはリンクテキストも併用する
- 公開範囲が令和5年以降と比較的少ないが、今後ページが追加される可能性がある
- PDF 内に開催日が明記されていないケースがあり、ファイル名（例: `20251203`）から日付を推定する必要がある
- 一部の古い PDF ではファイル名に日付が含まれない（例: `dai1gouhonbun.pdf`）ため、各定例会ページの情報と組み合わせて日付を特定する

---

## 推奨アプローチ

1. **トップダウンで収集**: トップページ → 各定例会ページ → PDF リンクの順に階層的にクロール
2. **本文 PDF のみ取得**: 名簿 PDF はスキップし、本文 PDF のみをダウンロード対象とする
3. **PDF テキスト抽出**: スキャン画像ではなくテキストベース PDF のため、OCR は不要
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: トップページの会議一覧を定期的にチェックし、新規追加分のみを取得する
