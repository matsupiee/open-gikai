# 今金町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.imakane.lg.jp/ass/kaigiroku/
- 分類: CMS による年度・会議種別ごとの PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8（HTML ページ）
- 特記: サイトに WAF/ボット対策あり（curl・自動アクセスがブロックされる可能性が高い）

## PDF ファイルの URL パターン

年度・時期によって複数の URL パターンが混在している。

### パターン 1: ハッシュ値ファイル名（`/ass/` 直下）

```
https://www.town.imakane.lg.jp/ass/{ハッシュ}.pdf
例: /ass/21a697f62d54c8acdf12c7514080ac70.pdf（令和7年第2回定例会）
例: /ass/1cf31664e7b8be429bc93e6d846d5a0a.pdf（令和7年第1回定例会 第3号）
例: /ass/7bf7eec0f4b6a08fa766b6d036f61e56.pdf（令和6年第1回定例会 第2号）
```

### パターン 2: 日本語ファイル名（`/ass/` 直下）

```
https://www.town.imakane.lg.jp/ass/{日本語ファイル名}.pdf
例: /ass/第１回定例２号.pdf（URL エンコード済み）
例: /ass/本文1.23 (2).pdf（総務産業常任委員会）
例: /ass/総務H28.9.13 ー本文.pdf（総務厚生常任委員会）
例: /ass/合同H29.12.11－本文.pdf（合同委員会）
```

### パターン 3: uploads/pdf + 日本語ファイル名

```
https://www.town.imakane.lg.jp/ass/uploads/pdf/{日本語ファイル名}.pdf
例: /ass/uploads/pdf/H27.8.21ー本文.pdf（平成27年第5回臨時会）
例: /ass/uploads/pdf/H27.3.9—本文　済.pdf（合同委員会）
例: /ass/uploads/pdf/H26.7.15ー本文(合同委員会）.pdf
```

### パターン 4: 年号プレフィックス + 日本語ファイル名

```
https://www.town.imakane.lg.jp/ass/{年号}.{月}.{日}　{種別}　{詳細}.pdf
例: /ass/R1.7.31　産業G　会議録　本文.pdf
```

## PDF 内の文書構造

PDF のヘッダーおよびスニペットから判明した構造:

### メタ情報（PDF 冒頭）

```
令和７年第２回今金町議会定例会　第１号
令和７年６月１２日（木）

〇議事日程
１　会議録署名議員の指名について
２　会期の決定について
```

- 開催年: `令和X年` / `平成XX年` の全角表記
- 会議名: `第X回今金町議会定例会（臨時会）　第X号`
- 開催日: `令和X年X月X日（曜日）`
- 議事日程: `〇議事日程` の見出しに続いて番号付きリスト

### 常任委員会の文書構造

```
総 務 産 業 常 任 委 員 会
期　日　令和６年１月２３日（火）
時　間　午前１０：００
場　所　議　場

〇議事日程
１．委員長あいさつ
```

- 委員会名は全角スペース区切りで表記
- 期日・時間・場所が個別行で記載

### 発言の構造（推定）

定例会 PDF のスニペットから推定される発言者パターン:

```
〇議長（○○○○）
〇副議長（○○○○）
〇○番（○○○○）
〇町長（○○○○）
```

#### パース用正規表現（案）

```typescript
// 開催情報の抽出
const sessionPattern = /(令和|平成)[０-９\d]+年第[０-９\d]+回今金町議会(定例会|臨時会)/;

const speakerPattern = /^〇(.+?)（(.+?)）/;
```

## スクレイピング戦略

### Step 1: 会議録一覧ページから PDF リンクの収集

会議録閲覧ページ `/ass/kaigiroku/` および各カテゴリページから PDF リンクを収集する。

**収集方法:**

1. `/ass/kaigiroku/` の HTML を取得
2. 各カテゴリページ（定例会、特別委員会、常任委員会、合同委員会）のリンクを辿る
3. 各ページの `<a>` タグから `.pdf` で終わるリンクを抽出
4. 相対パスの場合はベース URL と結合して絶対 URL に変換
5. リンクテキストから会議種別・回次・開催日をメタ情報として抽出

### Step 2: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容をパース

### Step 3: 会議録のパース

#### メタ情報の抽出

PDF 冒頭から以下を抽出:

- 会議種別（定例会 / 臨時会 / 常任委員会 / 合同委員会）
- 回次（第1回、第2回 ...）
- 号数（第1号、第2号 ...）
- 開催日
- 議事日程

#### 発言内容の抽出

PDF テキストから発言者と発言内容を分離してパースする。実際の PDF を取得して発言者パターンの詳細を確認する必要がある。

## 注意事項

- **WAF/ボット対策**: サイトに強力なボット対策が導入されており、curl や一般的な HTTP クライアントではアクセスがブロックされる（HTTP/2 stream cancel）。Playwright 等のヘッドレスブラウザの使用が必要になる可能性が高い
- PDF の URL パターンが年度・会議種別によって大きく異なる（4 種類以上のパターンが混在）
- 一部の PDF は日本語ファイル名やスペースを含むため、URL エンコードが必要
- 全角文字（数字・スペース・記号）が多用されている
- 委員会名称が時期によって変更されている（総務厚生常任委員会 → 総務産業常任委員会）
- 会議録はすべて PDF 形式であり、HTML の会議録本文は提供されていない

---

## 推奨アプローチ

1. **ヘッドレスブラウザの使用**: WAF 対策のため、Playwright 等でページを取得する
2. **一覧ページ起点でクロール**: URL パターンが多様なため、一覧ページから動的にリンクを辿る
3. **PDF 直接取得**: HTML の本文ページが無いため、PDF をダウンロードしてテキスト抽出する方式を採用
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページを定期的に再クロールし、新規 PDF リンクを検出する
6. **URL パターンの多様性に対応**: `.pdf` 拡張子でのリンク抽出を基本とし、特定のパスパターンに依存しない実装にする
