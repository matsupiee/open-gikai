- サイト: https://www.vill.morotsuka.miyazaki.jp/

## 議会だより PDF

### 公開状況

- 最新号ページ（908.html）: No.184〜No.187（2025年1月号〜2025年11月号）
- 過去号一覧ページ（166.html）: No.153〜No.183（2017年5月号〜2024年11月号）
- 発行頻度: 年4回（1月・5月・8月・11月）
- 欠番あり: No.169〜No.176（2021年5月〜2022年11月頃）が一覧に未掲載

### PDF URL パターン

PDF ファイルはすべて以下のディレクトリ配下に格納:

```
https://www.vill.morotsuka.miyazaki.jp/material/files/group/9/
```

ファイル名の命名規則は統一されておらず、以下の2パターンが混在:

1. **意味のあるファイル名**: `gikaidayori{YYYYMM}.pdf`（例: `gikaidayori202511.pdf`）
2. **ハッシュ値ファイル名**: `{MD5ハッシュ}.pdf`（例: `6f52f1e95d229e45658fb9405a5dfe57.pdf`）

→ URL に規則性がないため、HTML ページからリンクを抽出する必要がある。

### PDF の内容構成（No.187 / 2025年11月号の例）

| ページ | 内容 |
| --- | --- |
| p.1 | 表紙 |
| p.2 | 定例会の概要（決算認定額など） |
| p.3-4 | 議案一覧と議決結果 |
| p.5-6 | 総括質疑（歳入歳出に対する質疑応答） |
| p.7 | 総括意見・個別意見 |
| p.8 | 一般質問（議員名・写真付き、問答形式の要約） |

**重要**: 議会だよりは会議録（逐語記録）ではなく、議案概要・質疑応答の要約・一般質問の要約を掲載した広報紙である。発言の完全な記録は含まれない。

## スクレイピング戦略

### Step 1: PDF URL の収集

2つの一覧ページから PDF リンクを抽出する。

1. 最新号ページ `https://www.vill.morotsuka.miyazaki.jp/sonsei/gikai/908.html`
2. 過去号一覧ページ `https://www.vill.morotsuka.miyazaki.jp/soshiki/1008/1/166.html`

**収集方法:**

1. 各ページの HTML を取得
2. `<a>` タグから `/material/files/group/9/` を含む PDF リンクを Cheerio で抽出
3. リンクテキストから号数（No.XXX）と発行年月を対応付ける

### Step 2: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、`pdf-parse` 等でテキストを抽出する
- PDF はスキャン画像ではなくテキスト埋め込み型のため、テキスト抽出が可能

### Step 3: コンテンツのパース

#### 一般質問セクション

議員の質問と執行機関の答弁が問答形式で記載:

```
問  {議員名}議員
{質問内容の要約}

答  {役職名}（または{課長名}）
{答弁内容の要約}
```

- `問` で始まる行が質問者（議員名付き）
- `答` で始まる行が答弁者（村長・課長等）

#### 総括質疑セクション

歳入歳出に関する質疑応答:

```
問  {議員名}議員
{質問内容}

答  {課長名}
{答弁内容}
```

#### 議案一覧

表形式で議案番号・議案名・結果（可決/否決）が記載。

## 注意事項

- **会議録（逐語記録）はオンライン非公開**: 諸塚村は会議録検索システムを持たず、会議の逐語記録をウェブ上に公開していない
- **議会だよりは要約**: 発言内容は編集・要約されたものであり、完全な発言記録ではない
- **PDF ファイル名に規則性なし**: ハッシュ値ベースのファイル名が多く、URL の推測は不可能。必ず HTML ページからリンクを収集する
- **欠番の存在**: No.169〜No.176 が一覧ページに掲載されていない（サイト未掲載の可能性）

---

## 推奨アプローチ

1. **議会だより PDF を情報源として割り切る**: 会議録が非公開のため、議会だよりから取得可能な情報（議案一覧・質疑要約・一般質問要約）を構造化データとして抽出する
2. **2ページからリンク収集**: 最新号ページと過去号一覧ページの両方を巡回して全 PDF を網羅する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 号数（No.XXX）で管理し、前回取得済みの最大号数以降のみを取得する
