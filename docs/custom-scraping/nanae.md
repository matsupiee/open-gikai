## 概要

- サイト: https://www.town.nanae.hokkaido.jp/hotnews/category/471.html
- 分類: 自治体公式サイト（CMS）で PDF 形式の会議録を公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 対象範囲: 平成31年第1回定例会（2019年3月）以降の本会議会議録
- 会議録形式: PDF（日ごとに分割）
- 総件数: 49件（2025年6月時点）

## ページ構造

### 一覧ページ (`/hotnews/category/471.html`)

- 全件が**単一ページ**に表示される（ページネーションなし）
- `<ul>` リスト形式で各会議録へのリンクが新しい順に並ぶ
- 各リンクのテキスト形式: `{年号}({西暦})第{N}回七飯町議会{定例会|臨時会}会議録（{開催日}）`

### 詳細ページ (`/hotnews/detail/{ID}.html`)

- 各開催日ごとにセクション分けされている
- 各セクションに以下の情報が含まれる:
  - **議事日程**: 順序付きリストで議案を列挙
  - **会議録 PDF**: 日ごとの PDF ファイルへのリンク（ファイルサイズ表示あり）
- 一般質問の質問者名が記載されている場合がある

## スクレイピング戦略

### Step 1: 詳細ページ URL の収集

一覧ページ `https://www.town.nanae.hokkaido.jp/hotnews/category/471.html` から全会議録の詳細ページ URL を収集する。

- 全件が単一ページに表示されるため、ページネーション処理は不要
- `<ul>` 内の `<li>` > `<a>` から `href` を抽出
- URL パターン: `/hotnews/detail/{8桁ID}.html`

**収集方法:**

1. 一覧ページの HTML を取得
2. Cheerio で `a[href*="/hotnews/detail/"]` を抽出
3. リンクテキストから会議名・開催日のメタ情報もパース可能

### Step 2: PDF URL の収集

各詳細ページから PDF ファイルの URL を収集する。

- 各詳細ページ内の `a[href$=".pdf"]` リンクを抽出
- 1つの定例会に対して複数の PDF（日ごと）が存在する
- 臨時会は通常1日分の PDF のみ

**収集方法:**

1. 各詳細ページの HTML を取得
2. Cheerio で PDF リンクを抽出
3. リンク前後のテキストから日付情報を取得

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを抽出
- 会議録 PDF は議事の全文が含まれているため、発言者と発言内容の分離が必要

### Step 4: テキストのパース

PDF から抽出したテキストを構造化する。

#### メタ情報の抽出

詳細ページのタイトルおよびリンクテキストから以下を抽出:

```
令和6年(2024年)第4回七飯町議会定例会会議録（12月9日～12日）
```

- 年号・西暦
- 回次
- 会議種別（定例会 / 臨時会）
- 開催期間

#### パース用正規表現（案）

```typescript
// 一覧ページのタイトルからメタ情報を抽出
const titlePattern = /^(令和|平成)(\d+)年\((\d{4})年\)第(\d+)回七飯町議会(定例会|臨時会)会議録（(.+?)）$/;
// 例: 令和6年(2024年)第4回七飯町議会定例会会議録（12月9日～12日）
// → era="令和", eraYear="6", year="2024", session="4", type="定例会", dates="12月9日～12日"
```

## 注意事項

- 会議録は **PDF 形式のみ** で公開されており、HTML での本文提供はない
- PDF のテキスト抽出精度に依存するため、OCR 品質の検証が必要（テキスト埋め込み PDF かスキャン PDF かを確認）
- 一覧ページは全49件を単一ページで表示するが、今後件数が増えた場合にページネーションが追加される可能性がある
- PDF ファイル名の命名規則が時期によって異なる（日付ベース vs タイムスタンプベース）
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける

---

## 推奨アプローチ

1. **一覧ページから全量取得**: 単一ページに全件表示されるため、1リクエストで全詳細ページ URL を取得可能
2. **PDF テキスト品質の事前検証**: 実装前に数件の PDF をダウンロードし、テキスト抽出の品質を確認する（テキスト埋め込みかスキャン画像かで対応が大きく変わる）
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: 一覧ページの先頭（最新）から順に確認し、既に取得済みの会議録に到達したら停止する
