# 新温泉町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.shinonsen.hyogo.jp/page/index.php?mode=page_list&cate_id=C2608
- 分類: 町公式サイト上で年度別に PDF を掲載（外部の会議録検索システム未導入）
- 文字コード: UTF-8
- 特記: 年度ごとにランダムなハッシュ形式のページ ID でページが管理される。PDF は `/uppdf/[タイムスタンプ].pdf` 形式で提供。

## 会議種別

各年度ページは以下の会議種別で会議録を分類している。

- **定例会**: 年4回開催（3月・6月・9月・12月）
- **臨時会**: 随時開催

各定例会は複数日程に分割されており、日程ごとに PDF が提供される。

---

## PDF ファイル名の命名規則

PDF ファイルは `/uppdf/[Unixタイムスタンプ].pdf` 形式でアップロードされており、ファイル名からメタ情報を読み取ることができない。会議名・開催日などのメタ情報は年度ページ上のリンクテキストから取得する必要がある。

---

## HTML 構造

### 会議録一覧トップ（年度一覧）

```
ul
  li > a[href="/page/?mode=detail&page_id={ハッシュID}"] "令和7年会議録"
  li > a[href="/page/?mode=detail&page_id={ハッシュID}"] "令和6年会議録"
  ...
```

### 年度別一覧ページ

jQuery accordion で会議種別ごとに展開・折りたたみ。

```
h3 または div.accordion-title: "第135回定例会"（令和7年X月X日）
  ul
    li > a[href="/uppdf/{タイムスタンプ}.pdf"] "1日目 (450KB)"
    li > a[href="/uppdf/{タイムスタンプ}.pdf"] "2日目 (520KB)"
```

リンクテキスト形式の例:
- `令和7年[月]月[日]日 第135回定例会 1日目 (450KB)`

## スクレイピング戦略

### Step 1: 年度別ページ ID の収集

既知のページ ID テーブルを使用して各年度ページに直接アクセスする。新年度追加時はトップページから新たな ID を取得する必要がある。

**収集方法:**

1. トップページ `https://www.town.shinonsen.hyogo.jp/page/index.php?mode=page_list&cate_id=C2608` を取得
2. `a[href*="mode=detail&page_id="]` でページ ID とリンクテキスト（年度名）を抽出
3. 既知の ID と比較して新規追加があれば検出

### Step 2: 年度別ページから PDF リンクを収集

各年度ページにアクセスし、会議録 PDF へのリンクと会議名・開催日を収集する。

**収集方法:**

1. 年度別ページの HTML を取得
2. accordion の見出し（会議名・開催日）と配下の `a[href$=".pdf"]` を対応付けて抽出
3. リンクテキストから開催日・会議種別・日目を抽出

**リンクテキストからのメタ情報抽出:**

```typescript
// 会議見出し: "第135回定例会" / "第133回臨時会"
const sessionPattern = /第(\d+)回(定例会|臨時会)/;

// 開催日: "令和7年3月10日"
const datePattern = /(令和|平成)(\d+)年(\d+)月(\d+)日/;

// 日目: "1日目 (450KB)"
const dayPattern = /(\d+)日目/;
```

### Step 3: PDF のダウンロードとテキスト抽出

- PDF URL: `https://www.town.shinonsen.hyogo.jp/uppdf/{タイムスタンプ}.pdf`
- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用
- ファイルサイズは 275KB〜701KB 程度

## ページネーション

なし。各年度ページで当該年度の全会議録リンクが一覧表示される。

---

## 注意事項

- 年度別ページの ID はランダムなハッシュ値であり、ファイル名からは年度を推測できない
- 新年度のページ ID は事前に予測できないため、トップページを定期的にクロールして新規 ID を検出する必要がある
- PDF ファイル名（タイムスタンプ）からもメタ情報を読み取れないため、リンクテキストからの抽出が必須
- jQuery accordion により一部コンテンツが初期状態で非表示になっているが、静的 HTML には全データが含まれるため通常の HTTP 取得で対応可能

---

## 推奨アプローチ

1. **年度 ID のキャッシュ管理**: 既知の年度ページ ID リストを保持し、トップページから新規 ID を定期的に検出する
2. **リンクテキストからのメタ情報取得**: PDF ファイル名ではなく、年度ページ上の accordion 見出しとリンクテキストから会議種別・開催日・日目を抽出する
3. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使用
4. **レート制限**: リクエスト間に適切な待機時間（1〜2 秒）を設ける
