# 玉城町議会 カスタムスクレイピング方針

## 概要

- サイト: https://kizuna.town.tamaki.mie.jp/chosei/senkyogikai/gikai/shingi/gijiroku.html
- 分類: 公式ウェブサイト内で議事録を直接管理・公開（外部の標準的な会議録検索システム不使用）
- 文字コード: UTF-8
- 特記: 議事録は年度ごとの HTML 目次ページ経由で PDF ファイルとして公開されている

## PDF ファイル命名規則

PDF のファイル名は `documents/{年月日}{会議タイプ識別子}.pdf` 形式。

| パターン | 意味 | 例 |
| --- | --- | --- |
| `YYMMDD` + `jyoutei` | 定例会（上程） | `061210jyoutei.pdf` |
| `YYMMDD` + `situmonn` / `ippann` | 定例会（一般質問） | `061211situmonn.pdf` |
| `YYMMDD` + `touron` / `saiketu` | 定例会（討論・採決） | `061218touron.pdf` |
| `YYMMDD` + `yosann` | 予算決算常任委員会 | `061213yosann.pdf` |
| `YYMMDD` + `rinnji` | 臨時会 | `060701rinnji.pdf` |
| `h{年}-{月}-{番号}` | 平成期の旧形式 | `h19-3-1.pdf` |

- 年は西暦下2桁（令和6年 → `06`）
- 旧ファイルには上記と異なる命名規則のものも存在する（例: `0308T11.pdf`）

## スクレイピング戦略

### Step 1: 年度別目次 URL の収集

索引ページ `gijiroku.html` を取得し、リンクされている年度別目次 URL をすべて抽出する。

- 索引ページには全年度分のリンクが1ページにまとめて掲載されており、ページネーションはない
- `<a href="...">` を Cheerio で取得し、`.html` 拡張子を持つリンクを年度別目次として収集する

### Step 2: PDF リンクの収集

各年度別目次ページを取得し、`documents/` 配下の PDF リンクをすべて抽出する。

- 各目次ページ内の `<a href="documents/...pdf">` を Cheerio で収集する
- 相対パスのため、ベース URL（`https://kizuna.town.tamaki.mie.jp/chosei/senkyogikai/gikai/shingi/`）を補完して絶対 URL に変換する
- 会議名・開催日は PDF リンク周辺の `<h2>`・`<li>` テキストから抽出する

### Step 3: PDF の取得とテキスト抽出

取得した PDF URL から PDF をダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用する
- 発言者・発言内容の構造は PDF 内のテキストレイアウトに依存するため、実際の PDF を確認して解析パターンを決定する

---

## 注意事項

- 年度別目次の URL は、令和5年以降はアップロードタイムスタンプを含む不規則な形式のため、索引ページから動的に収集する必要がある（URL を推測して直接生成することは不可）
- 平成19年（2007年）が公開されている最古の年度
- PDF のファイル名命名規則は年度によって一貫していないため、索引ページ経由での収集を基本とする
- 議事録本文はすべて PDF 形式であり、HTML での本文公開はない

---

## 推奨アプローチ

1. **索引ページから年度 URL を動的収集**: 索引ページをクロールして年度別目次 URL を取得し、直接 URL 生成に依存しない
2. **PDF リンクを目次ページから網羅収集**: 各年度目次から `documents/*.pdf` リンクをすべて収集する
3. **PDF テキスト抽出**: ダウンロードした PDF から `pdf-parse` 等でテキストを抽出し、発言者・発言内容をパースする
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 既取得の PDF URL を記録しておき、次回クロール時に新規追加分のみを処理する
