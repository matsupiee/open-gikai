# にかほ市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.nikahoshigikai.akita.jp/
- 会議録ページ: https://www.nikahoshigikai.akita.jp/conference.html
- 分類: 議会独自サイトで PDF 形式の会議録を年度別に公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 形式: PDF ファイル

### テーブル構造

各タブ内はテーブル形式で会議録を表示。行ごとに1つの会議（定例会・臨時会）が並び、列に「1日目」〜「7日目」の PDF リンクが配置される。

- ページネーション: なし（全会議録を単一ページ内に格納）
- PDF リンク: 相対パス（`/pdf/...`）

## 会議種別

- **定例会**: 年 5〜8 回程度（2月/3月、6月、9月、11月/12月など）
- **臨時会**: 必要に応じて開催（通常 1 日のみ）

### 定例会の日数

- 最大 6〜7 日目まで
- 各日に「議案説明・市政報告」「一般質問」「議案質疑」「委員長報告」などが実施される

## PDF ファイル名パターン

### 新形式（令和2年〜令和7年）

```
令和{N}年第{回数}回定例会{日数}日目（{月}月{日}日）.pdf
令和{N}年第{回数}回臨時会（{月}月{日}日）.pdf
```

例:
- `令和7年第5回定例会1日目（9月2日）.pdf`
- `令和6年第8回定例会1日目（11月26日）.pdf`
- `令和5年第3回臨時会（1月13日）.pdf`

### 特殊表記付き

```
(訂正版)令和4年第5回定例会1日目（8月31日）.pdf
令和5年第8回定例会3日目【副本】.pdf
平成30年第6回定例会2日目【文字訂正後】.pdf
```

### 旧形式・月別サブディレクトリ（平成21年〜平成30年頃）

```
/pdf/{年度コード}/{月}/{年度コード}_{回数}_{日目}.pdf
```

例:
- `/pdf/30/09/30_05_1.pdf`
- `/pdf/29/12/29_06_1.pdf`

### 最古形式（平成17年〜平成20年頃）

```
/pdf/{年度コード}/{通し番号}_{日目}.pdf
```

例:
- `/pdf/21/306_1.pdf`
- `/pdf/17/335_1.pdf`

通し番号は会議ごとに連番で管理されている。

## スクレイピング戦略

### Step 1: 会議録一覧ページの取得

`conference.html` を取得し、全タブ内の PDF リンクを抽出する。

- 単一ページに全年度の会議録リンクが含まれるため、1 回のリクエストで全量取得可能
- Cheerio で各タブ内のテーブルをパースし、`<a>` タグの `href` 属性から PDF URL を収集

**収集方法:**

1. `conference.html` を GET で取得
2. 各タブ（`#tab01` 〜 `#tab21`）内のテーブルから PDF リンクを抽出
3. ファイル名から会議名・開催日・日目などのメタ情報をパース

### Step 2: PDF のダウンロード

抽出した PDF URL からファイルをダウンロードする。

- URL エンコードされた日本語ファイル名に注意（`%E4%BB%A4%E5%92%8C...`）
- `【副本】` `【副本改】` `(訂正版)` `【文字訂正後】` などの特殊表記が付くファイルがある

### Step 3: PDF からテキスト抽出

PDF ファイルからテキストを抽出し、発言単位にパースする。

- PDF パーサー（pdf-parse 等）でテキスト抽出
- 議会会議録のため、発言者と発言内容の構造的なパースが必要

#### メタ情報の抽出

ファイル名から以下を抽出:

```
令和7年第5回定例会1日目（9月2日）.pdf
```

- 年号: `令和7年` → 開催年の算出
- 回数: `第5回`
- 種別: `定例会` / `臨時会`
- 日目: `1日目`
- 開催日: `9月2日`

#### パース用正規表現（案）

```typescript
// 新形式ファイル名からメタ情報を抽出
const newFormatPattern = /^(?:\(訂正版\))?(?:令和|平成)(\d+)年第(\d+)回(定例会|臨時会)(?:(\d+)日目)?(?:（(\d+)月(\d+)日）)?/;
// 例: 令和7年第5回定例会1日目（9月2日）→ era=令和, year=7, num=5, type=定例会, day=1, month=9, date=2

// 旧形式ファイル名からメタ情報を抽出
const oldFormatPattern = /^(\d{2})_(\d{2})_(\d+)\.pdf$/;
// 例: 30_05_1.pdf → yearCode=30, meetingNum=05, dayNum=1

// 最古形式ファイル名からメタ情報を抽出
const legacyFormatPattern = /^(\d{3})_(\d+)\.pdf$/;
// 例: 306_1.pdf → serialNum=306, dayNum=1
```

---

## 注意事項

- PDF 形式のため、HTML スクレイピングとは異なり PDF パーサーが必要
- 古い年度ほどファイル名の命名規則が異なるため、3 つのパターン（新形式・旧形式・最古形式）を考慮する必要がある
- `【副本】` `【副本改】` `(訂正版)` `【文字訂正後】` などの訂正版ファイルが存在する場合、最新版を優先して取得する
- URL 内の日本語ファイル名は URL エンコードされている

---

## 推奨アプローチ

1. **一括取得が効率的**: 単一ページに全リンクがあるため、1 リクエストで全 PDF URL を収集可能
2. **PDF パーサーの選定**: pdf-parse や pdf.js など、日本語テキスト抽出に対応したライブラリを使用
3. **レート制限**: 自治体サイトのため、PDF ダウンロード時にリクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 取得済みの PDF URL をリストで管理し、新規追加分のみダウンロードする
5. **訂正版の処理**: 同一会議に対して通常版と訂正版が存在する場合、訂正版を優先する
