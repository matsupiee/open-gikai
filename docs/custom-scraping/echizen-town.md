# 越前町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.echizen.fukui.jp/chousei/04/06/index.html
- 分類: 公式ホームページによる PDF 直接公開（専用の会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 会議録は PDF 形式で直接掲載。検索機能は実装されていない。令和5年3月定例会以降が対象とされているが、実際には平成29年3月定例会から掲載されている。

## 掲載構造

### 会議録一覧ページ

- `index.html` に全会議録へのリンクが単一ページに列挙されている（ページネーションなし）
- 掲載件数: 約72件（平成29年3月〜令和7年12月）
- 各リンクのテキスト形式:
  - 本会議議事録: `令和X年X月 越前町議会定例会 議事録`
  - 臨時会議事録: `令和X年第X回 越前町議会臨時会 議事録`
  - 一般質問会議録: `令和X年X月定例会・一般質問会議録`

### 会議種別

| 種別 | 概要 |
| --- | --- |
| 定例会（本会議議事録） | 各定例会の開催日ごとに複数 PDF を掲載 |
| 臨時会（本会議議事録） | 臨時会の開催日ごとに PDF を掲載 |
| 一般質問会議録 | 定例会の一般質問を議員ごとに個別 PDF で掲載 |

### 各詳細ページの構造

#### 本会議議事録ページ

- 会期情報（例: `令和7年9月3日から9月12日までの10日間`）
- 開催日ごとに PDF リンクが列挙される（1会期につき複数 PDF）
- ファイル名パターン:
  - 新形式: `YYMMDD_[first/second/last].pdf`（例: `070903_first.pdf`）
  - 旧形式: `YYYYMMDD_teireikai_NN.pdf`（例: `20170316_teireikai_01.pdf`）

#### 一般質問会議録ページ

- 「質問順」「議員氏名」「質問事項」「議事録」「映像」の5列テーブル構成
- 議員ごとに個別 PDF が用意される
- ファイル名パターン: `R{年月}_{議員姓ローマ字}.pdf`（例: `R0712_ibe.pdf`）

## スクレイピング戦略

### Step 1: 会議録リンクの収集

一覧ページ `index.html` から詳細ページへのリンクを抽出する。

- ページネーションなしで全件が1ページに列挙されている
- `<a>` タグの `href` が `/chousei/04/06/p{6桁ID}.html` パターンのものを収集
- リンクテキストから会議種別（定例会・臨時会・一般質問）を判別する

### Step 2: 詳細ページの取得と PDF リンク収集

各詳細ページにアクセスし、PDF リンクを収集する。

- `.pdf` で終わる `<a>` タグの `href` を抽出
- 相対パスの場合は絶対 URL に変換（ベース URL: `https://www.town.echizen.fukui.jp/chousei/04/06/`）
- 会議録の種別（本会議 or 一般質問）に応じてメタ情報を取得

### Step 3: PDF の取得とテキスト抽出

- 各 PDF をダウンロードしてテキスト抽出処理を行う
- PDF は開催日ごとまたは議員ごとに分割されているため、ファイル単位で管理する

---

## メタ情報の抽出

### 本会議議事録

ページのテキストから以下を抽出:

- 会期・日付: `令和X年X月 越前町議会定例会 議事録` 形式のタイトル
- 開催日範囲: `令和X年X月X日から X月X日まで` 形式の会期説明文

### 一般質問会議録

テーブルの各行から以下を抽出:

- 質問順（番号）
- 議員氏名
- 質問事項（テキスト）
- PDF リンク

---

## 注意事項

- 検索機能が存在しないため、全量クロールが必要
- ファイル名の命名規則が年代によって異なる（旧形式: `YYYYMMDD_teireikai_NN.pdf`、新形式: `YYMMDD_first/second/last.pdf`）
- 一般質問会議録は定例会に紐づく別ページとして掲載されており、本会議議事録とは別途収集が必要
- YouTube 映像リンクも各一般質問に付随しているが、スクレイピング対象外

---

## 推奨アプローチ

1. **一覧ページから全リンクを収集**: `index.html` を起点に全詳細ページ URL を抽出
2. **詳細ページから PDF リンクを収集**: 各詳細ページの PDF リンクを抽出し、会議種別・開催日のメタ情報と紐付け
3. **PDF テキスト抽出**: ダウンロードした PDF からテキストを抽出して処理
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページのリンクと既取得 URL を比較し、未取得分のみを処理する
