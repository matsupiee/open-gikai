## ページ構造

### 一覧ページ

- 全年度の会議録が単一ページに掲載（ページネーションなし）
- 年度ごとに見出し（h2 相当）で区切られている
- 各会議録は PDF へのリンクとして表示される
- リンクテキストに会議種別・開催日が含まれる
- 掲載範囲: 平成25年〜令和8年（現在）

### 会議の分類

```
年度（令和8年〜平成25年）
 ├─ 定例会（第1回〜第4回）
 │   └─ 日ごと（第1日、第2日、第3日）
 └─ 臨時会（随時）
```

- 定例会は年4回、通常2〜3日間にわたって開催
- 臨時会は随時開催、通常1日で完結

### PDF ファイルの特徴

- ファイル名は連番の数値 ID（例: `4804.pdf`）で、日付や会議名は含まれない
- PDF バージョン: 1.7
- 1ファイルあたり約20〜30ページ程度

## PDF 内部構造

### メタ情報（1ページ目）

PDF の先頭に以下の構造でメタ情報が記載される:

```
令和８年第１回設楽町議会臨時会会議録

令和８年１月７日第１回設楽町議会臨時会が設楽町役場議場に招集された。
```

抽出すべき情報:
- 年度: `令和X年` / `平成XX年`（全角数字）
- 回数: `第X回`
- 会議種別: `定例会` / `臨時会`
- 開催日: `令和X年XX月XX日`

1ページ目には続けて以下の情報も記載される:
- 出席議員一覧（番号 + 氏名）
- 欠席議員
- 説明出席者（町長、副町長、教育長、各課長）
- 議会事務局出席職員
- 議事日程

### 発言の構造

発言者パターン:

```
議長 皆さん、明けましておめでとうございます。
町長 皆さんおはようございます。
副町長 おはようございます、今年もよろしくお願いいたします。
総務課長 それでは、説明をさせていただきます。
６今泉 おはようございます。
```

- 発言者は行頭に役職名または「番号+氏名」で記載
- 議長、町長、副町長、教育長などは役職名のみ
- 議員は「番号+氏名」形式（例: `６今泉`）
- 発言者と発言内容は同一行にスペース区切りで続く
- 議事進行の区切り線として罫線（空行 + スペース）が使用される

### 議場内の表記

```
(「異議なし」の声あり)
```

- 括弧内に議場の状況が記載される

### パース用正規表現（案）

```typescript
// PDF タイトルからメタ情報を抽出

const datePattern = /(令和|平成)([\d０-９]+)年([\d０-９]+)月([\d０-９]+)日/;

// 発言者の抽出（役職名パターン）
const officialSpeakerPattern = /^(議長|副議長|町長|副町長|教育長|総務課長|企画ダム対策課長|津具総合支所長|生活課長|産業課長|保健福祉センター所長|建設課長|町民課長|財政課長|教育課長|出納室長|事務局長)\s/;

// 発言者の抽出（議員番号+氏名パターン）

const chamberActionPattern = /^\(「(.+?)」の声あり\)/;
```

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `soshiki/9/1226.html` を取得し、全 PDF リンクを抽出する。

- ページネーションなし（単一ページに全件掲載）
- `<a href="/uploaded/attachment/{ID}.pdf">` 形式のリンクをすべて収集
- リンクテキストから会議種別・開催日を事前に取得可能

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]').map((_, el) => ({
  url: $(el).attr('href'),
  text: $(el).text().trim(),
}));
```

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse、pypdf 等）でテキスト化
- 1ページ目からメタ情報を抽出
- 2ページ目以降から発言内容を抽出

### Step 3: 発言のパース

テキスト抽出後、発言者と発言内容を分離する。

1. 1ページ目のメタ情報ブロック（出席議員〜議事日程）をスキップ
2. 「会議録」「開会」の記載以降を発言本文として処理
3. 行頭の発言者パターンを検出し、発言者の切り替わりを認識
4. 発言者行から次の発言者行までのテキストを発言内容として結合

---

## 注意事項

- PDF ファイル名（数値 ID）には規則性がなく、連番でもないため、ファイル名からメタ情報を推測することは不可能。必ず一覧ページのリンクテキストまたは PDF 内のテキストからメタ情報を取得すること
- PDF のテキスト抽出時、ページ番号（各ページ上部の数字）が本文に混入する可能性がある。除去ロジックが必要
- 発言者パターンの役職名リストは議会期によって変動する（課の統廃合等）。厳密なマッチングではなく、行頭の役職名パターンを柔軟に検出する方式が望ましい
- 正誤表が別 PDF で掲載されているケースがある（例: 令和4年第4回定例会）
- 定例会は複数日にわたるため、各日が個別の PDF として公開される

---

## 推奨アプローチ

1. **一覧ページの一括取得**: 単一ページに全件掲載されているため、1回のリクエストで全 PDF リンクを取得可能
2. **リンクテキストからのメタ情報事前取得**: PDF をダウンロードする前に、リンクテキストから会議種別・開催日を解析してメタ情報を構築する
3. **PDF テキスト抽出**: pdf-parse 等のライブラリで PDF からテキストを抽出し、発言者・発言内容をパースする
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 一覧ページの PDF リンク数と既取得済み件数を比較し、新規追加分のみダウンロードする
