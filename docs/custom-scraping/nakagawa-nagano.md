# 中川村議会（長野県） カスタムスクレイピング方針

## 概要

- サイト: https://www.vill.nakagawa.nagano.jp/site/gikai/list30-185.html
- 分類: 自治体 CMS による PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF 形式のみで公開。HTML 本文なし

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（トップ） | `https://www.vill.nakagawa.nagano.jp/site/gikai/list30-185.html` |
| 定例会・臨時会 詳細ページ | `https://www.vill.nakagawa.nagano.jp/site/gikai/{ページID}.html` |

## ページ構造

### 一覧ページ (`list30-185.html`)

- 全定例会・臨時会が単一ページにリスト表示（ページネーションなし）
- 新しい順に並んでいる
- 各エントリは定例会/臨時会名（例: 「令和7年9月定例会」）と詳細ページへのリンクで構成
- 平成23年～令和7年の会議録が掲載（約80件）

### 詳細ページ（各定例会・臨時会）

- 定例会ごとに 1 ページが割り当てられている
- ページ内に開催日ごとの PDF リンクが並ぶ
- 各 PDF は開催日付をラベルとして表示（例: 「令和7年9月8日」）
- ファイルサイズも併記

---

## スクレイピング戦略

### Step 1: 詳細ページ URL の収集

一覧ページ `list30-185.html` から全定例会・臨時会の詳細ページ URL を抽出する。

- リンクは `/site/gikai/{ページID}.html` 形式
- ページネーションなし（単一ページに全件掲載）
- Cheerio で `<a>` タグの `href` を抽出

**収集対象例:**

| 会議 | 詳細ページ URL |
| --- | --- |
| 令和7年9月定例会 | `/site/gikai/12426.html` |
| 令和7年6月定例会 | `/site/gikai/12147.html` |
| 平成31年3月定例会 | `/site/gikai/1072.html` |
| 平成23年臨時会 | `/site/gikai/1037.html` |

### Step 2: PDF URL の収集

各詳細ページにアクセスし、会議録 PDF の URL を抽出する。

- PDF リンクは `/uploaded/attachment/{添付ID}.pdf` 形式
- 各定例会ページに複数の PDF（開催日ごと）が含まれる
- 日付ラベルとセットで抽出し、メタ情報として保持する

**例（令和7年9月定例会）:**

| 開催日 | PDF URL |
| --- | --- |
| 令和7年9月8日 | `/uploaded/attachment/7695.pdf` |
| 令和7年9月9日 | `/uploaded/attachment/7696.pdf` |
| 令和7年9月26日 | `/uploaded/attachment/7697.pdf` |

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキストを抽出する
- PDF パーサー（pdf-parse 等）を使用してテキスト化

### Step 4: メタ情報の構築

詳細ページの会議名と PDF の日付ラベルから以下を構築:

- **自治体名**: 中川村（長野県）
- **会議種別**: 定例会 / 臨時会
- **開催時期**: 年月（例: 令和7年9月）
- **開催日**: PDF ラベルの日付（例: 令和7年9月8日）

---

## 注意事項

- 会議録は PDF のみで公開されており、HTML 本文は存在しない
- PDF 内のテキスト構造はファイルによって異なる可能性がある（レイアウト依存）
- 一覧ページにページネーションがないため、全件を一括で取得可能
- 添付 ID はおおむね連番だが、定例会ページ ID は不規則（CMS 自動採番）

---

## 推奨アプローチ

1. **2段階クロール**: 一覧ページ → 詳細ページ → PDF URL の順で収集
2. **PDF テキスト抽出**: pdf-parse 等でテキスト化し、発言者・発言内容をパース
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの更新日付を監視し、新規追加分のみをクロールする
