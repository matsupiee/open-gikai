# 綾川町議会 カスタムスクレイピング方針

## 概要

- 自治体公式サイト: https://www.town.ayagawa.lg.jp/soshiki/gikai/
- 議会専用サイト: https://ayagawa-gikai.jp/
- 分類: 独自静的 HTML サイトによる PDF 公開（標準的な検索システムなし）
- 文字コード: UTF-8
- 特記: 会議録は議会専用ドメイン `ayagawa-gikai.jp` で公開されており、自治体公式サイトからリンクされている

## 会議ページの命名規則

会議ページの HTML ファイル名は一貫した命名規則を持たず、複数のパターンが混在している。

### 定例会ページのパターン

| パターン | 例 | 対応する会議 |
| --- | --- | --- |
| `r{年度}{月}gatsuteireikai` | `r712gatsuteireikai.html` | 令和7年12月定例会 |
| `r{年度}nen{月}gatsuteireikai` | `r6nen9gatsuteireikai.html` | 令和6年9月定例会 |
| `{西暦年}{月2桁}teireikai` | `202503teireikai.html` | 令和7年3月定例会 |

### 臨時会ページのパターン

| パターン | 例 | 対応する会議 |
| --- | --- | --- |
| `r{年度}{月}gatsurinjikai` | `r75gatsurinjikai.html` | 令和7年5月臨時会 |
| `r{年度}{月}gatsurinnjikai` | `r65gatsurinnjikai.html` | 令和6年5月臨時会 |

## PDF ファイルの命名規則

PDF ファイルは `img/` ディレクトリに格納されており、複数の命名パターンが混在している。

### 本会議会議録 PDF のパターン

| パターン | 例 | 内容 |
| --- | --- | --- |
| `{西暦年月}kaigiroku.pdf` | `202509kaigiroku.pdf` | 令和7年9月定例会 会議録 |
| `{西暦年月}kaigiroku2.pdf` | `202406kaigiroku2.pdf` | 令和6年6月定例会 会議録（別バージョン） |
| `{西暦年月日}kaigiroku.pdf` | `20240502kaigiroku.pdf` | 令和6年5月臨時会 会議録（日付まで含む） |

### 一般質問 PDF のパターン

| パターン | 例 | 内容 |
| --- | --- | --- |
| `file{連番}.pdf` | `file36.pdf`〜`file46.pdf` | 令和7年12月定例会 一般質問（議員別） |
| `file{連番}.pdf` | `file47.pdf` | 令和7年12月定例会 本会議会議録 |

※ `file{連番}` 形式は一般質問 PDF に用いられることが多いが、本会議会議録にも使用される場合がある。連番は議会専用サイト全体で通し番号となっている。

## PDF の形式

- **作成ツール**: Microsoft Word LTSC（Word で作成後 PDF 出力）
- **テキスト抽出**: 可能（OCR 不要）
- **フォント**: MS Mincho、Century（埋め込みあり）
- **言語設定**: ja-JP
- **ページ数**: 本会議会議録は 78〜95 ページ程度（会議の規模による）

## スクレイピング戦略

### Step 1: 会議ページ URL の収集

`https://ayagawa-gikai.jp/teireikai.html` を取得し、各会議ページへのリンク（`*.html`）を抽出する。

- ページ内の年度別リンクを Cheerio でパースして収集
- 命名規則が混在しているため、テキストマッチングで年度・会議種別を判定する

### Step 2: 各会議ページの PDF リンク収集

各会議ページにアクセスし、`img/*.pdf` へのリンクをすべて抽出する。

- 本会議会議録: `img/{年月}kaigiroku.pdf` パターン
- 一般質問: `img/file{連番}.pdf` パターン（複数 PDF が議員ごとに存在する）
- 委員会: サブページ（`*iinkai.html`）に遷移して PDF リンクを収集

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL から PDF をダウンロードし、テキストを抽出する。

- テキスト埋め込み形式のため `pdf-parse` や `pdfjs-dist` で直接抽出可能
- OCR 処理は不要

---

## 注意事項

- 議会専用サイト（`ayagawa-gikai.jp`）と自治体公式サイト（`town.ayagawa.lg.jp`）は別ドメインで運用されており、会議録本体は前者にある
- HTML ファイルの命名規則が年度によって異なるため、`teireikai.html` の一覧ページを起点としてリンクを動的に収集する必要がある（ハードコードによる URL 生成は不完全になる）
- `file{連番}.pdf` は議会専用サイト全体での通し番号であり、会議や年度をまたいで付番されているため、ファイル番号から会議情報を逆引きすることはできない
- 一般質問は議員ごとに個別 PDF が作成されており、1 回の定例会で 10 件以上の PDF が存在する場合がある
- 音声（YouTube）へのリンクも一般質問ページに含まれているが、会議録 PDF が別途存在するため音声は対象外とする
- レート制限: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **起点は `teireikai.html`**: 命名規則が混在しているため、一覧ページから動的にリンクを収集する
2. **会議種別の判定**: ページのテキスト（「定例会」「臨時会」「一般質問」「委員会」）からメタ情報を抽出する
3. **PDF の種別判定**: `kaigiroku` を含むファイルは本会議会議録、`file{連番}` は一般質問または委員会として扱う
4. **差分更新**: `teireikai.html` の新着会議ページを前回取得時と比較して、未処理の会議のみを対象とする
