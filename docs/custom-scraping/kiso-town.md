# 木曽町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town-kiso.com/gikai/
- 分類: 自治体 CMS による HTML + PDF 公開（専用の会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 議決事項が年度別に整理されている。会議録本文はPDFで提供され、HTML上にテキストとして掲載されない。

---

| ページ | URL パターン |
| --- | --- |
| 議会トップ | `https://www.town-kiso.com/gikai/` |
| 議案・会議の結果（トップ） | `https://www.town-kiso.com/gikai/kekka/` |
| 議案（カテゴリ） | `https://www.town-kiso.com/gikai/kekka/100238/` |
| 請願・陳情及び意見書（カテゴリ） | `https://www.town-kiso.com/gikai/kekka/100237/` |
| 議決結果（カテゴリ） | `https://www.town-kiso.com/gikai/kekka/100312/` |
| 一般質問（カテゴリ） | `https://www.town-kiso.com/gikai/kekka/100413/` |
| 一般質問録画動画（カテゴリ） | `https://www.town-kiso.com/gikai/kekka/100414/` |
| 各会議の詳細ページ | `https://www.town-kiso.com/gikai/kekka/{カテゴリID}/{記事ID}/` |
| PDF ファイル | `https://www.town-kiso.com/files/file/box/{hash_prefix}/{hash}.pdf` |

### URL パターンの規則

- カテゴリ ID は固定（`100238`=議案, `100237`=請願・陳情, `100312`=議決結果, `100413`=一般質問, `100414`=録画動画）
- 記事 ID は数値連番（例: `102359`, `102348` など）で、新しい記事ほど大きい値
- PDF ファイルは `/files/file/box/{2桁ディレクトリ}/{SHA1ハッシュ}.pdf` の形式

## サイト構造

### カテゴリ一覧ページ

各カテゴリページ（例: `/gikai/kekka/100413/`）には、会議ごとのリンクが時系列で表示される。

- 表示形式: リスト形式で「令和X年Y月会議」のリンクが並ぶ
- ページネーション: あり（`?page=N` パラメータ）
- 各リンクは `/gikai/kekka/{カテゴリID}/{記事ID}/` の形式

### 詳細ページ

各詳細ページ（例: `/gikai/kekka/100413/102357/`）には以下が含まれる:

- 会議の概要テキスト（開催日、質問者数等の簡単な説明）
- PDF ファイルへのリンク（議案書、通告書、議決結果等）
- 質問内容や議決結果の本文は HTML 上には掲載されず、**PDF のみ**で提供

### 議決結果ページの年度別構造

議決結果カテゴリ（`/gikai/kekka/100312/`）は年度単位でまとめられている:

| 記事 | URL |
| --- | --- |
| 令和8年議決結果 | `/gikai/kekka/100312/102348/` |
| 令和7年議決結果 | `/gikai/kekka/100312/102185/` |
| 令和6年議決結果 | `/gikai/kekka/100312/102029/` |
| 令和5年議決結果 | `/gikai/kekka/100312/101888/` |

## スクレイピング戦略

### Step 1: カテゴリページから記事 URL を収集

各カテゴリの一覧ページをクロールし、詳細ページの URL を収集する。

**対象カテゴリ:**

| カテゴリ | URL |
| --- | --- |
| 議案 | `/gikai/kekka/100238/` |
| 請願・陳情及び意見書 | `/gikai/kekka/100237/` |
| 議決結果 | `/gikai/kekka/100312/` |
| 一般質問 | `/gikai/kekka/100413/` |

**収集方法:**

1. 各カテゴリページにアクセス
2. `<a href="/gikai/kekka/{カテゴリID}/{記事ID}/">` パターンのリンクを抽出
3. ページネーション（`?page=N`）がある場合は全ページを巡回

### Step 2: 詳細ページから PDF リンクを収集

各詳細ページにアクセスし、PDF ファイルへのリンクを抽出する。

- PDF リンクのパターン: `/files/file/box/{hash_prefix}/{hash}.pdf`
- 各 PDF のファイル名（リンクテキスト）からドキュメント種別を判別（「議事日程」「一般質問通告書」「議決結果」等）

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF からテキストを抽出（pdf-parse 等のライブラリを使用）
3. 抽出したテキストから以下のメタ情報をパース:
   - 会議名（例: 「令和8年3月会議」）
   - 開催日
   - 議案番号・議案名
   - 質問者名・質問内容（一般質問の場合）
   - 議決結果（可決/否決等）

---

## 注意事項

- 会議録本文（発言録）は公開されていない。公開されているのは議案書、一般質問通告書、議決結果等のPDFのみ。
- 専用の会議録検索システムは存在しない。
- PDF ファイルの URL はハッシュベースのため、ファイル一覧を事前に推測することはできない。必ず HTML ページ経由で PDF リンクを取得する必要がある。
- 一般質問録画動画（`/gikai/kekka/100414/`）は動画コンテンツのため、テキスト抽出の対象外とする。

---

## 推奨アプローチ

1. **カテゴリページ起点のクロール**: 4つのカテゴリページを起点に、全詳細ページの URL を収集する
2. **PDF 中心の抽出**: HTML ページにはメタ情報のみ記載されるため、PDF のダウンロードとテキスト抽出が必須
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 記事 ID は数値連番のため、前回取得済みの最大 ID 以降のみを取得する差分更新が可能
5. **PDF テキスト抽出の品質確認**: 自治体の PDF はスキャン画像の場合もあるため、テキスト抽出可能かを事前に確認する（OCR が必要な場合は別途対応）
