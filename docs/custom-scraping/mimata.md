## PDF ファイル名パターン

### 定例会単独

```
{年号}{年}年{月}月定例会.pdf
```

例: `令和7年9月定例会.pdf`

### 臨時会＋定例会の併録

```
{年号}{年}年{月}月臨時会ー{月}月定例会.pdf
{年号}{年}年{月}月臨時会－{月}月定例会.pdf
{年号}{年}年{月}月定例会−{月}月臨時会.pdf
```

例: `令和7年1月臨時会ー3月定例会.pdf`、`令和7年12月定例会−12月臨時会.pdf`

※ 区切り文字が「ー」（長音）、「－」（全角ハイフン）、「−」（マイナス）など統一されていない点に注意。

### 一般質問（平成年代のみ）

```
{年号}{年}年{月}月定例会_一般質問.pdf
```

例: `平成20年3月定例会_一般質問.pdf`

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録一覧トップページ (`/categories/index/kaigiroku`) から各年度ページへのリンクを収集する。

- リンクは `/contents/{ID}.html` 形式
- ページネーションなし（1ページに全年度が一覧表示）

### Step 2: PDF リンクの収集

各年度ページ (`/contents/{ID}.html`) にアクセスし、添付ファイルセクションから PDF リンクを抽出する。

- PDF リンクは `/upload/file/09gikai/` 配下を指す
- 各ページに年間 4〜8 個程度の PDF が掲載

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で本文を取得する
- ファイルサイズは 48KB〜6.1MB 程度

### Step 4: メタ情報の抽出

PDF ファイル名から以下を抽出する。

```typescript
// ファイル名からメタ情報を抽出
const filenamePattern = /^(令和|平成)(元|\d+)年(\d+)月(臨時会|定例会)/;
// 例: "令和7年9月定例会.pdf" → era="令和", year="7", month="9", type="定例会"
```

### Step 5: 本文のパース

PDF から抽出したテキストを解析し、発言者・発言内容を構造化する。

※ PDF のレイアウトや文字認識精度により、テキスト抽出の品質が変動する可能性がある。実際の PDF を取得して構造を確認した上でパーサーを設計する必要がある。

---

## 注意事項

- PDF ファイルの URL はファイル名が日本語のため、URL エンコードされている
- 区切り文字（「ー」「－」「−」「＝」等）がファイルごとに異なるため、リンク解析時は HTML のアンカー要素から直接 URL を取得すること（ファイル名の推測はしない）
- 平成年代は定例会の本体と一般質問が別 PDF に分かれている場合がある
- ファイルサイズが大きい（最大 6MB 超）ため、ダウンロード時のタイムアウト設定に注意
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける

---

## 推奨アプローチ

1. **リンク収集を優先**: まず全年度ページをクロールして PDF の URL リストを作成する
2. **ファイル名からメタ情報抽出**: PDF ダウンロード前にファイル名から年度・会議種別を判定する
3. **PDF テキスト抽出**: pdf-parse 等でテキストを抽出し、発言構造のパース方針は実データに基づいて決定する
4. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
5. **差分更新**: 年度ページの更新日を確認し、前回取得時から更新がある年度のみ再取得する
