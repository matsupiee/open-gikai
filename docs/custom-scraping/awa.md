# 阿波市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.awa.lg.jp/gikai/category/bunya/kaigiroku
- 分類: 市公式サイト内での直接公開（会議録検索システム未導入）
- 文字コード: UTF-8
- 特記: 会議録は PDF 形式で提供。会期（定例会・臨時会）ごとに個別の記事ページが作成され、各ページに開催日ごとの PDF が列挙される。専用の検索システムは存在しない。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（1ページ目） | `https://www.city.awa.lg.jp/gikai/category/bunya/kaigiroku` |
| 会議録一覧（追加ページ） | `https://www.city.awa.lg.jp/gikai/category/bunya/kaigiroku/more@docs_{N}.html` |
| 会期別会議録ページ | `https://www.city.awa.lg.jp/gikai/docs/{ID}/` |
| 会議録 PDF | `https://www.city.awa.lg.jp/gikai/docs/{ID}/file_contents/{ファイル名}.pdf` |

### 一覧ページのページネーション

- 1ページあたり約28〜30件表示
- `more@docs_1.html` から始まり、`more@docs_N.html` で追加ページにアクセス
- 現時点で令和2年〜令和7年分が計4ページ程度に分散

### 会期別ページ ID の構造

ID は公開日の日付 + 連番で構成される（例: `2026022700023` = 2026年02月27日公開・連番23）。

## ページ構造

### 会議録一覧ページ

- 各会期の会議録がリスト形式（`<li>` 要素）で表示される
- リンクテキスト例: `令和7年第4回定例会会議録`
- リンク先: `/gikai/docs/{ID}/`（ディレクトリ形式）

### 会期別会議録ページ

- ページタイトル（`<h1>`）に会期名が記載（例: `令和7年第4回定例会会議録`）
- 各開催日の PDF が `<a>` タグのシンプルなリストで列挙される（クラス名なし）
- リンクテキスト形式: `令和7年第4回定例会会議録11月25日[PDF：326KB]`
- PDF は相対 URL（`file_contents/{ファイル名}.pdf`）で参照される
- 1会期あたり通常 5〜6 日分の PDF が掲載される

### PDF ファイル名の命名規則

命名規則はページ（時期）によって異なる：

**新形式（令和3年以降）:**
```
kaigiroku{元号2桁}{月日4桁}.pdf
例: kaigiroku071125.pdf（令和7年11月25日）
    kaigiroku070904.pdf（令和7年9月4日）
```

**旧形式（令和2年以前）:**
```
{ID}_gikai_docs_{ID}_file_contents_{月日}.pdf
例: 2021022200023_gikai_docs_2021022200023_file_contents_241125.pdf（令和2年11月25日）
```

## スクレイピング戦略

### Step 1: 会期別ページ URL の収集

一覧ページ（`/gikai/category/bunya/kaigiroku`）からページネーションをたどり、全会期のページ URL を収集する。

```typescript
// 一覧ページからのリンク抽出
const sessionLinks = $('a[href*="/gikai/docs/"]');

// ページネーションリンクの抽出
const nextPageLinks = $('a[href*="more@docs_"]');
```

**手順:**
1. 一覧ページ（`more@docs_1.html`〜）を順にアクセス
2. 各ページから `/gikai/docs/{ID}/` 形式のリンクを抽出
3. 次ページリンク（`more@docs_{N}.html`）がなくなるまで繰り返す

### Step 2: 会期別ページからの PDF リンク収集

各会期別ページにアクセスし、PDF へのリンクを抽出する。

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]');

// 各リンクの情報を取得
pdfLinks.each((_, el) => {
  const href = $(el).attr('href'); // 相対URL: file_contents/kaigiroku071125.pdf
  const text = $(el).text();       // 例: 令和7年第4回定例会会議録11月25日[PDF：326KB]
  const absoluteUrl = new URL(href, pageUrl).toString();
});
```

**収集する情報:**
1. PDF の絶対 URL
2. リンクテキスト（会期名 + 開催日 + ファイルサイズ）
3. 会期名（ページタイトル `<h1>` から取得）
4. 開催日（リンクテキストの末尾「X月X日」部分から抽出）

### Step 3: メタ情報の抽出

リンクテキストと `<h1>` タイトルからメタ情報を抽出する。

```typescript
// ページタイトルから会期情報を抽出
const sessionPattern = /(令和|平成)(\d+)年(第\d+回(?:定例会|臨時会))/;
// 例: 令和7年第4回定例会会議録 → 令和7年・第4回定例会

// リンクテキストから開催日を抽出
const datePattern = /(\d+)月(\d+)日/;
// 例: 令和7年第4回定例会会議録11月25日[PDF：326KB] → 11月25日
```

### Step 4: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、pdftotext 等でテキストを抽出する
- PDF 内部の発言者パターン等は PDF のテキスト抽出後に分析が必要（PDF がスキャン画像の場合は OCR が必要な可能性あり）

---

## 注意事項

- PDF ファイルの命名規則が新旧で異なるため、URL を HTML から取得することが重要（ファイル名を推測してアクセスしてはならない）
- 相対 URL で記述された PDF リンクは、ベース URL（会期別ページの URL）と組み合わせて絶対 URL に変換すること
- 一覧ページのページネーション URL（`more@docs_N.html`）は `N=1` から始まるが、404 エラーが返る場合がある（`more@docs_2.html` 以降が 404 になるケースを確認）。実際の構造を事前に確認すること
- リクエスト間には適切な待機時間（1〜2秒）を設けること

---

## 推奨アプローチ

1. **全量取得を優先**: 一覧ページのページネーションをたどり、全会期のページ ID リストをまず構築する
2. **HTML からリンクを辿る**: PDF のファイル名命名規則が年代によって異なるため、必ず HTML ページから `<a href>` を辿って PDF URL を取得する
3. **PDF テキスト抽出**: ダウンロードした PDF から pdftotext 等でテキストを抽出し、発言内容をパースする
4. **差分更新**: 一覧ページの公開日（ページ ID の日付部分）を管理し、前回取得済みの最新 ID 以降のみを取得する差分更新が可能
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
