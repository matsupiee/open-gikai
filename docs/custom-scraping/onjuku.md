- サイト: https://www.town.onjuku.chiba.jp/sub5/4/gikai/gijiroku/

## サイト構造

### トップページ（会議録一覧）

- 全23年分の年別リンクが単一ページに `<ul><li>` で列挙
- ページネーションなし
- 令和7年（最新）から平成15年（最古）まで降順で表示

### 年別ページ

- `<h3>` 見出しで「定例会」と「臨時会」に区分
- 各会議は月別に整理（3月議会、6月議会、9月議会、12月議会）
- 会議録 PDF へのリンクが `<ul><li>` で列挙
- 複数日にわたる会議は日程ごとに分割された PDF を提供
- 目次 PDF と本文 PDF が分かれている場合あり

### 会議の種別

- **定例会**: 年4回（3月、6月、9月、12月）
- **臨時会**: 不定期開催（年によって0〜6回）

## スクレイピング戦略

### Step 1: 年別ページ URL の収集

トップページ `https://www.town.onjuku.chiba.jp/sub5/4/gikai/gijiroku/` から全年別ページへのリンクを抽出する。

- `<ul>` 内の `<a>` タグから `href` を取得
- 23件の年別ページ URL を収集

### Step 2: 各年別ページから PDF URL の収集

各年別ページにアクセスし、PDF へのリンクを全て抽出する。

- `<a>` タグの `href` が `.pdf` で終わるリンクを全て取得
- リンクテキストから会議種別（定例会/臨時会）、日付、日程番号を抽出
- `<h3>` 見出しで定例会/臨時会を判別

**収集するメタ情報:**

- 年（和暦・西暦）
- 会議種別（定例会/臨時会）
- 開催月
- 開催日（リンクテキストから取得可能な場合）
- 日程番号
- PDF URL

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキスト化
- 目次 PDF（`c.pdf`, `mokuji.pdf`）はスキップまたは別途処理

### Step 4: 会議録テキストのパース

抽出したテキストから発言情報を構造化する。

- 発言者パターンは PDF のテキスト抽出後に確認が必要
- 一般的な議会会議録の発言者パターン（`○議長（氏名）`、`○N番（氏名）` 等）を想定

---

## 注意事項

- **PDF 形式**: 会議録は全て PDF で公開されており、HTML ベースの会議録は存在しない
- **ファイル名の一貫性なし**: 年代によって PDF のファイル命名規則が大きく異なるため、ファイル名からのメタ情報抽出は補助的にのみ使用する
- **PDF のテキスト品質**: スキャン PDF の可能性がある（特に古い年度）。OCR が必要になる場合がある
- **目次と本文の分離**: 一部の年度では目次 PDF と本文 PDF が分かれているため、目次 PDF を除外するフィルタリングが必要
- **相対 URL**: PDF リンクは相対パスで記載されている場合があるため、ベース URL と結合して完全な URL を構成する

---

## 推奨アプローチ

1. **2段階クロール**: トップページ → 年別ページ → PDF URL の順で段階的に収集
2. **PDF リンクの全量リスト作成**: まず全 PDF URL のリストを作成し、ダウンロード対象を確定させる
3. **目次 PDF の除外**: ファイル名に `mokuji` や `c.pdf` パターンが含まれるものは目次として除外（またはメタ情報のみ利用）
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **PDF テキスト品質の事前確認**: 数件の PDF を手動でテキスト抽出し、品質（テキスト PDF かスキャン PDF か）を確認してから本格実装に進む
6. **差分更新**: 年別ページの ID が数値連番のため、最新の年別ページのみを再クロールすることで差分更新が可能
