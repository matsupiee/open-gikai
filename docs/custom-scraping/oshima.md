# 大島町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.oshima.tokyo.jp/soshiki/gikaijim/gikai-kekka.html
- 分類: PDF 直接公開（結果報告書のスキャン画像）
- 文字コード: UTF-8
- 特記: 会議録（発言録）ではなく「議案等の審議・決定結果報告」を PDF で公開。PDF はスキャン画像（OCR なし）のため、テキスト抽出にはOCR処理が必要。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 結果報告一覧 | `https://www.town.oshima.tokyo.jp/soshiki/gikaijim/gikai-kekka.html` |
| PDF ファイル | `https://www.town.oshima.tokyo.jp/uploaded/attachment/{ID}.pdf` |
| Word ファイル（一部） | `https://www.town.oshima.tokyo.jp/uploaded/attachment/{ID}.doc` |

- 一覧ページは単一ページ構成（ページネーションなし）
- 全会議の結果報告リンクが1ページに掲載されている（約70件）
- ファイル ID は連番ではなく不規則

---

## 掲載範囲

- 平成26年（2014年）第1回定例会 〜 現在
- 定例会: 年4回（3月、6月、9月、12月）
- 臨時会: 不定期開催

## コンテンツの種類

### 結果報告（現在公開されているもの）

ページタイトルに「議案等の審議・決定結果を掲載します」と明記されており、公開されているのは**会議録（発言録）ではなく結果報告書**である。

- 議案番号、議案名、審議結果（可決・否決等）を記載
- 各議員の賛否は記載されていない可能性がある
- 発言者ごとの発言内容は含まれない

### PDF の特徴

- 生成ツール: Haru Free PDF Library 2.4.0d（PDF 1.3）
- 内容: スキャン画像（白黒、2480x3507px、1bit深度）
- テキストレイヤー: なし（OCR 未処理）
- ファイルサイズ: 50KB 〜 300KB 程度
- ページ数: 3〜4ページ程度

### 一部 Word ファイル

平成28年の2件のみ `.doc` 形式で公開:

- `/uploaded/attachment/1193.doc`（平成28年第3回臨時会）
- `/uploaded/attachment/1150.doc`（平成28年第3回定例会）

## スクレイピング戦略

### Step 1: PDF URL の収集

一覧ページ `gikai-kekka.html` から全 PDF リンクを抽出する。

- ページネーションなし（単一ページに全件掲載）
- `<a>` タグの `href` 属性から `/uploaded/attachment/{ID}.pdf` パターンを抽出
- リンクテキストから会議名・開催時期のメタ情報を取得

**収集用正規表現（案）:**

```typescript
// リンクテキストからメタ情報を抽出
const meetingPattern = /(?:令和|平成)\d+年第\d+回(?:（(\d+)月）)?大島町議会(定例会|臨時会)結果報告/;

// PDF URL の抽出
const pdfUrlPattern = /\/uploaded\/attachment\/(\d+)\.pdf/;
```

### Step 2: PDF のダウンロードと OCR

PDF はスキャン画像のため、テキスト抽出には OCR が必要。

1. PDF をダウンロード
2. 画像を抽出（白黒、2480x3507px）
3. OCR エンジン（Tesseract 等）で日本語テキストを抽出
4. 抽出テキストから議案情報をパース

### Step 3: テキストのパース

OCR で抽出したテキストから以下を構造化:

- 議案番号
- 議案名
- 審議結果（原案可決、同意、採択など）

---

## 注意事項

- PDF がスキャン画像であるため、OCR の精度に依存する
- 白黒1bit画像のため、OCR 精度は比較的高いと期待できる
- `.doc` ファイル（2件）は別途テキスト抽出が必要
- 会議録（発言録）は公開されていないため、発言者ごとの発言内容の取得は不可
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける

---

## 推奨アプローチ

1. **一覧ページから全量取得**: 単一ページに全件掲載されているため、1回のリクエストで全 PDF URL を収集可能
2. **OCR パイプライン**: スキャン PDF → 画像抽出 → OCR → テキストパースの流れを構築
3. **差分更新**: 一覧ページの先頭に最新の会議が追加されるため、既知の URL リストとの差分で新規分のみ取得
4. **メタ情報はリンクテキストから取得**: PDF を開かずとも、リンクテキストから会議種別・開催時期を取得可能
