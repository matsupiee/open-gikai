# 山北町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.yamakita.kanagawa.jp/category/14-3-0-0-0.html
- 分類: 公式サイト上で PDF ファイルとして年度別に公開（専用の会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 検索機能なし。会議録本文は PDF のみで提供され、HTML 上に本文テキストは表示されない

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.town.yamakita.kanagawa.jp/category/14-3-0-0-0.html` |
| 会議録詳細（定例会・臨時会ごと） | `https://www.town.yamakita.kanagawa.jp/{ページID}.html` |
| PDF ファイル | `https://www.town.yamakita.kanagawa.jp/cmsfiles/contents/{上位ディレクトリ}/{ページID}/{ファイル名}.pdf` |

### ページ ID の例

- `0000006954` → 令和7年第3回定例会
- `0000006850` → 令和7年第1回定例会
- `0000004068` → 平成30年第4回定例会

### PDF パス構造

PDF は `./cmsfiles/contents/{7桁ディレクトリ}/{ページID}/` 配下に格納される。

```
./cmsfiles/contents/0000006/6954/20250903-3_minutes_day1.pdf
./cmsfiles/contents/0000004/4068/301204kaigiroku.pdf
```

上位ディレクトリはページ ID の上位4桁（例: ページ ID `6954` → `0000006`）。

## サイト構造

### 会議録一覧ページ

- 全会議録が単一ページにリスト表示（ページネーションなし）
- 平成30年〜令和7年の範囲で約60件の会議録が掲載
- 各リンクのテキストは「令和X年第Y回{定例会|臨時会}会議録」形式

### 会議録詳細ページ

各定例会・臨時会ごとに1ページが存在し、以下の構造で PDF リンクが掲載される:

```
h1: 令和7年第3回定例会会議録
h2: 本会議
  h3: 9月3日
    div.mol_attachfileblock
      ul > li > a (PDF リンク)
  h3: 9月4日
    ...
h2: 一般質問
  h3: 9月3日水曜日　質問者
    div.mol_attachfileblock
      ul > li > a (PDF リンク)
h2: 委員会
  h3: 決算特別委員会
    div.mol_attachfileblock
      ul > li > a (PDF リンク)
```

### PDF ファイルの種類

各日程につき最大3種類の PDF が存在:

| 種類 | リンクテキストのパターン | 例 |
| --- | --- | --- |
| 表紙 | `【表紙】令和X年第Y回定例会(M月D日).pdf` | `20250903-1_frontcover.pdf` |
| 議事日程 | `【議事日程】令和X年第Y回定例会(M月D日).pdf` | `20250903-2_agenda.pdf` |
| 会議録 | `【会議録】令和X年第Y回定例会(M月D日).pdf` | `20250903-3_minutes_day1.pdf` |

一般質問は議員ごとに個別 PDF:

| 種類 | リンクテキストのパターン | 例 |
| --- | --- | --- |
| 一般質問 | `N.【議員名】令和X年第Y回定例会一般質問.pdf` | `20250903-1_akinoriwada.pdf` |

## PDF ファイル命名規則

命名規則は年代により異なる:

### 新しい会議録（令和以降）

```
{YYYYMMDD}-{連番}_{種別}.pdf
例: 20250903-3_minutes_day1.pdf
```

### 古い会議録（平成30年頃）

```
{YYYYMMDD}{種別}.pdf  または  {YYMMDD}{種別}.pdf
例: 20181204hyoushi.pdf, 301204kaigiroku.pdf
```

## スクレイピング戦略

### Step 1: 会議録一覧の取得

会議録一覧ページ `category/14-3-0-0-0.html` から全会議録ページへのリンクを収集する。

- ページネーションなし（全件が1ページに表示）
- リンクの `href` 属性からページ ID を抽出
- リンクテキストから年度・回次・会議種別（定例会/臨時会）を抽出

**収集方法:**

1. 一覧ページを GET で取得
2. `a[href]` から `/{ページID}.html` パターンのリンクを Cheerio で抽出
3. リンクテキストを正規表現でパース

```typescript
// リンクテキストのパース
const titlePattern = /^(令和|平成)(\d+)年第(\d+)回(定例会|臨時会)会議録$/;
// 例: "令和7年第3回定例会会議録" → era="令和", year=7, session=3, type="定例会"
```

### Step 2: 各会議録ページから PDF リンクの収集

各会議録詳細ページにアクセスし、PDF ファイルへのリンクを収集する。

- `div.mol_attachfileblock` 内の `a[href$=".pdf"]` を抽出
- `h2`（本会議/一般質問/委員会）と `h3`（日付）で構造を把握
- リンクテキストから PDF の種類（表紙/議事日程/会議録/一般質問）を判別

**会議録 PDF の判別:**

```typescript
// 会議録PDFの判定（表紙・議事日程を除外）
const isMinutesPdf = (linkText: string) =>
  linkText.includes("【会議録】") ||
  linkText.includes("kaigiroku") ||
  linkText.includes("一般質問") ||
  linkText.includes("委員会");
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用
- 会議録 PDF のみを対象とし、表紙・議事日程 PDF はスキップ可能

---

## 注意事項

- 会議録は HTML テキストではなく PDF のみで提供されるため、PDF パーサーが必須
- PDF の命名規則が年代により統一されていない（新旧で異なるパターンが存在）
- 一覧ページにページネーションがないため、全件を一度に取得可能
- 臨時会は一般質問・委員会セクションがない場合がある
- サイトは HTTP から HTTPS へリダイレクトされる（`http://www.town.yamakita.kanagawa.jp` → `https://www.town.yamakita.kanagawa.jp`）

---

## 推奨アプローチ

1. **一覧ページから全量取得**: 一覧ページにページネーションがないため、1リクエストで全会議録リンクを取得可能
2. **会議録 PDF のみを対象**: 表紙・議事日程 PDF はスキップし、会議録本文の PDF のみをダウンロード
3. **PDF テキスト抽出**: `pdf-parse` 等で PDF からテキストを抽出し、発言者・発言内容をパース
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページのリンク数を前回取得時と比較し、新規追加分のみを処理する
