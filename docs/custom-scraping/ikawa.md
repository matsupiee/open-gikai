## PDF 内部構造

議会だよりは約8〜10ページの広報紙形式 PDF。会議録の全文ではなく、要約・抜粋で構成される。

### 表紙（1ページ目）

- 号数（例: No.163）、発行日（例: 2025.7.1）
- 定例会の月、会期
- 目次（定例会、常任委員会、一般質問、条例改正等のページ番号）

### 定例会報告（2〜3ページ目）

- 会期情報（例: 「第2回定例会 会期 6月10日〜13日」）
- 一般会計補正予算の概要（歳入・歳出の主な内容）
- 議案一覧と審議結果（議案番号、補正内容、賛否）
- 臨時会・全員協議会の報告が含まれる場合もある

### 常任委員会審査（4ページ目）

- 総務産業常任委員会、教育民生常任委員会
- 「問」「答」形式で質疑応答の要旨を掲載
- 委員会の写真付き

### 一般質問（5〜8ページ目）

- 冒頭に質問者一覧（議員名、質問方式、質問項目）
- 各議員の質問を1〜2ページで紹介
- 議員の顔写真付き
- 発言者の表記パターン:
  - 議員名（姓名）のみの見出し
  - 「町長」等の役職名で答弁者を表記
- 質問方式: 一括質問方式、一問一答方式

### その他（9〜10ページ目）

- 条例改正・陳情の概要
- 「聞かせて町の声」（住民の声）
- 議会のうごき

## スクレイピング戦略

### Step 1: PDF URL の収集

議会だより一覧ページ `1305.html` から PDF リンクを抽出する。

**収集方法:**

1. `https://www.town.ikawa.akita.jp/site/gikai/1305.html` の HTML を取得
2. `<a>` タグから `/uploaded/attachment/{ID}.pdf` 形式のリンクを Cheerio で抽出
3. リンクテキストから年月情報を正規表現で抽出（`令和\d+年\d+月議会` or `平成\d+年\d+月議会`）

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF は画像ベース（写真・レイアウト重視）の部分と、テキスト埋め込み部分が混在
- UD新ゴシリーズフォントを使用
- `pdf-parse` 等でテキスト抽出を試み、抽出できない場合は OCR が必要
- ファイルサイズが 1.4MB〜15MB と幅がある（画像の多さによる）

### Step 3: テキストのパース

#### メタ情報の抽出

表紙または定例会報告ページから以下を抽出:

```
第2回定例会 会期 6月10日〜13日
```

- 回数: `第{N}回` 形式
- 種別: `定例会` / `臨時会`
- 会期: 日付範囲

#### 一般質問の抽出

一般質問ページから質問者名と質問項目を抽出:

```
{姓} {名} 議員（{質問方式}）
```

#### パース用正規表現（案）

```typescript
// 議会だよりのタイトルから年月を抽出
const titlePattern = /いかわ町議会だより（(.+?)年(\d+)月議会）/;

// 定例会回数の抽出
const sessionPattern = /第(\d+)回\s*(定例会|臨時会)/;

// 会期の抽出
const periodPattern = /(\d+)月(\d+)日[〜～](\d+)日/;

// 議員名の抽出（一般質問目次）
const memberPattern = /^(.+?)\s+議員（(.+?)方式）/;
```

## 注意事項

- 会議録の全文テキストは公開されていないため、議会だより PDF からの要約情報のみが取得可能
- PDF は広報紙レイアウト（段組み・写真・装飾）のため、テキスト抽出の精度が低い可能性がある
- 令和4年以前の PDF はファイルサイズが大きい（10MB超）ものがあり、画像中心の構成と推測される
- 議員定数は11名の小規模議会
- 一般質問は1定例会あたり3名程度

---

## 推奨アプローチ

1. **議会だより一覧ページからの PDF URL 収集を優先**: 単一ページ（1305.html）のみをスクレイピングすれば全 PDF URL を取得可能
2. **PDF テキスト抽出**: `pdf-parse` でテキスト抽出を試み、品質が低い場合は OCR（Tesseract 等）を検討
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: PDF リンク一覧を取得し、既存データベースと照合して未取得分のみダウンロード
5. **データ品質の制約**: 会議録全文ではなく議会だよりの要約情報であることをデータの属性として記録する
