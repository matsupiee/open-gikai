# 八代市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.yatsushiro.kumamoto.jp/VOICES/
- 分類: 独自システム「VOICES/Web」による CGI 公開（既存アダプターでは対応不可）
- 文字コード: Shift_JIS（`charset=Shift_JIS` / `charset=x-sjis`）
- 特記: フレームセット構造。CGI エンドポイント `CGI/voiweb.exe` が全機能を担う。データは昭和62年（1987年）から現在まで、975件以上の日程を収録。

## 検索パラメータ

### ACT（アクション種別）

| ACT | 意味 |
| --- | --- |
| `1` | 簡易検索フォーム |
| `2` | 詳細検索フォーム |
| `100` | 会議日程一覧（GET/POST 両対応） |
| `200` | 会議録詳細フレームセット |
| `201` | 会議録ヘッダーフレーム |
| `202` | 発言一覧サイドバーフレーム |
| `203` | 会議録本文フレーム |
| `204` | サイドバーヘッダーフレーム |

### KTYP（発言種別）

| 値 | 説明 |
| --- | --- |
| `0` | 目次・その他 |
| `1` | 議長 |
| `2` | 質問者 |
| `3` | 答弁者 |

デフォルトは `KTYP=2,3`（質問者・答弁者）。全発言を取得する場合は `KTYP=0,1,2,3` または省略。

### KGTP（会議種別）

| 値 | 説明 |
| --- | --- |
| `1` | 定例会 |
| `2` | 臨時会 |

デフォルトは `KGTP=1,2`。

### SORT（表示順）

| 値 | 説明 |
| --- | --- |
| `0` | 新しい会議日程から（降順） |
| `1` | 古い会議日程から（昇順） |

### FINO / KGNO（ファイル・会議番号）

- `FINO`: 会議録ファイルの一意 ID（数値連番）。`FINO=1`（昭和62年3月定例会）〜 `FINO=977`（令和7年9月定例会）程度が確認済み
- `KGNO`: 会議セッション番号。`KGNO=1` から始まる連番

### PAGE / HIT（ページネーション）

- `PAGE`: ページ番号（1 始まり）
- `HIT`: 総件数（一覧取得時に返却される）
- 1ページあたり約20件を表示

### 詳細検索専用パラメータ（ACT=2 フォーム）

| パラメータ | 説明 |
| --- | --- |
| `FYY` / `TYY` | 開始/終了年（西暦4桁）。1987〜2025 |
| `FMM` / `TMM` | 開始/終了月（01〜12） |
| `FDD` / `TDD` | 開始/終了日（01〜31） |
| `TITL` | 会議名キーワード |
| `FBKEY1`〜`FBKEY4` | 発言キーワード（最大4件） |
| `FBMODE1`〜`FBMODE4` | キーワード検索モード |
| `NAMES1`〜`NAMES4` | 発言者氏名 |
| `YAKUS1`〜`YAKUS4` | 発言者役職 |

## スクレイピング戦略

### Step 1: 全会議録の FINO 収集

ACT=100 で全件を一覧取得し、各エントリの `FINO` と `KGNO` を収集する。

```
GET /VOICES/CGI/voiweb.exe?ACT=100&KTYP=0,1,2,3&KGTP=1,2&SORT=1&PAGE=1
```

- 総件数は最初のページの「N件の日程がヒットしました」から取得
- 1ページ20件。`PAGE` を 1 からインクリメントして全ページを取得
- 各行から `FINO=(\d+)` と `KGNO=(\d+)` を正規表現で抽出
- `TITL_SUBT` パラメータ（URL エンコードされた会議名）もメタ情報として取得可能

**一覧行の HTML 例:**

```html
令和　７年　９月定例会,<A HREF="voiweb.exe?ACT=200&...&KGNO=220&FINO=973" TARGET="HLD_WIN">10月03日-01号</A>
```

- 会議名: `令和　７年　９月定例会`（全角スペース区切り）
- 開催日: `10月03日-01号`（号数は同一定例会内の開催回）

### Step 2: 会議録本文の取得

`ACT=203` を直接呼び出して本文 HTML を取得する（フレームセット経由不要）。

```
GET /VOICES/CGI/voiweb.exe?ACT=203&FINO={FINO}&KENSAKU=0&SORT=0&KTYP=0,1,2,3&KGTP=1,2&HATSUGENMODE=1&HYOUJIMODE=0&STYLE=0
```

- レスポンスは Shift_JIS
- 発言ブロックは `<A NAME="HUID{数値}"></A>` アンカーで区切られる
- 各発言はテキストノードとして `<BR>` タグで改行されている

### Step 3: 会議録本文のパース

#### メタ情報

ページ冒頭の `<A NAME="HUID{最初の番号}">` 直後のテキストに以下が含まれる:

```
令和　７年　９月定例会

　　　　　　　令和７年９月八代市議会定例会会議録（第１号）

・令和７年１０月３日（金曜日）
```

- 会議名: `令和X年X月定例会` / `令和X年X月臨時会`（全角スペースあり）
- 開催日: `令和X年X月X日（曜日）` 形式
- 旧市名義のデータには `旧八代市` が含まれる（昭和62年〜平成17年合併前）

#### 発言の構造

発言者の記号と形式:

```
○議長（高山正夫君）　これより令和７年９月定例会を開会いたします。
◎副市長（平井宏英君）　お許しをいただきましたので...
◆木村博幸君　皆様、こんにちは。
```

| 記号 | 意味 |
| --- | --- |
| `○` | 議長・一般発言者 |
| `◎` | 答弁者（行政側） |
| `◆` | 質問者（議員） |

発言者名の形式:
- 役職あり: `役職名（氏名君）`（例: `議長（高山正夫君）`、`市長（小野泰輔君）`）
- 役職なし（議員の質問等）: `（氏名君）` または `氏名君` のみ
- 発言記号の後にスペース（全角）を挟んで本文が続く

非発言ブロック（議事日程等）:
- `（名簿）`、`（議題）` のようなラベルが `<B>` タグで表示される

#### パース用正規表現（案）

```typescript
// 発言者行の判定（○◎◆で始まる）
const speakerLinePattern = /^[○◎◆](.+?)[\u3000\s]/u;

// 役職あり発言者の抽出
const speakerWithRolePattern = /^[○◎◆](.+?)（(.+?)君）/u;
// 例: ○議長（高山正夫君） → role="議長", name="高山正夫"
// 例: ◎理事兼水道局長（吉永哲也君） → role="理事兼水道局長", name="吉永哲也"

// 役職なし発言者（議員名のみ）
const speakerOnlyPattern = /^◆(.+?)君/u;
// 例: ◆木村博幸君 → name="木村博幸"

// 開催日の抽出
const datePattern = /・(?:令和|平成|昭和)(\d+)年(\d+)月(\d+)日（(.+?)）/;

// HUID アンカーで発言ブロックを分割
const huidPattern = /<A NAME="HUID(\d+)"><\/A>/g;
```

## 注意事項

- **文字コード**: 全レスポンスが Shift_JIS（`charset=Shift_JIS` または `charset=x-sjis`）。iconv や TextDecoder での変換が必要
- **フレームセット**: トップページ・閲覧ページはすべてフレームセット構造。直接 CGI エンドポイントにアクセスすること
- **旧市名義のデータ**: 平成17年（2005年）の合併前データには `旧八代市` や `旧坂本村` 等の表記が含まれる
- **全角スペース**: 発言者と本文の区切りに全角スペース（`\u3000`）が使われる
- **`君` 敬称**: 全発言者の氏名末尾に `君` が付く慣習がある（`高山正夫君`）
- **システム起動年**: SINCE 1999/11/17 の表示があるが、データ自体は昭和62年（1987年）から収録

## 推奨アプローチ

1. **FINO 連番を活用した全量取得**: `ACT=100&SORT=1` でページネーションを辿り FINO の全リストを取得してから、`ACT=203` で本文を順次取得する
2. **差分更新**: FINO は数値連番のため、前回取得済みの最大 FINO 以降のみを対象にした差分更新が容易
3. **ACT=203 を直接呼び出す**: フレームセット（ACT=200）は不要。本文フレーム（ACT=203）を直接取得することでパース負荷を軽減できる
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
5. **Shift_JIS デコード**: Node.js では `iconv-lite` や `TextDecoder('shift_jis')` を使用する
