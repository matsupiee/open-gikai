## 年度別ページの構造

各年度ページには、会議種別ごとにセクションが設けられており、それぞれの本会議の開催日ごとに PDF リンクが箇条書きで列挙される。

### 会議種別

- 定例会（3月、6月、9月、12月）
- 臨時会（随時）

### エントリーのテキスト形式

```
[会議種別セクション見出し]
  令和X年X月X日定例会 本会議
  令和X年X月X日 (PDFファイル: 498.6KB)
  令和X年X月X日 (PDFファイル: ...)
  ...
```

具体例（令和6年度）：

- 令和7年3月定例会 本会議 → 3日・10日・24日・26日の4件
- 令和7年1月臨時会 本会議 → 27日の1件
- 令和6年12月定例会 本会議 → 4日・11日・18日・20日の4件
- 令和6年10月臨時会 本会議 → 23日の1件
- 令和6年9月定例会 本会議 → 3日・10日・19日・24日の4件
- 令和6年8月臨時会 本会議 → 19日の1件
- 令和6年6月定例会 本会議 → 7日・13日・24日・26日の4件
- 令和6年5月臨時会 本会議 → 16日の1件

## スクレイピング戦略

### Step 1: 年度別ページの URL を収集

インデックスページ（`456.html`）を取得し、年度別ページへのリンクを Cheerio で抽出する。

- `.pageLink` 要素内の `<a>` タグの `href` を収集
- リンクテキストが `【令和X年度】会議録` / `【平成X年度】会議録` に一致するものを対象とする

**ページネーションなし**のため、1リクエストで全年度のリンクが取得できる。

### Step 2: 各年度ページから PDF リンクを収集

各年度ページを取得し、PDF へのリンクを Cheerio で抽出する。

- `a[href$=".pdf"]` で PDF リンクを抽出
- リンクテキストから開催日（`令和X年X月X日`）を取得
- リンク直前のセクション見出しテキストから会議種別を判定

### Step 3: メタ情報の構築

PDF ファイル名（日付コード）からメタ情報を補完できる。

```typescript
// 日付コードの解析例（R070605 → 令和7年6月5日）
const dateCodePattern = /^R(\d{2})(\d{2})(\d{2})\.pdf$/;
// グループ1: 和暦年(07=令和7年), グループ2: 月(06), グループ3: 日(05)
```

- 会議録テキスト（HTML）は存在せず、**PDF のみ**が提供される
- PDF の内容解析には別途 PDF パーサーが必要

---

## 注意事項

- **テキスト形式の会議録は存在しない**: すべて PDF 形式のみの提供
- **委員会の会議録は存在しない**: 本会議（定例会・臨時会）のみが公開対象
- **差分更新の方針**: 年度別ページIDは連番ではなく不規則なため、インデックスページの変更検知（更新日の比較など）によって新規年度の追加を検出する
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- **PDF URL のプロトコル**: ページ内ではプロトコルなし（`//www.town...`）の形式で記述されていることがあるため、`https:` を補完する

---

## 推奨アプローチ

1. **インデックスページを起点とする**: 年度別URLは固定ではなく随時追加されるため、インデックスページを定期クロールして新年度ページを検出する
2. **年度単位で差分更新**: 年度別ページ内の全PDFリンクを取得し、未収録のものだけを処理する
3. **PDF解析は別ジョブで**: PDFのダウンロードとテキスト抽出は、リンク収集とは分離して非同期で処理する
