# 玉東町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.gyokuto.kumamoto.jp/list00507.html
- 分類: 町公式サイト上に PDF ファイルとして会議録を掲載（既存アダプターでは対応不可）

- 特記: 会議録は年度ごとのページに分かれており、各ページから PDF を直接ダウンロードする形式

## ページ構造

### 一覧ページ（list00507.html）

- 年度別会議録ページへのリンクが「全N件」のリスト形式で掲載
- JavaScript による「もっと見る」（autopager）を使用した動的読み込みに対応
- 各リンクには更新日が表示される

### 年度別会議録ページ（kiji{ID}/index.html）

- 会議種別ごとに `<h4>` 見出しで区分
- 各会議の下に `<ul>/<li>` で目次 PDF と日付別 PDF がリスト化
- PDF リンクは `<a>` タグで直接記述

| --- | --- | --- |

## PDF のファイル命名規則

```
3_{kijiID}_{連番}_up_{ランダム8文字}.pdf
```

- `kijiID`: 年度別ページの ID（例: `1559`, `1376`, `1229`）
- `連番`: 数字 4〜5 桁（例: `4054`, `3086`）。連続していない場合もある
- `ランダム8文字`: アップロード時に生成されるランダム文字列

連番やランダム文字列は予測不可能なため、HTML を解析して PDF リンクを収集する必要がある。

---

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

一覧ページ `list00507.html` を取得し、`kiji{ID}/index.html` 形式のリンクを抽出する。

- リスト全件を取得するため、JavaScript による動的読み込みに注意
  - autopager を使用しているが、全3件程度のため初期表示で全件取得できる可能性が高い
  - 取得できない場合は Playwright 等のブラウザ自動化を検討
- 各リンクに付随する更新日をメタ情報として記録しておくと差分更新に活用できる

### Step 2: 年度別ページから PDF リンクの収集

各 `kiji{ID}/index.html` を取得し、PDF リンクと会議メタ情報を抽出する。

**抽出する情報:**

- 会議種別（`<h4>` テキスト、例: `第1回(3月)定例会`）
- 会議日付（`<li>` テキスト、例: `3月6日`）
- PDF URL（`<a href="...">` の href 属性）
- PDF 種別（目次か本文か）

**HTML 解析方法（Cheerio 使用例）:**

```typescript
// 会議種別ごとに h4 でセクションを分割
$('h4').each((_, h4) => {
  const sessionTitle = $(h4).text().trim();
  // 例: "第1回(3月)定例会"

  // h4 直後の ul から PDF リンクを抽出
  $(h4).next('ul').find('a[href$=".pdf"]').each((_, a) => {
    const pdfUrl = new URL($(a).attr('href')!, baseUrl).toString();
    const label = $(a).text().trim();
    // 例: "目次" / "3月6日"
  });
});
```

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL から PDF をダウンロードし、テキストを抽出する。

- 目次 PDF は本文テキストの抽出対象外とし、会議構成の参照用に使用
- 本文 PDF は日付ごとに 1 ファイルが対応
- PDF テキスト抽出には `pdf-parse` 等のライブラリを使用

---

## 注意事項

- PDF の URL パターンにランダム文字列が含まれるため、URL の予測による直接アクセスは不可能
- 必ず HTML を解析して PDF リンクを動的に収集する
- 一覧ページの件数が少ない（現時点で3件）ため、ページネーション処理は不要と思われるが、今後追加される可能性を考慮して全件取得の実装を推奨
- 目次 PDF は各会議の先頭に掲載されており、本文 PDF と同一 `<ul>` 内にリストされている
- 令和5年のページでは第1回（3月）定例会が掲載されていない年があるなど、年度によって会議数が異なる

---

## 推奨アプローチ

1. **一覧ページから年度 URL を収集**: `list00507.html` をパースして `kiji{ID}/index.html` リンクを全件取得
2. **年度別ページから PDF リンクを収集**: Cheerio で `<h4>` セクション単位に会議メタ情報と PDF URL を抽出
3. **目次と本文を区別**: リンクテキストが「目次」であれば目次 PDF として分類
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページの更新日をキャッシュし、前回取得時から更新のあった年度ページのみ再クロールする
