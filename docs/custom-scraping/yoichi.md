# 余市町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.yoichi.hokkaido.jp/gikai/kaigiroku/
- 分類: 公式ウェブサイト上で PDF 形式の会議録を年度別・会議種別で公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: JustSystems Homepage Builder で作成された HTML 4.01 Transitional ページ

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（トップ） | `https://www.town.yoichi.hokkaido.jp/gikai/kaigiroku/` |
| 会議録 PDF（定例会） | `https://www.town.yoichi.hokkaido.jp/gikai/kaigiroku/files/R{和暦年}.{回数}tei{日程番号}.pdf` |
| 会議録 PDF（臨時会） | `https://www.town.yoichi.hokkaido.jp/gikai/kaigiroku/files/R{和暦年}.{回数}rin.pdf` |

---

## PDF ファイル命名規則

### 令和 5 年以降（統一形式）

- **定例会**: `R{和暦年}.{回数}tei{日程番号}.pdf`
  - 例: `R7.3tei1.pdf` = 令和7年第3回定例会 第1日目
  - 例: `R6.4tei1.pdf` = 令和6年第4回定例会 第1日目
- **臨時会**: `R{和暦年}.{回数}rin.pdf`
  - 例: `R7.5rin.pdf` = 令和7年第5回臨時会
  - 例: `R6.5rin.pdf` = 令和6年第5回臨時会

### 令和 5 年（一部旧形式）

- 定例会: `{回数}tei.R05.{月}.{日}.pdf`
  - 例: `4tei.R05.12.12.pdf` = 令和5年第4回定例会 12月12日

### 令和 4 年（旧形式）

- 定例会: `kaigiroku{回数}tei{日程番号}R04.{月}.{日}.pdf`
  - 例: `kaigiroku4tei1R04.12.13.pdf` = 令和4年第4回定例会 12月13日

## ページ構造

### 一覧ページの HTML 構造

- HTML 4.01 Transitional、JustSystems Homepage Builder で生成
- 年度ごとに `【 令和X年 】` の見出し（`<strong>` タグ）で区切り
- 各会議は `〇令和X年第Y回定例会（日付リンク・日付リンク...）` の形式
- PDF リンクは日付テキストに `<a href="files/xxx.pdf">` として埋め込み
- テーブルやリスト構造ではなく、段落ベースのフリーテキスト形式
- ページネーションなし（全年度の会議録を 1 ページに掲載）

### 掲載範囲

- 令和 4 年（2022 年）〜 現在
- 令和 3 年以前のデータは掲載なし

### 会議種別

| 種別 | 説明 |
| --- | --- |
| 定例会 | 年 3〜4 回開催、複数日程（各日程ごとに個別 PDF） |
| 臨時会 | 年 3〜5 回開催、単一日程（1 PDF） |

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `https://www.town.yoichi.hokkaido.jp/gikai/kaigiroku/` から全 PDF リンクを抽出する。

- ページネーションがないため、1 リクエストで全データを取得可能
- `<a href="files/...pdf">` パターンでリンクを抽出
- リンクテキスト（日付）と前後のテキスト（会議名・年度）からメタ情報を推定

**収集方法:**

1. 一覧ページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` セレクタにマッチするリンクを全て抽出
3. 各リンクの周辺テキストから年度・会議種別・開催日を抽出

### Step 2: PDF のダウンロード

収集した PDF URL に対してファイルをダウンロードする。

- ベース URL: `https://www.town.yoichi.hokkaido.jp/gikai/kaigiroku/`
- 相対パス `files/xxx.pdf` を結合してフル URL を生成

### Step 3: PDF からのテキスト抽出

ダウンロードした PDF から会議録本文を抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを抽出
- PDF 内のレイアウトに依存するため、抽出精度の検証が必要

### Step 4: メタ情報の抽出

#### ファイル名からの情報抽出

```typescript
// 新形式（令和5年以降）
const newFormatPattern = /^R(\d+)\.(\d+)(tei|rin)(\d*)\.pdf$/;
// 例: R7.3tei1.pdf → 年=7, 回=3, 種別=定例会, 日程=1

// 旧形式（令和5年一部）
const oldFormat1Pattern = /^(\d+)(tei|rin)\.R(\d+)\.(\d+)\.(\d+)\.pdf$/;
// 例: 4tei.R05.12.12.pdf → 回=4, 種別=定例会, 年=05, 月=12, 日=12

// 旧形式（令和4年）
const oldFormat2Pattern = /^kaigiroku(\d+)(tei|rin)(\d*)R(\d+)\.(\d+)\.(\d+)\.pdf$/;
// 例: kaigiroku4tei1R04.12.13.pdf → 回=4, 種別=定例会, 日程=1, 年=04, 月=12, 日=13
```

#### リンク周辺テキストからの情報抽出

```typescript
// 会議名パターン
const meetingPattern = /〇令和(\d+)年第(\d+)回(定例会|臨時会)/;
// 例: 〇令和７年第３回定例会 → 年=7, 回=3, 種別=定例会

// 日付パターン（リンクテキスト）
const datePattern = /(\d+)月(\d+)日/;
```

---

## 注意事項

- PDF 形式のため、テキスト抽出の精度は PDF の作成方法に依存する（画像 PDF の場合は OCR が必要）
- ファイル命名規則が年度によって異なるため、複数のパターンに対応する必要がある
- 令和 4 年以前のデータはサイトに掲載されていない
- HTML が Homepage Builder で生成されており、セマンティックな構造（h1〜h6 等）が使われていないため、テキストの位置関係でメタ情報を推定する必要がある

---

## 推奨アプローチ

1. **1 ページ完結**: 一覧ページが 1 ページで完結しているため、クロール処理はシンプルに実装可能
2. **PDF パース**: HTML ではなく PDF からのテキスト抽出が主要な課題。まず数件の PDF をダウンロードしてテキスト抽出の品質を検証する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの PDF リンク数を前回取得時と比較し、新規リンクのみをダウンロードする差分更新が可能
5. **ファイル名パターンの柔軟対応**: 年度によって命名規則が異なるため、複数パターンのパーサーを用意する
