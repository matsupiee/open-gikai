## PDF ファイル命名規則

ファイル名の命名規則は年度によって異なり、統一されていない。

### 令和4年以降のパターン（比較的統一）

```
R{和暦年}-{回}tei-{日目}.pdf
```

例:
- `R7-1tei-2.pdf` → 令和7年第1回定例会 2日目
- `R6-3tei-3.pdf` → 令和6年第3回定例会 3日目
- `R6-3tei_2.pdf` → 令和6年第3回定例会 2日目（ハイフンではなくアンダースコアの場合あり）

### 令和3年以前のパターン（不統一）

```
R3{MMDD}kennsaku.pdf    （例: R30309kennsaku.pdf）
R3-{回}tei{日目}.pdf    （例: R3-2tei2.pdf）
R3yosann{日目}.pdf      （例: R3yosann2.pdf — 予算審査）
R0{和暦年}{MMDD}kessann.pdf  （例: R031111kessann.pdf — 決算審査）
```

**注意**: 命名規則が統一されていないため、PDF の URL はファイル名パターンから推測するのではなく、年度別ページの HTML からリンクを直接抽出する必要がある。

## HTML 構造

### 年度別ページの構造

```
<h2> 会議種別見出し（例: 令和7年第1回定例会大綱質疑）
  <h3> 日付（例: 令和7年3月4日）
    <a href="...pdf"> PDF リンク（ファイルサイズ表記付き）
    <h4> 議員名（会派名）
      <ol> 質問項目リスト（ページ番号付き）
```

- `<h2>`: 会議種別の大見出し
- `<h3>`: 開催日ごとのセクション（PDF リンクを含む）
- `<h4>`: 各議員の発言セクション
- `<ol>`: 質問項目一覧

### PDF リンクの抽出

PDF リンクは `<a>` タグの `href` 属性に格納されている。

```
//www.city.shibetsu.lg.jp/material/files/group/36/{ファイル名}.pdf
```

- プロトコルスキーム省略形式（`//` で始まる）で記載されている
- すべての PDF は `/material/files/group/36/` 配下に格納

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

検索トップページ（`index.html`）から年度別ページへのリンクを抽出する。

- `<a>` タグの `href` から `/kaigirokukennsaku/{ページID}.html` パターンを収集
- `889.html`（説明ページ）は除外する

### Step 2: PDF リンクの収集

各年度別ページの HTML をパースし、PDF リンクとメタ情報を抽出する。

**抽出対象:**

| 項目 | 抽出元 |
| --- | --- |
| 会議種別 | `<h2>` テキスト |
| 開催日 | `<h3>` テキスト |
| PDF URL | `<a href="...pdf">` |
| 質問議員名 | `<h4>` テキスト |
| 質問項目 | `<ol><li>` テキスト |

**抽出方法:**

1. 各年度ページの HTML を Cheerio でパース
2. `<a>` タグから `.pdf` で終わる `href` を抽出
3. 直前の `<h2>` から会議種別、`<h3>` から開催日を取得
4. `<h4>` と `<ol>` から議員名・質問項目を取得

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 発言者ごとにテキストを分割・構造化

### Step 4: 発言データの構造化

PDF から抽出したテキストを発言単位に分割する。

- HTML ページの `<h4>` から得た議員名・会派情報を PDF 内テキストと照合
- 質問項目リスト（`<ol>`）をもとに発言内容を紐付け

## ページネーション

## 注意事項

- **PDF 形式**: 会議録本文は HTML ではなく PDF で公開されているため、テキスト抽出に PDF パーサーが必要
- **命名規則の不統一**: PDF ファイル名の命名規則が年度・会議種別によって異なるため、HTML からのリンク抽出が必須
- **プロトコルスキーム省略**: PDF リンクが `//` で始まるため、`https:` を先頭に付与する必要がある
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- **差分更新**: 年度別ページの URL は固定のため、最新年度のページのみを定期的にチェックすることで差分更新が可能
- **委員会の会議録**: 年度によって掲載範囲が異なる可能性がある（令和3年は委員会あり、令和6年は定例会のみ確認）

---

## 推奨アプローチ

1. **HTML からのリンク抽出を優先**: PDF ファイル名のパターン推測ではなく、年度別ページの HTML から確実にリンクを収集する
2. **PDF テキスト抽出**: pdf-parse 等を使用して PDF からテキストを抽出し、発言者・発言内容を構造化する
3. **メタ情報の補完**: PDF 内のテキストだけでなく、HTML ページの構造化情報（議員名、会派、質問項目）を活用して精度を高める
4. **段階的な実装**: まずは定例会（大綱質疑・一般質問）を対象にし、委員会は後から追加する
