# 座間味村議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.vill.zamami.okinawa.jp/info/kaigiroku.html
- 分類: 単一 HTML ページに PDF リンクを一覧掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 公開形式: PDF
- 対象期間: 平成19年（2007年）〜 令和6年（2024年）

| ページ | URL パターン |

| PDF ファイル（新形式） | `https://www.vill.zamami.okinawa.jp/info/{40桁ハッシュ}.pdf` |

### PDF URL パターンの詳細

- **新形式**（主に平成26年以降）: ファイル名が40文字の16進ハッシュ値
  - 例: `d96f9e12a9c9e54b48c141887748094ced5c735d.pdf`
  - 例: `bfab1f92ceb403efb8aa49dc92aac48b9468998a.pdf`
- **旧形式**（主に平成25年以前）: `YYMMDD` + `gijiroku.pdf`
  - 例: `250308gijiroku.pdf`（平成25年3月8日）
  - 例: `240307gijiroku.pdf`（平成24年3月7日）
- **旧形式・別パス**: `/politics/` 配下に格納されているケースもある

## スクレイピング戦略

### Step 1: 一覧ページの取得

`https://www.vill.zamami.okinawa.jp/info/kaigiroku.html` を取得する。

- 単一ページにすべての会議録リンクが掲載されているため、ページネーション処理は不要
- 1回の HTTP リクエストで全データのリンクを取得可能

### Step 2: PDF リンクの抽出

HTML をパースし、年度・会期ごとに PDF リンクを収集する。

**抽出方法:**

1. 年度見出し（h4 相当）を走査し、年度を特定
2. 各年度配下の会期見出し（strong/太字）から会期名を抽出
3. 各会期配下の `<a>` タグから PDF の URL とリンクテキスト（開催日・号数）を抽出
4. リンクテキストから開催日・号数・一般質問の有無をパース

**メタ情報の抽出用正規表現（案）:**

```typescript
// リンクテキストから開催日を抽出
const datePattern = /(?:令和|平成)(\S+?)年(\d+)月(\d+)日/;
// 例: 「令和7年12月16日（第1号）」 → 年="7", 月="12", 日="16"

// 号数の抽出
const issuePattern = /第(\d+)号/;
// 例: 「（第1号）」 → "1"

// 会期見出しから会期情報を抽出
const sessionPattern = /(?:令和|平成)\S+?年第(\d+)回(定例会|臨時会)/;
// 例: 「令和7年第4回定例会」 → 回="4", 種別="定例会"

// 一般質問の検出
const generalQuestionPattern = /一般質問/;
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 抽出した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容をパース

**PDF テキストのパース（想定）:**

座間味村は小規模自治体のため、PDF の構造は比較的シンプルと想定される。テキスト抽出後の発言者パターンは PDF の内容を実際に確認して決定する必要がある。

---

## 注意事項

- すべての会議録は PDF 形式で公開されており、HTML 本文は存在しない
- PDF のテキスト抽出精度は PDF の作成方法（テキスト埋め込み or 画像スキャン）に依存する
- 旧形式の PDF（平成25年以前）は画像スキャンの可能性があり、OCR が必要になる場合がある
- 平成18年以前の会議録は問い合わせ対応のため、スクレイピング対象外
- `/info/` と `/politics/` の2つのパスに PDF が分散しているため、両方のパターンに対応する必要がある

---

## 推奨アプローチ

1. **単一ページ取得**: 一覧ページ1回の取得で全リンクを収集できるため、クロール負荷は最小限
2. **PDF テキスト確認を先行**: 実装前に数件の PDF をダウンロードし、テキスト抽出可能か（画像スキャンでないか）を確認する
3. **レート制限**: PDF ダウンロード時はリクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの HTML を前回取得分と比較し、新規追加された PDF リンクのみをダウンロードする
5. **旧形式 PDF の優先度を下げる**: テキスト抽出が困難な場合は、新形式（平成26年以降）を優先的に対応する
