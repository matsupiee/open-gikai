# 横浜町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.yokohama.lg.jp/index.cfm/10,0,47,html
- 分類: 町公式サイト（ColdFusion CMS）の単一ページに PDF リンクを直接掲載
- 文字コード: UTF-8
- 特記: 会議録をPDFファイルとして直接提供。定例会（3月、6月、9月、12月）と臨時会の記録がダウンロード可能。年度ごとにページが分かれておらず、1ページに当該年の全会議録が掲載される。会議録検索システムは未導入。

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（令和7年） | `https://www.town.yokohama.lg.jp/index.cfm/10,0,47,html` |
| PDF ファイル | `https://www.town.yokohama.lg.jp/index.cfm/10,12437,c,html/12437/{filename}.pdf` |

- URL パス `/index.cfm/10,0,47,html` のパラメータ: `10` = グループID（議会）、`47` = カテゴリID（会議録）
- ContentID `12437` が現在の会議録ページに対応
- 過去年度の会議録が同一ページに追加されるか、別の ContentID で管理されるかは不明（現時点では令和7年分のみ確認）

## HTML 構造

### 会議録一覧ページ

```
div#Center > div#CenterArea
  div.content > section
    h2.titleOfContent: "令和7年会議録"
    div.contentBodyBox > div.contentBody
      h6.content_p_01: "第2回臨時会（12月19日（金））"
      p > a[href="/index.cfm/10,12437,c,html/12437/{filename}.pdf"]: "１２月１９日（金）　本会議１号（議事日程）"
      p > a[href="..."]: "１２月１９日（金）　本会議１号（開会、提案理由、議案審議）"

      h6.content_p_01: "第4回定例会（12月1日（月）から12月2日（火））"
      p > a[href="..."]: "１２月１日（月）　本会議１号（議事日程）"
      p > a[href="..."]: "１２月１日（月）　本会議１号（開会、提案理由、委員長報告、請願審議）"
      p > a[href="..."]: "１２月２日（火）　本会議２号（議事日程）"
      p > a[href="..."]: "１２月２日（火）　本会議２号（一般質問、議案審議）"
      ...
```

### 会議の区切り

- `<h6>` タグ（class は `content_p_01` またはなし）が会議の区切りを示す
- h6 テキスト例:
  - `"第2回臨時会（12月19日（金））"`
  - `"第4回定例会（12月1日（月）から12月2日（火））"`
  - `"第３回定例会（９月５日（金）から９月１０日（水））"`
  - `"第１回定例会（3月3日（月）から3月6日（木））"`

### PDF リンクの構造

- 各 `<h6>` の後に `<p>` タグが続き、その中の `<a>` タグが PDF へのリンク
- リンクテキスト例:
  - `"１２月１日（月）　本会議１号（議事日程）"`
  - `"１２月１日（月）　本会議１号（開会、提案理由、委員長報告、請願審議）"`
  - `"９月９日（火）　決算審査特別委員会１号（決算審議）"`
- 全角・半角が混在している（日付やカッコの表記が統一されていない）

### PDF ファイル名の命名規則

ファイル名はタイムスタンプ形式で統一:
- `{YYYYMMDD}-{HHMMSS}.pdf`（例: `20260311-132035.pdf`, `20250606-085155.pdf`）

### 会議録の種類

各定例会・臨時会に対して、以下の種類の PDF が掲載される:

- **議事日程**: 本会議の議事日程
- **本会議**: 開会、提案理由、委員長報告、請願審議、一般質問、議案審議 など
- **予算審査特別委員会**: 議事日程、予算審議（3月定例会）
- **決算審査特別委員会**: 議事日程、決算審議（9月定例会）

すべての PDF がスクレイピング対象。

## ページネーション

なし。会議録は1ページにすべて掲載される。

ただし、年が変わると別ページ（別 ContentID）になる可能性がある。その場合はカテゴリページ `/index.cfm/10,0,47,html` 内に複数のコンテンツへのリンクが表示される構造になると推測される。

## スクレイピング方針

### 1. 会議録ページの取得

会議録カテゴリページ (`/index.cfm/10,0,47,html`) にアクセスし、`div.contentBody` 内のコンテンツを取得する。

### 2. 会議情報の解析

`<h6>` タグから会議の区切りを検出し、以下を抽出:
- 会議種別: 「定例会」または「臨時会」
- 回次: 「第{N}回」
- 開催期間: カッコ内の日付情報

### 3. PDF リンクの収集

各 `<h6>` の後続する `<p>` 内の `<a>` タグから PDF の URL を取得する。

- href 属性から PDF URL を構築（相対パスの場合はベース URL を付与）
- リンクテキストから日付と内容（議事日程、本会議、委員会等）を抽出

### 4. PDF のダウンロードとテキスト抽出

- PDF URL パターン: `/index.cfm/10,12437,c,html/12437/{filename}.pdf`
- ベース URL: `https://www.town.yokohama.lg.jp`
- PDF をダウンロードし、テキスト抽出を行う

### 5. メタデータの構成

`<h6>` テキストとリンクテキストから以下を解析:
- 会議種別: 「定例会」「臨時会」
- 回次: 「第{N}回」
- 開催日: リンクテキスト冒頭の日付（例: 「１２月１日（月）」）
- 内容区分: リンクテキスト末尾のカッコ内（例: 「議事日程」「一般質問、議案審議」）

### 注意事項

- 全角・半角の数字やカッコが混在しているため、日付解析時に正規化が必要
- `<h6>` タグの class 属性が `content_p_01` の場合と属性なしの場合があるため、タグ名で判定する
- ColdFusion CMS の URL 構造（`/index.cfm/10,12437,c,html/12437/...`）はリダイレクトなしで直接アクセス可能
- 過去年度のデータが別ページになる場合、カテゴリページから各年度ページへのリンクを辿る必要がある
