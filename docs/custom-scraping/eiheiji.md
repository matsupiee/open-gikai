# 永平寺町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.eiheiji-gikai.jp/
- 会議録ページ: https://www.eiheiji-gikai.jp/minutes
- 分類: 独自議会ホームページによる PDF 公開（HTML 形式の会議録は存在しない）
- 文字コード: UTF-8
- 特記: Google Analytics（G-FMS07L6XKE）使用

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.eiheiji-gikai.jp/minutes` |
| 会議録詳細（PDF 一覧） | `https://www.eiheiji-gikai.jp/assembly/minutes/{会議ID}` |
| PDF ファイル | `https://www.eiheiji-gikai.jp/files/admin/{会議ID}/{ハッシュ}.pdf` |

### 会議 ID の命名規則

| 会議種別 | パターン | 例 |
| --- | --- | --- |
| 定例会（近年） | `teirei{年度2桁}{月2桁}` | `teirei0612`（令和6年12月）|
| 定例会（連番付き） | `teirei{年度2桁}{月2桁}-{N}` | `teirei07012-2`（令和7年12月）|
| 臨時会 | `rinnji{年度2桁}{月2桁}` | `rinnji0801`（令和8年1月）|
| 臨時会（旧表記） | `teirei{年度2桁}{月2桁}rinji` | `teirei2905rinji` |
| 旧形式（平成23年） | `gijiroku{年度2桁}{月2桁}` | `gijiroku2309` |

※ 年度は元号ベース（令和7年 → `07`、平成28年 → `28`）

## 会議録一覧ページの構造

一覧ページ（`/minutes`）は **ページネーションなし** で、全 86 件（平成23年〜令和8年）が単一ページに表示される。

HTML 構造:

```html
<ul>
  <li><a href="/assembly/minutes/rinnji0801">令和8年1月臨時会</a></li>
  <li><a href="/assembly/minutes/teirei07012-2">令和7年12月定例会</a></li>
  ...
</ul>
```

---

## 会議録詳細ページの構造

各会議録詳細ページには、その会議の PDF ファイルが複数掲載される。

- **臨時会**: 表紙・目次・本文の計 3 ファイル程度
- **定例会**: 表紙・目次・各開催日の本文（複数日分）の計 6〜7 ファイル程度

PDF ファイルのリンクテキストから開催日が判別できる（例: `R071202` → 令和7年12月2日）。

## PDF 会議録の内容構造

### メタ情報（冒頭部）

```
令和８年第１回永平寺町議会臨時会議事日程
（１日目）
令和８年１月２１日（水）
午前１１時００分　開　議
```

- 会議名: `令和X年第N回永平寺町議会{定例会|臨時会}議事日程`
- 開催日: `令和X年X月X日（曜日）`
- 開始時刻: `午前/午後XX時XX分　開　議`

### 出席者リスト

```
出席議員（１２名）
　１番　中　村　勘太郎　君
　２番　長　岡　千惠子　君
  ...
欠席議員（２名）
  ...

　副　　町　　長　和　田　真　生　君
　教　　育　　長　竹　内　康　高　君
```

### 発言者パターン

発言者行は `○` で始まり、役職と括弧内に氏名（`君` 付き）が続く:

```
○議長（酒井圭治君）
○町長（河合永充君）
○副町長（和田真生君）
○教育長（竹内康高君）
○総務課長（多田和憲君）
○９番（滝波登喜男君）
○７番（森山　充君）
```

- 議員は議席番号で表記（氏名なし）
- 執行部職員は役職名で表記
- 議長・副議長も役職名で表記

## スクレイピング戦略

### Step 1: 会議 ID の収集

`/minutes` ページから `<a href="/assembly/minutes/{ID}">` の全リンクを抽出する。

- ページネーションなし → 1 回のリクエストで全件取得可能
- 現在 86 件（平成23年〜令和8年）

### Step 2: PDF URL の収集

各詳細ページ `/assembly/minutes/{ID}` にアクセスし、`/files/admin/{ID}/` 配下の PDF リンクを全て抽出する。

- ファイル名やリンクテキストから開催日（`R071202` 等）を読み取る
- 表紙・目次 PDF はスキップし、本文 PDF のみを対象とする

### Step 3: PDF のダウンロードとテキスト抽出

PDF を直接ダウンロードし、`pdftotext` 等のライブラリでテキストを抽出する。

- `-layout` オプション相当で空白を保持するとレイアウト把握が容易
- 文字が分割されている場合（例: `河　合　永　充`）は結合処理が必要

### Step 4: 会議録のパース

#### 発言者の抽出

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○９番（滝波登喜男君） → role="９番", name="滝波登喜男"
// 例: ○議長（酒井圭治君）   → role="議長", name="酒井圭治"
// 例: ○総務課長（多田和憲君） → role="総務課長", name="多田和憲"
```

#### 開催日の抽出

```typescript
// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;
```

#### 進行マーカーの検出

```typescript
// 休憩・再開・開閉会の検出
const sessionPattern = /（(?:午前|午後)\d+時\d+分\s*(?:開会|閉会|休憩|再開)）/;
```

---

## 注意事項

- PDF 内の氏名は文字間にスペース（`　`）が入ることがある（例: `多 田 和 憲`）→ 正規化が必要
- 議席番号は全角数字（`１番`、`１０番` 等）
- 会議 ID に命名の揺れがあり（`rinnji` と `teirei*rinji` が混在）、一覧ページからの収集が確実
- 令和5年以降の一部 ID には連番サフィックス（`-1`、`-2`）が付くケースがある

---

## 推奨アプローチ

1. **一覧から全 ID を取得**: `/minutes` ページを 1 回スクレイピングして全 86 件の会議 ID を収集
2. **詳細ページから PDF URL を取得**: 各詳細ページで本文 PDF の URL を収集（表紙・目次はスキップ）
3. **PDF をダウンロードしてテキスト抽出**: `pdf-parse` や `pdftotext` を使用してテキスト化
4. **文字正規化**: 全角スペースで分割された氏名を結合し、全角数字を含む議席番号を正規化
5. **レート制限**: リクエスト間に適切な待機時間（1〜2 秒）を設ける
