# 夕張市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.yubari.lg.jp/site/gikai/
- 分類: 自治体公式サイト CMS による PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF 形式で年度別に公開。年度ごとの一覧ページからPDFファイルへ直接リンク。

## ページ構造

### 年度別ページの HTML 構造

- `h3` 見出しで会議種別（定例市議会、臨時市議会、行政常任委員会など）を分類
- 各見出し下に `<ul><li>` のリスト形式で個別会議録を列挙
- 各リストアイテム内に PDF ファイルへの `<a>` リンクが含まれる
- リンクテキストは `第N回定例市議会　M月D日` の形式
- ページネーションなし（全会議録が単一ページに一覧表示）

### PDF ファイルの URL パターン

```
https://www.city.yubari.lg.jp/uploaded/attachment/{数値ID}.pdf
```

- ID は数値の連番だが、年度・会議種別との対応に規則性はない
- Content-Type: `application/pdf`

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録一覧ページ `list13.html` から各年度ページへのリンクを収集する。

**収集方法:**

1. `https://www.city.yubari.lg.jp/site/gikai/list13.html` をフェッチ
2. 会議録セクション内の年度リンク（`/site/gikai/{ID}.html`）を Cheerio で抽出
3. 新しい年度ページが追加される可能性があるため、固定リストではなく動的に取得する

### Step 2: PDF リンクの収集

各年度ページから PDF ファイルへのリンクとメタ情報を抽出する。

**収集方法:**

1. 各年度ページをフェッチ
2. `h3` 見出しから会議種別を判定
3. 各 `<a>` タグから PDF URL（`/uploaded/attachment/{ID}.pdf`）を抽出
4. リンクテキストから会議名・日付を抽出

**リンクテキストのパターン:**

```
第1回定例市議会　3月5日
第2回臨時市議会　5月15日
行政常任委員会　6月1日
予算審査委員会　3月17日
決算審査特別委員会　9月15日
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

**方法:**

1. PDF URL から直接ダウンロード
2. `pdf-parse` 等のライブラリでテキスト抽出
3. 抽出したテキストから発言者・発言内容をパース

### Step 4: メタ情報の構築

リンクテキストと年度ページの情報から以下のメタ情報を構築:

- 開催日: 年度（年度ページから） + 月日（リンクテキストから）
- 会議名: `h3` 見出し + リンクテキストから会議種別を判定
- 自治体: 夕張市

**日付の構築例:**

```typescript
// 年度ページ: 令和6年 → 2024年
// リンクテキスト: "3月7日"
// → 2024-03-07

// 注意: 1月〜3月の会議は年度の翌年に該当する場合がある
// 例: 令和5年度ページの1月20日 → 2024-01-20 ではなく文脈に応じて判断
```

## パース用正規表現（案）

```typescript
// PDF URL の抽出
const pdfLinkPattern = /\/uploaded\/attachment\/(\d+)\.pdf/;

// リンクテキストから会議名と日付を抽出
const meetingPattern = /(.+?)[\s　]+(\d+)月(\d+)日/;
// 例: "第1回定例市議会　3月5日" → name="第1回定例市議会", month=3, day=5

// 年度の抽出（ページタイトルから）
const yearPattern = /(令和|平成)(\d+)年/;
```

---

## 注意事項

- PDF 形式のため、テキスト抽出の精度は PDF の作成方法に依存する
- スキャン PDF（画像ベース）の場合は OCR が必要になる可能性がある
- 年度をまたぐ会議（1月〜3月）の日付処理に注意が必要
- Imperva（CDN/WAF）を使用しているため、過度なリクエストはブロックされる可能性がある
- レート制限: リクエスト間に適切な待機時間（2〜3 秒）を設ける

---

## 推奨アプローチ

1. **一覧ページから動的取得**: `list13.html` から年度ページ URL を動的に収集し、新年度の追加に自動対応
2. **PDF のキャッシュ**: ダウンロード済み PDF はローカルにキャッシュし、再取得を避ける
3. **差分更新**: 最新年度ページのみを再スクレイピングし、過去年度は初回取得後スキップ
4. **レート制限**: 自治体サイトのため、リクエスト間に 2〜3 秒の待機時間を設ける
5. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使用し、テキスト抽出が失敗した場合はログに記録して手動確認
