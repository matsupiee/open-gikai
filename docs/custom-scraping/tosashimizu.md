
## 各詳細ページの構造

各詳細ページ（例: `/kurashi/section/gikai/22293.html`）には、1会議の会議録が複数 PDF に分割されて掲載されている。

### PDF の分割パターン

会議録は通常、以下のように分割されている：

- **目次**: 会議の目次
- **開会日 / 再開日**: 最初の会議日（提案理由説明等）
- **質疑・一般質問**: 各開催日分（複数ファイルに分かれる場合あり）
- **一般質問**: 各開催日分
- **最終日 / 散会日**: 最終会議日
- **付録**: 議員名簿等の付属資料

## スクレイピング戦略

### Step 1: 詳細ページ URL の収集

一覧ページ（042.html）を取得し、リスト内の全リンクを抽出する。

- ページネーションなし、単一ページに全95件が掲載
- CSS セレクタまたは正規表現で `href="/kurashi/section/gikai/\d+\.html"` 形式を抽出
- 旧形式（`/kurashi/section/gikai/0\d+\.html`）も同様に抽出

```typescript
// 詳細ページへのリンク抽出（Cheerio 使用例）
const links = $('a[href*="/kurashi/section/gikai/"]')
  .map((_, el) => $(el).attr('href'))
  .get()
  .filter(href => /\/kurashi\/section\/gikai\/\d+\.html/.test(href));
```

### Step 2: 詳細ページから PDF URL を収集

各詳細ページを取得し、PDF ファイルへのリンクを全件抽出する。

- PDF リンクは `/fs/` で始まる相対パス
- `href` 属性が `.pdf` で終わるリンクを抽出

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]')
  .map((_, el) => ({
    title: $(el).text().trim(),
    url: new URL($(el).attr('href')!, 'https://www.city.tosashimizu.kochi.jp').href,
  }))
  .get();
```

### Step 3: 会議メタ情報の抽出

詳細ページのタイトル・見出しから会議名と年月を取得する。

- ページタイトルまたは `<h1>` / `<h2>` から会議名を取得
- 例: `令和7年定例会6月会議会議録`

```typescript
// 会議名のパース
const titlePattern = /(?:令和|平成)(\d+)年(?:第\d+回)?定例会(\d+)月(?:第\d+回)?会議会議録/;
// または
const oldTitlePattern = /(?:令和|平成)(\d+)年(\d+)月定例会会議録/;
```

### Step 4: PDF の取得とテキスト抽出

取得した PDF URL から PDF をダウンロードし、テキストを抽出する。

## 注意事項

- 会議録は複数 PDF に分割されているため、1会議につき複数ファイルをダウンロードする必要がある
- 平成25年以前は「定例会」形式（年4回）、平成26年以降は「月会議」形式（年複数回）と表記が変わっている
- 「第2回会議」が同月に追加開催される場合がある（9月第2回会議など）
- 旧形式の URL（022.html〜043.html 台）は3桁の連番で、新形式と混在している
- 一覧ページの1件目（令和7年6月）は絶対 URL（`http://`）、他は相対パスで記載されている

## 推奨アプローチ

1. **一覧ページから全詳細 URL を一括収集**: 042.html を1回取得するだけで全95件の URL が揃う
2. **詳細ページを順次クロール**: 各詳細ページから PDF リンクを収集し、会議メタ情報を取得
3. **PDF ダウンロードと統合**: 同一会議の複数 PDF をまとめて1会議分の会議録として扱う
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 一覧ページの最上位（最新）エントリを確認し、未取得の詳細ページのみ処理する
