## ページ構造

### 年度索引ページ（2938.html）

- 平成24年〜令和7年の14年度分が箇条書き（`<ul>/<li>`）で一覧表示
- 各年度が年度別ページへのリンクになっている
- ページネーションなし（単一ページに全年度を掲載）

### 年度別ページ

- 各定例会・臨時会が `<h2>` 見出しで区切られる（例: `<h2>令和6年12月定例会</h2>`）
- 各見出しの下にテーブル（2列構成）が配置される
  - 列1: 月日（曜日）（例: `12月2日（月曜日）`）
  - 列2: 主な内容説明 + PDF リンク
- PDF リンク形式: `<a href="/uploaded/attachment/{ファイルID}.pdf">[内容テキスト [PDFファイル／サイズKB]]</a>`
- 1つの定例会あたり通常5日程度（質疑、一般質問、採決等）、臨時会は1日程度

### 会議種別

- **定例会**: 3月、6月、9月、12月（年4回）
- **臨時会**: 不定期（年1〜2回程度）

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

年度索引ページ `site/council/2938.html` から全年度のリンクを取得する。

```
// 年度別ページリンクの抽出
const yearLinks = $('a[href*="/soshiki/34/"]');
```

- URL は既に上記の表で確定しているが、将来の年度追加に対応するため動的に取得する
- 全14年度分（平成24年〜令和7年）のリンクを収集

### Step 2: PDF リンクの収集

各年度別ページにアクセスし、会議録 PDF へのリンクを抽出する。

```
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]');
```

**収集する情報:**

1. PDF の URL（`/uploaded/attachment/{ファイルID}.pdf`）
2. 所属する定例会・臨時会の名前（直前の `<h2>` 見出しから取得）
3. 開催日（テーブルの月日列から取得）
4. 内容の説明テキスト（リンクテキストから取得）

### Step 3: メタ情報の抽出

`<h2>` 見出しとテーブル構造から以下を抽出:

- 年度: `令和X年` / `平成XX年`
- 会議種別: `定例会` / `臨時会`
- 開催月: `3月` / `6月` / `9月` / `12月` / `第X回`
- 日付: テーブルの月日列から `X月X日` 形式で取得

```typescript
// h2 見出しからのメタ情報抽出
const sessionPattern = /(令和|平成)\d+年(\d+月定例会|第\d+回臨時会)/;

// テーブルの月日列から日付抽出
const datePattern = /(\d+)月(\d+)日/;
```

### Step 4: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、pdftotext 等でテキストを抽出する
- PDF 内部の発言者パターン等は PDF のテキスト抽出後に分析が必要（PDF がバイナリ圧縮されており、事前のパターン確認は困難）

---

## 注意事項

- PDF ファイルは FlateDecode 圧縮されており、フォントが埋め込まれている。テキスト抽出には pdftotext 等の専用ツールが必要
- PDF のファイル ID は連番だが、年度やページとの対応関係は固定されていないため、必ず HTML ページからリンクを辿って取得すること
- 年度別ページの URL パターンが2種類ある（令和5年以降は5桁 ID、令和4年以前は4桁 ID）が、取得方法に影響はない
- リクエスト間には適切な待機時間（1〜2秒）を設けること

---

## 推奨アプローチ

1. **年度索引からの全量取得**: 年度索引ページから全年度のリンクを取得し、各年度ページから PDF リンクを一括収集する
2. **h2 見出しによるメタ情報補完**: PDF リンクの所属する定例会・臨時会を `<h2>` 見出しから特定し、メタ情報として付与する
3. **PDF テキスト抽出**: ダウンロードした PDF から pdftotext 等でテキストを抽出し、発言内容をパースする
4. **差分更新**: 年度索引ページの最終更新日や、取得済み PDF の attachment ID を管理し、新規追加分のみを取得する
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
