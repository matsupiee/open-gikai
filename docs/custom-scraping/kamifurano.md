## ページネーション

- 各一覧ページは 20 件ずつ表示
- ページ番号はクエリパラメータ `dpgndg1` で指定（1 始まり）
- 本会議一覧は 12 ページ、予算・決算特別委員会は各 2 ページ程度
- ページネーションリンクはページ下部に数値リンクとして表示

## スクレイピング戦略

### Step 1: PDF リンクの収集

各一覧ページ（id=152, 153, 154）を全ページ巡回し、PDF の URL を収集する。

**収集方法:**

1. 一覧ページ（`id=152`, `id=153`, `id=154`）にアクセス
2. テーブル内の PDF リンク（`/contents/20down/gikai-ka/.../*.pdf`）を Cheerio で抽出
3. 同時にテーブルのタイトル列から会議名・開催日のメタ情報を取得
4. ページネーション（`dpgndg1=1` 〜 最終ページ）を順に辿り、全件を収集
5. ページネーションの最終ページは、リンクが存在しなくなるか空結果で判定

### Step 2: PDF のダウンロード

収集した URL から PDF ファイルをダウンロードする。

- PDF は 1 ファイルに会議全体の議事録がまとまっている（数百 KB 〜 数 MB）
- 定例会は複数日分が 1 ファイルに結合されている場合がある

### Step 3: PDF からのテキスト抽出

PDF からテキストを抽出し、会議録として構造化する。

#### メタ情報

一覧ページのテーブルから以下を抽出:

- 会議種別: 定例会 / 臨時会 / 予算特別委員会 / 決算特別委員会
- 回次: 第○回
- 開催日: 和暦形式（例: `R07/3/11～14`）
- 年度: 和暦年度

#### PDF テキストの構造（想定）

PDF 内のテキスト構造は PDF ごとに異なる可能性があるが、一般的な議会会議録の形式として以下を想定:

- 冒頭に会議の基本情報（日時、出席議員等）
- 発言者名 + 発言内容の繰り返し
- 議長の進行発言を含む

---

## 注意事項

- 全ての会議録が PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF のレイアウトや文字コードにより、テキスト抽出の品質が変動する可能性がある
- ファイル名の命名規則に微妙な揺れがある（例: `r07_1.pdf` vs `r07_1all.pdf`、`h14_rinji1.pdf` vs `r05_rinji01.pdf` のゼロ埋め有無）
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **一覧ページの全量スキャン**: 3 つの一覧ページ（id=152, 153, 154）を全ページ巡回し、PDF URL の完全なリストを作成
2. **メタ情報は一覧テーブルから取得**: PDF 内部のパースに依存せず、一覧ページのテーブル行からメタ情報（会議名、日付、種別）を抽出
3. **PDF テキスト抽出**: pdf-parse 等のライブラリで PDF からテキストを抽出し、発言者・発言内容を構造化
4. **レート制限**: リクエスト間に 1〜2 秒の待機時間を設ける
5. **差分更新**: 一覧ページ 1 ページ目（最新）のみを確認し、新規追加分のみダウンロードする方式で差分更新が可能
