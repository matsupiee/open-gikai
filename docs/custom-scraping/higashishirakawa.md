## リンクテキストのパターン

### 目次

```
令和○年 第○回 東白川村議会定例会会議録(目次)(PDF版:○○KB)
令和○年 第○回 東白川村議会臨時会会議録(目次)(PDF版:○○KB)
```

### 定例会 本文

```
令和○年 第○回 東白川村議会定例会会議録(第○号 令和○年○月○日)(PDF版:○○KB)
```

### 臨時会 本文

```
令和○年 第○回 東白川村議会臨時会会議録(令和○年○月○日)(PDF版:○○KB)
```

- 定例会は号数あり（第1号、第2号、…）
- 臨時会は号数なし（開催日のみ）

## スクレイピング戦略

### Step 1: 一覧ページの取得

`https://www.vill.higashishirakawa.gifu.jp/sonsei/gikai/kaigiroku/` を 1 回取得するだけで全件の PDF リンクが得られる。

### Step 2: PDF リンクとメタ情報の抽出

Cheerio 等で `<a>` タグを全件抽出し、`href` が `/files/upload/` で始まるものを対象とする。

各リンクについて、リンクテキストから以下を解析する:

| 情報 | 抽出元 |
| --- | --- |
| 開催年 | `令和○年` / `平成○年` |
| 回次 | `第○回` |
| 会議種別 | `定例会` / `臨時会` |
| 号数（定例会のみ） | `第○号` |
| 開催日 | `令和○年○月○日` |
| 目次/本文の区別 | `(目次)` の有無 |
| ファイルサイズ | `(PDF版:○○KB)` |

#### パース用正規表現（案）

```typescript
// 会議種別と回次
const sessionPattern = /第(\d+)回\s*東白川村議会(定例会|臨時会)/;

// 目次の判定
const isToc = linkText.includes("(目次)");

// 定例会の号数と開催日
const regularPattern = /第(\d+)号\s*(令和|平成)(\d+)年(\d+)月(\d+)日/;

// 臨時会の開催日（号数なし）
const specialPattern = /臨時会会議録\((令和|平成)(\d+)年(\d+)月(\d+)日\)/;

// 年号変換（令和・平成 → 西暦）
// 令和: 2018 + 年号年数
// 平成: 1988 + 年号年数
```

### Step 3: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、PDF テキスト抽出ライブラリ（例: `pdf-parse`）でテキストを取得する。

## PDF 内の会議録構造

PDF は紙面の議事録をスキャン・電子化したもの、または Word 等で作成した文書を PDF 化したものと推測される。

### 注意事項

- 個人情報保護の観点から、氏名・住所等の記載部分に伏字処理がされている
- 発言者パターンや本文構造は PDF の内容を実際に確認して決定する必要がある
- 目次 PDF は本文を含まないため、スクレイピング対象から除外する

---

## 推奨アプローチ

1. **単一ページ取得**: 一覧ページを 1 回取得するだけで全 PDF リンクが得られるため、クロールの複雑さがない
2. **リンクテキストのパース**: URL に年・会期情報が含まれないため、リンクテキストから必要なメタ情報をすべて抽出する
3. **目次 PDF の除外**: `(目次)` を含むリンクはスキップし、本文 PDF のみを対象とする
4. **差分更新**: 一覧ページを定期取得し、新たに追加された UUID の PDF のみをダウンロードする
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
