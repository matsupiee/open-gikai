# 足寄町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.ashoro.hokkaido.jp/gikai/kaigiroku/
- 分類: 自治体公式サイトで PDF 形式の会議録を年度別に公開
- 文字コード: UTF-8
- 特記: 会議録は PDF ファイルで提供。一覧ページは単一ページに全年度分が掲載（ページネーションなし）

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録トップ | `https://www.town.ashoro.hokkaido.jp/gikai/kaigiroku/` |
| 会議録一覧 | `https://www.town.ashoro.hokkaido.jp/gikai/kaigiroku/page_6.html` |
| 会議録 PDF | `https://www.town.ashoro.hokkaido.jp/gikai/assets/{ファイル名}.pdf` |

---

## 一覧ページの HTML 構造

`page_6.html` に全年度・全会議種別の会議録リンクが掲載されている。

```
h2（年号: "令和８年", "令和７年", ... "平成29年"）
  h4（会議種別: "第１回臨時会", "第1回定例会", "予算審査特別委員会" など）
    ul
      li
        a[href="/gikai/assets/xxx.pdf"]（リンクテキスト: "3月5日" など日付）
```

- 年度は `h2` 見出しで区切られる
- 会議種別は `h4` 見出しで区切られる
- 各会議日の PDF へのリンクは `ul > li > a` で構成される
- リンクテキストは開催日（例: "3月5日"）

---

## PDF ファイル名の命名規則

ファイル名パターン: `{年号プレフィックス}{会議種別コード}kaigiroku{月}.{日}.pdf`

### 年号プレフィックス

| プレフィックス | 年 |
| --- | --- |
| `29` | 平成29年（2017年） |
| `30` | 平成30年（2018年） |
| `31` | 平成31年/令和元年（2019年） |
| `r2` | 令和2年（2020年） |
| `r3` | 令和3年（2021年） |
| `r4` | 令和4年（2022年） |
| `r5` | 令和5年（2023年） |
| `r6` | 令和6年（2024年） |
| `r7` | 令和7年（2025年） |
| `r8` | 令和8年（2026年） |

### 会議種別コード

| コード | 会議種別 |
| --- | --- |
| `t1` ~ `t4` | 定例会（第1回~第4回） |
| `r1` ~ `r6` | 臨時会（第1回~第6回） |
| `yosan` / `yosann` | 予算審査特別委員会 |
| `kessan` | 決算審査特別委員会 |

### ファイル名の例

| ファイル名 | 意味 |
| --- | --- |
| `r7t1kaigiroku3.4.pdf` | 令和7年 第1回定例会 3月4日 |
| `r7r1kaigiroku1.24.pdf` | 令和7年 第1回臨時会 1月24日 |
| `r7yosan3.17.pdf` | 令和7年 予算審査特別委員会 3月17日 |
| `29t1kaigiroku3.2.pdf` | 平成29年 第1回定例会 3月2日 |
| `29kessan9.14.pdf` | 平成29年 決算審査特別委員会 9月14日 |

**注意:** ファイル名の命名規則にゆらぎがある。

- 予算審査特別委員会は `yosan` と `yosann` の2パターンが存在（例: `r6yosann3.14_1.pdf`）
- 一部のファイルに `_1`, `_2` のサフィックスが付く（例: `r6r1kaigiroku2.19_2.pdf`）
- 平成30年の一部に `h30` プレフィックスが存在（例: `h30r2kaigiroku5.8.pdf`）
- 日付のゼロ埋めが不統一（`06.3` と `6.3` が混在）

## スクレイピング戦略

### Step 1: 一覧ページから PDF リンクを収集

`page_6.html` を取得し、全 PDF リンクとメタ情報を抽出する。

- 単一ページに全データが掲載されているため、ページネーション処理は不要
- `h2` から年度、`h4` から会議種別、`a[href]` から PDF URL と開催日を抽出

**収集方法:**

1. `page_6.html` を Cheerio でパース
2. `h2` 要素を走査し、年度を特定
3. 各 `h2` 配下の `h4` 要素から会議種別を特定
4. 各 `h4` 配下の `ul > li > a` から PDF URL とリンクテキスト（開催日）を抽出
5. 年度 + 会議種別 + 開催日 + PDF URL をセットで保存

```typescript
// パースのイメージ
const $ = cheerio.load(html);
const records: Array<{
  year: string;       // "令和7年" など
  category: string;   // "第1回定例会" など
  date: string;       // "3月4日" など
  pdfUrl: string;     // "/gikai/assets/r7t1kaigiroku3.4.pdf"
}> = [];

$("h2").each((_, h2) => {
  const year = $(h2).text().trim();
  // h2 の後続要素を走査
  $(h2).nextUntil("h2").filter("h4").each((_, h4) => {
    const category = $(h4).text().trim();
    $(h4).nextUntil("h4, h2").find("a[href$='.pdf']").each((_, a) => {
      records.push({
        year,
        category,
        date: $(a).text().trim(),
        pdfUrl: $(a).attr("href")!,
      });
    });
  });
});
```

### Step 2: PDF のダウンロード

収集した PDF URL からファイルをダウンロードする。

- ベース URL: `https://www.town.ashoro.hokkaido.jp`
- PDF パス: `/gikai/assets/{ファイル名}.pdf`

### Step 3: PDF からテキストを抽出

ダウンロードした PDF から本文テキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを取得
- メタ情報（年度、会議種別、開催日）は Step 1 で HTML から取得済みのものを使用

---

## 注意事項

- PDF ファイルのため、HTML パースではなく PDF パーサーが必要
- ファイル名の命名規則にゆらぎがあるため、ファイル名からのメタ情報抽出は補助的に使い、主に HTML の構造（h2/h4/リンクテキスト）からメタ情報を取得する
- 全データが単一ページに掲載されているため、差分更新は前回取得済みの PDF URL リストとの差分で判定する
- 自治体サイトのため、リクエスト間に適切な待機時間（1～2 秒）を設ける

---

## 推奨アプローチ

1. **一覧ページ起点**: `page_6.html` の HTML 構造から全 PDF リンクとメタ情報を一括収集する
2. **メタ情報は HTML から取得**: PDF ファイル名の命名規則にゆらぎがあるため、年度・会議種別・開催日は HTML の見出し構造から取得する
3. **PDF テキスト抽出**: pdf-parse 等のライブラリで PDF からテキストを抽出する
4. **レート制限**: PDF ダウンロード時にリクエスト間隔を 1～2 秒空ける
5. **差分更新**: 前回取得済みの PDF URL セットを保持し、新規追加分のみダウンロードする
