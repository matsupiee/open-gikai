# 木古内町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.kikonai.hokkaido.jp/gikai/kaigiroku/
- 分類: 自治体公式サイトに PDF 形式で年度別に公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: Google Analytics（G-P6RH1X3J9D）使用、Adobe Reader 必須の注記あり

## PDF ファイルの URL パターン

PDF ファイルは `/files/` ディレクトリ配下に格納されており、会議種別ごとに異なるディレクトリ ID が割り当てられている。

### 定例会の例（令和6年）

```
/files/00005200/00005207/第1回定例会.pdf
/files/00005200/00005207/第2回定例会.pdf
/files/00005200/00005207/第3回定例会.pdf
/files/00005200/00005207/第4回定例会.pdf
```

### 臨時会の例（令和6年）

```
/files/00005200/00005204/R6.1（臨時会）.pdf
/files/00005200/00005204/R6.2（臨時会）.pdf
```

### 常任委員会の例（令和6年）

```
/files/00005400/00005407/第1回.pdf
/files/00005400/00005407/第2回.pdf
...
/files/00005400/00005407/第11回.pdf
```

> **注意:** ディレクトリ ID（`00005200/00005207` 等）は会議種別・年度ごとに異なる可能性がある。PDF の URL はページ内の `<a>` タグから動的に取得する必要がある。

## サイト構造（3 階層）

```
会議録トップ（年度一覧）
  └─ 年度別ページ（会議種別一覧）
       └─ 会議種別ページ（PDF リンク一覧）
            └─ PDF ファイル（会議録本文）
```

- **第1階層:** 年度一覧 — 平成25年〜令和7年のリンク
- **第2階層:** 会議種別一覧 — 定例会・臨時会・各委員会へのリンク
- **第3階層:** PDF リンク一覧 — 各回の会議録 PDF へのリンクと開催日情報

---

## スクレイピング戦略

### Step 1: 年度一覧の取得

会議録トップページ `/gikai/kaigiroku/` から年度別ページへのリンクを収集する。

- リンクパターン: `/gikai/kaigiroku/{年号}/`
- HTML 構造: `<ul><li>` によるリスト形式

### Step 2: 会議種別ページの URL 収集

各年度ページから会議種別ページへのリンクを収集する。

- リンクパターン: `/gikai/kaigiroku/{年号}/{種別}.html`
- **ファイル名が年度ごとに異なるため、ハードコードせずにページ内のリンクを動的に取得すること**

### Step 3: PDF リンクの収集

各会議種別ページから PDF ファイルへのリンクと付随するメタ情報を収集する。

**収集する情報:**

- PDF ファイルの URL
- 会議名（定例会・臨時会・委員会名）
- 開催日（ページ内テキストから抽出）
- 回数（第N回）

**開催日の抽出パターン:**

```
令和6年3月4日～3月11日    （期間指定の場合）
令和6年6月20日            （単日の場合）
```

### Step 4: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で本文を取得する
- PDF 内に発言者情報・議案情報が含まれるが、PDF の構造はファイルごとに異なる可能性がある

---

## ページネーション

なし。各ページに全件が表示される。

---

## 注意事項

- PDF ファイル名に日本語が含まれるため、URL はパーセントエンコーディングされている
- 全角括弧（`（）`）がファイル名に使用されているケースがある
- 会議種別ページのファイル名が年度ごとに統一されていないため、リンクのハードコードは不可
- PDF の内容構造（発言者パターン等）はダウンロード後に別途分析が必要
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設けること

---

## 推奨アプローチ

1. **3 階層クロール**: トップ → 年度 → 会議種別の順にリンクを再帰的に辿り、全 PDF の URL を収集する
2. **リンク動的取得**: ファイル名の命名規則が不統一なため、必ずページ内の `<a>` タグからリンクを取得する
3. **PDF テキスト抽出**: ダウンロードした PDF から pdf-parse 等でテキストを抽出し、発言単位にパースする
4. **レート制限**: リクエスト間に 1〜2 秒の待機時間を設ける
5. **差分更新**: 年度ページの構成が変わることは少ないため、最新年度のみを定期的にチェックする運用が効率的
