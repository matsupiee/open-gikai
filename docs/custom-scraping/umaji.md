# 馬路村議会 カスタムスクレイピング方針

## 概要

- サイト: https://vill.umaji.lg.jp/about/category/parliament/
- 分類: WordPress サイトで PDF ファイルによる情報公開（会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 会議録（発言録）のオンライン公開なし。議決の状況・一般質問の状況・議会の開催状況を年度別 PDF で提供

---

## サイト構造

WordPress（テーマ: uniontheme）で構築された公式ホームページ内に議会カテゴリページがある。

### カテゴリページ

`https://vill.umaji.lg.jp/about/category/parliament/`

各投稿ページへのリンク一覧が表示される。ページネーションなし。

### 投稿ページ

| ページ | URL パターン | 内容 |
| --- | --- | --- |
| 議会の開催状況 | `/about/parliament/3408/`, `/about/parliament/3360/` | 年度別の開催日程 PDF |
| 議決の状況 | `/about/parliament/{ID}/` | 年度別の議決結果 PDF |
| 一般質問の状況 | `/about/parliament/574/` | 年度別の一般質問 PDF |
| 馬路村議会について | `/about/parliament/560/` | 議員紹介・組織紹介 PDF |

### PDF の URL パターン

```
https://vill.umaji.lg.jp/wp/wp-content/uploads/{YYYY}/{MM}/{ハッシュ文字列}.pdf
```

- ファイル名はハッシュ値ベース（内容から推測不可）
- `/wp/wp-content/uploads/` 配下に WordPress の標準アップロード構造で格納

---

## 公開されている情報

### 1. 議会の開催状況（PDF）

年度別に議会の開催日・会議名を記載した PDF。平成25年～令和8年分が存在。

### 2. 議決の状況（PDF）

定例会・臨時会ごとの議決結果を記載した PDF。各会ごとに個別の PDF が用意されている。

- 例: 「第1回臨時会（令和7年1月20日）」「第2回定例会（令和7年3月6日）」等
- 平成25年～令和8年分が存在

### 3. 一般質問の状況（PDF）

年度別に一般質問の質問者・質問内容をまとめた PDF。平成25年～令和7年分が存在。

### 4. 会議録（発言録）

**オンラインでは公開されていない。** 会議録検索システム（dbsearch、discuss 等）は導入されていない。

---

## スクレイピング戦略

### Step 1: カテゴリページから投稿ページ URL を収集

`https://vill.umaji.lg.jp/about/category/parliament/` にアクセスし、`/about/parliament/{ID}/` 形式のリンクを抽出する。

- ページネーションなし（全件が1ページに表示される）
- Cheerio で `a[href]` を抽出し、`/about/parliament/` にマッチするものをフィルタ

### Step 2: 各投稿ページから PDF URL を収集

各投稿ページにアクセスし、PDF へのリンクを抽出する。

- PDF リンクは `/wp/wp-content/uploads/` 配下を指す `a[href]` タグ
- リンクテキストに会議名・年度が含まれる（例: 「第1回臨時会（令和7年1月20日）」）

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、`pdf-parse` 等でテキストを抽出
- 議決の状況 PDF には議案番号・議案名・議決結果が含まれる
- 一般質問 PDF には質問者・質問事項が含まれる

---

## 注意事項

- 会議録（発言の全文記録）はオンライン公開されていないため、発言内容のスクレイピングは不可
- 取得可能な情報は議決結果・一般質問の概要・開催日程に限定される
- PDF のファイル名がハッシュ値のため、URL からコンテンツを推測できない（必ず投稿ページ経由でリンクを取得する必要がある）
- WordPress サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **PDF 経由の情報取得**: 会議録テキストは取得不可のため、議決の状況・一般質問の状況 PDF からメタ情報（開催日・会議名・議案・質問者等）を抽出する方針とする
2. **カテゴリページ起点**: カテゴリページ → 投稿ページ → PDF の3段階でクロールする
3. **差分更新**: 投稿ページの ID（数値）が増加するため、前回取得済みの最大 ID 以降の投稿のみを確認する差分更新が可能
4. **レート制限**: 小規模自治体サイトのため、リクエスト間隔は 1〜2 秒を確保する
