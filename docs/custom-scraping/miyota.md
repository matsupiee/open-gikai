# 御代田町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.miyota.nagano.jp/
- 分類: 自治体公式サイト内の静的 HTML ページに PDF を掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF 形式で公開。年度ごとにカテゴリページが分かれ、各定例会ページから日別の PDF をダウンロードする構造

---

## サイト構造の詳細

### 3 階層構造

```
会議録一覧（kaigiroku/index.html）
  └─ 年度別一覧（{年度カテゴリ}/index.html）
       └─ 定例会詳細（{年度カテゴリ}/{ページID}.html）
            └─ PDF ファイル（file/{ファイルID}.pdf）
```

### 年度別一覧ページ

- 各年度に原則 4 回の定例会（第1回〜第4回）が掲載される
- 臨時会の掲載は確認されていない
- 各定例会は `{年度カテゴリ}/{ページID}.html` へのリンクで表示される
- ページ ID は数値（例: `168536`, `166187`）

### 定例会詳細ページ

- 会議の開催期間が説明文に記載（例: 「令和6年11月29日（金）～12月16日（月）」）
- 日別の PDF ファイルがリスト形式（`<li>`）で並ぶ
- 各 PDF リンクには日付・内容・ファイルサイズが併記される

### PDF リンクの構造

各定例会ページでの PDF 掲載パターン:

| 内容区分 | 説明 |
| --- | --- |
| 議事日程 | 各開催日ごとに 1 つ |
| 諸般報告・招集挨拶・議案説明 | 初日に掲載 |
| 一般質問 | 質問日ごとに 1 つ（通常 2 日間） |
| 議案質疑・委員会付託 | 質疑日に 1 つ |
| 委員長報告・採決 | 最終日に 1 つ |

PDF の URL パターン: `https://www.town.miyota.nagano.jp/file/{ファイルID}.pdf`

- ファイル ID は数値（例: `154352`, `6005`）
- 古い会議録ほどファイル ID が小さい

検索システムは存在しない。年度別カテゴリページを順に辿る階層ナビゲーション方式。

## スクレイピング戦略

### Step 1: 年度別カテゴリ URL の収集

会議録一覧ページ `category/kaigiroku/index.html` から全年度のカテゴリ URL を取得する。

- カテゴリスラッグに規則性がないため、一覧ページからリンクを抽出する必要がある
- `<a>` タグの `href` 属性から `category/{スラッグ}/index.html` パターンを収集

### Step 2: 定例会ページ URL の収集

各年度カテゴリページから定例会詳細ページの URL を取得する。

- 各ページに第1回〜第4回定例会のリンクが掲載される
- `<a>` タグの `href` 属性から `category/{スラッグ}/{ページID}.html` パターンを収集
- リンクテキストから会議名（例: 「令和６年第４回御代田町議会定例会会議録」）を抽出

### Step 3: PDF URL の収集

各定例会詳細ページから PDF ファイルの URL を取得する。

- `<a>` タグの `href` 属性から `file/{ファイルID}.pdf` パターンを収集
- リンクテキストから日付・内容区分を抽出（例: 「一般質問（12月3日）」）
- 開催期間は説明文テキストから抽出

### Step 4: PDF のダウンロードとテキスト抽出

- 収集した PDF URL からファイルをダウンロード
- PDF パーサー（pdf-parse 等）でテキストを抽出
- PDF 内のテキストから発言者・発言内容を構造化

### Step 5: 会議録のパース

#### メタ情報

定例会詳細ページの説明文から以下を抽出:

```
令和6年11月29日（金）～12月16日（月）に行われた
『令和６年第４回御代田町議会定例会』の会議録を次のとおり公開します。
```

- 開催期間: `{年}年{月}日{日}（{曜日}）～{月}月{日}日（{曜日}）`
- 会議名: `令和{X}年第{N}回御代田町議会定例会`

各 PDF リンクのテキストから:
- 開催日: `{月}月{日}日`
- 内容区分: 議事日程 / 一般質問 / 委員長報告・採決 等

#### 発言の構造

PDF 内のテキスト構造は PDF パーサーで抽出後に分析が必要。一般的な町議会の会議録パターンとして以下が想定される:

```
○議長（氏名）
○{番号}番（氏名）
○町長（氏名）
```

※ 実際のパターンは PDF テキスト抽出後に確認・調整が必要。

## ページネーション

ページネーションは存在しない。各階層のページにすべてのリンクが一覧表示される。

---

## 注意事項

- 年度カテゴリのスラッグに規則性がないため、ハードコードではなく一覧ページからの動的取得が必須
- 会議録は PDF 形式のみで HTML 版は存在しない
- PDF からのテキスト抽出精度に依存するため、OCR が必要なスキャン PDF が含まれる可能性に注意（特に古い年度）
- 対象範囲は平成18年（2006年）〜現在
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **階層的クロール**: 会議録一覧 → 年度別一覧 → 定例会詳細 → PDF の順に 3 段階でリンクを収集
2. **PDF テキスト抽出**: `pdf-parse` 等のライブラリで PDF からテキストを抽出し、発言者パターンを解析
3. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
4. **差分更新**: ファイル ID は数値連番のため、前回取得済みの最大ファイル ID 以降のみを取得する差分更新が可能
5. **年度カテゴリの動的取得**: スラッグが不規則なため、必ず一覧ページから最新のリンクを取得する
