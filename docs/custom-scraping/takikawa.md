# 滝川市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.takikawa.lg.jp/page/2872.html
- 分類: 静的 HTML ページ + PDF 形式の会議録（既存アダプターでは対応不可）
- 文字コード: UTF-8（推定）
- 特記: 会議録は全て PDF ファイルで提供。HTML テキスト版は存在しない

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録トップ（一覧） | `https://www.city.takikawa.lg.jp/page/2872.html` |
| 年度別会議録ページ | `https://www.city.takikawa.lg.jp/page/{ページID}.html` |
| 会議録 PDF | `https://www.city.takikawa.lg.jp/uploaded/attachment/{ファイルID}.pdf` |

- ページ ID・ファイル ID は数値だが連番ではなく、規則性はない
- 全てのナビゲーションは `/page/{数字}.html` 形式

## 年度別ページの構造

各年度別ページ（例: `/page/18437.html`）は以下の構造:

- h2 見出しで会議回次・種別を区切り（例: `第4回臨時会`、`第1回定例会`）
- 各セクション内に PDF リンクのリスト
  - **目次 PDF**: 各会議セッションの冒頭に配置（32〜99KB）
  - **日付別 PDF**: `{月}月{日}日 [PDFファイル／{サイズ}KB]` の形式

### PDF リンクの例

```
第4回定例会
  - 目次 [PDFファイル／55KB]     → /uploaded/attachment/18555.pdf
  - 12月3日 [PDFファイル／350KB]  → /uploaded/attachment/18556.pdf
  - 12月9日 [PDFファイル／280KB]  → /uploaded/attachment/18557.pdf
  - 12月10日 [PDFファイル／310KB] → /uploaded/attachment/18558.pdf
```

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

トップページ `/page/2872.html` から全ての年度別ページへのリンクを抽出する。

- トップページにはページネーションがなく、全リンクが 1 ページに掲載
- `/page/{数字}.html` 形式のリンクを Cheerio で抽出
- 会議種別はリンクの前後のテキスト（見出し等）から判定

### Step 2: PDF URL の収集

各年度別ページから PDF リンクを抽出する。

- h2 見出しから会議の回次・種別（定例会/臨時会）を取得
- `/uploaded/attachment/{数字}.pdf` 形式のリンクを抽出
- リンクテキストから日付情報を取得（`{月}月{日}日` 形式）
- 「目次」PDF と「本文」PDF を区別する（目次はリンクテキストが「目次」）

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを抽出
- 目次 PDF からは会議のメタ情報（議題、出席者等）を取得可能
- 日付別 PDF から発言内容を抽出

### Step 4: 発言のパース

PDF から抽出したテキストを発言単位に分割する。

- PDF のテキスト構造は実際のファイルを確認して決定する必要がある
- 一般的な議会会議録 PDF では、発言者名が行頭に記載されるパターンが多い

---

## ページネーション

なし。トップページ・年度別ページともに、全てのリンクが 1 ページに掲載される。

---

## 注意事項

- **PDF のみ提供**: HTML テキスト形式の会議録は存在しない。PDF からのテキスト抽出が必須
- **ページ ID に規則性がない**: 年度やカテゴリとページ ID の間に規則性がないため、トップページからのリンク収集が必須
- **過去データの範囲**: 本会議は平成16年から、委員会は平成19年〜20年から掲載
- **目次 PDF の活用**: 各会議の目次 PDF には議題一覧や発言者一覧が含まれる可能性があり、メタ情報の抽出に活用できる

---

## 推奨アプローチ

1. **2 段階クロール**: トップページ → 年度別ページ → PDF URL の 2 段階でリンクを収集
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使用して PDF からテキストを抽出
3. **目次 PDF を先に解析**: 目次 PDF からメタ情報を取得し、本文 PDF のパースに活用
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: トップページの更新日（ページ下部に表示）を監視し、変更があった場合のみ再クロール
