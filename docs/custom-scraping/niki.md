## PDF ファイル命名規則

ファイル名の命名規則は年度によって異なり、統一されていない。

### パターン 1: 日付ベース（西暦 8 桁）

- 例: `20251218.pdf` = 2025 年 12 月 18 日開催
- 例: `250619.pdf` = 2025 年 6 月 19 日開催（6 桁の場合もあり）

### パターン 2: 和暦日付ベース

- 例: `R07.09.24.pdf` = 令和 7 年 9 月 24 日開催
- 例: `R07.08.05.pdf` = 令和 7 年 8 月 5 日開催
- 例: `R7.5.29.pdf` = 令和 7 年 5 月 29 日開催（ゼロ埋めなしの場合もあり）

### パターン 3: 平成日付ベース

- 例: `H230427.pdf` = 平成 23 年 4 月 27 日開催
- 例: `H230531.pdf` = 平成 23 年 5 月 31 日開催

### パターン 4: CMS 自動生成 ID

- 例: `btsphm0000002pgc.pdf`
- 例: `immd6j00000083gc.pdf`
- 例: `irv97600000007nv.pdf`
- ファイル名から日付情報を推定できないため、リンクテキストからメタ情報を取得する必要がある

## スクレイピング戦略

### Step 1: PDF リンクとメタ情報の収集

一覧ページ `irv97600000004s6.html` から全 PDF リンクとメタ情報を抽出する。

- ページネーションがないため、1 リクエストで全データを取得可能
- `<a href="irv97600000004s6-att/...pdf">` パターンでリンクを抽出
- リンクテキストから会議種別・回数・日程・開催日を抽出
- セクション見出し（`<h3>`）から年度情報を取得

**収集方法:**

1. 一覧ページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` セレクタにマッチするリンクを全て抽出
3. 各リンクのテキストから以下を正規表現で抽出:
   - 会議種別（定例会 / 臨時会）
   - 回数
   - 日程番号（複数日の場合）
   - 開催日

### Step 2: PDF のダウンロード

収集した PDF URL に対してファイルをダウンロードする。

- ベース URL: `https://www.town.niki.hokkaido.jp/section/gikai/`
- 相対パス `irv97600000004s6-att/xxx.pdf` を結合してフル URL を生成

### Step 3: PDF からのテキスト抽出

ダウンロードした PDF から会議録本文を抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを抽出
- PDF 内のレイアウトに依存するため、抽出精度の検証が必要

### Step 4: メタ情報の抽出

#### リンクテキストからの情報抽出

```typescript
// リンクテキストから会議情報を抽出
const meetingPattern = /第(\d+)回(定例会|臨時会)(.*?)（(令和|平成)(\d+)年(\d+)月(\d+)日開催）/;
// 例: 第4回定例会（令和7年12月18日開催）
//   → 回=4, 種別=定例会, 年=令和7, 月=12, 日=18

// 日程番号の抽出（複数日の場合）
const dayPattern = /(\d+)日目/;
// 例: 第1回定例会3日目 → 日程=3
```

## 注意事項

- PDF 形式のため、テキスト抽出の精度は PDF の作成方法に依存する（画像 PDF の場合は OCR が必要）
- ファイル命名規則が年度によって大きく異なる（4 パターン以上）ため、メタ情報はファイル名ではなくリンクテキストから取得するのが確実
- 括弧の種類が年度によって異なる場合がある（全角括弧 `（）` と半角括弧 `()` の混在）
- 一部の会議録で「初議会」「〔初議会〕」などの特殊表記がある

## 推奨アプローチ

1. **1 ページ完結**: 一覧ページが 1 ページで完結しているため、クロール処理はシンプルに実装可能
2. **リンクテキスト優先**: ファイル名の命名規則が不統一なため、メタ情報の取得はリンクテキストを主要ソースとする
3. **PDF パース**: HTML ではなく PDF からのテキスト抽出が主要な課題。まず数件の PDF をダウンロードしてテキスト抽出の品質を検証する
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページの PDF リンク数を前回取得時と比較し、新規リンクのみをダウンロードする差分更新が可能
