### 会議種別

- **定例会**: 年複数回開催（例: 第3回定例会、第5回定例会）
- **臨時会**: 必要に応じて開催（例: 第1回臨時会、第4回臨時会）

### PDF ファイル名パターン

ファイル名の命名規則にはばらつきがある:

```
R7　第1回臨時会.pdf
R7　第3回　定例会.pdf
R7.4月　第4回臨時会　会議録.pdf
R6.2月　第1回臨時会.pdf
会議録　R6　第7回臨時会.pdf
```

- 年号の表記: `R7`（令和）、`H30`（平成）など略称
- 月の有無: 含まれる場合と含まれない場合がある
- 「会議録」の位置: 先頭・末尾・なしの3パターン
- 全角スペースが多用されている

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録一覧ページ（単一ページ）から全 PDF リンクを抽出する。

- ページネーションがないため、1回のリクエストで全リンクを取得可能
- `<a>` タグの `href` 属性から `.pdf` で終わるリンクを抽出
- リンクテキストから会議名・年度情報を取得

**収集方法:**

1. 会議録一覧ページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` を抽出
3. 各リンクのテキストと URL をペアで保存

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]')
  .map((_, el) => ({
    text: $(el).text().trim(),
    url: $(el).attr("href"),
  }))
  .get();
```

### Step 2: メタ情報の抽出

リンクテキストおよびファイル名から会議のメタ情報を解析する。

```typescript
// リンクテキストからの年号・回次・会議種別の抽出
const metaPattern = /[RH](\d+)(?:\.(\d+)月)?.*?第(\d+)回\s*(定例会|臨時会)/;
// 例: "R7.4月　第4回臨時会　会議録" → era="R", year=7, month=4, number=4, type="臨時会"
```

- 年号: `R`（令和）または `H`（平成）+ 数字
- 月: 省略されている場合がある
- 回次: `第X回`
- 種別: `定例会` or `臨時会`

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

- PDF パーサー（`pdf-parse` 等）を使用してテキストを取得
- PDF 内の構造は会議録形式（発言者名 + 発言内容）

## 注意事項

- **平成19年以前**: 会議録はサイト上に公開されていない。議会事務局への問い合わせが必要（TEL: 0980-44-3117、Mail: gikai@vill.ogimi.lg.jp）
- **ファイル名のばらつき**: 命名規則が統一されていないため、正規表現は柔軟に設計する必要がある
- **全角文字**: ファイル名・URL に全角スペースや全角数字が含まれるため、URL エンコーディングに注意
- **PDF 形式**: HTML ではなく PDF での公開のため、テキスト抽出の精度に依存する
- **令和3年・令和2年の一般質問分割**: 一部の年度では一般質問が別 PDF に分割されている

## 推奨アプローチ

1. **一括リンク取得**: ページネーションがないため、1回のリクエストで全 PDF リンクを収集
2. **ファイル名からメタ情報を推定**: リンクテキストとファイル名の両方からメタ情報を抽出し、補完する
3. **PDF テキスト抽出**: `pdf-parse` 等を用いて PDF からテキストを抽出。発言者・発言内容の分離は PDF 内のテキスト構造に依存
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 取得済み PDF の URL リストを保持し、新規追加分のみをダウンロードする
