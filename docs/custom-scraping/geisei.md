# 芸西村議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.vill.geisei.kochi.jp/a02b07.php
- 分類: 村公式サイトに PDF 形式で年度別に公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF ファイルで提供。議案一覧（議決結果）は HTML テーブルで掲載。

## ページ構造

各年度ページは以下の構造で構成される:

1. **ページタイトル** (`h3.cs_pagetitle`): `会議録及び議会の結果(令和X年)`
2. **会議区分** (`h6.cs_komidasi`): `【第N回　定例会】` または `【第N回　臨時会】`
3. **開催期間** (`p.cs_dantext`): `令和X年X月X日（曜）～X月X日（曜）` または単日
4. **議案一覧テーブル** (`div.cs_hyo > table`): 議案番号・件名・議決年月日・結果
5. **PDF リンク** (`div.cs_file > a`): 会議録 PDF へのリンク

### PDF ファイルの種類

PDF リンクのテキストから以下の種類を判別できる:

| リンクテキストのパターン | 内容 |
| --- | --- |
| `会議録表紙・目次` / `RX.XX会議録表紙・目次` | 表紙・目次（スクレイピング対象外） |
| `RX.XX.XX会議録` | 本会議の会議録（主要なスクレイピング対象） |
| `RX.XX.XX会議録(一般質問)` | 一般質問の会議録（スクレイピング対象） |
| `一般質問通告一覧表` | 一般質問の通告一覧（参考情報） |
| `会議録` | 臨時会の会議録（単独ファイル） |

### PDF URL のパターン

```
https://www.vill.geisei.kochi.jp/pbfile/m00XXXX/pbfYYYYMMDDHHMMSS.pdf
```

- `m00XXXX`: 年度ページの ID と一致
- `pbfYYYYMMDDHHMMSS`: ファイルのアップロード日時（会議開催日とは異なる）

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

村政ページ `a02b07.php` から「会議録及び議会の結果」セクションのリンクを収集する。

**収集方法:**

1. `https://www.vill.geisei.kochi.jp/a02b07.php` を取得
2. `会議録及び議会の結果` を含むリンクテキストを持つ `<a>` タグから `pages/m00XXXX.php` を抽出
3. 新しい年度ページが追加された場合も自動的に検出可能

### Step 2: PDF リンクの収集

各年度ページから会議録 PDF のリンクを収集する。

**収集方法:**

1. 各年度ページの HTML を取得
2. `div.cs_file > a[href$=".pdf"]` で PDF リンクを抽出
3. リンクテキストから種類を判別し、「表紙・目次」や「一般質問通告一覧表」を除外
4. 直前の `h6.cs_komidasi` と `p.cs_dantext` から会議区分・開催日を取得

```typescript
// PDF リンクの抽出（Cheerio 使用）
const pdfLinks = $("div.cs_file a[href$='.pdf']")
  .map((_, el) => ({
    url: new URL($(el).attr("href")!, baseUrl).toString(),
    label: $(el).text().trim(),
  }))
  .get();

// 会議録 PDF のフィルタリング（表紙・目次・通告一覧を除外）
const minutesPdfs = pdfLinks.filter(
  (link) =>
    link.label.includes("会議録") &&
    !link.label.includes("表紙") &&
    !link.label.includes("目次") &&
    !link.label.includes("通告")
);
```

### Step 3: メタ情報の抽出

PDF リンク周辺の HTML 構造からメタ情報を取得する。

```typescript
// 会議区分の抽出
const sessionPattern = /【(第\d+回)\s*(定例会|臨時会)】/;
// 例: 【第１回　定例会】 → 回次="第１回", 種別="定例会"

// 開催期間の抽出
const datePattern = /令和(\d+)年(\d+)月(\d+)日/;
// 例: 令和４年３月４日（金）～３月10日（木）

// PDF ファイル名から開催日を推定
const pdfDatePattern = /[RＲ](\d+)\.(\d+)\.(\d+)/;
// 例: R4.3.4会議録 → 令和4年3月4日
```

### Step 4: PDF のテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

- PDF パーサー（`pdf-parse` 等）を使用してテキストを抽出
- PDF は議会事務局が作成したもので、テキスト埋め込み型（画像 PDF ではない可能性が高い）
- 画像 PDF の場合は OCR（Tesseract 等）が必要になる

---

## 注意事項

- PDF のリンクは相対パス（`../pbfile/m00XXXX/...`）で記述されているため、ベース URL からの解決が必要
- 年度ページの ID（`m00XXXX`）は連番だが不規則なため、一覧ページから動的に取得する必要がある
- 臨時会は PDF が1ファイルのみの場合がある（リンクテキストが単に「会議録」）
- PDF ファイルのサイズは数十 KB〜約 850 KB 程度
- リンクテキストの表記ゆれあり（全角・半角混在: `Ｒ4` vs `R4`、括弧: `(一般質問)` vs `(一般質問）`）

---

## 推奨アプローチ

1. **一覧ページから年度ページを動的収集**: `a02b07.php` を起点に年度ページの URL を自動検出し、新年度追加時にも対応
2. **HTML 構造でメタ情報を補完**: PDF ファイル名だけでなく、周辺の HTML 要素（`h6.cs_komidasi`, `p.cs_dantext`）から会議区分・開催日を取得
3. **PDF テキスト抽出の事前検証**: 最初の数ファイルでテキスト抽出可否を確認し、画像 PDF の場合は OCR 対応を検討
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: PDF ファイル名にアップロード日時が含まれるため、既知のファイル名リストと比較して新規分のみ取得可能
