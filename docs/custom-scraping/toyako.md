## HTML 構造

### ページレイアウト

各年度ページには `<table class="color1">` 形式のテーブルが会議ごとに配置されている。

```html
<table class="color1">
  <thead>
    <tr>
      <td colspan="7"><b><i>洞爺湖町議会令和７年９月会議</i></b></td>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><a href="...目次.pdf">目次等</a></td>
      <td><a href="...1号.pdf">第１日目（９月１０日）</a></td>
      <td><a href="...2号.pdf">第２日目（９月１１日）</a></td>
      <td>－</td>
      ...
    </tr>
  </tbody>
</table>
```

- `thead` 内に会議名（例: `洞爺湖町議会令和７年９月会議`）
- `tbody` の各 `td` に PDF リンクまたは `－`（該当日なし）
- 1列目は「目次等」、2列目以降が「第N日目（M月D日）」
- ページネーションなし（1ページに全会議が一覧表示）

### PDF リンクの形式

PDF の URL は年度によって形式が異なる:

- **古い年度（平成26年頃）**: 相対パス `/upload/files/04_ town_administration/town_council/H2612_01.pdf`
- **新しい年度（令和以降）**: 絶対 URL `http://www.town.toyako.hokkaido.jp/upload/files/20250305洞爺湖町３月会議１号.pdf`

## スクレイピング戦略

### Step 1: 年度別ページの巡回と PDF URL の収集

1. トップページ（令和7年）と各年度ページ（ページ ID 一覧参照）を順番に取得
2. 各ページ内の `<table class="color1">` を Cheerio で全て取得
3. 各テーブルの `thead` から会議名を抽出
4. 各テーブルの `tbody` 内の `<a>` タグから PDF リンクを抽出
5. リンクテキスト（`第１日目（９月１０日）` 等）から開催日を抽出

**収集情報:**

- 年度（ページから判定）
- 会議名（例: `洞爺湖町議会令和７年９月会議`）
- 開催日（リンクテキストから抽出、例: `９月１０日`）
- PDF URL（絶対 URL に正規化）
- 種別（目次 / 本文）

### Step 2: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 目次 PDF は会議のメタ情報として利用可能（議事日程、出席議員等）

### Step 3: 会議録のパース

#### メタ情報

テーブルの `thead` から抽出:

```
洞爺湖町議会令和７年９月会議
```

- 会議体名: `洞爺湖町議会`（固定）
- 年度: `令和７年` → 西暦に変換
- 月: `９月会議`

リンクテキストから開催日を抽出:

```
第１日目（９月１０日）
```

#### メタ情報抽出の正規表現（案）

```typescript
// テーブル見出しから会議名を抽出
const meetingPattern = /洞爺湖町議会((?:令和|平成)\S+年\S+月会議)/;

// リンクテキストから開催日を抽出
const dayPattern = /第(\S+)日目（(\S+)月(\S+)日）/;

// 和暦を西暦に変換
const eraPattern = /(令和|平成)([\d０-９]+)年/;
```

#### 発言のパース

PDF からテキスト抽出後の発言構造は PDF の内部フォーマットに依存する。PDF テキスト抽出後に発言者と発言内容の分離ロジックを実装する必要がある。

---

## 注意事項

- PDF 形式のため、テキスト抽出の精度が HTML 公開の自治体と比較して低い可能性がある
- PDF のファイル名に日本語（全角文字）が含まれるため、URL エンコード処理が必要
- 古い年度の PDF パスにスペースが含まれるケース（`04_ town_administration`）がある
- 相対パスと絶対 URL が混在するため、ベース URL を用いた正規化が必要
- ページネーションはないが、年度ごとにページが分かれているため全ページを巡回する必要がある

---

## 推奨アプローチ

1. **全量取得を優先**: トップページ + 年度別ページ全11ページを巡回し、PDF URL の全量リストを作成
2. **PDF テキスト抽出ライブラリの選定**: `pdf-parse` や `pdfjs-dist` 等を使い、テキスト抽出精度を事前に検証
3. **目次 PDF の活用**: 目次 PDF に議事日程・出席議員等のメタ情報が含まれるため、補助データとして活用
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 新しい年度のトップページのみを定期的にチェックし、新規 PDF の追加を検出
