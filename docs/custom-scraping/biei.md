# 美瑛町議会 カスタムスクレイピング方針

## 概要

- サイト: https://town.biei.hokkaido.jp/administration/parliament/proceedings/
- 分類: 年度別ページに PDF リンクを掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: Google Tag Manager（GTM-KQK9TTX）使用。会議録は PDF 形式で提供。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録トップ | `https://town.biei.hokkaido.jp/administration/parliament/proceedings/` |
| 年度別ページ（令和8年） | `https://town.biei.hokkaido.jp/administration/parliament/proceedings/proceedings8.html` |
| 年度別ページ（令和5年） | `https://town.biei.hokkaido.jp/administration/parliament/proceedings/r5.html` |
| 年度別ページ（令和4年） | `https://town.biei.hokkaido.jp/administration/parliament/proceedings/r4.html` |
| 年度別ページ（平成31年） | `https://town.biei.hokkaido.jp/administration/parliament/proceedings/h31.html` |
| PDF ファイル | `https://www.town.biei.hokkaido.jp/files/{parent_dir}/{sub_dir}/{filename}.pdf` |

### 年度別ページの URL 命名規則

| 年度 | URL ファイル名 |
| --- | --- |
| 令和8年 | `proceedings8.html` |
| 令和7年以前 | `r7.html`, `r6.html`, `r5.html`, `r4.html`, `r3.html`, `r2.html` |
| 平成31年以前 | `h31.html`, `h30.html`, `h29.html`, `h28.html`, `h27.html`, `h26.html` |

※ URL 命名規則に一貫性がないため（令和8年のみ `proceedings8.html`）、年度ごとのページ URL はトップページからリンクを収集して取得する。

## PDF ファイルの URL パターン

PDF は `/files/{parent_dir}/{sub_dir}/{filename}.pdf` の形式で格納される。ディレクトリは年度ごとに異なる。

| 年度 | ディレクトリパス |
| --- | --- |
| 令和8年 | `/files/00006500/00006508/` |
| 令和5年 | `/files/00005100/00005171/` |
| 令和4年 | `/files/00004600/00004617/` |

ファイル名はタイムスタンプ形式（例: `20250108150408.pdf`）または日付+種別形式（例: `20231215_kaigiroku.pdf`）。

---

## 文書の種別

各年度ページには以下の種別の PDF が掲載される。

| 種別 | ファイル名の特徴 | スクレイピング対象 |
| --- | --- | --- |
| 会議録 | `kaigiroku` を含む、またはタイムスタンプのみ | **対象** |
| 議案 | `gian` を含む | 対象外 |
| 資料 | `shiryo` / `siryou` を含む | 対象外 |
| 審議結果 | `shingikekka` / `kekka` / `kakka` を含む | 対象外 |
| 予算書 | `yosansho` / `yosansetsumeisho` を含む | 対象外 |

※ ファイル名の命名規則が統一されていないため、リンクテキスト（アンカーテキスト）も併用して会議録 PDF を判別する必要がある。

## 会議の種類

| 種類 | 説明 |
| --- | --- |
| 定例会 | 年4回程度開催。複数日にわたる場合あり |
| 臨時会 | 不定期に開催。通常1日 |
| 常任委員会 | 産業経済常任委員会など（令和8年から掲載あり） |

## HTML 構造

各年度ページは以下の構造:

- `<h3>` 見出しで会議区分（例: 「第7回定例会（12月14日～15日）」）
- 見出しの下に `<p>` または `<li>` で PDF リンクを列挙
- テーブルは使用されていない
- ページネーションなし（1ページに年度の全会議を掲載）

---

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

トップページ `proceedings/` から全年度ページへのリンクを収集する。

- サイドメニューまたは本文内に過去年度へのリンクが存在
- `href` 属性から `.html` で終わるリンクを抽出
- 対象範囲: 令和2年〜令和7年（指定に従う）

### Step 2: 各年度ページから会議録 PDF リンクの収集

各年度ページにアクセスし、会議録 PDF のリンクを収集する。

**収集方法:**

1. `<h3>` 見出しから会議名・開催日を抽出
2. 見出し配下のリンクからアンカーテキストに「会議録」を含むものを抽出
3. ファイル名に `kaigiroku` を含むリンクも対象に含める
4. 各 PDF リンクに対して、会議名・開催日・年度のメタ情報を紐付ける

**会議名・開催日の抽出パターン:**

```typescript
// h3 見出しから会議名と開催日を抽出
// 例: "第7回定例会（12月14日～15日）"
// 例: "第1回臨時会（1月23日）"
const meetingPattern = /第(\d+)回(定例会|臨時会|.+常任委員会)（(.+?)）/;
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言内容をパースする

### Step 4: 会議録のパース

PDF から抽出したテキストの構造は PDF の作成方法に依存するため、実際のテキスト内容を確認した上でパース戦略を決定する。

一般的な自治体議会 PDF のパターン:

- 開催日・出席者情報がヘッダー部分に記載
- 発言者名と発言内容が交互に記載
- ページ番号のフッターが含まれる

---

## 注意事項

- PDF ベースのため、テキスト抽出の品質は PDF の作成方法に依存する（スキャン PDF の場合は OCR が必要）
- ファイル名の命名規則が年度・会議によって統一されていない（タイムスタンプ形式、日付+種別形式が混在）
- 会議録以外の文書（議案・資料・審議結果）が混在するため、リンクテキストとファイル名の両方で判別する
- ドメインが `town.biei.hokkaido.jp` と `www.town.biei.hokkaido.jp` の2パターンで参照される場合がある
- 令和8年のページ URL のみ命名規則が異なる（`proceedings8.html`）ため、トップページからのリンク収集が必須

---

## 推奨アプローチ

1. **トップページ起点でクロール**: URL 命名規則が不統一なため、ハードコードせずトップページからリンクを動的に収集する
2. **リンクテキストで文書種別を判別**: ファイル名だけでなくアンカーテキスト（「会議録」等）を優先的に使用する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **PDF テキスト品質の事前確認**: 実装前に数件の PDF をダウンロードしてテキスト抽出の品質を確認する
5. **差分更新**: 年度ページ単位で管理し、過去年度は再取得不要とする
