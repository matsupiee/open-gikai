# 川南町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.kawaminami.miyazaki.jp/site/gikai/
- 分類: 自治体公式サイト内で PDF 形式の会議録を公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は全て PDF ファイルで提供。一般質問は議員ごとに個別 PDF に分割

## スクレイピング戦略

### Step 1: 年度別一覧ページの URL 収集

会議録一覧ページ `/site/gikai/1028.html` から年度ごとのリンクを収集する。

- 各年度は `/soshiki/11/{記事ID}.html` 形式のリンクで列挙されている
- リスト形式で表示されており、ページネーションなし

### Step 2: 個別会議録ページの URL 収集

各年度別一覧ページ `/soshiki/11/{記事ID}.html` から個別の会議録ページへのリンクを収集する。

- リンクテキストに「令和X年第Y回（Z月）定例会」等の会議名が含まれる
- ページネーションなし

**収集方法:**

1. 年度別一覧ページの HTML を取得
2. `/soshiki/11/{記事ID}.html` パターンの内部リンクを Cheerio で抽出
3. リンクテキストから会議名・開催時期を抽出

### Step 3: PDF URL の収集

各個別会議録ページから PDF ファイルの URL を収集する。

- PDF は `/uploaded/attachment/{ファイルID}.pdf` 形式
- 定例会・臨時会: 1ページにつき1つの PDF
- 一般質問: 1ページにつき議員数分の PDF（議員名がリンクテキストに含まれる）

**収集方法:**

1. 個別会議録ページの HTML を取得
2. `/uploaded/attachment/{ファイルID}.pdf` パターンのリンクを Cheerio で抽出
3. リンクテキストから議員名（一般質問の場合）を取得

### Step 4: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用
- 抽出したテキストから発言者・発言内容を構造化

---

## PDF 内の会議録構造（想定）

### メタ情報

PDF のタイトルや冒頭部分から以下を抽出:

- 開催日
- 会議名（第X回定例会/臨時会）
- 開催回次

### 一般質問 PDF の構造

- 議員ごとに個別 PDF として分割済み
- リンクテキストに議員名が含まれるため、PDF ダウンロード前に議員名を把握可能

---

## 注意事項

- 会議録は全て PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- 一般質問は議員ごとに個別 PDF に分割されており、1回の定例会で7〜10ファイル程度になる
- 記事 ID は連番ではなく不規則なため、一覧ページからのリンク収集が必須
- PDF ファイル ID（`/uploaded/attachment/{ファイルID}.pdf`）も不規則な採番

---

## 推奨アプローチ

1. **3段階クロール**: 年度一覧 → 個別会議録ページ → PDF URL の順に段階的に収集
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使用して PDF からテキストを抽出
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度別一覧ページの更新日を確認し、新規追加分のみを取得する
5. **一般質問の議員名紐付け**: リンクテキストから議員名を取得し、PDF の内容と紐付ける
