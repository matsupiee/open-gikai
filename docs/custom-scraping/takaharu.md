## 概要

- サイト: https://www.town.takaharu.lg.jp/site/gikai/list18.html
- 分類: 自治体公式サイトに PDF を直接掲載（会議録検索システムなし）
- 形式: PDF（定例会・臨時会ごとに 1 ファイル）
- 特記: バックナンバーページ（`/site/gikai/296447.html`）に全会議録が一覧掲載

## サイト構造

### 会議録トップページ（list18.html）

- 「会議録」という見出しのハブページ
- PDF への直接リンクはなく、「バックナンバー」ページへの導線のみ

### バックナンバーページ（296447.html）

- 全年度の会議録 PDF が単一ページに一覧掲載（ページネーションなし）
- 年度（令和X年）ごとにセクション分けされている
- 各セクション内に定例会・臨時会の PDF リンクが時系列順に並ぶ

### PDF リンクの形式

リンクテキスト: `令和X年　第X回[定例会|臨時会]会議録 [PDFファイル／ファイルサイズ]`
リンク先: `/uploaded/attachment/{数値ID}.pdf`

## スクレイピング戦略

### Step 1: PDF リンクの収集

バックナンバーページ `https://www.town.takaharu.lg.jp/site/gikai/296447.html` から全 PDF リンクを抽出する。

- 単一ページに全年度分が掲載されているため、ページネーション処理は不要
- `<a>` タグの `href` 属性から `/uploaded/attachment/{ID}.pdf` パターンを抽出
- リンクテキストから年度・回数・会議種別（定例会/臨時会）を取得

**収集方法:**

1. バックナンバーページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` を選択し、全 PDF リンクを抽出
3. リンクテキストから以下のメタ情報をパース:
   - 年号（令和X年）
   - 回数（第X回）
   - 会議種別（定例会/臨時会）

```typescript
// リンクテキストのパース
const meetingPattern = /(?:令和|平成)(\d+)年\s*第(\d+)回(定例会|臨時会)/;
// 例: "令和6年　第1回定例会会議録" → 年=6, 回=1, 種別="定例会"
```

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF を `https://www.town.takaharu.lg.jp/uploaded/attachment/{ID}.pdf` からダウンロードし、テキストを抽出する。

- PDF は定例会・臨時会ごとに 1 ファイル（複数日の会議が含まれる場合あり）
- ファイルサイズは 199KB〜2.78MB 程度
- PDF パーサー（pdf-parse 等）でテキスト抽出後、発言単位に分割

### Step 3: テキストのパース

PDF から抽出したテキストを解析し、会議情報と発言データに分割する。

- PDF 内の具体的なフォーマットは PDF をダウンロードして確認が必要
- 一般的な議会会議録 PDF では、以下の構造が想定される:
  - 開会日時・閉会日時
  - 出席議員一覧
  - 発言者名と発言内容

---

## 注意事項

- PDF 形式のため、テキスト抽出の精度は PDF の作成方法に依存する（スキャン画像の場合は OCR が必要）
- 定例会と臨時会が混在しており、回数の付番は年度通しの連番
- PDF ID（attachment ID）は連番ではなく、アップロード順に振られるため、ID の大小で時系列を判断できない場合がある
- バックナンバーページの URL（`296447.html`）が変更される可能性があるため、トップページ（`list18.html`）からの導線も確認する

---

## 推奨アプローチ

1. **バックナンバーページを起点にする**: 単一ページに全 PDF リンクがまとまっているため、クロール対象は 1 ページのみ
2. **PDF テキスト抽出を事前検証**: 1 件の PDF をダウンロードしてテキスト抽出の品質を確認し、OCR が必要かどうかを判断する
3. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（2〜3 秒）を設ける
4. **差分更新**: バックナンバーページの PDF リンク一覧と取得済みリストを比較し、新規追加分のみダウンロードする
