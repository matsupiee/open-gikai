## ページ構成

### セクション６「定例会・臨時会の会議録」

会議録リンクは **年度（列）× 回次（行）** のテーブル形式で掲載。

- 列: 令和2年〜令和7年（メインページ）、平成27年〜令和元年（旧資料室）

- 各セルに「○月定例会」または「○月臨時会」の種別と PDF リンクが記載

## PDF URL パターン

**ドメイン**: `https://www.town.yokoze.saitama.jp`

**パス**: `/wp-content/uploads/{YYYY}/{MM}/{ファイル名}.pdf`

ファイル名の命名規則は統一されておらず、年代によって以下の形式が混在する:

| 時期 | ファイル名パターン | 例 |
| --- | --- | --- |
| R2〜R6 | `令和{年}年第{回}回{定例/臨時}会議事録.pdf` | `令和6年第2回定例会議事録.pdf` |
| R7 | `{年}.{月}横瀬町表紙､目次､本文､署名.pdf` | `７．６横瀬町表紙､目次､本文､署名-1.pdf` |
| H29〜R1 | `H{年}.{月}kaigiroku.pdf` 等 | `H30.3kaigiroku.pdf` |
| H27〜H28 | 数字のみ・略称混在 | `2806.pdf`, `270309.pdf` |

ファイル名から URL を推定することはできないため、**必ず一覧ページから直接 URL を取得する**。

## スクレイピング戦略

### Step 1: 一覧ページから PDF リンクを収集

2つのページから PDF リンクを取得する:

1. **メインページ** (`/yokoze/about-yokoze/5696`): R2〜R7 の会議録
2. **旧資料室ページ** (`/yokoze/shiryo/120`): H27〜R1 の会議録

各ページの会議録テーブルから `<a href="...pdf">` を全件抽出する。

**収集方法:**

```typescript
// Cheerio でテーブルから PDF リンクを抽出する例
const $ = cheerio.load(html);
const links: { url: string; text: string }[] = [];

$('a[href$=".pdf"]').each((_, el) => {
  const href = $(el).attr('href');
  const text = $(el).text().trim();
  if (href) {
    links.push({
      url: href.startsWith('http') ? href : `https://www.town.yokoze.saitama.jp${href}`,
      text,
    });
  }
});
```

### Step 2: セルのコンテキストからメタ情報を補完

テーブルの行列構造から「年度」「回次」「会議種別（定例会/臨時会）」「開催月」を抽出する。

- 列ヘッダー: 「令和○年」「令和○年（R○/西暦）」形式
- 行ヘッダー: 「第○回」形式
- セル本文: 「○月定例会」「○月臨時会」形式

```typescript
// テーブル解析イメージ
// <th> から年度を取得、<td> のテキストから会議種別・月を取得
const sessionPattern = /(\d+)月(定例会|臨時会)/;
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF は本文テキストを含む議事録形式。テキスト抽出には `pdf-parse` 等を使用する。

## 注意事項

- ファイル名にマルチバイト文字（日本語）を含む URL は URL エンコードされている場合がある
- 旧資料室ページ（H27〜R1）はメインページとは別 URL のため、個別にクロールが必要
- H23〜H26 の会議録はサイト上で確認できない（備考の「H23〜」は掲載開始の起点と推定されるが、現時点ではH27以降のみ確認）
- PDF のファイル名は命名規則が統一されておらず、URL からメタ情報を推定することは困難
- 定例会は3月・6月・9月・12月の年4回が基本（各年の第1〜4回に相当するが、臨時会が挟まる場合は回次がずれる）
- ページネーションなし・単一ページ構成のため、クロール負荷は低い

---

## 推奨アプローチ

1. **2ページクロール**: メインページと旧資料室ページの計2ページから PDF リンクを収集
2. **テーブル構造解析**: 行列のヘッダーと各セルのテキストを組み合わせてメタ情報（年度・回次・種別・月）を補完
3. **URL 正規化**: 相対パスを絶対 URL に変換し、URL エンコードを適切に処理
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: PDF URL のリストを前回取得と比較し、新規追加された URL のみをダウンロード対象とする
