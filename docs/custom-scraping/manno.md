## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録一覧ページ `https://www.town.manno.lg.jp/life/4/25/88/` をフェッチし、`/site/gikai/{ID}.html` 形式のリンクを抽出する。

- 平成18年（2006年）から令和8年（2026年）まで21年分
- 年度追加時はこのページに新たなリンクが加わる

### Step 2: 年度別ページから PDF リンクを収集

各年度ページ（`/site/gikai/{ID}.html`）をフェッチし、`/uploaded/attachment/{ID}.pdf` 形式のリンクを全て抽出する。

- 各リンクに付随するテキスト（会議名・開催日）をメタ情報として保持する
- 会議名の例: `第1回3月定例会（3月3日）`、`第1回1月臨時会（1月18日）`

**収集方法:**

1. 年度別ページを Cheerio でパースし、`a[href*="/uploaded/attachment/"]` を抽出
2. リンクテキストまたは直前のテキストノードから会議名・開催日を取得
3. PDF の `attachment/{ID}` の ID は連番ではなく任意の数値のため、URL を直接収集する

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、`pdf-parse` 等のライブラリでテキストを抽出する
- 圧縮 PDF のため、テキスト抽出ツールが必要（WebFetch では内容取得不可）

### Step 4: テキストのパース

#### メタ情報

PDF 冒頭や表紙から以下を抽出:

```
令和7年第1回まんのう町議会3月定例会会議録
令和7年3月3日
```

- 開催日: `令和X年X月X日` または `平成X年X月X日` 形式
- 会議名: 年度別ページのリンクテキストを補完情報として利用

#### 発言の構造

まんのう町議会の会議録は PDF 形式のため、テキスト抽出後に発言者パターンを解析する。

一般的な町議会 PDF の発言者パターン（要実物確認）:

```
○議長（氏名）
○○○番（氏名）
○町長（氏名）
```

- `○` で始まり、役職または議席番号 + 括弧内に氏名
- 発言内容は発言者行の後に続くテキスト

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（田中太郎） → role="議長", name="田中太郎"

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;
```

---

## 注意事項

- **発言者情報の有無**: PDF テキスト抽出後に確認が必要。スキャン PDF の場合は OCR が必要になる可能性がある。
- **PDF サイズ**: 一部のファイル（平成18年等）は 2MB を超えるため、タイムアウト設定に注意。
- **旧形式**: 平成時代の PDF はスキャン画像の可能性があり、テキスト抽出できない場合がある。
- **ページ構造の変動**: 平成18年の定例会は第1〜3回の3回制、近年は第1〜4回の4回制と異なる。

---

## 推奨アプローチ

1. **全量 URL 収集を優先**: 一覧ページ → 年度ページ → PDF URL の2段階でリンクを収集
2. **メタ情報はリンクテキストから取得**: 会議名・開催日は PDF 本文ではなく年度別ページのリンクテキストを利用
3. **PDF テキスト抽出の検証**: まず数ファイルをサンプルとしてテキスト抽出し、発言者パターンを実物確認してからパーサーを実装
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 年度別ページの PDF リンク一覧を保存し、新規追加分のみをダウンロードする
