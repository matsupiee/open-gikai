# 由仁町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.yuni.lg.jp/chosei/gikai/teireikai
- 分類: WordPress による PDF 公開（会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 会議録はすべて PDF 形式で公開。定例会・臨時会で別ページに分かれている

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 定例会の結果 | `https://www.town.yuni.lg.jp/chosei/gikai/teireikai` |
| 臨時会の結果 | `https://www.town.yuni.lg.jp/chosei/gikai/rinjikai` |
| 会議録 PDF | `https://www.town.yuni.lg.jp/wp-content/uploads/{年}/{月}/{ファイル名}.pdf` |

---

## PDF ファイル命名規則

### 会議録（近年の統一的なパターン）

```
R{年度}_{回数}{定|臨}_{日数}.pdf
```

例:
- `R07_1定_01.pdf` → 令和7年第1回定例会 1日目
- `R07_1定_02.pdf` → 令和7年第1回定例会 2日目
- `R07_１臨_01.pdf` → 令和7年第1回臨時会 1日目
- `R06_３定_01.pdf` → 令和6年第3回定例会 1日目

### 注意事項

- 全角数字と半角数字が混在する（例: `R07_1定_01` vs `R07_３定_01`）
- 一部ファイルにサフィックスがつく（例: `R07_４定_01-1.pdf`, `R06_２定_01-1.pdf`）
- 令和4年以前はファイル名の規則が不統一（例: `R04_1teireikai01.pdf`, `R040117kaigiroku.pdf`, `20221206-kaigiroku_1.pdf`）

## ページ構造

### 定例会ページ（teireikai）

- 年度ごとに表形式で会議結果・会議録の PDF リンクを掲載
- 令和4年〜令和8年が掲載（令和8年は会議結果のみ、会議録は未掲載の場合あり）
- 各定例会は年4回（第1回〜第4回）
- ページネーションなし（全件が単一ページに表示）
- 検索機能なし

### 臨時会ページ（rinjikai）

- 同様に年度ごとの表形式
- 令和4年〜令和8年が掲載
- 回数は年度によって異なる（1〜5回）

---

## PDF 内部構造

### メタ情報（1〜3ページ目）

1ページ目冒頭に以下の形式でタイトル・日付が記載:

```
令和７年由仁町議会第１回定例会　第１号

令和７年３月４日（火）
```

続いて議事日程、出席議員、出席説明員、出席事務局職員の一覧。

### 出席議員の表記

```
○出席議員（９名）
　議長　９番　後　藤　篤　人　君　　副議長　８番　早　坂　寿　博　君
　　　　１番　浮　田　孝　雄　君　　　　　　２番　加　藤　重　夫　君
```

- 議員数は9名（小規模議会）

### 発言の構造（4ページ目以降）

発言者パターン:

```
○議長（後藤篤人君）
○町長（松村　論君）
○２番（加藤重夫君）
○教育長（石井　洋君）
○総務課長（河合髙弘君）
```

- `○` で始まり、役職または番号 + 括弧内に氏名 + 「君」
- 発言内容は発言者行の後に続くテキスト

### 議事の区切り

```
◎開会の宣告
◎開議の宣告
◎日程第１　会議録署名議員の指名
◎日程第２　会期の決定
◎日程第３　諸般の報告
◎日程第４　行政報告
```

- `◎` で始まる行が議事の区切り

### その他の表記

```
（「なし」の声あり）
（「異議なし」の声あり）
◎開会　午前　９時３０分
◎散会　午前１１時１５分
```

## スクレイピング戦略

### Step 1: PDF URL の収集

定例会ページ・臨時会ページの HTML から会議録 PDF のリンクを抽出する。

- 定例会: `https://www.town.yuni.lg.jp/chosei/gikai/teireikai`
- 臨時会: `https://www.town.yuni.lg.jp/chosei/gikai/rinjikai`

**収集方法:**

1. 各ページの HTML を取得
2. `<a>` タグから `wp-content/uploads/` を含む PDF リンクを Cheerio で抽出
3. 会議結果 PDF（`議決結果` を含むリンクテキスト）は除外し、会議録 PDF のみを対象にする

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF はテキスト埋め込み型（画像 PDF ではない）のため、`pdf-parse` 等でテキスト抽出可能
- ページ数は 20〜40 ページ程度

### Step 3: 会議録のパース

#### メタ情報の抽出

PDF 冒頭から以下を抽出:

```
令和７年由仁町議会第１回定例会　第１号
令和７年３月４日（火）
```

- 開催日: `令和X年X月X日` 形式（全角数字）
- 会議名: `由仁町議会第{N}回{定例会|臨時会}　第{M}号`

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（後藤篤人君） → role="議長", name="後藤篤人"
// 例: ○２番（加藤重夫君） → role="２番", name="加藤重夫"

// 議事区切りの検出
const agendaPattern = /^◎(.+)/;
// 例: ◎日程第１　会議録署名議員の指名

// 開催日の抽出
const datePattern = /令和(\d+)年(\d+)月(\d+)日/;

// 会議タイトルの抽出
const titlePattern = /令和\d+年由仁町議会第(.+?)回(定例会|臨時会)/;
```

---

## 注意事項

- 会議録はすべて PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF ファイル名の命名規則が年度によって不統一（令和4年以前は特にばらつきが大きい）
- 会議結果 PDF と会議録 PDF が同じページに混在しているため、リンクテキストで区別する必要がある
- 議員数が9名の小規模議会のため、データ量は比較的少ない
- 全角・半角数字が混在するため、パース時に正規化が必要

---

## 推奨アプローチ

1. **HTML からの PDF URL 収集を優先**: 定例会・臨時会の2ページのみをスクレイピングすれば全 PDF URL を取得可能
2. **PDF テキスト抽出**: `pdf-parse` 等のライブラリでテキストを抽出し、正規表現でパース
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: HTML ページの PDF リンク一覧を取得し、既存データベースと照合して未取得分のみダウンロード
