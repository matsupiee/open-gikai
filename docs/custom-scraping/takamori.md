## ページ構造

### 会議録一覧ページ

- `<ul>` リスト形式で年度別リンクを列挙
- ページネーションなし（全25年度が単一ページに表示）
- 各 `<li>` 内の `<a>` タグに年度別ページへのリンクが含まれる

### 年度別ページ

- 各年度のページに、その年の会議録 PDF へのリンクが一覧表示される
- 会議は定例会（年4回）と臨時会（年複数回）に分類される
- 1つの会議が複数 PDF に分割されている場合あり（例: `（1）`, `（2）`, `（3）`）

**リンクテキストの例（令和6年）:**

```
令和6年第1回臨時会（1月）
令和6年第2回臨時会（2月）
令和6年第1回定例会（3月）（1）
令和6年第1回定例会（3月）（2）
令和6年第1回定例会（3月）（3）
令和6年第3回臨時会（3月）
令和6年第4回臨時会（4月）
令和6年第2回定例会（6月）（1）
...
令和6年第4回定例会（12月）（3）
```

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録一覧ページ `list32-138.html` から各年度別ページへのリンクを収集する。

- 年度別ページ ID は連番（5885〜5910）で、上記テーブルで管理
- 一覧ページをパースして `<a href="/site/gikai/\d+\.html">` を抽出することでも取得可能

### Step 2: 年度別ページから PDF リンクを収集

各年度別ページ（`/site/gikai/{ページID}.html`）にアクセスし、PDF リンクを収集する。

- PDF へのリンクは `<a href="/uploaded/attachment/\d+\.pdf">` の形式
- リンクテキストから会議名・会議種別・開催月・分割番号を抽出する
- 1ページに年間全ての PDF リンクが列挙される

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、テキストを抽出する。

- PDF は `https://www.town.kumamoto-takamori.lg.jp/uploaded/attachment/{添付ID}.pdf` からダウンロード
- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用

#### メタ情報の抽出（リンクテキストから）

リンクテキストのパターン:

```
{元号}{年}年第{N}回{種別}（{月}月）
{元号}{年}年第{N}回{種別}（{月}月）（{分割番号}）
```

**抽出用正規表現（案）:**

```typescript
// 会議名の抽出
const titlePattern = /^((?:令和|平成)\d+年第\d+回(?:定例会|臨時会)(?:（\d+月）)?(?:（\d+）)?)/;

// 開催年の抽出
const yearPattern = /(令和|平成)(\d+)年/;

// 会議種別の抽出
const typePattern = /第(\d+)回(定例会|臨時会)/;

// 開催月の抽出
const monthPattern = /（(\d+)月）/;
```

---

## 注意事項

- PDF テキスト抽出の精度は PDF の作成方法に依存する（スキャン画像の場合は OCR が必要な可能性あり）
- 古い年度（平成13〜20年頃）の PDF はスキャン画像の可能性があるため、事前に確認が必要
- 1会議が複数 PDF に分割されている場合、分割番号（`(1)`, `(2)`, `(3)`）を保持して管理する
- レート制限: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける

---

## 推奨アプローチ

1. **年度別ページ URL は固定管理**: 上記テーブルの25年度分を静的に管理し、新年度は追加する
2. **PDF リンクの全量収集**: 各年度別ページをパースして PDF リンクとリンクテキストを収集
3. **差分更新**: 取得済みの PDF 添付 ID を記録し、新規追加分のみダウンロードする
4. **PDF テキスト品質確認**: 古い年度の PDF はスキャン画像の可能性があるため、テキスト抽出前にサンプルチェックを実施
