## ページ構造

### 会議録一覧ページ

会議録一覧ページ (`378.html`) に全年度の会議録が掲載されている。

- 年度ごとに `<h3>` 相当の見出しで区切られている（例: 「令和7年」「令和6年」）
- 各年度内に PDF ダウンロードリンクが新しい順に列挙されている
- リンクテキストに会議名とファイルサイズが含まれる

**リンクテキストの例:**

```
令和7年 第7回12月定例会 (PDFファイル: 1.1MB)
令和6年 第1回1月臨時会 (PDFファイル: 149.1KB)
```

## スクレイピング戦略

### Step 1: 会議録一覧ページから PDF URL を収集

会議録一覧ページ (`378.html`) を 1 回取得するだけで、全年度の PDF リンクを収集できる。

```typescript
// PDF リンクの抽出例
const pdfLinks = $('a[href*="/material/files/group/14/"][href$=".pdf"]')
  .map((_, el) => ({
    url: new URL($(el).attr('href')!, BASE_URL).toString(),
    label: $(el).text().trim(),
  }))
  .toArray();
```

### Step 2: 会議メタ情報の抽出

`<a>` テキストから会議名・年度・回次・種別を抽出する。

**テキスト例:**

- `令和7年 第7回12月定例会 (PDFファイル: 1.1MB)`
- `令和6年 第1回1月臨時会 (PDFファイル: 149.1KB)`
- `平成28年 第1回3月定例会 (PDFファイル: 1.2MB)`

```typescript
// 会議名から年度・回次・月・種別を抽出
const sessionPattern = /(?:令和|平成)(\d+)年\s*第(\d+)回(\d+)月(定例会|臨時会)/;
// 例: "令和7年 第7回12月定例会" → era="令和", year=7, count=7, month=12, type="定例会"
```

### Step 3: 各 PDF を取得・テキスト抽出

PDF ファイルを取得し、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

## 注意事項

- **1 ページ完結**: 全年度の会議録が 1 ページに掲載されているため、複数ページのクロールは不要
- **PDF URL は推測不可**: ファイル ID に規則性がないため、必ず HTML から `<a href>` を抽出して URL を収集する
- **プロトコル省略**: PDF リンクが `//www.town.wake.lg.jp/...` のようにプロトコル省略形式で記載されている場合がある。`https:` を先頭に付与して正規化すること
- **レート制限**: 自治体サイトのため、PDF ダウンロード時にリクエスト間で適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **1 リクエストで全量収集**: 会議録一覧ページを 1 回取得するだけで全 PDF リンクを収集可能。年度別ページの巡回は不要
2. **リンクから直接収集**: PDF URL は HTML の `<a href>` から取得し、URL を推測・構築しない
3. **テキストからメタ情報取得**: リンクテキストに年度・回次・月・種別が含まれるため、正規表現で抽出する
4. **差分更新**: 既取得 URL のリストと比較し、新規 URL のみ PDF を取得する
5. **レート制限**: PDF ダウンロード時に各リクエスト間で 1〜2 秒の待機を設ける
