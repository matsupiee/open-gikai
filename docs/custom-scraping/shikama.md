## 会議録 PDF の構造

### PDF ファイル名パターン

ファイル名の命名規則は年度によって異なり、一貫性がない。

**令和4年以降（定例会）:**

- `gijirokuR6MM-N.pdf` — 令和6年MM月第N日目（例: `gijirokuR612-1.pdf`）
- `kaigirokuR6MM-N.pdf` — 一部月で `kaigiroku` プレフィックス（例: `kaigirokuR606-1.pdf`）
- `gikaiR8MMDD.pdf` — 令和7年以降の一部（例: `gikaiR80327.pdf`）

**令和3年:**

- `YYYYMMDD-N.pdf` — 西暦年月日-連番（例: `20211207-1.pdf`）

**特別委員会:**

- `yosanR7MMDD.pdf` — 予算審査（例: `yosanR70310.pdf`）
- `kextusanR6-N.pdf` — 決算認定審査（例: `kextusanR6-1.pdf`）

### PDF ベース URL

```
https://www.town.shikama.miyagi.jp/material/files/group/15/
```

全ての PDF はこのディレクトリ配下に格納されている。

### PDF 内部構造（想定）

PDF は Word 等から生成されたものと推定され、テキストレイヤーが含まれている可能性が高い。

#### 発言者パターン（想定）

```
○議長（○○○○君）
○町長（○○○○君）
○○番（○○○○君）
```

- `○` で始まり、役職または議席番号 + 括弧内に氏名

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（○○○○君） → role="議長", name="○○○○君"

// 開催日の抽出
const datePattern = /(?:令和|平成)\s*(\S+?)年\s*(\S+?)月\s*(\S+?)日/;
```

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録トップページから各年度ページへのリンクを収集する。

1. トップページ `https://www.town.shikama.miyagi.jp/chosei/gikai/2363.html` を取得
2. `/soshiki/gikai/1/kaigiroku/{ID}.html` パターンのリンクを全て抽出
3. ページネーションなし（全件が1ページに表示される）

### Step 2: 会議録 PDF URL の収集

各年度ページから PDF ダウンロードリンクを収集する。

1. 各年度ページ（`kaigiroku/{ID}.html`）を取得
2. `/material/files/group/15/*.pdf` パターンのリンクを全て抽出
3. 各リンクの周辺テキストから会議種別・開催日を取得
4. 予算・決算特別委員会ページ（`2233.html`）も同様に処理

### Step 3: PDF のダウンロードとテキスト抽出

1. 会議録 PDF をダウンロード
2. `pdftotext` 等のツールで PDF からテキストを抽出
3. Word 由来の PDF であればテキスト抽出の精度は比較的良好（OCR 不要の可能性が高い）

### Step 4: 会議録のパース

抽出したテキストから以下を構造化する:

1. **メタ情報の抽出**
   - 会議名: PDF 冒頭のタイトル行または周辺テキストから
   - 開催日: リンクテキストまたは PDF 内の日付情報から
   - 会議種別: 定例会 / 臨時会 / 予算審査特別委員会 / 決算認定審査特別委員会

2. **発言の抽出**
   - `○` で始まる行を発言者の区切りとして分割
   - 発言者の役職と氏名を正規表現で抽出
   - 発言者行以降、次の発言者行までを発言内容として取得

---

## 注意事項

- PDF ファイル名の命名規則が年度ごとに異なるため、ファイル名からのメタ情報推定は困難。HTML ページのリンクテキストからメタ情報を取得すること
- 令和3年以前のデータが存在するかは不明（トップページに令和3年以降のリンクのみ確認）
- ページ ID（`2233.html`、`2235.html` など）は連番ではないため、差分更新にはトップページの再クロールが必要
- PDF のテキストレイヤーの品質は実際にダウンロードして確認する必要がある（OCR が必要な場合もあり得る）

---

## 推奨アプローチ

1. **トップページから全量収集**: 会議録トップページを起点に、全年度ページの URL を収集
2. **年度ページから PDF リンク収集**: 各年度ページを巡回し、PDF の URL と周辺テキスト（日付・会議種別）を取得
3. **PDF ダウンロード**: 全 PDF をダウンロードし、`pdftotext` でテキスト変換
4. **正規表現でパース**: `○` パターンで発言者を分割し、メタ情報と合わせて構造化
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
6. **差分更新**: トップページを定期的に再クロールし、新しい年度ページや PDF リンクの追加を検出する
