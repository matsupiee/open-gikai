### PDF の URL パターン

```
https://www.city.misawa.lg.jp/index.cfm/24,11423,c,html/11423/{ファイル名}.pdf
```

ファイル名の命名規則は時期によって異なる:

| 時期 | ファイル名パターン | 例 |
|------|-------------------|-----|
| 近年（令和〜） | `YYYYMMDD-HHMMSS.pdf`（アップロード日時） | `20250709-093446.pdf` |
| 平成後期 | `YYYYMMDD-HHMMSS.pdf` | `20181106-115944.pdf` |
| 平成中期 | `H{年}.{回}.{日目}.pdf` や任意の名前 | `H26.1.1.pdf`, `rinnjikai.pdf` |

## スクレイピング方針

### 全体フロー

```
1. 一覧ページを取得
   ↓
2. 年度セクションごとにPDFリンクを抽出
   ↓
3. リンクテキスト・位置から会議名・日目を判定
   ↓
4. PDFをダウンロード
   ↓
5. PDFからテキストを抽出
```

### Step 1: 一覧ページの取得

- `https://www.city.misawa.lg.jp/index.cfm/24,11423,118,420,html` を取得
- ページネーションがないため、1回のリクエストで全データを取得可能

### Step 2: PDFリンクの抽出

- `/index.cfm/24,11423,c,html/11423/` で始まる `.pdf` リンクを全て抽出
- 各リンクの周辺テキストから年度・会次・日目を判定
- テーブル構造（行: 会次、列: 日目）を利用してメタデータを紐付け

### Step 3: PDFのダウンロードとテキスト抽出

- PDFファイルをダウンロード
- PDF解析ライブラリ（pdf-parse等）でテキストを抽出
- 発言者・発言内容の構造化が必要（PDF内のレイアウトに依存）

## 注意事項

- 詳細ページは存在せず、一覧ページから直接PDFにリンクされている
- ファイル名の命名規則が時期によって異なるため、URLパターンだけでメタデータを抽出するのは困難
- PDFリンクの周辺HTML構造（テーブルの行・列位置）からメタデータを取得する必要がある
- 古い会議録（平成20年代前半）はファイル名が日本語や和暦を含む場合がある
- サイトへのアクセス間隔は適切に設定すること（1秒以上のウェイト推奨）

## メタデータ抽出

| フィールド | 抽出元 | 抽出方法 |
|-----------|--------|---------|
| 年度 | 一覧ページのセクション見出し | `令和○年` / `平成○年` を正規表現で抽出 |
| 会議種別 | テーブル行のラベル | `定例会` / `臨時会` を判定 |
| 会次 | テーブル行のラベル | `第○回` を正規表現で抽出 |
| 日目 | テーブル列の位置 | 1列目=1日目、2列目=2日目、... |
| ファイルサイズ | リンクテキスト | `[ XXX KB pdf]` を正規表現で抽出 |
