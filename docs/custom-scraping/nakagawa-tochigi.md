# 那珂川町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.tochigi-nakagawa.lg.jp/05gikai/kaigiroku/
- 分類: 町独自 CMS による HTML + PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録本文は PDF ファイルで提供。年別にページが分かれており、平成21年（2009年）から令和7年（2025年）まで掲載。

## ページ階層

### Level 1: 会議録トップページ

トップページには以下が掲載される:

- 最新の会議録 5 件へのリンク
- 年別一覧へのリンク（平成21年〜令和7年）

### Level 2: 年別一覧ページ

各年のページに、その年に開催された定例会・臨時会の一覧がリスト表示される。

- ページネーションなし（1 ページに全会議を表示）
- 各エントリは `{年号}年第{N}回那珂川町議会{定例会|臨時会}（{月}月）会議録` の形式
- リンク先は会議録詳細ページ（日程表）

### Level 3: 会議録詳細ページ（日程表）

テーブル形式で会期中の各日の議事内容を表示:

- 列構成: 月日、曜日、時間、議事・議案
- 月日のセルが PDF ファイルへのリンクになっている
- 一般質問の議員名と質問項目が議事欄に記載

### Level 4: PDF ファイル（会議録本文）

発言の全文は PDF ファイルに格納されている。

## PDF ファイル命名規則

PDF のファイル名は年代によって命名規則が異なる:

### 近年（令和4年頃〜）

`R{和暦年}.{月}.{通し番号}.pdf`

例:
- `R6.12.3.pdf` → 令和6年12月3日
- `R6.6.4.pdf` → 令和6年6月4日

### 中期（令和元年〜令和3年頃）

`teireikai{通し番号}t{日番号}.pdf` のような形式

例:
- `teireikai3t1.pdf` → 定例会3回目の1日目

### 初期（平成21年〜）

`{西暦下2桁}kaigiroku{月}gatuteirei{通し番号}.pdf`

例:
- `09kaigiroku3gatuteirei1.pdf` → 平成21年3月定例会1日目

※ 命名規則が統一されていないため、各詳細ページから PDF リンクを直接抽出する必要がある。

## スクレイピング戦略

### Step 1: 年別一覧から会議録詳細ページの URL を収集

会議録トップページから年別一覧ページ（2009〜2025）を巡回し、各年の会議録詳細ページへのリンクを収集する。

**収集方法:**

1. `https://www.town.tochigi-nakagawa.lg.jp/05gikai/kaigiroku/{西暦}/index.html` を 2009〜2025 の範囲で取得
2. 各ページのリスト要素から会議録詳細ページへのリンクを抽出
3. リンクテキストから会議名（定例会/臨時会、回次、月）を抽出

### Step 2: 会議録詳細ページから PDF リンクとメタ情報を収集

各詳細ページのテーブルから以下を抽出:

- PDF ファイルへのリンク（月日セルの `<a>` タグの `href` 属性）
- 開催日（月日 + 年別一覧の年から算出）
- 議事内容（一般質問の議員名・質問項目など）

**収集方法:**

1. テーブルの各行を走査
2. 月日セル内の `<a>` タグから PDF の相対パスを取得し、絶対 URL に変換
3. 議事欄のテキストから議員名を正規表現で抽出

### Step 3: PDF ファイルのダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

**抽出方法:**

1. PDF をダウンロード（`files/` ディレクトリ配下）
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者と発言内容を分離

### Step 4: 会議録のパース

#### メタ情報

詳細ページの見出しから抽出:

```
令和６年第７回那珂川町議会定例会（１２月）会議録
```

- 年号: `令和X年` or `平成X年`
- 回次: `第N回`
- 会議種別: `定例会` or `臨時会`
- 月: `（M月）`

#### メタ情報パース用正規表現（案）

```typescript
// 会議名の抽出
const meetingPattern = /(?:令和|平成)(\S+?)年第(\S+?)回那珂川町議会(定例会|臨時会)（(\S+?)月）/;

// 開催期間の抽出
const periodPattern = /(\d+)月(\d+)日.+?から(\d+)月(\d+)日/;
```

#### 発言者の抽出（PDF テキストから）

PDF から抽出したテキストの発言者パターンは PDF の書式に依存するため、実際の PDF テキスト出力を確認した上でパーサーを実装する必要がある。

一般的な議会会議録 PDF の発言者パターン:

```typescript
// 発言者の抽出（要実 PDF テキスト確認後に調整）
const speakerPattern = /^○(.+?)(?:（(.+?)）)?/;
// 例: ○議長（矢後紀夫） → role="議長", name="矢後紀夫"

// 議員番号付きパターン
const numberedSpeakerPattern = /^○(\d+番)\s*（(.+?)）/;
```

※ 上記は仮のパターン。実際の PDF テキスト出力を確認して調整が必要。

## 注意事項

- 会議録本文は PDF のみで提供されており、HTML での本文閲覧はできない
- PDF の命名規則が年代によって異なるため、URL の直接生成ではなく詳細ページからのリンク抽出が必須
- 一部の古い会議録詳細ページの URL パターンが `{日付}-{時刻}-64.html` 形式（例: `2009-0716-1135-64.html`）と、それ以外の形式（例: `3gasuteireikaikaigiroku.html`）が混在している
- Google Custom Search（`cx = '005398719686345367285:nr_nq7jnn54'`）が実装されているが、会議録の全文検索には使えない
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **トップダウンで巡回**: トップページ → 年別一覧 → 詳細ページ → PDF の順に階層的にクロール
2. **PDF リンクは詳細ページから取得**: 命名規則が不統一のため、URL を直接生成せず必ず HTML から抽出する
3. **PDF テキスト抽出ライブラリの選定**: `pdf-parse` や `pdfjs-dist` などを使用し、テキスト抽出の精度を事前検証する
4. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
5. **差分更新**: 年別一覧ページの構成が変わらないため、最新年のみを再クロールする差分更新が可能
