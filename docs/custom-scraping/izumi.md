# 出水市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.kagoshima-izumi.lg.jp/gikai/gijiroku/
- 分類: 出水市独自の議事録検索システム（HTML 公開）
- 文字コード: UTF-8
- 特記: DiscussVision 社の映像配信システム（https://smart.discussvision.net/smart/tenant/kagoshima-izumi/WebView/rd/council_1.html）も存在するが、**テキスト会議録は出水市公式サイト側が提供**。映像配信システムはテキスト会議録を提供していないため、スクレイピング対象外とする。

---

## テキスト会議録の所在

| 項目 | 詳細 |
| --- | --- |
| 議事録検索トップ | `https://www.city.kagoshima-izumi.lg.jp/gikai/gijiroku/` |
| 会議録詳細 | `https://www.city.kagoshima-izumi.lg.jp/gikai/gijiroku/detail_select/{会議ID}/{発言者ID}` |
| データ期間 | 平成18年（2006年）第1回定例会〜現在 |

## URL 構造

### 会議録詳細ページ

```
https://www.city.kagoshima-izumi.lg.jp/gikai/gijiroku/detail_select/{会議ID}/{発言者ID}
```

- `{会議ID}`: 整数の連番。平成18年第1回定例会は `1`、令和7年第4回定例会（6日分）は `525` 前後
- `{発言者ID}`: 各会議内の発言者に割り当てられた整数（`1` が最初の発言者＝議長）
- 1 つの `{会議ID}` に対して複数の `{発言者ID}` が存在する（1 会議日あたり多数の発言者）
- `{発言者ID}` が存在しない番号は 404 を返す

### 会議録の単位

1 つの会議日 = 1 つの `{会議ID}` に対応。定例会は 1 回あたり複数日開催のため、複数の `{会議ID}` が存在する。

例（令和7年第4回定例会）:

| 開催日 | 会議ID |
| --- | --- |
| 令和7年11月21日 第1日 | 520 前後 |
| 令和7年12月4日 第2日 | 521 前後 |
| …（計6日） | … |

※ 実際の ID は調査フェーズで特定する。連番のため ID 範囲を総当たりで確認可能。

## スクレイピング戦略

### Step 1: 会議 ID の収集

検索トップページ（`/gikai/gijiroku/`）の会議名 select 要素から全会議の ID を収集する。

1. `/gikai/gijiroku/` を取得
2. `name=search_form` フォーム内の会議名 `<select>` の `<option value="{会議ID}">` を抽出
3. 全 option を列挙し、会議 ID のリストを作成

### Step 2: 各会議の発言者 ID の収集

各会議詳細ページの目次（`<ul><li>` リスト）から発言者 ID を収集する。

1. `detail_select/{会議ID}/1` を取得（議長の発言ページ）
2. ページ内の目次リスト（`<li><a href="javascript:void(0);">` 形式）から発言者番号を抽出
3. 発言者番号の一覧を取得（例: 10, 20, 30 など）

または、`detail_select/{会議ID}/{発言者ID}` を `1` から順に試行し、404 が返るまで取得する方法も有効。

### Step 3: 会議録本文の取得とパース

各 `detail_select/{会議ID}/{発言者ID}` ページから発言テキストを抽出する。

**メタ情報の取得（会議ID/1 のページから）:**

```
出水市令和7年第4回定例会会議録第1号
令和7年11月21日（金曜日）午前10時00分開議
出水市議会議場
```

- 会議名・定例会名・日付は本文冒頭のテキストから抽出
- 日付形式: `令和X年XX月XX日（曜日）`

**発言のパース:**

発言者と発言内容はテキストベースの形式で記述されている:

```
○議長（杉本尚喜議員）　ただいまの出席議員は...
○椎木伸一市長　提案理由を...
```

- `○` で始まる行が発言者を示す
- 役職・氏名の形式: `○役職（氏名）` または `○氏名役職`
- 発言本文は `○` 行の後に続くテキスト
- セクション区切り: `△　開　会`、`△　日程第1...` 形式

**JavaScript による表示制御:**

```javascript
function dispChange(cbox, to_checked) {
  var text_id = cbox.val();
  // #text_{発言者ID} の要素を show/hide
}
```

- 各発言者の内容は `id="text_{発言者ID}"` の要素に格納されている
- HTML 取得後は jQuery の show/hide に関わらず全テキストを取得可能

**パース用正規表現（案）:**

```typescript
// 発言者の抽出
const speakerWithRole = /^○(.+?)（(.+?)）/; // ○役職（氏名）
const speakerWithoutParens = /^○(.+?)(?:　|$)/; // ○氏名役職（括弧なし）

// セクション区切り
const sectionPattern = /^△\s+(.+)/;

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;
```

## DiscussVision システムについて

出水市議会は DiscussVision 社の映像配信システム（`smart.discussvision.net`）も導入している。

| 項目 | 詳細 |
| --- | --- |
| URL | `https://smart.discussvision.net/smart/tenant/kagoshima-izumi/WebView/rd/council_1.html` |
| 用途 | 会議映像の配信・アーカイブ |
| テキスト会議録 | **提供なし**（映像配信のみ） |

DiscussVision システムはキーワード検索・発言者絞り込み等の UI を持つが、テキスト会議録はなく映像へのアクセスが主目的。**スクレイピング対象は出水市公式サイトのみ**とする。

---

## 注意事項

- 会議 ID は連番だが欠番がある可能性があるため、404 エラーを適切にハンドリングする
- `detail_select/{会議ID}/{発言者ID}` の `{発言者ID}` は会議ごとに異なる。目次リストから正確に取得する
- 議事録全体は `{会議ID}/1` のページにまとめて記載されているため、発言者フィルタリングを使わず全文取得が効率的な場合がある
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- Google Tag Manager（GTM-M569953）が使用されているが、スクレイピングには影響なし

---

## 推奨アプローチ

1. **会議 ID の全量収集**: 検索トップの select 要素から全会議 ID を取得
2. **会議録全文は `{会議ID}/1` から取得**: 議長ページに会議録全文が記載されているため、発言者 ID を個別に叩く必要がない場合がある。実装前に確認する
3. **差分更新**: 会議 ID の最大値を記録し、前回取得済み ID 以降のみを対象にする
4. **レート制限**: リクエスト間隔 1〜2 秒を設ける
