## PDF ファイル命名規則

### H19〜現在（`files/` 直下）

```
files/{年号}{月}-{種別}.pdf
```

- 年号: `R07`, `R06`, `H28` 等
- 月: 開催月（`.` 区切り、ただし一部 `-` 区切りあり）
- 種別:
  - `m` = 目次
  - `1`, `2`, `3` ... = 議事録（分割番号）
  - `k` = 審議結果
  - `s` = 参考資料

例:
- `R06.12-m.pdf` → 令和6年12月 目次
- `R06.12-1.pdf` → 令和6年12月 議事録-1
- `R06.12-k.pdf` → 令和6年12月 審議結果
- `H28.3-m.pdf` → 平成28年3月 目次
- `H28.5r-m.pdf` → 平成28年5月臨時会 目次（`r` 付きは臨時会の可能性）

### H17〜H18（`H{年数}/files/` 配下）

```
H{年数}/files/{年数}-{回数}{種別}.pdf
```

例:
- `H17/files/17-3m.pdf` → 平成17年第3回 目次
- `H17/files/17-3g-1.pdf` → 平成17年第3回 議事録-1（`g` = 議事録）

## 会議種別

各年度ページから以下の2種別が確認できる:

- **定例会**: 年4回程度（3月、6月、9月、12月）
- **臨時会**: 不定期（年により回数が異なる）

会議名のフォーマット:
```
【みなかみ町{年号}年第{N}回（{月}月）{定例会|臨時会}】
```

例:
- `【みなかみ町令和6年第2回（3月）定例会】`
- `【みなかみ町令和6年第1回（2月）臨時会】`
- `みなかみ町平成17年第3回（12月）定例会`（H17 は【】なし）

## PDF 内部構造

### ヘッダー情報

各 PDF の右上にヘッダーあり:
```
R6-7 (12.3) 第１号
```
- `R6-7`: 令和6年第7回
- `(12.3)`: 12月3日
- `第１号`: 会議録番号

### タイトル（1ページ目）

```
令和６年第７回（１２月）みなかみ町議会定例会会議録第１号

令和６年１２月３日（火曜日）
```

### メタ情報セクション

以下の順序で記載:

1. **議事日程** - 日程番号と議案一覧
2. **本日の会議に付した事件**
3. **出席議員** - `{番号}番 {氏名} 君` 形式
4. **欠席議員**
5. **会議録署名議員**
6. **職務のため議場に出席した事務職員の職氏名**
7. **説明のため出席した者** - `{役職} {氏名} 君` 形式

### 発言の構造

発言者パターン:
```
議　長（小林　洋君）　おはようございます。
町　長（阿部賢一君）　皆さん、おはようございます。
```

- `{役職}（{氏名}）` 形式で太字
- 役職と氏名の間に全角スペースが入る場合あり
- 役職例: `議　長`, `町　長`, `副町長`, `教育長`, `総務課長`, `財政課長` 等

一般質問の発言者:
```
◇　石坂　武　君・・・１．町長、公約実現への取り組み
◇　鈴木美香　君・・・１．アピアランスケア事業の周知と柔軟な対応を
```

- `◇` で始まり、`{氏名}　君` + 質問項目

登壇表記:
```
（町長　阿部賢一君登壇）
```

### パース用正規表現（案）

```typescript
// タイトルから会議情報を抽出
const titlePattern = /(?:令和|平成)([\d０-９]+)年第([\d０-９]+)回（([\d０-９]+)月）みなかみ町議会(定例会|臨時会)会議録第([\d０-９]+)号/;

// 開催日の抽出
const datePattern = /(?:令和|平成)([\d０-９]+)年([\d０-９]+)月([\d０-９]+)日/;

// 発言者の抽出（太字の役職 + 括弧内に氏名）
const speakerPattern = /^(.+?)（(.+?)君）/;
// 例: 議　長（小林　洋君） → role="議　長", name="小林　洋"
// 例: 町　長（阿部賢一君） → role="町　長", name="阿部賢一"

// 一般質問者の抽出
const questionerPattern = /^◇\s*(.+?)\s*君/;

// 登壇の検出
const podiumPattern = /（(.+?)君登壇）/;
```

## スクレイピング戦略

### Step 1: 年度別一覧ページの URL 収集

議事録トップページ `index.html` から各年度ページへのリンクを収集する。

- トップページに H17〜R7 の全年度リンクが列挙されている
- リンクは相対パスで `<a href="{filename}.html">` 形式
- URL パターンは2種類（CMS 生成 ID 形式 / `h{年数}.html` 形式）

### Step 2: PDF リンクの収集

各年度ページから PDF リンクを抽出する。

- 議事録 PDF は `<a href="files/{ファイル名}.pdf">` または `<a href="H{年数}/files/{ファイル名}.pdf">` 形式
- 会議名テキストは PDF リンクの前の行に `【...】` または平文で記載
- 種別の判別:
  - `目次` → `-m.pdf`
  - `議事録-N` → `-{N}.pdf` または `-g-{N}.pdf`
  - `審議結果` → `-k.pdf`
  - `参考資料` → `-s.pdf`
- **議事録本文の PDF のみを対象とする**（目次・審議結果・参考資料は除外可）

### Step 3: PDF のダウンロードとテキスト抽出

- PDF は Word 由来のレイアウトで、テキスト抽出は比較的容易
- `pdf-parse` 等のライブラリでテキスト抽出が可能
- 全角数字・全角スペースが多用されているため、正規化処理が必要

### Step 4: 会議録のパース

1. **タイトル行からメタ情報を抽出**: 年号、回数、月、定例会/臨時会、会議録番号
2. **開催日を抽出**: `令和X年X月X日（X曜日）` 形式
3. **出席議員リストを抽出**: `{番号}番 {氏名} 君` のパターン
4. **発言を分割**: `{役職}（{氏名}君）` パターンで発言者を検出し、次の発言者までをその人の発言とする
5. **一般質問を識別**: `◇` で始まる行で質問者を検出

## 注意事項

- 会議録は PDF のみで公開されており、HTML 版は存在しない
- PDF のテキスト抽出時に全角スペースや改行位置がずれる可能性がある
- H17〜H18 のファイルパスが他の年度と異なる（`H{年数}/files/` 配下）
- 定例会は議事録が複数ファイルに分割される（最大3分割程度）、臨時会は1ファイルが多い
- ファイル命名の区切り文字に揺れがある（`.` と `-` の混在: `R06.12-1.pdf` vs `R06-2-1.pdf`）
- ページネーションは存在しない（各年度ページに全会議が一括表示）

---

## 推奨アプローチ

1. **2段階クロール**: トップページ → 年度別ページ → PDF リンク収集 → PDF ダウンロード
2. **PDF テキスト抽出**: `pdf-parse` または `pdfjs-dist` でテキスト化
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度別ページの PDF リンク数を比較し、新規追加分のみダウンロード
5. **全角正規化**: 全角数字・全角スペースを半角に変換してからパースすると正規表現がシンプルになる
