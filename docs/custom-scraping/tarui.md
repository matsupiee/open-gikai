- サイト: https://www.town.tarui.lg.jp/site/gikai/list32.html
- 分類: 公式ホームページ内での年度別 PDF 直接公開（既存アダプターでは対応不可）

## HTML 構造（年度別ページ）

年度別ページは定義リスト形式（`dl/dt/dd`）で会議録を列挙している。

```html
<dl>
  <dt>[会議名]（例: 第5回定例会）</dt>
  <dd><a href="/uploaded/attachment/{ID}.pdf">目次 [PDFファイル／76KB]</a></dd>
  <dd><a href="/uploaded/attachment/{ID}.pdf">会議録　12月3日（水曜日） [PDFファイル／255KB]</a></dd>
  <dd><a href="/uploaded/attachment/{ID}.pdf">会議録　12月10日（水曜日）（一般質問） [PDFファイル／480KB]</a></dd>
  <dd><a href="/uploaded/attachment/{ID}.pdf">会議録　12月12日（金曜日） [PDFファイル／271KB]</a></dd>
</dl>
```

- `dt` に会議名（例: 「第5回定例会」「第3回臨時会」）
- `dd` に目次 PDF と各開催日の会議録 PDF を列挙
- アンカーテキストに開催日・曜日・議事内容（一般質問、提案説明 等）を記載
- ファイルサイズ（KB）も表示される

## 検索パラメータ

URL パラメータによる絞り込みや検索機能はない。年度ごとに固定 URL のページが存在し、PDF リンクが静的に列挙されている。

## スクレイピング戦略

### Step 1: 年度別ページの PDF リンク収集

インデックスページまたは上記の年度別 URL リストから、各年度ページへアクセスし、`/uploaded/attachment/` を含む PDF リンクを抽出する。

- `dt` に会議名が入り、直後の `dd` 群がその会議の PDF（目次・各日分）
- 目次 PDF（`目次` を含むアンカーテキスト）と会議録 PDF を区別する
- 会議録 PDF のアンカーテキストから開催日・議事内容を取得する

**収集方法:**

1. 各年度ページを取得し、Cheerio で `dl > dt` と `dl > dd > a[href*="/uploaded/attachment/"]` を抽出
2. `dt` のテキストを会議名として記録し、後続の `dd` の PDF を紐付ける
3. アンカーテキストから「目次」か「会議録」かを判別し、会議録 PDF のみ処理対象とする

### Step 2: PDF テキストの抽出

各会議録 PDF を取得し、`pdftotext` 相当のライブラリでテキストを抽出する。

- PDF は「JUST PDF 4」で生成されたもので、テキスト埋め込みあり（OCR 不要）
- 1 ファイルあたり 30〜70 ページ程度

### Step 3: 会議録のパース

#### メタ情報

PDF 冒頭の 1 行目に開催日と開始時刻が記載される:

```
令和６年12月11日（水曜日）午前９時
```

- 開催日: `令和X年MM月DD日（曜日）`
- 開始時刻: `午前/午後HH時`

#### 発言の構造

発言者は `○` で始まる行として出力される:

```
○議長（若山隆史君）
○３番（水野忠宗君）
○１番（江上裕子君）
○１２番（中村ひとみ君）
○町長（早野博文君）
○総務課長（藤塚正博君）
○健康福祉課長（酒井明美君）
○上下水道課長（川瀬桂一郎君）
○教育次長兼学校教育課長（小川裕司君）
```

- `○` + 役職または議席番号 + `（` + 氏名 + `君）`
- 議員は `○X番（氏名君）`（全角数字）
- 役職者は `○役職名（氏名君）`（町長、副町長、各課長など）

登壇・着席の表記:

```
〔３番　水野忠宗君登壇〕
```

- `〔` で始まり `登壇〕` で終わる行として出現する
- 議員・説明員ともに同パターン

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○３番（水野忠宗君） → role="３番", name="水野忠宗"
// 例: ○町長（早野博文君） → role="町長", name="早野博文"

// 登壇の検出
const podiumPattern = /^〔(.+?)君登壇〕/;

// 開催日の抽出（PDF冒頭）
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日（.+?）/;
```

#### 区切り線

発言ブロック間は罫線で区切られる:

```
―――――――――――――――――――――――――――――――――――――――――――――――――――
```

## 注意事項

- PDF ファイルは会議ごとに複数日分に分割されており、1 ファイル = 1 開催日分
- 目次 PDF（`目次` とアンカーテキストに含まれるもの）は本文を含まないためスキップする
- PDF の `Title` メタデータに会議名と開催日が含まれる（例: `Microsoft Word - 垂井町定例会第２号　令和６年12月11日`）
- 議席番号は全角数字（`１番`、`１２番`）で出力されるが、氏名内は通常の文字
- 氏名の後に必ず「君」が付く
- ページ区切り位置でテキストが分断される場合があるため、行単位の処理が必要

---

## 推奨アプローチ

1. **静的 URL リストを使用**: 年度ページは URL が固定のため、上記リストを起点に全年度を網羅する
2. **目次 PDF をスキップ**: アンカーテキストに「目次」を含む PDF は処理対象外
3. **PDF メタデータを活用**: `Title` フィールドから会議名・開催日を補完可能
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 取得済みの PDF ファイル ID（数値）を記録し、新規 ID のみを差分取得する
