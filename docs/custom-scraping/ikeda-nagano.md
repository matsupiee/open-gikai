## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（単一ページ） | `https://www.ikedamachi.net/0000000115.html` |
| 会議録 PDF | `https://www.ikedamachi.net/cmsfiles/contents/0000000/115/{ファイル名}.pdf` |

### PDF ファイル名の命名規則

```
{元号略称}{年}.{月}{会議種別}.pdf
```

| 要素 | 値 | 例 |
| --- | --- | --- |
| 元号略称 | `R`（令和）/ `H`（平成） | `R7`, `H30` |
| 年 | 元号の年数 | `7`, `31` |
| 月 | 開催月（1〜12） | `3`, `12` |
| 会議種別 | `T`（定例会）/ `R`（臨時会） | `T`, `R` |

**例:**

- `R7.12T.pdf` → 令和7年12月定例会
- `R7.10R.pdf` → 令和7年10月臨時会
- `H30.9T.pdf` → 平成30年9月定例会

**注意:** 大文字・小文字が混在している（`R2.9t.pdf`, `H30.12t.pdf` など小文字の `t`/`r` も存在する）。

## ページ構造

### 会議録一覧ページ

- 全年度が **単一ページ** に掲載（ページネーションなし）
- 年度ごとに `<h3>` 見出し（例: `令和7年`、`平成30年`）
- 各年度の会議録は `<ul>` > `<li>` > `<a>` のリスト形式
- リンクテキスト: `{月}月定例会 (PDF形式、{サイズ})` または `{月}月臨時会 (PDF形式、{サイズ})`

### 掲載年度

令和7年〜平成25年（約13年分、合計約97件の PDF）

## PDF 内部構造

### 全体構成

各 PDF は1つの会議全体の会議録を含む（複数日にわたる場合も1ファイル）。

1. **表紙**: `令和７年３月定例会` のようなタイトル
2. **目次**: 議事日程の一覧（ページ番号付き）
3. **招集告示**: 招集日・場所・付議事件
4. **応招・不応招議員**: 議員名簿
5. **議事日程**: 日程一覧
6. **出席議員・説明員・事務局職員**: 名簿
7. **本文**: 議事進行の全文

### ページ番号

`−1−`、`−2−` のような全角ハイフンで囲まれた形式で各ページに付与。

### セクション見出し（◎マーク）

議事の大項目は `◎` で始まる:

```
◎開会及び開議の宣告
◎諸般の報告
◎会議録署名議員の指名
◎会期の決定
◎町長あいさつ
◎承認第１号の上程、説明、質疑、討論、採決
◎議案第１号の上程、説明、質疑、討論、採決
```

### 発言者パターン（○マーク）

発言者は `○` で始まり、役職または番号 + 括弧内に氏名（`君` 付き）:

```
○議長（横澤はま君）
○副議長（和澤忠志君）
○１番（矢口結以君）
○１０番（服部久子君）
○振興文教委員長（大出美晴君）
○総務福祉委員長（大噐美秋君）
○建設水道課長（山本利彦君）
○総務課財政係長（寺島靖成君）
○会計管理者兼会計課長（丸山光一君）
```

### 発言者の分類

| パターン | 例 |
| --- | --- |
| 議長・副議長 | `○議長（横澤はま君）` |
| 議員（番号） | `○１番（矢口結以君）` |
| 委員長 | `○振興文教委員長（大出美晴君）` |
| 行政職員（課長等） | `○建設水道課長（山本利彦君）` |

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録一覧ページ `https://www.ikedamachi.net/0000000115.html` から全 PDF リンクを抽出する。

**収集方法:**

1. 一覧ページの HTML を取得
2. `<a>` タグから `./cmsfiles/contents/0000000/115/*.pdf` パターンの `href` を抽出
3. 相対パスをベース URL `https://www.ikedamachi.net/` で解決して絶対 URL を構築
4. リンクテキストから会議種別（定例会/臨時会）と開催月を抽出
5. 親要素の `<h3>` から年度を取得

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF は `pdftotext` 等のツールでテキスト抽出可能（テキスト埋め込み型）
- フォント: MS-Mincho、MS-Gothic、Century
- エンコーディング: Identity-H（日本語 CID フォント）

### Step 3: 会議録のパース

#### メタ情報の抽出

PDF 冒頭部分から以下を抽出:

```
令和７年３月定例会
令和７年３月池田町議会定例会会議録目次
```

- 元号 + 年 + 月 + 会議種別

#### 開催日の抽出

議事日程セクションから:

```
令和７年３月４日（火曜日）午前１０時開会
```

#### 発言の抽出

##### パース用正規表現（案）

```typescript
// セクション見出しの抽出
const sectionPattern = /^◎(.+)/;

// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○１番（矢口結以君） → role="１番", name="矢口結以"
// 例: ○議長（横澤はま君） → role="議長", name="横澤はま"
// 例: ○建設水道課長（山本利彦君） → role="建設水道課長", name="山本利彦"

// ページ番号の検出（除外用）
const pageNumberPattern = /^−\d+−$/;

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;
```

##### パース手順

1. `pdftotext` で全文テキストを取得
2. ページ番号行（`−N−`）を除外
3. `◎` で始まる行をセクション区切りとして認識
4. `○` で始まる行を発言者として認識し、次の `○` または `◎` までのテキストを発言内容とする
5. 発言者名から `君` を除去して氏名を取得

## 注意事項

- PDF ファイル名の大文字・小文字が統一されていない（`R2.9t.pdf` vs `R7.9T.pdf`）。URL 比較時は case-insensitive で扱う必要がある
- 1つの PDF に複数日の会議が含まれる場合がある（定例会は複数日にわたることが多い）
- PDF テキスト抽出時、全角スペースによるレイアウト調整が含まれる（例: `令 和 ７ 年 ３ 月 定 例 会`）。冒頭のタイトル部分は文字間にスペースが挿入されている
- 議員名に環境依存文字が含まれる場合がある（例: `大噐`、`山﨑`）

## 推奨アプローチ

1. **一覧ページから全量取得**: 単一ページに全 PDF リンクが掲載されているため、1回のリクエストで全リンクを収集可能
2. **PDF テキスト抽出**: `pdftotext`（poppler-utils）または同等のライブラリで抽出。テキスト埋め込み型のため OCR は不要
3. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの PDF リンク一覧と既存取得済みリストを比較し、新規分のみダウンロードする
5. **複数日会議の分割**: 1つの PDF 内に複数日の議事日程が含まれる場合、`◎開会及び開議の宣告` や日付パターンで日単位に分割する
