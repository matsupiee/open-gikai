# 信濃町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.shinano.lg.jp/chosei/gikai/
- 分類: CMS（SHIRASAGI）による HTML ページ + PDF/DOC/DOCX ファイル直接提供
- 文字コード: UTF-8
- 特記: 会議録検索システムなし。年度別の一覧ページから個別の PDF/DOC/DOCX ファイルをダウンロードする方式

---

## 議会の構成

信濃町議会は「通年議会」制度を採用しており、年1回の定例会を開催する。

- 会期: 1月〜12月（例: 第424回信濃町議会定例会 令和7年1月6日〜12月17日）
- 定例月会議: 主に1月・2月・3月・5月・6月・9月・11月・12月に開催
- 一般質問がある月: 主に3月・6月・9月・12月（2月に実施される年もあり）
- 臨時会議: 必要に応じて追加開催

---

## 提供されるファイルの種類

各月の会議について、以下のファイルが提供される:

| 種類 | ファイル形式 | 備考 |
| --- | --- | --- |
| 町長あいさつ / 再開あいさつ | PDF | 一部の月のみ |
| 会議日程 | PDF | |
| 提出議案案件名 / 議案審議結果 | PDF | |
| 一般質問通告書内容一覧 | PDF | 一般質問がある月のみ |
| 一般質問会議録（議員別） | PDF / DOC / DOCX | 議員ごとに個別ファイル |

### ファイル形式の混在

- 令和6年以降: 主に PDF 形式
- 令和4年: PDF と DOC/DOCX が混在（同一月内でも議員により異なる）
- スクレイパーは PDF・DOC・DOCX すべてに対応する必要がある

## スクレイピング戦略

### Step 1: 年度別一覧ページの URL 収集

議会トップページ `https://www.town.shinano.lg.jp/chosei/gikai/` から年度別一覧ページへのリンクを収集する。

- リンクは `/docs/{ページID}.html` 形式
- ページネーションなし（全年度が1ページに表示）

**収集方法:**

1. トップページの HTML を取得
2. `/docs/\d+\.html` パターンのリンクを Cheerio で抽出
3. リンクテキストから年度（令和X年）を判定

### Step 2: 各年度ページから会議録ファイル URL の収集

各年度ページ `/docs/{ページID}.html` から個別の会議録ファイルへのリンクを収集する。

- ファイルリンクは `/fs/` で始まるパス
- 拡張子は `.pdf`, `.doc`, `.docx` のいずれか
- リンクテキストに議員名や文書種別が含まれる

**収集方法:**

1. 年度ページの HTML を取得
2. `a[href*="/fs/"]` セレクタで全ファイルリンクを抽出
3. リンクテキストと周囲の見出し（`<h2>`, `<h3>` 等）から以下のメタ情報を推定:
   - 回号（第420回〜第424回等）
   - 月（1月〜12月）
   - 文書種別（町長あいさつ、日程、議案審議結果、一般質問会議録）
   - 議員名（一般質問の場合）

### Step 3: ファイルのダウンロードとテキスト抽出

#### PDF ファイル

- PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で本文を取得
- 一般質問会議録は議員別に1ファイルのため、発言者の分離は不要

#### DOC/DOCX ファイル

- DOC/DOCX をダウンロードし、テキスト抽出ライブラリ（mammoth 等）で本文を取得
- DOC（旧形式）と DOCX（新形式）の両方に対応が必要

## メタ情報の抽出

### 見出し構造からの抽出

年度ページ内の見出し階層から会議の回号・月を特定する:

```
第424回12月会議  ← h2/h3 見出し
├── 町長再開あいさつ (PDF)
├── 日程 (PDF)
├── 提出議案案件名 (PDF)
├── 一般質問通告書内容一覧 (PDF)
└── 一般質問会議録
    ├── 1. 北村富貴夫議員 (PDF)
    ├── 2. 佐藤博一議員 (PDF)
    └── ...
```

### ファイル名からの抽出

ファイル名にはエンコードされた日本語が含まれるが、URLデコード後に以下のパターンが現れる:

```
第{回号}回{月}月会議{種別}.pdf
{通告順}.{議員名}.pdf
```

## 注意事項

- CMS は SHIRASAGI を使用（`SS.config` が HTML 内に存在）
- ファイルの URL パスは `/fs/{1桁数字}/{1桁数字}/.../_/{ファイル名}` という独特な構造（SHIRASAGI の GridFS ストレージ）
- ファイル名は日本語を含み URL エンコードされている
- 年度ページの ID に規則性はないため、トップページからのリンク収集が必須
- 一般質問がない月（5月、11月等）は日程と議案審議結果のみ掲載

---

## 推奨アプローチ

1. **2段階クロール**: まずトップページから年度ページ URL を収集し、次に各年度ページからファイル URL を収集する
2. **ファイル形式の判定**: 拡張子（`.pdf`, `.doc`, `.docx`）でファイル形式を判定し、適切なパーサーを使い分ける
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: 年度ページの URL リストを保持し、新しい年度ページが追加された場合のみ差分取得する
5. **一般質問を優先**: 一般質問会議録（議員別 PDF）が最も議事録としての価値が高いため、優先的に取得する
