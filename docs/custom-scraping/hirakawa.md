## 概要

- サイト: https://www.city.hirakawa.lg.jp/jouhou/gikai/nittei/kaigiroku.html
- 分類: 市公式サイト内で PDF ファイルを年度別・会議別に静的掲載
- 文字コード: UTF-8
- 特記: 会議録検索システム未導入。単一の HTML ページに全年度の PDF リンクを一括掲載

## ページ構造

全会議録が単一の HTML ページに掲載されている。検索機能やページネーションはない。

### 見出し階層

- `<h2>`: 年度単位（例: `令和７年会議録一覧`、`平成30年会議録一覧`）
- `<h3>`: 会議種別（`定例会` / `臨時会`）
- `<h4>`: 個別会議（例: `令和７年第４回定例会`、`平成30年第1回臨時会`）
- `<p><a href="...pdf">`: 各日の会議録 PDF リンク

### 会議種別

| 種別 | 説明 |
| --- | --- |
| 定例会 | 年4回（第1回〜第4回） |
| 臨時会 | 不定期開催 |

### 掲載年度範囲

- 最古: 平成24年（2012年）第1回定例会
- 最新: 令和7年（2025年）第4回定例会
- 平成24年〜平成26年は旧命名規則、平成27年以降は新命名規則

---

## PDF ファイル命名規則

### 新形式（令和元年〜現在 / 一部平成27年〜）

```
{YYNN}_{type}_kaigiroku_{MMDD}.pdf
{YYNN}_{type}_kaigiroku_mokuji.pdf
```

- `YY`: 和暦年の下2桁（令和7年 → `07`、平成31年 → `31`）
- `NN`: 第N回（`01`〜`04`）
- `type`: `teirei`（定例会） / `rinnji` or `rinji`（臨時会、表記揺れあり）
- `MMDD`: 開催日の月日（例: `1202` → 12月2日）
- `mokuji`: 目次ファイル

例:
- `0704_teirei_kaigiroku_1202.pdf` → 令和7年第4回定例会 12月2日
- `0201_rinji_kaigiroku_0120.pdf` → 令和2年第1回臨時会 1月20日

### 旧形式（平成24年〜平成26年の一部）

```
H{YY}-{N}-{type}-mokuji.pdf
H{YY}-{N}-{type}{連番}.pdf
```

- `YY`: 和暦年の下2桁
- `N`: 第N回
- `type`: `teirei` / `rinji`
- 連番: `1`, `2`, `3`...（日ごとではなく通し番号）

例:
- `H26-2-teirei-mokuji.pdf` → 平成26年第2回定例会 目次
- `H26-2-teirei1.pdf` → 平成26年第2回定例会 第1日

## スクレイピング方針

### Step 1: 一覧ページの取得

単一ページ `kaigiroku.html` を取得し、全 PDF リンクを抽出する。

- ページネーションなし（全データが1ページに収まっている）
- `<a href="files/...pdf">` のリンクをすべて収集

### Step 2: メタ情報の抽出

HTML の見出し階層からメタ情報を構造化する。

```
<h2> → 年度（令和7年、平成30年 etc.）
  <h3> → 会議種別（定例会 / 臨時会）
    <h4> → 会議名（令和7年第4回定例会 etc.）
      <p><a> → PDF リンク + リンクテキスト（「第１日12月２日」「目次」etc.）
```

リンクテキストから開催日を抽出:
```typescript
// 開催日の抽出
const datePattern = /第\d+日(\d+)月(\d+)日/;
// 例: 「第１日12月２日」 → month=12, day=2
```

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で本文を取得
- 目次ファイル（`mokuji`）はスキップまたは別途メタ情報として利用

---

## 注意事項

- 臨時会の `type` に表記揺れあり（`rinnji` と `rinji` が混在）。両方のパターンを考慮すること
- 旧形式（H26 以前）と新形式でファイル名の規則が異なる
- PDF の内容はテキスト抽出可能かどうか事前確認が必要（画像 PDF の可能性あり）
- 単一ページに全データがあるため、差分更新は前回取得済みの PDF ファイル名と比較して新規分のみダウンロードする方式が有効

---

## 推奨アプローチ

1. **一括取得が容易**: 単一 HTML ページに全 PDF リンクがあるため、クロール対象のページは1つだけ
2. **PDF テキスト抽出の検証を先行**: PDF が機械可読テキストを含むかどうかを数件サンプルで確認する
3. **レート制限**: PDF ダウンロード時にリクエスト間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: ファイル名リストを保持し、新規追加分のみをダウンロードする
