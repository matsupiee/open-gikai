# 多良木町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.taragi.lg.jp/
- 分類: 町公式サイトで PDF を直接公開（専用の会議録検索システムは使用していない）
- 文字コード: UTF-8
- 特記: 会議録一覧ページに個別詳細ページへのリンクが並び、各詳細ページから PDF をダウンロードする 2 段階構造

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.town.taragi.lg.jp/gyousei/soshiki/gikai/gikaikaigiroku/index.html` |
| 各会議詳細ページ | `https://www.town.taragi.lg.jp/gyousei/soshiki/gikai/gikaikaigiroku/{ID}.html` |
| PDF ファイル | `https://www.town.taragi.lg.jp/material/files/group/12/{ファイル名}.pdf` |

詳細ページの ID は数値で、新しい会議ほど大きい値になる（例: 3720 が最新、1198 が古い）。

---

## ページ構造

### 会議録一覧ページ

単一の HTML ページ。全会議録が新しい順にリスト表示されており、ページネーションはない。

各エントリのリンクテキスト形式:

```
{元号}X年度第Y回多良木町議会（{月}会議）
{元号}X年度第Y回多良木町議会（{月}定例会議）
```

例:
- `令和7年度第2回多良木町議会（7月会議）`
- `令和6年度第5回多良木町議会（12月定例会議）`
- `令和元年度第1回多良木町議会（5月会議）初議会`

### 各会議詳細ページ

各詳細ページに以下の 2 つの PDF リンクが掲載されている:

1. **会議日程** (`nittei*.pdf` または日付ベースのファイル名)
2. **会議会議録** (`kaigiroku*.pdf` または日付ベースのファイル名)

## PDF ファイルの命名規則

ファイルは `https://www.town.taragi.lg.jp/material/files/group/12/` 配下に格納されている。

ファイル名には 2 つのパターンが確認されている:

### パターン A: 用途＋月日形式

```
nittei{MMDD}.pdf      # 会議日程
kaigiroku{MMDD}.pdf   # 会議録
```

例（令和7年7月10日開催）:
- `nittei0710.pdf`
- `kaigiroku0710.pdf`

### パターン B: 日付連番形式

```
{YYYYMMDD連番}.pdf
```

例（令和7年3月28日開催）:
- `20250311.pdf`（会議日程）
- `20250312.pdf`（会議録）

命名規則に揺れがあるため、**ファイル名を推測・構築するのではなく、各詳細ページの `<a href>` から URL を直接収集する**ことを強く推奨する。

---

## 会議種別

多良木町議会は「定例会議」と「会議（臨時会議）」が混在している。

| 種別 | リンクテキスト内表記 | 備考 |
| --- | --- | --- |
| 定例会議 | `{月}定例会議` | 年3〜4回（6月・9月・12月・3月が標準） |
| 会議（臨時） | `{月}会議` | 年複数回開催 |

委員会別の会議録は掲載されていない。すべて本会議のみ。

## スクレイピング戦略

### Step 1: 一覧ページから詳細ページ URL を収集

`index.html` を取得し、`/gikaikaigiroku/` を含む `<a href>` をすべて抽出する（`index.html` 自身は除外）。

```typescript
// Cheerio での収集例
const detailUrls = $('a[href*="/gikaikaigiroku/"]')
  .map((_, el) => {
    const href = $(el).attr('href')!;
    if (href.includes('index.html')) return null;
    return new URL(href, 'https://www.town.taragi.lg.jp/').toString();
  })
  .toArray()
  .filter(Boolean);
```

各リンクのテキストから会議のメタ情報（年度・回次・会議種別・開催月）を取得できる。

### Step 2: 各詳細ページから PDF URL を収集

各詳細ページを取得し、`/material/files/group/12/` を含む `<a href>` を抽出する。

```typescript
// Cheerio での収集例
const pdfUrls = $('a[href*="/material/files/group/12/"]')
  .map((_, el) => {
    const href = $(el).attr('href')!;
    return new URL(href, 'https://www.town.taragi.lg.jp/').toString();
  })
  .toArray();
// pdfUrls[0]: 会議日程 PDF
// pdfUrls[1]: 会議録 PDF
```

1 詳細ページにつき通常 2 つの PDF（日程・会議録）が掲載される。

### Step 3: 会議録 PDF を取得・テキスト抽出

会議録 PDF（`kaigiroku*.pdf` または 2 番目の PDF）を取得し、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

- 会議録ファイルサイズは数百 KB〜1 MB 程度
- 日程 PDF は会議録のテキスト抽出には不要

### Step 4: メタ情報の取得

一覧ページのリンクテキストからメタ情報を取得する。

```typescript
// リンクテキストから年度・回次・会議種別・開催月を抽出
const pattern = /(?:令和|平成)(\d+)年度第(\d+)回多良木町議会（(\d+)月(定例会議|会議)）/;
// 例: "令和6年度第5回多良木町議会（12月定例会議）"
//   → era="令和", fiscalYear=6, sessionNum=5, month=12, type="定例会議"
```

## 注意事項

- **2 段階構造**: 一覧ページ → 詳細ページ → PDF という 2 ホップのアクセスが必要
- **PDF が 2 種類**: 各詳細ページに日程・会議録の 2 PDF があり、会議録のみを対象とする
- **ファイル名の揺れ**: パターン A（`kaigiroku0710.pdf`）とパターン B（`20250312.pdf`）が混在するため、URL は必ずページから取得する
- **ページネーションなし**: 一覧ページで全件が取得できる（2026年3月時点で 61 件）
- **最古のデータ**: 平成29年度第3回（2017年9月定例会議）

---

## 推奨アプローチ

1. **一覧ページを単一取得**: `index.html` の 1 リクエストで全詳細ページ URL とリンクテキストを収集
2. **詳細ページを順次取得**: 各詳細ページから PDF URL を収集（61 件 × 1 リクエスト）
3. **会議録 PDF のみを対象**: 日程 PDF はスキップし、会議録 PDF のみをダウンロード
4. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機を設ける
5. **差分更新**: 取得済み詳細ページ ID の最大値を記録し、新規 ID のみを対象にする
