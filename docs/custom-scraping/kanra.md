# 甘楽町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.kanra.lg.jp/gikai/kaigiroku/index.html
- 分類: 自治体 CMS による PDF 年度別公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF のみで提供（HTML 本文なし）。Google Analytics（G-BVYZLCQFXM）使用

---

| ページ | URL パターン |

| 年度別一覧（令和） | `https://www.town.kanra.lg.jp/gikai/kaigiroku/R{N}/index.html` |

## サイト構造

### 3階層構造

1. **トップページ** (`kaigiroku/index.html`): 年度リンク一覧（平成22年〜令和8年、計17年度）
2. **年度別一覧** (`kaigiroku/R6/index.html` 等): 定例会・臨時会のリスト（`<ul>` リスト形式）
3. **会議録詳細** (`gikai-jimu/gikai/kaigi/XXXXX.html`): 目次 PDF + 号別 PDF のダウンロードリンク

### 検索機能

なし。年度別のリンク一覧を辿る構造。

### ページネーション

なし。各年度ページに当該年度の全会議が一覧表示される。

---

## 会議種別

- **定例会**: 第1回〜第4回（3月、6月、9月、12月）
- **臨時会**: 不定期（年に0〜3回程度）

年度一覧ページのリンクテキストから会議種別と開催月を取得可能。

例: `第４回定例会（令和６年１２月）`、`第１回臨時会（令和６年１月）`

## PDF ファイルの命名規則

PDF のファイル名パターンは時期により異なる。

### 新しい記事（令和以降）

詳細ページと同じディレクトリに配置。

| 種類 | ファイル名パターン | 例 |
| --- | --- | --- |
| 目次 | `{YYYYMMDD}mokuji.pdf` | `20241206mokuji.pdf` |
| 号別 | `{YYYYMMDD}{N}gou.pdf` | `202412061gou.pdf`、`202412122gou.pdf` |

### 古い記事（平成期）

`/gikai-jimu/sozai/other/` ディレクトリに配置。

| 種類 | ファイル名パターン | 例 |
| --- | --- | --- |
| 目次 | `kaigi_{YYMM}_mokuji.pdf` | `kaigi_2203_mokuji.pdf` |
| 号別 | `kaigi_{YYMMDD}.pdf` | `kaigi_220309.pdf`、`kaigi_220311.pdf` |

### 中間期（一部の年度）

| 種類 | ファイル名パターン | 例 |
| --- | --- | --- |
| 目次 | `r{NN}_{N}t.mokuji.pdf` | `r06_3t.mokuji.pdf` |
| 号別 | `{YYYYMMDD}.{N}gou.pdf` | `20240906.1gou.pdf` |

## スクレイピング戦略

### Step 1: 年度一覧ページから会議リンクを収集

トップページ `kaigiroku/index.html` から全年度ページの URL を取得し、各年度ページから会議録詳細ページへのリンクを収集する。

**収集方法:**

1. トップページの年度リンク（`./R{N}/index.html`、`./h{NN}/index.html`）を全て取得
2. 各年度ページのリスト要素から詳細ページへのリンクを抽出
3. リンクテキストから会議種別と開催月を取得

### Step 2: 詳細ページから PDF リンクを収集

各会議録詳細ページにアクセスし、PDF ファイルへのリンクを全て収集する。

**収集方法:**

1. 詳細ページの `<a>` タグから `.pdf` で終わるリンクを抽出
2. 相対パスを絶対 URL に変換
3. ファイル名から目次 / 号別を判別

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

**注意点:**

- PDF は議事録の本文を含むため、PDF パーサー（pdf-parse 等）でテキスト抽出が必要
- 目次 PDF は会議のメタ情報（議題一覧等）を含むため、別途処理する

### Step 4: テキストのパース

PDF から抽出したテキストを構造化する。

#### メタ情報

年度一覧ページのリンクテキストから以下を抽出:

```
第４回定例会（令和６年１２月）
```

- 回次: `第{N}回`
- 会議種別: `定例会` / `臨時会`
- 開催年月: `令和{N}年{MM}月` / `平成{NN}年{MM}月`

PDF ファイル名から開催日を取得:

```
202412061gou.pdf → 2024年12月6日
kaigi_220309.pdf → 2022年3月9日
```

#### 発言の構造

PDF 内のテキスト構造は実際の PDF を解析して確認が必要。一般的な議会 PDF では以下のパターンが想定される:

- 発言者名が行頭に記載
- 発言内容がその後に続く
- 議長の発言で区切られる

#### パース用正規表現（案）

```typescript
// リンクテキストから会議情報を抽出
const meetingPattern = /第(\d+)回(定例会|臨時会)（(令和|平成)(\d+)年(\d+)月）/;

// PDF ファイル名から日付を抽出（新形式）
const newDatePattern = /^(\d{4})(\d{2})(\d{2})/;

// PDF ファイル名から日付を抽出（旧形式）
const oldDatePattern = /kaigi_(\d{2})(\d{2})(\d{2})/;
```

---

## 注意事項

- 会議録は全て PDF 形式で提供されており、HTML 本文はない
- PDF のテキスト抽出精度はフォントやスキャン品質に依存する
- 詳細ページの URL パターンが新旧で異なる（タイムスタンプ ID vs 連番）
- PDF の配置パスも新旧で異なる（同一ディレクトリ vs `/gikai-jimu/sozai/other/`）
- PDF ファイル名の命名規則が時期により統一されていない

---

## 推奨アプローチ

1. **リンク収集を優先**: まず全年度 × 全会議の詳細ページ URL と PDF URL のリストを作成
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使って PDF からテキストを抽出
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度一覧ページの更新を監視し、新しい会議録詳細ページのみを取得する
5. **ファイル名パターンの統合**: 新旧のファイル名パターンを統一的に処理できるパーサーを用意する
