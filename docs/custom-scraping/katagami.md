## HTML 構造

年代別ページへのリンクをリスト形式で掲載。

### 年代別ページ（各 ID.html）

```html
<ul>
  <li>
    <!-- 年度見出し（例: 令和6年会議録） -->
    <ul>
      <li>
        <!-- 定例会/臨時会見出し（例: 第3回定例会） -->
        <ul>
          <li>

              1日目（令和6年9月4日）(PDFファイル:XXX.XKB)
            </a>
          </li>
        </ul>
      </li>
    </ul>
  </li>
</ul>
```

- リンクテキストに日付情報（`令和○年○月○日`）とファイルサイズが含まれる
- 会議種別（定例会 / 臨時会 / 委員会）は親要素のテキストから判定
- ページネーションなし（1 ページに全件表示）

## スクレイピング戦略

### Step 1: 年代別ページの PDF リンク収集

5 つの年代別ページ（4189, 2769, 2768, 2767, 2766）を巡回し、全 PDF リンクを収集する。

**収集方法:**

1. 各年代別ページの HTML を取得
2. `<a>` タグから `material/files/group/13/*.pdf` パターンの URL を抽出
3. リンクテキストから開催日・日目情報を抽出
4. 親要素のテキストから会議種別（定例会 / 臨時会 / 委員会）と回次を特定

**リンクテキストからのメタ情報抽出:**

```typescript
// リンクテキストの例: "1日目（令和6年9月4日）(PDFファイル:546.3KB)"
const linkPattern = /(.+?)（((?:令和|平成)\d+年\d+月\d+日)）/;
// → label="1日目", date="令和6年9月4日"

// 親要素テキストの例: "第3回定例会"
const sessionPattern = /第(\d+)回(定例会|臨時会)/;
```

### Step 2: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、PDF パーサーでテキストを抽出する。

- PDF は会議録の全文を含む（発言者名・発言内容）
- テキスト抽出には `pdf-parse` 等のライブラリを使用

### Step 3: 会議録のパース

PDF から抽出したテキストを解析し、発言者と発言内容を分離する。

#### メタ情報

- 開催日: リンクテキストから抽出済み（`令和○年○月○日` 形式）
- 会議名: 親要素テキストから抽出済み（`第○回定例会` 等）
- 自治体: 潟上市

#### 発言の構造（PDF 内テキスト）

PDF 内の発言者パターンは PDF のフォーマットに依存するため、実際の PDF をダウンロードして確認する必要がある。一般的な議会会議録 PDF では以下のようなパターンが想定される:

```
○議長（○○○○）
○○番（○○○○）
○市長（○○○○）
```

---

## 注意事項

- 全ての会議録が PDF 形式のため、HTML パースだけでは本文を取得できない
- PDF のレイアウト（段組み、ヘッダー/フッター）によってテキスト抽出の精度が変わる可能性がある
- ファイル名の命名規則が年代によって統一されていないため、ファイル名からのメタ情報抽出は困難。リンクテキストからの抽出を推奨
- PDF の URL はプロトコル相対（`//www.city.katagami.lg.jp/...`）で記述されているため、`https:` を付与する必要がある

---

## 推奨アプローチ

1. **全量リスト作成を優先**: 5 つの年代別ページから全 PDF リンクとメタ情報（日付・会議種別）を収集してリスト化
2. **PDF テキスト抽出**: `pdf-parse` 等を使い PDF からテキストを抽出。段組み対応が必要な場合は `pdf2json` も検討
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 最新の年代別ページ（4189.html）のみを定期的にチェックし、新規 PDF リンクがあれば追加取得する
5. **PDF サンプル確認**: 実装前に各年代から数件の PDF をダウンロードし、テキスト抽出の精度と発言者パターンを確認する
