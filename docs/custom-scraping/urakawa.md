# 浦河町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.urakawa.hokkaido.jp/gyosei/council/
- 分類: 公式サイト内の「審議の結果」ページで会議録 PDF を掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 会議録形式: PDF（日付ごとに1ファイル）
- 特記: 会議録本文はすべて PDF で提供されており、HTML 上には議案一覧のみ掲載

## スクレイピング戦略

### Step 1: 年度別カテゴリ ID の収集

審議の結果トップページ `?category=220` から年度別カテゴリへのリンクを抽出する。

- リンクテキストに「20XX年（令和X年分）」の形式で年度が記載
- URL パターン: `?category={ID}`

### Step 2: 会議種別カテゴリ ID の収集

各年度カテゴリページから会議種別（本会議、各委員会）のカテゴリリンクを抽出する。

- 会議種別: 本会議、総務産業建設常任委員会、厚生文教常任委員会
- カテゴリ ID は年度ごとに異なるため、毎回取得が必要

### Step 3: 会議録詳細ページ（content ID）の収集

各会議種別カテゴリページから個別の会議録詳細ページへのリンクを抽出する。

- 本会議: 「第X回浦河町議会（定例会/臨時会）［令和X年X月X日～X日］」形式
- 委員会: 「令和X年X月X日」形式
- URL パターン: `?content={ID}`
- ページネーションなし（1ページにすべて表示）

### Step 4: PDF リンクの収集

各会議録詳細ページから PDF ファイルへのリンクを抽出する。

- 本会議は複数日程の場合、日付ごとに個別の PDF がある（例: 3月4日、3月12日、3月13日、3月14日）
- 委員会は通常1日1ファイル
- PDF の URL パターン: `../../assets/images/content/content_{タイムスタンプ}.pdf`
  - 絶対パス: `https://www.town.urakawa.hokkaido.jp/gyosei/assets/images/content/content_{YYYYMMDD_HHMMSS}.pdf`
- リンクテキストに日付とファイルサイズが記載: 「令和X年X月X日 [PDF｜XXX KB]」

### Step 5: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを抽出
- 発言者・発言内容の構造は PDF 内のテキストレイアウトに依存

## メタ情報の抽出

### 会議録詳細ページから取得可能な情報

- **会議名**: ページタイトルから抽出（例: 「第8回浦河町議会定例会」）
- **開催日**: リンクテキストまたはページ見出しから抽出（和暦表記）
- **議案一覧**: テーブル形式で議案番号・件名・議決年月日が掲載
- **議決結果**: 「原案可決」「承認」等

### 日付の表記パターン

- 和暦: 「令和6年12月10日」
- 複数日程: 「令和6年12月10日~11日」「令和6年3月4日~14日」

---

## 注意事項

- 会議録はすべて PDF 形式のため、HTML パースだけでは本文を取得できない
- PDF の URL はタイムスタンプベース（`content_YYYYMMDD_HHMMSS.pdf`）で、content ID からは推測不可能
- 年度別・会議種別のカテゴリ ID は不規則で、ハードコーディングではなく動的取得が必要
- ページネーションは確認されていない（各カテゴリページですべての会議録が一覧表示される）
- 2019年（平成31年）以降のデータが公開されている

---

## 推奨アプローチ

1. **階層的クロール**: トップ → 年度 → 会議種別 → 詳細ページ → PDF の順に辿る
2. **PDF テキスト抽出**: pdf-parse 等のライブラリで PDF からテキストを抽出する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: content ID は数値連番に近いため、前回取得済みの最大 ID 以降を取得する差分更新が可能
5. **メタ情報の活用**: 詳細ページの議案テーブルから会議名・日付・議案情報を構造化データとして取得し、PDF 本文と紐付ける
