## 概要

- サイト: https://www.vill.soni.nara.jp/info/632
- 分類: 公式サイト内に PDF を直接掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 2023年5月（令和5年第1回臨時会）以降の会議録のみ掲載

---

## URL 構造

| ページ | URL |
| --- | --- |
| 会議録一覧（PDF リンク集） | `https://www.vill.soni.nara.jp/info/632` |
| PDF ファイル | `https://www.vill.soni.nara.jp/div/gikai/pdf/HP/{filename}.pdf` |

PDF の相対パスは `/div/gikai/pdf/HP/` 配下が基本だが、一部ファイルは `/div/gikai/pdf/`（`HP/` なし）に置かれている例外がある（例: `/div/gikai/pdf/7_3giketu.pdf`）。

## 会議録の提供形式

- **議決結果**: 各会議につき 1 件の PDF

- ページネーションなし。全会議録が 1 ページに列挙される

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `https://www.vill.soni.nara.jp/info/632` から会議録 PDF のリンクを取得する。

- ページネーションなし。単一ページに全リンクが掲載される
- `<a href>` 属性から `.pdf` で終わるリンクを Cheerio で抽出する
- 議決結果（`giketu` を含むファイル名）と会議録（`kaigiroku`・`teirei`・`rinji` 等を含むファイル名）を区別する
- リンクテキストで「議決結果」「会議録」の種別を判定できる

**PDF URL の組み立て:**

```
https://www.vill.soni.nara.jp{href}
```

例:
- `/div/gikai/pdf/HP/7_12kaigiroku1.pdf` → `https://www.vill.soni.nara.jp/div/gikai/pdf/HP/7_12kaigiroku1.pdf`

### Step 2: メタ情報の抽出

一覧ページの HTML からリンク周辺のテキストを解析し、各 PDF に対応する年度・会議情報を取得する。

- `<h3>` タグで年度（令和X年）を取得
- 各リンク直前のテキストノードまたは親要素から会議名（定例会・臨時会・開催月）を取得
- リンクテキスト（「会議録➀」「会議録➁」「議決結果」）で種別を識別

### Step 3: PDF のダウンロードとパース

収集した URL から PDF をダウンロードし、テキストを抽出する。

- PDF パーサーを使用してテキストを抽出（`pdfjs-dist` 等）
- 会議録テキストから発言者・発言内容を構造化する

---

## 注意事項

- **掲載期間の制限**: 2023年5月（令和5年第1回臨時会）以降の会議録のみ。それ以前の会議録は掲載されていない
- **ファイル名の規則性が低い**: ファイル名にパターンがなく（例: `dayori69.pdf`、`61titirei.pdf` 等）、ファイル名からメタ情報を推測できない。一覧ページのリンクテキストと HTML 構造からメタ情報を取得する必要がある
- **PDF のパスに例外あり**: 大半は `/div/gikai/pdf/HP/` 配下だが、一部 `/div/gikai/pdf/`（`HP/` なし）に置かれているファイルが存在する。href をそのまま使用すること
- **差分更新**: 一覧ページに新しいリンクが追加される形で更新される。前回取得済み PDF URL のセットと比較して差分のみ処理することを推奨

---

## 推奨アプローチ

1. **単一ページクロール**: 一覧ページ 1 件のみ取得すれば全 PDF リンクを収集できる
2. **リンクテキストでメタ情報を補完**: ファイル名が不規則なため、HTML のリンクテキストと見出し構造から年度・会議名・種別を取得する
3. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 取得済み PDF URL を記録し、次回は新規 URL のみをダウンロードする
