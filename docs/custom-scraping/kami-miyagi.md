## PDF ファイルの命名規則

PDF ファイル名には **2 種類のパターン** が存在する。

### パターン 1: 構造化ファイル名（令和4年〜）

```
r{年度}-{種別}{回数}-{日数}-{月日}.pdf
```

| 要素 | 説明 | 例 |
| --- | --- | --- |
| `r{年度}` | 令和の年度番号 | `r04`, `r06` |
| `{種別}` | 会議種別コード | `01t`〜`04t`（定例会）、`01r`〜`05r`（臨時会）、`y`（予算）、`k`（決算） |
| `{日数}` | 開催日の通し番号 | `01`, `02`, ... |
| `{月日}` | 月日（4桁） | `0308`（3月8日） |

**例:**
- `r04-01t-01-0308.pdf` → 令和4年第1回定例会第1日（3月8日）
- `r06-y-05-0315.pdf` → 令和6年予算審査特別委員会第5日（3月15日）
- `r04-01r-0329.pdf` → 令和4年第1回臨時会（3月29日）

### パターン 2: タイムスタンプファイル名（〜平成29年頃）

```
YYYYMMDD-HHMMSS.pdf
```

**例:**
- `20201105-142525.pdf` → アップロード日時がファイル名（内容との対応は HTML テーブルのリンクテキストから判別）

## HTML 構造

### トップページ（年度一覧）

- 年度名をリンクテキストとした `<a>` タグのリスト
- ページネーションなし（全年度が 1 ページに表示）

### 年度別ページ

- 会議種別ごとにセクション（見出し）で区切られたテーブル形式
- テーブルは 2 列構成: 「会議名」「ファイル」
- 「会議名」列: 定例会第○回第○日、臨時会第○回、予算審査○日目 等
- 「ファイル」列: PDF ファイルへのリンク（ファイルサイズ表記付き）
- PDF リンクは `//www.town.kami.miyagi.jp/material/files/group/42/{ファイル名}.pdf` 形式（プロトコル相対 URL）

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

議事録トップページ `index.html` から全年度ページへのリンクを収集する。

**収集方法:**

1. `index.html` を取得
2. `/choseijoho/kamimachigikai/gijiroku/` 配下へのリンクを抽出
3. リンクテキストから年度情報を取得

### Step 2: 各年度ページから PDF リンクの収集

各年度ページのテーブルから PDF リンクとメタ情報を抽出する。

**収集方法:**

1. 年度ページの HTML を取得
2. セクション見出しから会議種別を判定
3. テーブルの各行から以下を抽出:
   - 会議名（リンクテキスト or セル内テキスト）
   - PDF URL（`<a>` タグの `href`）
4. 会議名テキストから開催日・回次・日次を正規表現でパース

```typescript
// 会議名テキストの例:
// "第1回定例会 第1日目(3月8日)"
// "第3回臨時会(7月28日)"
// "予算審査特別委員会 1日目(3月11日)"

// 会議名パース用正規表現（案）
const teireikaiPattern = /第(\d+)回定例会\s*第(\d+)日目?\((\d+)月(\d+)日\)/;
const rinjikaiPattern = /第(\d+)回臨時会\((\d+)月(\d+)日\)/;
const yosanPattern = /予算審査特別委員会\s*(\d+)日目?\((\d+)月(\d+)日\)/;
const kesanPattern = /決算審査特別委員会\s*(\d+)日目?\((\d+)月(\d+)日\)/;
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

**抽出方法:**

1. PDF ファイルを `fetch` でダウンロード（プロトコル相対 URL に `https:` を付与）
2. `pdf-parse` 等のライブラリで PDF からテキストを抽出
3. 抽出されたテキストから発言者・発言内容をパース

### Step 4: 会議録テキストのパース

PDF から抽出したテキストの構造は PDF 内のレイアウトに依存するため、実際の PDF を確認した上でパース処理を実装する必要がある。

## 注意事項

- **PDF のみでの公開**: 会議録は全て PDF ファイルで提供されており、HTML テキストは存在しない。PDF からのテキスト抽出が必須。
- **ファイル名の命名規則が年度により異なる**: 令和4年以降は構造化された命名規則、それ以前はタイムスタンプ形式。ファイル名からのメタ情報抽出は令和4年以降のみ可能。
- **プロトコル相対 URL**: PDF リンクは `//www.town.kami.miyagi.jp/...` 形式のため、`https:` を先頭に付与する必要がある。
- **ページ ID の不規則性**: 年度別ページの ID（URL の数値部分）に規則性がないため、トップページからのリンク収集が必須。
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける。
- **SMART CMS**: サイトは SMART CMS プラットフォーム上で構築されている。

---

## 推奨アプローチ

1. **2 段階クロール**: まずトップページから年度別ページ URL を収集し、次に各年度ページから PDF リンクを収集する
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使って PDF からテキストを抽出する
3. **メタ情報は HTML から取得**: 会議名・開催日等のメタ情報は PDF 内ではなく HTML テーブルのリンクテキストから取得する方が確実
4. **レート制限**: PDF ダウンロード時は特にファイルサイズが大きいため、リクエスト間隔を十分に取る（2〜3 秒）
5. **差分更新**: 最新年度のページのみを再クロールし、新規 PDF リンクがあれば取得する
