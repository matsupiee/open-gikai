# 飯綱町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.iizuna.nagano.jp/gikai/kaigiroku/
- 分類: 独自 CMS による HTML + PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF ファイルで提供。一覧ページにページネーションなし（全件を単一ページに表示）

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.town.iizuna.nagano.jp/gikai/kaigiroku/` |
| 会議録詳細 | `https://www.town.iizuna.nagano.jp/docs/{ID}.html` |
| 会議録 PDF | `https://www.town.iizuna.nagano.jp/fs/{a}/{b}/{c}/{d}/{e}/_/___.pdf` |

- 詳細ページの `{ID}` は数値（例: `13282`, `11454`）
- PDF の URL は `/fs/` 配下に数字ディレクトリが5階層あり、ファイル名は `___.pdf` で固定

---

## 一覧ページの構造

会議録一覧ページ（`/gikai/kaigiroku/`）は `<h2>` タグ内の `<a>` タグで全会議録へのリンクを提供する。

```html
<h2><a href="/docs/13282.html">令和7年12月定例会 会議録</a></h2>
<h2><a href="/docs/13544.html">令和8年第1回（1月16日）臨時会　会議録</a></h2>
<h2><a href="/docs/11454.html">令和6年12月定例会 会議録</a></h2>
```

- ページネーションなし（2026年3月時点で73件が単一ページに全件表示）
- 降順（新しい会議録が上）で表示
- 平成29年3月定例会〜令和8年第1回臨時会までの記録が存在

---

## 会議の種類

リンクテキストから以下の2種類を識別できる:

| 種類 | テキストパターン | 例 |
| --- | --- | --- |
| 定例会 | `{和暦}年{月}月定例会 会議録` | `令和7年12月定例会 会議録` |
| 臨時会 | `{和暦}年第{N}回（{月}月{日}日）臨時会　会議録` | `令和8年第1回（1月16日）臨時会　会議録` |

### パース用正規表現（案）

```typescript
// 定例会
const teireikaiPattern = /^((?:令和|平成)\d+年)\d+月定例会/;

// 臨時会
const rinjikaiPattern = /^((?:令和|平成)\d+年)第(\d+)回（(\d+)月(\d+)日）臨時会/;
```

---

## 詳細ページの構造

各詳細ページ（`/docs/{ID}.html`）には以下が含まれる:

1. **ページタイトル**: 会議名（例: `令和7年12月定例会 会議録`）
2. **公開日**: テキストで記載
3. **会議録 PDF リンク**: `会議録 (PDF {サイズ})` の形式で1件
4. **一般質問一覧テーブル**（定例会のみ）: 順番・議席番号・氏名・発言事項
5. **YouTube 動画リンク**（定例会のみ）: 一般質問の録画動画
6. **関連記事リンク**: 表決結果ページへのリンク

### PDF リンクの抽出

```typescript
// 詳細ページから PDF リンクを抽出
const pdfPattern = /href="(\/fs\/[^"]+\.pdf)"/;
// 例: /fs/8/5/8/8/7/_/___.pdf
```

### 一般質問テーブル（定例会のみ）

定例会の詳細ページには一般質問の発言者テーブルが含まれる:

| 順 | 議席 | 氏名 | 発言事項 |
| --- | --- | --- | --- |
| 1 | 5 | 近藤正 | 1. 項目A 2. 項目B |

※ 臨時会の詳細ページにはテーブルなし、PDF リンクのみ

## スクレイピング戦略

### Step 1: 会議録一覧の取得

一覧ページ `/gikai/kaigiroku/` から全会議録のリンクを収集する。

**収集方法:**

1. 一覧ページの HTML を取得
2. `<h2><a href="/docs/{ID}.html">` パターンで全リンクを抽出
3. リンクテキストから会議種別（定例会/臨時会）と和暦の開催時期を取得
4. ページネーションなしのため、1リクエストで全件取得可能

```typescript
// 一覧ページからリンクを抽出
const linkPattern = /<h2><a href="(\/docs\/\d+\.html)">(.+?)<\/a><\/h2>/g;
```

### Step 2: 各詳細ページから PDF URL を取得

各詳細ページにアクセスし、会議録 PDF のダウンロード URL を抽出する。

**収集方法:**

1. `/docs/{ID}.html` にアクセス
2. PDF リンク（`/fs/.../___.pdf`）を抽出
3. PDF をダウンロード

### Step 3: PDF からテキストを抽出

ダウンロードした PDF からテキストを抽出し、発言データを構造化する。

- PDF パーサー（例: `pdf-parse`）でテキスト化
- 発言者と発言内容を分離

---

## 注意事項

- 会議録本文は HTML ではなく **PDF でのみ提供** されるため、PDF パース処理が必要
- 一覧ページに全件表示されるため、ページネーション処理は不要
- リンクテキストの表記ゆれあり（全角数字と半角数字の混在: `令和４年` と `令和4年`、`６月` と `6月`）
- PDF の URL パターンは `/fs/{数字5階層}/_/___.pdf` で、詳細ページの HTML からしか取得できない（ID から推測不可）

---

## 推奨アプローチ

1. **一覧ページから全量取得**: 1リクエストで全会議録リンクを取得できるため、一覧ページの定期クロールで差分検出が容易
2. **PDF ダウンロード + テキスト抽出**: 会議録本文は PDF のため、`pdf-parse` 等でテキスト抽出が必要
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの `<h2><a>` タグ数の変化、または未知の `/docs/{ID}.html` の出現で新規会議録を検出
