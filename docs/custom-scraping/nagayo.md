# 長与町議会 カスタムスクレイピング方針

## 概要

- サイト: https://webtown.nagayo.jp/gikai/list00423.html
- 分類: 独自 Web サイトで会議録を PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 本会議・各常任委員会・議会運営委員会の会議録を PDF ファイルで提供

### PDF URL パターン詳細

PDF ファイル名は以下の2種類が存在する:

1. 新形式: `3_{記事番号}_{連番}_up_{ランダム文字列}.pdf`
   - 例: `3_1007_30636_up_abc12345.pdf`
2. 旧形式: `3_{記事番号}_{連番}_{年号}_{月}_{日}.pdf`
   - 例: `3_1007_12345_R06_03_04.pdf`

## ページ構造

### 本会議 会議録一覧ページ

- 単一ページに全年度・全会議の PDF リンクが掲載（ページネーションなし）
- 階層構造: 年度 → 定例会/臨時会 → 開催日（号数）
- 各リンクテキスト: `{月}月{日}日（第{号数}号）`
- 各リンクに PDF アイコンと「別ウインドウで開きます」アイコンが付与
- ファイルサイズが明記されている

### 委員会 会議録一覧ページ

- 委員会ページは `list00902.html` 等の一覧ページから `kiji{ID}/index.html` の詳細ページにリンク
- 詳細ページ内に年度 → 定例会/臨時会 → 開催日の階層で PDF リンクが掲載
- 委員会は複数日開催の場合あり（「第1日目」〜「第6日目」等）
- jQuery autopager による動的読み込み機能あり（`list00902.html` 等）

## スクレイピング戦略

### Step 1: 会議録一覧ページから PDF リンクを収集

各会議種別の一覧ページ（`kiji{ID}/index.html`）にアクセスし、PDF リンクを抽出する。

**収集方法:**

1. 本会議・各委員会の一覧ページ URL（5ページ）を順次取得
2. HTML を Cheerio でパースし、`<a>` タグから `.pdf` で終わるリンクを抽出
3. リンクテキストから開催日（月日）と号数を取得
4. 親要素の見出し（`<h3>` 等）から年度・定例会/臨時会の情報を取得

**取得対象ページ一覧:**

```typescript
const pages = [
  { type: "本会議", url: "https://webtown.nagayo.jp/gikai/kiji0031007/index.html" },
  { type: "総務厚生常任委員会", url: "https://webtown.nagayo.jp/gikai/kiji0032948/index.html" },
  { type: "産業文教常任委員会", url: "https://webtown.nagayo.jp/gikai/kiji0032949/index.html" },
  { type: "議会広報広聴常任委員会", url: "https://webtown.nagayo.jp/gikai/kiji0031008/index.html" },
  { type: "議会運営委員会", url: "https://webtown.nagayo.jp/gikai/kiji0031006/index.html" },
];
```

### Step 2: PDF ファイルのダウンロード

収集した PDF URL からファイルをダウンロードする。

- PDF ファイルサイズは 100KB〜800KB 程度
- `Content-Type: application/pdf` を確認

### Step 3: PDF からテキスト抽出

PDF ファイルからテキストを抽出する。

- `pdf-parse` 等のライブラリで PDF をテキスト化
- PDF は議事録形式のため、発言者と発言内容の構造を持つと想定

### Step 4: メタ情報の構築

HTML の階層構造とリンクテキストからメタ情報を構築する。

#### メタ情報の抽出元

| 情報 | 抽出元 |
| --- | --- |
| 会議種別 | 一覧ページの種別（本会議、各委員会） |
| 年度 | 見出し要素のテキスト（例: `令和6年`） |
| 定例会/臨時会 | 見出し要素のテキスト（例: `第1回定例会`） |
| 開催日 | リンクテキスト（例: `3月4日（第1号）`） |
| 号数/日目 | リンクテキスト（例: `第1号` or `第1日目`） |

#### 開催日パース用正規表現（案）

```typescript
// 本会議のリンクテキストから月日・号数を抽出
const honkaigiPattern = /(\d+)月(\d+)日（第(\d+)号）/;
// 例: "3月4日（第1号）" → month=3, day=4, number=1

// 委員会のリンクテキストから月日・日目を抽出
const committeePattern = /(\d+)月(\d+)日（第(\d+)日目）/;
// 例: "3月15日（第1日目）" → month=3, day=15, day_number=1

// 年度見出しから年を抽出
const yearPattern = /(令和|平成)(\d+)年/;
```

## 注意事項

- 会議録は PDF 形式のみで提供されており、HTML 本文は存在しない
- PDF のテキスト抽出精度はファイルの作成方法に依存する（スキャン画像の場合は OCR が必要になる可能性がある）
- 旧体制の常任委員会（令和3年4月以前）は別ページに掲載されている可能性がある
- 一覧ページが単一ページに全データを含むため、ページネーション処理は不要

---

## 推奨アプローチ

1. **全量取得を優先**: 5つの一覧ページから全 PDF リンクをまず収集し、メタ情報付きの URL リストを作成
2. **PDF テキスト抽出の品質確認**: 数件の PDF を手動でテキスト抽出し、発言者・発言内容の構造が機械的にパース可能か事前検証する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: PDF ファイル名の連番部分を利用し、前回取得済みの最大連番以降のみを取得する差分更新が可能
5. **旧体制委員会の調査**: 令和3年4月以前の常任委員会データが必要な場合は、サイドメニューから旧委員会ページを別途調査する
