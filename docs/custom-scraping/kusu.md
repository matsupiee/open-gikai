# 玖珠町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.kusu.oita.jp/choseijoho/kusuchogikai/1/index.html
- 分類: 自治体公式サイト上で年度別に PDF 形式で会議録を公開（外部の会議録検索システムは未導入）
- 文字コード: UTF-8
- CMS: SMART CMS プラットフォーム（JavaScript による動的コンテンツ読み込み）

---

| ページ | URL パターン |
| --- | --- |

### 年度別一覧ページ → 議事録詳細ページ

年度別一覧ページから議事録詳細ページへのリンクが1つ含まれる。議事録詳細ページの URL は以下の形式:

```
/soshiki/gikaijimukyoku/1/1/1/{記録名}/{page_id}.html
```

記録名の例:
- `reiwa3nenkaigiroku_1` → 令和5年（`_1` は連番）
- `reiwa3nenkaigiroku_2` → 令和6年

### 議事録詳細ページ → PDF ファイル

議事録詳細ページに各会議の PDF ファイルへのリンクが掲載されている。

## 会議種別

定例会と臨時会が公開されている。委員会の会議録は確認できない。

### 定例会

年4回（3月・6月・8-9月・12月）開催。各定例会で以下の種別の会議録が公開される:

- 開会（開会日）
- 議案質疑
- 一般質問表
- 一般質問（日数分に分割、通常2日間）
- 閉会（閉会日）

### 臨時会

不定期開催。通常は「開閉会」として1ファイルに統合されている。

## PDF ファイル名パターン

PDF ファイルは `/material/files/group/18/` 配下に格納されている。ファイル名には2種類のパターンが混在:

### パターン1: 略称 + 連番

```
R5{連番}{内容}.pdf
```

例:
- `R541kaikaibi.pdf` → 令和5年第4回定例会・開会日
- `R542giannsitugi.pdf` → 令和5年第4回定例会・議案質疑
- `R543ippannsitumonnhyou.pdf` → 令和5年第4回定例会・一般質問表
- `R544ippannsitumonn1.pdf` → 令和5年第4回定例会・一般質問1日目
- `R546heikaibi.pdf` → 令和5年第4回定例会・閉会日

### パターン2: 日付ベース

```
{YYYYMMDD}{内容}.pdf
```

例:
- `20230831kaikaibi.pdf` → 2023年8月31日・開会日
- `20230905giannsitugi.pdf` → 2023年9月5日・議案質疑
- `20230906ippannsitumonn.pdf` → 2023年9月6日・一般質問

※ 同一年度内でも命名規則が混在している。

## スクレイピング戦略

### Step 1: 年度別一覧ページの URL 収集

会議録トップページ（`index.html`）から各年度ページへのリンクを収集する。

- リンクは `/choseijoho/kusuchogikai/1/{page_id}.html` 形式
- 年度数は約11件（平成27年以前〜令和7年）で固定的
- ページは JavaScript で動的に読み込まれるため、SMART CMS の API またはヘッドレスブラウザが必要な可能性あり

### Step 2: 議事録詳細ページの URL 収集

各年度ページから議事録詳細ページへのリンクを取得する。

- 年度ページには議事録詳細ページへのリンクが1つ含まれる
- 詳細ページの URL は `/soshiki/gikaijimukyoku/1/1/1/{記録名}/{page_id}.html` 形式

### Step 3: PDF リンクの収集

議事録詳細ページから PDF ファイルへのリンクをすべて収集する。

- PDF リンクは `/material/files/group/18/{ファイル名}.pdf` 形式
- 各定例会あたり約6ファイル、臨時会は1ファイル
- リンクテキストに会議名・日付・内容種別の情報が含まれる

### Step 4: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用
- 発言者パターンや会議メタ情報は PDF 内のテキスト構造に依存する

---

## 注意事項

- **動的コンテンツ**: SMART CMS を使用しており、ページの一覧コンテンツが JavaScript で動的に読み込まれる。`fetch` による単純な HTML 取得ではリンク一覧が取得できない可能性がある
- **PDF のみ**: 会議録はすべて PDF 形式で公開されており、HTML 形式の本文ページは存在しない
- **ファイル名パターンの不統一**: 同一年度内でも PDF ファイル名の命名規則が混在している（略称+連番 と 日付ベース）。ファイル名からメタ情報を抽出する場合は両パターンに対応が必要
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
- **PDF テキスト抽出の品質**: PDF の生成方法によってはテキスト抽出が困難な場合がある（画像 PDF、特殊フォント等）

---

## 推奨アプローチ

1. **ヘッドレスブラウザの検討**: SMART CMS の動的コンテンツ読み込みに対応するため、Playwright 等のヘッドレスブラウザの使用を検討する。または CMS の JSON API エンドポイントを特定して直接データを取得する
2. **PDF 解析パイプライン**: PDF ダウンロード → テキスト抽出 → 発言者・日付・会議種別のパース、という3段階のパイプラインを構築する
3. **メタ情報はリンクテキストから取得**: PDF ファイル名よりも、議事録詳細ページのリンクテキスト（例:「開会」「一般質問（12月7日）」）からメタ情報を取得する方が信頼性が高い
4. **差分更新**: 年度ページの ID リストを保持し、新しいページ ID が追加された場合のみ差分取得を行う
