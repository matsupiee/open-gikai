# 竜王町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.ryuoh.shiga.jp/parliament/gijiroku/gijiroku.html
- 分類: 独自 HTML 公開（検索機能なし・PDF 直接ダウンロード形式）
- 文字コード: UTF-8
- 特記: 検索システムを採用しておらず、年度・会期ごとに整理された PDF リンクを一覧ページから直接提供している

## PDF ファイルの命名規則

### ディレクトリ名パターン

| 会議種別 | 令和（r） | 平成（H） |
| --- | --- | --- |
| 定例会 | `r{年号2桁}teirei/` | `H{年号2桁}teirei/` または `h{年号2桁}teirei/` |
| 臨時会 | `r{年号2桁}rinji/` | `H{年号2桁}rinji/` または `h{年号2桁}rinji/` |

例:
- `r07teirei/` → 令和7年 定例会
- `r06rinji/` → 令和6年 臨時会
- `H31teirei/` → 平成31年 定例会
- `h16rinji/` → 平成16年 臨時会

### ファイル名パターン

```
{年号2桁}_{回数}_{日数}.pdf
```

例:
- `07_3_4.pdf` → 令和7年 第3回定例会 第4日
- `06_4_1.pdf` → 令和6年 第4回臨時会 第1日
- `31_1_5.pdf` → 平成31年 第1回定例会 第5日

## 会議構成

- **定例会**: 年4回（第1〜4回）、各定例会は複数日程（最大5日程度）に分割
- **臨時会**: 年間開催数は変動（通常1〜4回）、各臨時会は通常1日構成

## スクレイピング戦略

### Step 1: 一覧ページから PDF リンクを収集

2つの一覧ページ（現在分・過去分）を取得し、ページ内の全 PDF リンクを抽出する。

- ページネーションなし（全データが単一ページに列挙）
- 対象リンク: `href` 属性が `.pdf` で終わるアンカータグ
- リンクテキストから会議名・日程情報を取得する

```typescript
// Cheerio でのリンク抽出例
$('a[href$=".pdf"]').each((_, el) => {
  const href = $(el).attr('href');   // 相対パス（例: r07teirei/07_3_4.pdf）
  const text = $(el).text().trim();  // リンクテキスト（例: 第3回定例会第4日）
  // ...
});
```

### Step 2: PDF URL の組み立て

収集した相対パスからフル URL を組み立てる。

```
https://www.town.ryuoh.shiga.jp/parliament/gijiroku/{相対パス}
```

### Step 3: メタ情報の抽出

PDF リンク周辺の HTML 構造（見出しタグ等）から年度・会議種別を取得し、ファイル名パターンから日程を補完する。

ファイル名パターンによる解析例:

```typescript
// ファイル名から年号・回数・日数を抽出
const filePattern = /^(\d{2})_(\d+)_(\d+)\.pdf$/;
// 例: "07_3_4.pdf" → 年号=07, 回数=3, 日数=4

// ディレクトリ名から年度・種別を判定
const dirPattern = /^[rRhH](\d{2})(teirei|rinji)\/$/;
// teirei → 定例会, rinji → 臨時会
```

### Step 4: PDF のダウンロードとテキスト抽出

- PDF を直接ダウンロードし、テキスト抽出ライブラリ（`pdf-parse` 等）でテキストを取得する
- 会議録の本文テキスト・発言者情報を抽出する

---

## 注意事項

- **ディレクトリ名の大文字/小文字の揺れ**: 平成時代のディレクトリは `H##` と `h##` が混在する可能性がある。URL 取得時はページ上のリンクをそのまま使用し、独自に URL を組み立てない
- **検索機能なし**: キーワード検索や絞り込み機能は存在しないため、全 PDF を一括取得するアプローチのみ有効
- **PDF 形式のみ**: HTML 形式の会議録は提供されていない。テキスト抽出には PDF パーサーが必須
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- **差分更新**: 新規 PDF は一覧ページへの追加により確認できる。前回取得済みの URL リストと比較することで差分更新が可能

---

## 推奨アプローチ

1. **2ページを起点とした全量収集**: `gijiroku.html` と `gijiroku_kako.html` の2ページから全 PDF リンクを収集する
2. **リンクテキストと URL の両方を活用**: リンクテキストから会議名・日程を取得し、ファイル名パターンで補完する
3. **PDF テキスト抽出**: ダウンロードした PDF から `pdf-parse` 等でテキストを抽出し、発言単位でパースする
4. **差分更新**: 既取得 URL のリストを管理し、一覧ページに新規追加された PDF のみを処理する
