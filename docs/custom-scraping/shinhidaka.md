# 新ひだか町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.shinhidaka-hokkaido.jp/gikai/detail/00000185.html
- 分類: 公式サイト内で HTML フレームセット形式の会議録を年度・会期ごとに整理して公開（検索機能なし）
- 文字コード: Shift_JIS
- 特記: 令和7年以降は PDF 形式に移行。令和6年以前は HTML（`<pre>` タグ + フレームセット）で公開。旧ドメイン（shinhidaka.hokkai.jp）と新ドメイン（www.shinhidaka-hokkaido.jp）が混在。

| ページ | URL パターン |

| 会議録フレームセット（令和2年〜令和6年） | `https://www.shinhidaka-hokkaido.jp/gikai/kaigiroku/{年号}/{年号}{月}{種別}.html` |
| 会議録フレームセット（平成21年〜令和元年） | `http://shinhidaka.hokkai.jp/gikai/kaigiroku/{年号}/{年号}{月}{種別}.html` |

### URL 命名規則

- 年号部分: `R06`（令和6年）、`H31`（平成31年）、`h29`（平成29年、小文字の場合あり）
- 種別: `T` = 定例会、`R` = 臨時会
- 例: `R0612T` = 令和6年12月定例会、`R0611R` = 令和6年11月臨時会

| 期間 | ドメイン | ディレクトリ命名例 | ファイル命名例 |

## HTML 構造

### フレームセット構成

各会期の会議録はフレームセット（`<FRAMESET>`）で構成される:

- **上部フレーム**: タイトルバー（`{年号}{月}{種別}_top.html`）
- **左フレーム**: 目次（`{年号}{月}{種別}M.html`）
- **右フレーム**: 本文（`{年号}{月}{種別}_{号数}.html`）

### 目次フレーム（M ファイル）

```html
議会定例会会議録<br>
<a href="R0612T_01.html" target="main">１号（１２月１０日）</a><br>
<a href="R0612T_01.html#議事日程" target="main">議事日程</a><br>
<a href="R0612T_01.html#出席議員" target="main">出席議員</a><br>
<a href="R0612T_01.html#一般質問" target="main">一般質問</a><br>
<a href="R0612T_01.html#池田一也議員" target="main">池田一也議員</a><br>
```

- 号数（日程）ごとのリンクと、各議事項目・議員名のアンカーリンクを含む
- 号数リンクのテキストから開催日を抽出可能（例: `１号（１２月１０日）`）

### 本文フレーム（号数ファイル）

- `<pre>` タグ内にプレーンテキスト形式で会議録を記載
- 一部 HTML タグ（`<b>`, `<br>`, `<a name>`, `<CENTER>`）が混在

#### メタ情報

ページ冒頭の `<CENTER><PRE>` 内にタイトル:

```
令和６年第７回新ひだか町議会定例会会議録
```

その後に議事日程として開催日:

```
令和６年１２月１０日(火)　午前９時３０分開会
```

#### 発言の構造

発言者パターン（`<b>` タグで囲まれる）:

```html
〇<b>議長(福嶋尚人君)</b>　おはようございます。
〇<b>町長(大野克之君)</b>　お手元に配付されております...
〇<b>教育長(久保田達也君)</b>　教育行政報告を申し上げます。
〇<b>２番(池田一也君)</b>　通告に従い、質問をさせていただきます。
〇<b>13番(川端克美君)</b>　私は、令和５年度の...
〇<b>企業会計決算審査特別委員長(池田一也君)</b>　報告をさせていただきます。
〇<b>地域包括支援センター長(及川啓明君)</b>　御答弁いたします。
〇<b>地域包括支援センター長補佐(戸子台弘一君)</b>　お答えいたします。
```

議事項目の見出し:

```html
◎<b>開会の宣告</b>
◎<b>一般質問</b>
```

#### パース用正規表現（案）

```typescript
// 発言者の抽出（〇 + <b> + 役職/番号 + (氏名君) + </b>）
const speakerPattern = /〇<b>(.+?)\((.+?)君\)<\/b>/;
// 例: 〇<b>議長(福嶋尚人君)</b> → role="議長", name="福嶋尚人"
// 例: 〇<b>２番(池田一也君)</b> → role="２番", name="池田一也"

// 議事項目見出しの抽出
const agendaPattern = /◎<b>(.+?)<\/b>/;

// タイトルの抽出
const titlePattern = /(?:令和|平成)\d+年第\d+回新ひだか町議会(?:定例会|臨時会)会議録/;

// 開催日の抽出（全角数字）
const datePattern = /(?:令和|平成)([\d１-９０]+)年([\d１-９０]+)月([\d１-９０]+)日/;
```

## スクレイピング戦略

### Step 1: 会議録 URL の収集

会議録一覧ページ `https://www.shinhidaka-hokkaido.jp/gikai/detail/00000185.html` から全会議録へのリンクを抽出する。

- 単一ページに全年度・全会期のリンクが列挙されている（ページネーションなし）
- HTML 会議録リンクと PDF リンクが混在
- HTML リンクはフレームセットページへのリンク

**収集方法:**

1. 一覧ページの HTML を取得
2. `kaigiroku/` を含むリンクを抽出（HTML 会議録）
3. `/hotnews/files/` を含むリンクを抽出（PDF）
4. HTML 会議録については、フレームセットの `<frame name=index>` の `src` 属性から目次ファイル URL を特定
5. 目次ファイルから各号数の本文ファイル URL を抽出

### Step 2: 本文 HTML の取得

各号数の本文ファイル（例: `R0612T_01.html`）を直接取得する。

- フレームセットを介さず、本文ファイルに直接アクセス可能
- 文字コードは Shift_JIS のため、デコード処理が必要
- `<pre>` タグ内のテキストが本文

### Step 3: 会議録のパース

#### メタ情報の抽出

1. `<CENTER><PRE>` 内のタイトルから会議名・回数を抽出
2. 議事日程の冒頭行から開催日を抽出
3. 出席議員一覧から議員情報を抽出

#### 発言の抽出

1. `〇<b>` で始まる行を発言者の開始として検出
2. 発言者行から役職（or 議席番号）と氏名を分離
3. 次の `〇<b>` または `◎<b>` までのテキストを発言内容として取得
4. `<br>` タグを改行に変換し、HTML タグを除去

### Step 4: PDF の処理（令和7年以降）

令和7年以降の会議録は PDF 形式で公開されているため、別途 PDF パーサーが必要。

- PDF ファイル名はタイムスタンプ形式（`YYYYMMDDHHmmss.pdf`）
- 一覧ページのリンクテキストから会期・日程情報を取得可能

---

## 注意事項

- 文字コードが Shift_JIS のため、UTF-8 への変換処理が必須
- 旧ドメイン（`shinhidaka.hokkai.jp`）のリンクがリダイレクトされるか、またはアクセス不可の可能性がある（要確認）
- 平成24年以前の旧命名規則（`h2106t4-1-n.html`）ではフレームセット構成が異なる可能性がある
- 議席番号は全角（`２番`）と半角（`13番`）が混在する
- 発言者の氏名には全角スペースが含まれる場合がある（例: `川合　清`）
- `〇`（漢数字のゼロ）が発言者マーカーとして使われている（`○`（丸）ではない）
- `◎` は議事項目の見出しマーカー

---

## 推奨アプローチ

1. **一覧ページから全量収集**: 単一ページに全リンクがあるため、1 回のリクエストで全会議録 URL を取得可能
2. **フレームセットをスキップ**: 目次ファイル（M ファイル）に直接アクセスし、本文ファイル URL を効率的に収集
3. **Shift_JIS デコード**: `iconv` や Node.js の `iconv-lite` パッケージで UTF-8 に変換
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **旧ドメイン対応**: 旧ドメインのリンクは新ドメインに書き換えてアクセスを試みる
6. **PDF 対応は後回し**: まず HTML 形式の会議録（平成21年〜令和6年）を優先的に対応し、PDF は後続タスクとする
