## 会議の種別

- **定例会**: 年4回（3月・6月・9月・12月）
- **臨時会**: 不定期開催（年1〜9回程度）

回次の命名規則: `第{N}回（{月}月）{定例会|臨時会}`

例:
- 第1回（3月）定例会
- 第3回（9月）臨時会

## ページ構造

一覧ページは年度別 → 会議種別別 → 日付別の三層構造。

```
令和7年
  ├─ 第4回（12月）定例会
  │   ├─ 12月5日会議録 [PDF] → /uploaded/attachment/9646.pdf
  │   ├─ 12月3日会議録 [PDF] → /uploaded/attachment/9645.pdf
  │   └─ 12月2日会議録 [PDF] → /uploaded/attachment/9644.pdf
  ├─ 第3回（9月）定例会

  │   └─ ...
  └─ 第1回（1月）臨時会
```

- 年度見出しは太字テキスト
- 会議種別見出しも太字テキスト
- PDF リンクのテキスト形式: `{月}月{日}日会議録 [PDF]`
- 1日1PDF（複数日の会議は別々の PDF）

## PDF の構造

### ヘッダー・メタ情報

PDF 冒頭に以下の情報が記載される:

```
令和７年第３回（９月）大郷町議会定例会会議録第３号
令和７年１２月５日（金）
```

- 1行目: 会議名（年・回次・会議種別・会議録号数）
- 2行目: 開催日・曜日

### 出席者情報

```
応招議員（１２名）
　　１番　鈴木安則　君　　　２番　赤間繁幸　君
　　...

出席議員（１２名）
　　応招議員と同じ

欠席議員（　０名）
　　な　　し
```

- 議員は番号 + 氏名 + 「君」で表記
- 説明員（町長・課長等）も役職 + 氏名 + 「君」で列挙

### 議事日程

```
議事日程第３号
　令和７年１２月５日（金曜日）　午前１０時開議
　日程第１　会議録署名議員の指名
　日程第２　議案第５０号　大郷町議会議員の議員報酬及び費用弁償等に...
```

### 発言の構造

発言者パターン:

```
議長（石垣正博君）　皆さん、おはようございます。
保健福祉課長（小野純一君）　皆様、おはようございます。
```

- `{役職}（{氏名}君）` の形式で発言者を表記
- `○` は使用しない（中野区方式とは異なる）
- 役職は「議長」「町長」「{部署名}課長」「{N}番」など
- 氏名の後に「君」が付く
- 発言内容は発言者行の後に続くテキスト

場内のト書き:

```
［「なし」と呼ぶ者あり］
［「省略」と呼ぶ者あり］
〔賛成者起立〕
```

- `［...］` または `〔...〕` で囲まれた記述

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `kaigiroku.html` から全 PDF リンクを抽出する。

- 単一ページに全データが掲載されているため、1回の HTTP リクエストで全リンクを取得可能
- `<a>` タグの `href` 属性から `/uploaded/attachment/{ID}.pdf` パターンを抽出
- 各リンクの親要素から会議種別（定例会/臨時会）と年度を取得

**収集方法:**

1. `kaigiroku.html` を取得
2. Cheerio でパースし、PDF リンク（`/uploaded/attachment/*.pdf`）を全件抽出
3. 各リンクの前後のテキストから年度・会議種別・開催日を紐づけ

### Step 2: PDF のダウンロード

収集した URL リストに基づき、PDF ファイルをダウンロードする。

- URL: `https://www.town.miyagi-osato.lg.jp/uploaded/attachment/{ID}.pdf`
- ファイルサイズは 100KB〜300KB 程度

### Step 3: PDF のテキスト抽出

PDF からテキストを抽出する。`pdf-parse` 等のライブラリを使用。

### Step 4: テキストのパース

#### メタ情報の抽出

```typescript
// 会議名の抽出
const titlePattern = /(?:令和|平成)[０-９]+年第[０-９]+回（[０-９]+月）大郷町議会(?:定例会|臨時会)会議録(?:第[０-９]+号)?/;

// 開催日の抽出（全角数字）
const datePattern = /(?:令和|平成)([０-９]+)年([０-９]+)月([０-９]+)日/;
```

#### 発言者の抽出

```typescript
// 発言者パターン: 役職（氏名君）
const speakerPattern = /^(.+?)（(.+?)君）/;
// 例: 議長（石垣正博君） → role="議長", name="石垣正博"
// 例: 保健福祉課長（小野純一君） → role="保健福祉課長", name="小野純一"
// 例: ７番（鈴木恵子君） → role="７番", name="鈴木恵子"

// ト書きの検出
const stageDirectionPattern = /^[［〔](.+?)[］〕]/;
```

## 注意事項

- PDF 形式のため、テキスト抽出時にレイアウト由来の不要な改行・空白が混入する可能性がある
- 全角数字が多用されている（議員番号、日付、議案番号等）
- 発言者表記に `○` は使われない（行頭の役職名で判別する）
- 議員名の後に必ず「君」が付く（敬称として統一）
- 出席者一覧のテーブル部分は2列レイアウトのため、PDF テキスト抽出時に列の混在が起こりうる

---

## 推奨アプローチ

1. **一覧ページ1回取得で完結**: 全 PDF リンクが単一ページに集約されているため、クロール負荷が極めて低い
2. **PDF テキスト抽出の品質確認**: 複数の PDF でテキスト抽出結果を検証し、レイアウト崩れのパターンを把握する
3. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: 一覧ページの PDF リンク数を前回と比較し、新規追加分のみダウンロード・パースする
5. **全角→半角変換**: 日付・番号の全角数字は半角に正規化してからデータベースに格納する
