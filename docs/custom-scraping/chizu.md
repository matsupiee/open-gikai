# 智頭町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www1.town.chizu.tottori.jp/chizu/gikaijimukyoku/gijiroku/
- 分類: 町公式サイトで年度別 PDF を直接公開（外部の専用検索システムは使用していない）
- 文字コード: UTF-8
- 特記: PDF は Microsoft Word 2021 から生成されており、フォントは MS-Mincho（明朝体）を使用

## 会議種別

| 種別 | 備考 |
| --- | --- |
| 定例会 | 年4回が標準（3月、6月/7月、9月、12月） |
| 臨時会 | 年によって回数が異なる |

委員会別の会議録は掲載されていない。すべて本会議（定例会・臨時会）のみ。

各定例会は複数日（通常3日間：初日、2日目（一般質問）、最終日）に分かれて掲載される。臨時会は「1日限り」として1ファイルで掲載される。

## PDF の URL パターン

年代により 3 種類の PDF ホスティング形式が存在する。

### パターン 1: `/user/filer_public/` 形式（令和4年以降）

```
/user/filer_public/{2文字}/{2文字}/{UUID}/{ローマ字ファイル名}.pdf
```

例: `/user/filer_public/29/51/2951832d-9e7e-4dc4-a5ce-2da2977a53a5/zhi-tou-ting-ling-he-6nian-di-4hui-ding-li-hui-1.pdf`

- UUID を含むパスで、推測不可能
- ファイル名は日本語のローマ字変換（ピンイン風）

### パターン 2: `/photolib/chizu_gikai/` 形式（平成27〜令和3年）

```
/photolib/chizu_gikai/{数値ID}.pdf
```

例: `/photolib/chizu_gikai/12805.pdf`

- 数値 ID は連番

### パターン 3: `/system/site/upload/live/1716/` 形式（平成24〜25年）

```
/system/site/upload/live/1716/atc_{タイムスタンプ}.pdf
```

例: `/system/site/upload/live/1716/atc_1392732629.pdf`

- 一部は `.doc` 形式

## ページ構造

### 会議録トップページ

トップページ (`/chizu/gikaijimukyoku/gijiroku/`) に以下が含まれる:

- 令和8年〜平成26年: 年度別一覧ページへのリンク
- 平成24〜25年: トップページに直接 PDF リンクが掲載

ページネーションなし。全年度が単一ページに表示される。

### 年度別会議録一覧ページ

各年度ページ（例: `/chizu/gikaijimukyoku/gijiroku/s166/`）には:

- 各定例会・臨時会がセクション（h3）で区切られる
- 各セクション内に PDF リンクが「初日」「2日目」「最終日」等で掲載
- ページネーションなし

## PDF 内部構造

### メタ情報（PDF 冒頭）

```
智頭町議会定例会会議録

令和６年１２月５日開議
```

- 1行目: `智頭町議会{定例会|臨時会}会議録`
- 2行目: 全角数字で `{和暦}年{月}月{日}日開議`

### 議事日程

冒頭に議事日程が番号付きリストで記載される。

### 出席者情報

以下の順序で記載:

1. `１．会議に出席した議員（{N}名）` - 議員番号と氏名の一覧
2. `１．会議に欠席した議員（{N}名）`
3. `１．会議に出席した説明員（{N}名）` - 役職と氏名の一覧
4. `１．会議に出席した事務局職員（{N}名）`

### 発言の構造

発言者パターン:

```
○議長（谷口雅人）
○町長（金兒英夫）
○副町長（矢部整）
○教育長（田中靖）
○総務課長（國岡厚志）
○５番（宮本行雄）
```

- `○` で始まり、役職 or 議員番号 + 括弧内に氏名
- 発言内容は発言者行の後に続くテキスト
- 議事の区切りとして `日程第{N}．{議題名}` が挿入される

### その他の表記

```
（「異議なし」と呼ぶ者あり）
```

- 議場の反応は括弧内に記載

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（谷口雅人） → role="議長", name="谷口雅人"
// 例: ○５番（宮本行雄） → role="５番", name="宮本行雄"

// 開催日の抽出
const datePattern = /(?:令和|平成)(.+?)年(.+?)月(.+?)日/;

// 会議種別の抽出（PDF 冒頭から）
const sessionTypePattern = /智頭町議会(定例会|臨時会)会議録/;

// 議事日程の区切り
const agendaPattern = /^日程第(.+?)．(.+)/;
```

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録トップページを取得し、年度別ページへのリンクを抽出する。

```typescript
// 年度別ページリンクの抽出
const yearLinks = $('a[href*="/gijiroku/"]')
  .filter((_, el) => {
    const href = $(el).attr('href')!;
    return href !== '/chizu/gikaijimukyoku/gijiroku/' && !href.endsWith('.pdf');
  })
  .map((_, el) => ({
    url: new URL($(el).attr('href')!, BASE_URL).toString(),
    label: $(el).text().trim(),
  }))
  .toArray();
```

### Step 2: 各年度ページから PDF URL を収集

各年度別ページを取得し、PDF リンクとそのアンカーテキストを抽出する。

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]')
  .map((_, el) => ({
    url: new URL($(el).attr('href')!, BASE_URL).toString(),
    label: $(el).text().trim(),
  }))
  .toArray();
```

平成24〜25年はトップページから直接 PDF リンクを収集する。

### Step 3: 会議メタ情報の抽出

アンカーテキストから開催日と会議の日目を抽出する。

**テキスト例:**

- `初　日（R6.12.05）`
- `２日目（R6.12.06）　一般質問`
- `最終日（R6.12.12）`
- `１日限り（R6.06.14）`

```typescript
// アンカーテキストから日付を抽出
const dateFromLabel = /[（(][RH](\d+)\.(\d+)\.(\d+)[)）]/;
// 例: "初　日（R6.12.05）" → era="R", year=6, month=12, day=5

// 会議名はページの見出し（h3等）から取得
const sessionPattern = /第(\d+)回(定例会|臨時会)/;
```

### Step 4: PDF を取得・テキスト抽出

PDF ファイルをダウンロードし、pdf-parse 等でテキスト化する。

### Step 5: PDF テキストのパース

テキスト化した内容から発言者と発言内容を抽出する。

## 注意事項

- **URL スラッグに規則性がない**: 年度別ページの URL は推測できないため、必ずトップページから収集する
- **3 種類の PDF ホスティング**: 年代により PDF の URL 構造が異なる（`/user/filer_public/`、`/photolib/`、`/system/site/upload/`）
- **平成24年の一部は .doc 形式**: PDF ではなく Word ファイルで公開されているものがある
- **PDF テキスト抽出の品質**: Word 由来の PDF のため、pdftotext でのテキスト抽出時に文字間にスペースが入る場合がある（特に出席者名簿部分）
- **ページネーションなし**: トップページも年度別ページも単一ページに全情報が掲載される
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **2段階でリンク収集**: トップページ → 年度別ページの順に PDF URL を収集する（平成24〜25年はトップページから直接収集）
2. **リンクから直接収集**: PDF URL は HTML の `<a href>` から取得し、URL を推測・構築しない
3. **テキストからメタ情報取得**: アンカーテキストと PDF 冒頭のヘッダーから会議名・開催日を抽出する
4. **差分更新**: 既取得 URL のリストと比較し、新規 URL のみ PDF を取得する
5. **レート制限**: 各リクエスト間に 1〜2 秒の待機を設ける
6. **.doc ファイルの取り扱い**: 平成24年の一部は .doc 形式のため、別途変換処理が必要（優先度は低い）
