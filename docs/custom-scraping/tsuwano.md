# 津和野町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.tsuwano.lg.jp/www/contents/1707956534539/index.html
- 分類: 公式ウェブサイト上で会議録を PDF ファイルとして直接提供（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 全会議録が単一ページに一覧表示される。検索システムなし。PDF 形式での提供。

| 年代 | 命名規則 | 例 |

## ページ構造

### 一覧ページの HTML 構造

- 全会議録が単一ページに掲載（ページネーションなし）
- 年度ごとに `<h3>` タグで区切り（例: `令和６年`, `平成30年`）
- 各年度内の会議は `<h6>` タグで表記（例: `第３回3月定例会`, `第１回２月臨時会`）
- `<h6>` の下に PDF リンクが直接配置（リスト・テーブルは不使用）

### リンクテキストのパターン

単一日程の会議:
```
第１回２月臨時会（PDF／273KB）
```

複数日程の定例会:
```
第１日目   令和６年３月８日（PDF／1247KB）
第２日目　令和６年３月13日（PDF／1126KB）
```

## スクレイピング戦略

### Step 1: 一覧ページから PDF リンクの収集

一覧ページ `https://www.town.tsuwano.lg.jp/www/contents/1707956534539/index.html` を取得し、全 PDF リンクとメタ情報を抽出する。

**収集方法:**

1. 一覧ページの HTML を取得
2. `<h3>` タグから年度を抽出
3. `<h6>` タグから会議名（回数・月・種別）を抽出
4. 各 `<h6>` の後続にある `<a>` タグから PDF の URL とリンクテキストを抽出
5. リンクテキストから開催日（複数日程の場合）を抽出

**抽出する情報:**

| フィールド | 抽出元 |
| --- | --- |
| 年度 | `<h3>` テキスト（例: `令和６年`） |
| 会議名 | `<h6>` テキスト（例: `第３回3月定例会`） |
| 開催日 | リンクテキスト内の日付（例: `令和６年３月８日`） |
| PDF URL | `<a>` の `href` 属性 |

### Step 2: PDF ファイルのダウンロード

収集した PDF URL から各ファイルをダウンロードする。

- 全 PDF が同一ディレクトリ（`/www/contents/1707956534539/simple/`）に格納されている
- ファイルサイズは 200KB〜1.5MB 程度

### Step 3: PDF からテキスト抽出

PDF ファイルからテキストを抽出し、会議録データとしてパースする。

- PDF パーサー（例: `pdf-parse`, `pdfjs-dist`）を使用してテキストを抽出
- 抽出したテキストから発言者と発言内容を分離

#### メタ情報の抽出

一覧ページのリンクテキストおよび `<h6>` テキストから:

```typescript
// 会議名の抽出（h6テキスト）
const meetingPattern = /第(\d+)回(\d+)月(定例会|臨時会)/;
// 例: 第３回3月定例会 → 回数=3, 月=3, 種別="定例会"

// 年度の抽出（h3テキスト）
const yearPattern = /(令和|平成)\s*(\d+)年/;

// 開催日の抽出（リンクテキスト内、複数日程の場合）
const datePattern = /(令和|平成)(\d+)年(\d+)月(\d+)日/;

// 日程番号の抽出（複数日程の場合）
const dayPattern = /第(\d+)日目/;
```

#### 発言の構造（PDF 内テキスト）

PDF 内のテキスト構造は PDF の生成方法に依存するため、実際にダウンロードした PDF を解析して発言者パターンを特定する必要がある。一般的な町議会の会議録では以下のようなパターンが想定される:

```
○議長（氏名）
○副議長（氏名）
○○番（氏名）
○町長（氏名）
```

## 注意事項

- 全会議録が PDF 形式のため、HTML パースではなく PDF パースが必要
- PDF のテキスト抽出精度は PDF の生成方法（スキャン画像 vs テキスト埋め込み）に依存する
- 一覧ページが単一ページのため、クロール対象は 1 ページ + 各 PDF ファイル
- ファイル名の命名規則が年度により異なるため、ファイル名からのメタ情報抽出は信頼性が低い（一覧ページのテキストを正とする）
- 令和5年の一部ファイルは `gijirokuNN.pdf` という別命名規則を使用

---

## 推奨アプローチ

1. **一覧ページから全量取得**: 単一ページに全 PDF リンクが掲載されているため、1 リクエストで全リンクを収集可能
2. **メタ情報は一覧ページから取得**: PDF ファイル名ではなく、`<h3>` / `<h6>` / リンクテキストからメタ情報を構築する
3. **PDF テキスト抽出の事前検証**: 数件の PDF をダウンロードし、テキスト抽出の品質を確認してからパース戦略を確定する
4. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページの PDF リンク数を前回取得時と比較し、新規追加分のみをダウンロードする
