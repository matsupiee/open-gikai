# 忠岡町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.tadaoka.osaka.jp/gyousei/gikai/2110.html
- 分類: 町公式サイト内での直接公開（外部の会議録検索システムは未導入）
- 文字コード: UTF-8
- CMS: Smart LGov
- 特記: 平成27年以降の会議録を PDF ファイルとして掲載。一覧ページから各会議の詳細ページへ遷移し、詳細ページ内で日ごとに分割された PDF をダウンロードする構成。一部の臨時会は一覧ページから直接 PDF にリンクされている。

## ページ構造

### 一覧ページ（2110.html）

- 全会議録が1ページ内に表示される（ページネーションなし）
- 会議録は年度ごとにグループ化されているが、HTML 上は明示的なセクション区切りなし
- 各会議録は `<a>` タグによるリンクで、リンクテキストに会議名が含まれる
- リンク先は主に詳細ページ（`.html`）だが、一部は直接 PDF ファイルにリンク

### 詳細ページ

- `<h1>` タグにページタイトル（会議名）が記載
- 本会議と委員会で `<h3>` 見出しによるセクション分け（「本会議 会議録」「委員会 会議録」）
- 各 PDF は `<a>` タグでリンクされ、リンクテキストに開催日が含まれる
- ファイルサイズが括弧内に併記される

- **本会議 会議録**: 日ごとに分割された PDF（通常3〜4日分）
- **委員会 会議録**: 総務事業常任委員会、福祉文教常任委員会の PDF

## スクレイピング戦略

### Step 1: 一覧ページから詳細ページ URL の収集

一覧ページ `2110.html` から全会議録のリンクを抽出する。

1. `https://www.town.tadaoka.osaka.jp/gyousei/gikai/2110.html` を取得
2. ページ内の `<a>` タグからリンクを収集
3. リンク先を2種類に分類:
   - **詳細ページ**（`.html` で末尾が数字ID）→ Step 2 へ
   - **直接 PDF**（`.pdf`）→ Step 3 へ直接進む
4. リンクテキストから会議名・年度を抽出

**収集対象のリンクパターン:**

```typescript
// 詳細ページへのリンク
const detailPattern = /\/soshiki\/gikai_jimukyoku\/1\/(\d+)\.html/;
const detailPattern2 = /\/gyousei\/gikai\/[^/]+\/(\d+)\.html/;

// 直接PDFリンク
const directPdfPattern = /\/material\/files\/group\/21\/[^"]+\.pdf/;
```

### Step 2: 詳細ページから PDF リンクの取得

各詳細ページにアクセスし、PDF ファイルへのリンクを抽出する。

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]');
```

- 詳細ページの `<h1>` タグから会議名を取得
- `<h3>` タグから本会議 / 委員会の区分を取得
- 各 PDF リンクのテキストから開催日を取得

### Step 3: メタ情報の抽出

#### 一覧ページのリンクテキストから

```
令和6年第3回定例会会議録
→ 年度: 令和6年, 回数: 第3回, 種別: 定例会

第6回ごみ処理施設調査特別委員会会議録（令和5年9月26日開催）
→ 種別: ごみ処理施設調査特別委員会, 回数: 第6回, 開催日: 令和5年9月26日
```

#### メタ情報抽出用正規表現

```typescript
// 定例会・臨時会パターン
const sessionPattern = /^(平成|令和)\d+年第\d回(定例会|臨時会|臨時議会)会議録/;

// 特別委員会パターン
const committeePattern = /^(?:第(\d+)回)?(.+?特別委員会)会議録(?:（(.+?)開催）)?/;

// 年度の抽出
const yearPattern = /(平成|令和)(\d+)年/;
```

### Step 4: PDF のダウンロードとテキスト抽出

- 収集した PDF URL からファイルをダウンロード
- PDF パーサー（pdf-parse 等）でテキストを抽出
- 抽出したテキストから発言者・発言内容をパース

## 注意事項

- 一覧ページにページネーションはないため、1回のリクエストで全件取得可能
- PDF ファイル名の命名規則が時期によって異なる（ハッシュ型 / 日付型 / 数字ID型）ため、ファイル名からのメタ情報推定は補助的にのみ使用し、主にリンクテキストやページタイトルからメタ情報を取得すること
- 一部の臨時会（令和2年第2回・第3回臨時会）は一覧ページから直接 PDF にリンクされており、詳細ページが存在しない
- PDF の URL はプロトコル相対パス（`//www.town.tadaoka.osaka.jp/...`）で記載される場合がある。`https:` を付与して正規化すること
- リクエスト間には適切な待機時間（1〜2秒）を設けること

---

## 推奨アプローチ

1. **一覧ページの一括取得**: ページネーションがないため、`2110.html` を1回取得するだけで全会議録リンクを収集できる
2. **詳細ページの巡回**: 各詳細ページから PDF リンクとメタ情報を抽出する
3. **PDF のテキスト抽出**: ダウンロードした PDF からテキストを抽出し、発言データを構造化する
4. **差分更新**: 詳細ページの ID（数字部分）が増加するため、前回取得済みの最大 ID 以降の新規ページのみを巡回する差分更新が可能
