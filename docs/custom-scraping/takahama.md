## ページ構造

### 定例会・臨時会ページ

- 全年度の会議録が **単一ページ** に掲載（ページネーションなし）
- 年度ごとに `<h3>` 見出し + `id` 属性でセクション分割（例: `#r7`, `#r6`）
- 各年度内は月別（12月、9月、6月、3月）に分類
- 各会議日ごとに PDF リンクが列挙される

### リンクテキストの形式

定例会:
```
第1日  11月25日 [PDF413KB]
第2日  11月27日 [PDF512KB]
```

臨時会:
```
第1日  1月30日 [PDF310KB]
```

委員会:
```
12月9日 総務建設委員会 [PDF282KB]
```

### PDF の URL パターン

```
/uploaded/attachment/{数値ID}.pdf
```

- ID は連番ではなく、アップロード順の通し番号
- 同一定例会の複数日でも ID は連番にならない場合がある

## スクレイピング戦略

### Step 1: 各カテゴリページから PDF リンクを収集

対象ページ（定例会・臨時会・委員会・特別委員会）の HTML を取得し、PDF リンクを抽出する。

**収集方法:**

1. 定例会ページ (`/site/gikai/1529.html`) の HTML を取得
2. 臨時会ページ (`/site/gikai/16707.html`) の HTML を取得
3. 委員会一覧ページ (`/site/gikai/1497.html`) から各委員会の個別ページ URL を取得し、各ページの HTML を取得
4. 特別委員会一覧ページ (`/site/gikai/2050.html`) から各特別委員会の個別ページ URL を取得し、各ページの HTML を取得
5. 各ページから `/uploaded/attachment/{ID}.pdf` 形式のリンクを Cheerio で抽出
6. リンクテキストから会議日・会議種別を併せて取得

**抽出対象:**

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href*="/uploaded/attachment/"]')
  .map((_, el) => ({
    url: $(el).attr("href"),
    text: $(el).text().trim(),
  }))
  .get();
```

### Step 2: メタ情報の抽出

各 PDF リンクの周辺 HTML 構造から以下を抽出する:

- **開催日**: リンクテキスト内の日付（例: `11月25日`）+ 年度セクションの見出しから年を補完
- **会議種別**: ページ種別（定例会/臨時会）またはリンクテキスト内の委員会名
- **定例会月**: 月別セクションの見出しから取得（定例会の場合）

```typescript
// 年度の抽出（セクション見出しから）
const yearPattern = /(?:令和|平成)(\S+?)年/;

// 開催日の抽出（リンクテキストから）
const datePattern = /(\d+)月(\d+)日/;

// 委員会名の抽出（委員会ページのリンクテキストから）
const committeePattern = /\d+月\d+日\s+(.+委員会)/;
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL から PDF ファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容をパース

---

## PDF 内テキストのパース方針

PDF から抽出したテキストの構造は、PDF の作成方法に依存するため実際のファイルで検証が必要。一般的な議会会議録 PDF では以下のパターンが想定される:

### 発言者パターン（想定）

```
○議長（氏名）
○副議長（氏名）
○XX番（氏名）
○市長（氏名）
```

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（山田太郎） → role="議長", name="山田太郎"

// 番号付き議員
const numberedSpeakerPattern = /^○(\d+)番（(.+?)）/;
// 例: ○5番（鈴木一郎） → number="5", name="鈴木一郎"
```

※ PDF からのテキスト抽出結果によっては、改行位置やスペースの扱いが異なる場合がある。実際の PDF で検証し、正規表現を調整する必要がある。

---

## 注意事項

- 掲載会議録は「公式記録ではない」旨がページに明記されている
- PDF のファイルサイズは数百 KB〜1 MB 程度（リンクテキストに `[PDF###KB]` で表示）
- 古い年度の定例会は HTML 形式で提供されている場合があり、別途パース処理が必要
- PDF の内容構造は作成ツール（Word → PDF 変換等）に依存するため、テキスト抽出精度に注意

---

## 推奨アプローチ

1. **一覧ページからの全量収集を優先**: 定例会・臨時会は単一ページに全年度が掲載されているため、1 リクエストで全 PDF リンクを取得可能
2. **委員会・特別委員会は 2 段階取得**: 一覧ページ → 個別委員会ページの順に遷移して PDF リンクを収集
3. **PDF テキスト抽出の事前検証**: 実際の PDF ファイルを数件ダウンロードし、テキスト抽出の品質・発言者パターンを確認してからパーサーを実装
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: PDF の attachment ID は増加傾向にあるため、前回取得済みの最大 ID 以降のリンクのみを新規取得する差分更新が可能
