## PDF リンクのパターン

PDF は各会議種別ページから直接リンクされる。ベース URL は以下のとおり。

```
https://www.town.ikaruga.nara.jp/cmsfiles/contents/0000000/419/{ファイル名}.pdf
```

ページ ID（419 等）は会議種別に対応する。

## スクレイピング戦略

### Step 1: PDF リンクの収集

各会議種別ページ（8 URL）を取得し、PDF リンクを抽出する。

- ページネーションなし。全年度が単一ページに降順で掲載される
- `<a href="./cmsfiles/contents/...pdf">` パターンで PDF リンクを抽出
- リンクテキストから会議名・日程を取得（例: `第1日目`、`目次`）
- `<h2>` 見出しから年度（令和/平成）を取得
- ページ内アンカーリンクでセクションが区切られている

```typescript
// PDF リンクの抽出例（Cheerio）
$('a[href$=".pdf"]').each((_, el) => {
  const href = $(el).attr('href');
  const text = $(el).text().trim();
  // href から絶対 URL を生成
});
```

### Step 2: メタ情報の解析

PDF リンクテキストおよび前後の HTML 要素から以下を取得する。

| 情報 | 取得元 |
| --- | --- |
| 年度 | `<h2>` 見出し（例: `令和7年`、`平成13年`） |
| 会議名 | セクション見出しまたはリンク直前のテキスト（例: `12月定例会`、`5月臨時会`） |
| 日程 | リンクテキスト（例: `第1日目`、`目次`） |
| PDF URL | `href` 属性の絶対 URL |

### Step 3: PDF のダウンロードとテキスト抽出

PDF からテキストを抽出する場合は PDF パーサー（`pdf-parse` 等）を使用する。

- 本会議は目次 PDF と日程別 PDF を別々に処理する
- 容量は 300KB〜950KB 程度

---

## 注意事項

- ファイル名の命名規則が年代・会議種別によって統一されておらず、URL からのメタ情報解析は困難。HTML の前後テキストから取得すること
- 令和と平成で日付フォーマットが異なる（`R7.12.11` と `H13.12.03`）
- 本会議の目次 PDF は会議録本文ではないため、インデックス対象から除外するか別扱いにする
- 「その他の委員会」（ページ ID: 434）は廃止済みのため更新はない
- 全データが単一ページに掲載されるため、クロール負荷は低い（8 リクエストで全 PDF URL を取得可能）

---

## 推奨アプローチ

1. **会議種別ページを順次取得**: 8 つの一覧ページを取得し、PDF リンクを収集
2. **HTML から年度・会議名を解析**: `<h2>` および周辺テキストでメタ情報を付与
3. **PDF をダウンロードしてテキスト抽出**: 本会議の日程別 PDF を優先処理
4. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
5. **差分更新**: PDF URL のリストを保持し、既取得 URL はスキップする
