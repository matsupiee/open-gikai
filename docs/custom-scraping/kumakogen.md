```

{元号}{年}年第{回}回{月}月臨時会
```

例（令和元年）:
- 令和元年第6回12月定例会

- 令和元年第3回6月定例会

- 平成31年第1回3月定例会

## HTML 構造

### 年度別会議一覧ページ（list149-{ID}.html）

各会議がリスト形式で表示される。

```html
<li><a href="/site/gikai/{ID}.html">{会議名}</a> / {YYYY}年{M}月{D}日更新</li>
```

例:
```html
<li><a href="/site/gikai/23389.html">令和6年第4回12月定例会</a> / 2025年2月20日更新</li>
```

### 個別会議ページ（{ID}.html）

各開催日の PDF へのリンクがリスト形式で提供される。

```html
<a href="/uploaded/attachment/{ID}.pdf">{会議名}（{日程}）</a>
```

例:
```html
<a href="/uploaded/attachment/13081.pdf">令和6年12月定例会（1日目）</a>  <!-- 909KB -->
<a href="/uploaded/attachment/13083.pdf">令和6年12月定例会（2日目）</a>  <!-- 686KB -->
<a href="/uploaded/attachment/13109.pdf">令和6年12月定例会（最終日）</a> <!-- 625KB -->
```

PDF は日別単位で提供される（1日目、2日目、最終日など）。

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

年度一覧トップ（`list149.html`）をフェッチし、`/site/gikai/list149-{ID}.html` 形式のリンクをすべて抽出する。

### Step 2: 個別会議ページ URL の収集

各年度別ページ（`list149-{ID}.html`）をフェッチし、`/site/gikai/{数字}.html` 形式のリンクを抽出する。アンカーテキストから会議名（例: 「令和6年第4回12月定例会」）を取得する。

**収集方法:**

```
/site/gikai/{4〜5桁の数字}.html
```

のパターンに一致するリンクを抽出する（`list149` や `list31` などのサブディレクトリ形式は除外する）。

### Step 3: PDF リンクの収集

各個別会議ページをフェッチし、`/uploaded/attachment/{ID}.pdf` 形式の PDF リンクをすべて抽出する。

アンカーテキストから日程情報（「1日目」「2日目」「最終日」など）を取得し、メタ情報として保存する。

### Step 4: PDF のダウンロード

抽出した PDF URL から会議録 PDF をダウンロードする。PDF はテキスト抽出処理（pdftotext 等）でテキスト化する。

## ページネーション

ページネーションは存在しない。年度別ページには当該年度の会議がすべて単一ページに表示される。

---

## 注意事項

- 会議録はすべて PDF 形式で提供されており、HTML テキストでの提供はない
- PDF のファイル名（`{ID}.pdf`）は連番ではなく CMS 内部 ID であるため、番号の連続性は保証されない
- 年度別ページの ID（`list149-{ID}.html`）も CMS 内部 ID のため、年度と ID の対応は年度一覧トップから毎回取得する
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **3段階クロール**: 年度一覧 → 年度別会議一覧 → 個別会議ページの順に URL を収集し、最後に PDF をダウンロードする
2. **差分更新**: 個別会議ページの URL（`/site/gikai/{ID}.html`）を既取得リストと照合し、新規分のみ処理する
3. **PDF テキスト化**: PDF から pdftotext 等でテキストを抽出し、発言者・発言内容を解析する
4. **レート制限**: 各リクエスト間に 1〜2 秒の待機を設ける
