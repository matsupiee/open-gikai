# 江田島市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.etajima.hiroshima.jp/cms/categories/show/473
- 分類: 市公式サイトで年度別に PDF ファイルとして会議録を公開（会議録検索システム未導入）
- 文字コード: UTF-8
- 特記: 旧 SSP（ssp.kaigiroku.net/tenant/etajima/）は廃止済み。平成17年（2005年）〜令和7年（2025年）の会議録を年度別ページで PDF 公開。PDF ファイル名の命名規則は年度によって異なる。

## ページ構造

### 会議録一覧ページ（トップ）

- 年度別リンクが新しい順に一覧表示（全21件）
- ページネーションなし（単一ページに全年度を表示）
- 各リンクは `/cms/articles/show/{記事ID}` 形式

### 年度別会議録ページ

- 定例会・臨時会ごとにセクション分けされている
- セクション見出し形式: `令和X年第Y回定例会（令和X年Z月）` / `令和X年第Y回臨時会（令和X年Z月）`
- 各セクション内に表紙・目次・日目別の PDF リンクが配置される
- PDF リンクテキスト形式: `第X回定例会会議録（表紙　PDF）` / `第X回定例会会議録（Y日目　PDF）`

---

## PDF ファイルの構成

### 各会議の PDF 構成

1. **表紙** - 会議の基本情報
2. **目次** - 発言者・議題の一覧（表紙と一体の場合あり）
3. **X日目** - 各開催日の会議録本文（1日目、2日目、3日目...）

### PDF ファイル名のパターン

年度によってファイル名の命名規則が異なる:

#### 新しい年度（令和以降の一部）

- ディレクトリ: `/cms/files/uploads/`
- ファイル名: 日本語 URL エンコード（例: `第１回定例会　表紙.pdf`）
- 一部は `江田島市令和７年第３回定例会１R070825.pdf` のように日付コード付き

#### 古い年度（平成24年頃以前の一部）

- ディレクトリ: `/cms/migration/uploads/smartsection/`
- ファイル名: 英数字のみ（例: `1209_h24_no01_hyoshi.pdf`, `h24no04_1stday.pdf`）
- パターン: `{接頭辞}_h{年}_no{回}_{種別}.pdf`

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録一覧ページ `/cms/categories/show/473` から全年度の記事 ID を取得する。

1. 一覧ページの HTML を取得
2. `/cms/articles/show/{ID}` 形式のリンクを抽出
3. リンクテキストから年度情報を取得

```typescript
// 年度ページリンクの抽出
const yearLinks = $('a[href*="/cms/articles/show/"]');
```

### Step 2: 各年度ページから PDF リンクの収集

各年度ページにアクセスし、PDF ファイルへの全リンクを抽出する。

1. 年度ページの HTML を取得
2. PDF リンク（`.pdf` で終わるもの）を全件抽出
3. リンクテキストから会議種別（定例会/臨時会）・回数・内容種別（表紙/目次/X日目）を取得
4. セクション見出しから開催年月を取得

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]');

// リンクテキストからメタ情報を抽出
const linkPattern = /第(.+?)回(定例会|臨時会)会議録（(.+?)　PDF）/;
// 例: "第１回定例会会議録（２日目　PDF）" → 回="１", 種別="定例会", 内容="２日目"

// セクション見出しから開催年月を抽出
const sectionPattern = /(令和|平成)\S+年第.+回(定例会|臨時会)（(令和|平成)\S+年(\d+)月）/;
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 表紙・目次 PDF は基本的にスキップ（メタ情報としては利用可能）
2. 「X日目」の PDF をダウンロード
3. PDF からテキストを抽出（pdf-parse 等を使用）

### Step 4: 会議録テキストのパース

PDF から抽出したテキストを構造化する。

- 開催日・会議名はセクション見出しまたは PDF 内の冒頭部分から取得
- 発言者・発言内容は PDF のテキストレイアウトに依存するため、PDF の構造を確認してパースロジックを調整する必要がある

---

## 注意事項

- PDF ファイル名の命名規則が年度・時期によって統一されていない。ファイル名からメタ情報を抽出するのではなく、リンクテキストやセクション見出しからメタ情報を取得すること
- PDF ファイルは日本語ファイル名が URL エンコードされている場合がある
- 古い年度のファイルは `/cms/migration/uploads/smartsection/` 配下に格納されており、新しい年度は `/cms/files/uploads/` 配下に格納されている
- 一部の PDF は同一年度内でもファイル名パターンが異なる（手動アップロードによるばらつき）
- リクエスト間には適切な待機時間（1〜2秒）を設けること

---

## 推奨アプローチ

1. **一覧ページから全量取得**: 21年度分の全記事 ID は固定的（上記一覧参照）なので、まず全年度ページから PDF リンクを一括収集する
2. **PDF テキスト抽出**: 表紙・目次を除く「X日目」PDF をダウンロードし、テキスト抽出を行う
3. **メタ情報はリンクテキストから取得**: PDF ファイル名の命名規則が不統一なため、年度ページのセクション見出しとリンクテキストからメタ情報（年度・回・種別・開催月）を取得する
4. **差分更新**: 記事 ID と PDF URL の組み合わせをキーとして、未取得の PDF のみダウンロードする差分更新が可能
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
