# 大洗町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.oarai.lg.jp/oaraigikai/
- 分類: WordPress サイトで PDF 形式の議事録を公開（検索システムなし、スクレイピング対応アダプターなし）
- 文字コード: UTF-8
- 特記: 会議録は PDF のみの提供、HTML 形式の会議録ページなし

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 議事録トップ | `https://www.town.oarai.lg.jp/oaraigikai/%e8%ad%b0%e4%ba%8b%e9%8c%b2` |
| PDF ダウンロード | `https://www.town.oarai.lg.jp/oaraigikai/wp/wp-content/uploads/{YEAR}/{MONTH}/{FILENAME}.pdf` |

### PDF ファイル名パターン

- `目次.pdf` - 会議の目次
- `{N}日目（{月}月{日}日）.pdf` - {N} 日目の会議録
  - 例: `1日目（12月22日）.pdf`

### アップロードパス構造

`uploads/{YEAR}/{MONTH}/` - 例: `2026/03/`（実装時点での最新アップロード）

---

## 検索パラメータ

**検索パラメータなし**。サイトには検索機能がないため、以下の戦略で会議録一覧を取得する必要がある：

1. 議事録ページを解析し、会議ごとのセクション（セッション）を抽出
2. 各セッションの PDF リンクを全て抽出
3. 日付情報はファイル名内に埋め込まれているため、正規表現でパース

---

## スクレイピング戦略

### Step 1: 会議一覧ページの取得と解析

議事録トップページ（`%e8%ad%b0%e4%ba%8b%e9%8c%b2`）を取得し、以下の情報を抽出する：

1. **会議セッション情報**
   - 令和X年第X回定例会 / 令和X年臨時会 のパターン
   - 会議実施日（例: 「令和7年12月2日～5日」）

2. **PDF リンク**
   - 各セッション下の `uploads` パス内のすべての PDF リンク
   - 正規表現例: `/oaraigikai/wp/wp-content/uploads/(\d{4})/(\d{2})/([^"]+\.pdf)`

3. **ファイル種別の分類**
   - `目次.pdf` → メタデータ抽出用
   - `\d日目（` → 会議本文（複数日の場合あり）

### Step 2: PDF ファイルの取得と解析

取得したメタデータ（ファイル名、URL、更新日時）をデータベースに保存する。

**メタ情報の抽出:**

- 会議開催日時: ファイル名内の日付 `{月}月{日}日` から取得
- 会議種別: ページ内の会議名タイトルから抽出
- 会議実施期間: ページ上の会議タイトル行から「令和X年第X回定例会」など

**PDF 本文の解析:**

PDF テキスト抽出ツール（pdftotext、pdf-parse など）を使用して：

1. テキストを行単位で処理
2. 発言者パターンを検出（以下参照）
3. 発言内容を抽出

#### 発言の構造・パターン

大洗町議会 PDF では以下の形式で発言が記録される（想定）：

```
○議長（氏名）
　発言内容...

○{番号}番（氏名）
　発言内容...

○町長（氏名）
　発言内容...
```

パターン（想定）:

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（田中太郎） → role="議長", name="田中太郎"
// 例: ○１番（山田花子） → role="１番", name="山田花子"

// 日付の抽出（ファイル名から）
const dateFromFilename = /(\d+)日目（(\d+)月(\d+)日）/;
// 例: 1日目（12月22日） → day=1, month=12, date=22
```

### Step 3: 差分更新の実装

毎回の実行で全 PDF をダウンロードするのではなく、以下の差分戦略を採用：

1. **前回取得済みのセッションリストを保持**
   - セッション ID（令和X年第X回定例会など）をキーに
2. **新規・更新セッションのみダウンロード**
   - ページの更新日時から判断
3. **PDF メタデータ キャッシュ**
   - ファイル URL と更新日時を記録して重複ダウンロードを回避

---

## 技術的注意事項

### 1. PDF 抽出精度

- **PDF テキスト抽出の信頼性**: PDF から直接テキストを抽出するため、フォント埋め込みやエンコーディング問題に注意
- **複雑な表組み**: 委員会情報などが表形式の場合、行単位での抽出では構造が失われる可能性あり
- **推奨ツール**: `pdfjs-dist` or `pdf-parse` (Node.js), または `pdftotext` コマンド

### 2. 文字コード・正規表現

- 発言者名に全角数字を使用（例: `１番`）
- 括弧は全角括弧 `（）` を使用
- 正規表現は Unicode 対応版を使用

### 3. レート制限

自治体サイトのため、連続ダウンロードを避ける：

- PDF ダウンロード間隔: 1～2 秒
- ページ取得後、リンク抽出 → 間隔を置く → ダウンロード、という順序で実施

### 4. URL エンコーディング

議事録ページ URL が `%e8%ad%b0%e4%ba%8b%e9%8c%b2` のようにパーセントエンコードされている：

- `%e8%ad%b0%e4%ba%8b%e9%8c%b2` = `議事録`
- 取得・キャッシュ時にデコード to 正規化を推奨

---

## 推奨アプローチ

1. **段階的な導入**
   - まず最新の 1～2 セッションで動作確認
   - PDF 抽出精度・発言者パターンの確認
2. **PDF メタデータの先行取得**
   - `files.json` のような中間ファイルに「セッション ID」「PDF URL」「更新日時」を記録
   - 実際のテキスト抽出は別ステップで実施
3. **エラーハンドリング**
   - PDF 取得失敗時のリトライ
   - テキスト抽出失敗時のログ記録
4. **検証データの保持**
   - 取得した生 PDF と抽出テキストのサンプルを検証用に保管
   - 精度改善の際の参考データとして活用
