# 本部町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.motobu.okinawa.jp/doc/2023103100062/
- 分類: PDF/DOCX ファイル直接公開（Joruri CMS 使用、会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 会議録は HTML ではなく DOCX/PDF ファイルとして公開されている。全年度が単一ページに掲載。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（全年度） | `https://www.town.motobu.okinawa.jp/doc/2023103100062/` |
| 会議録ファイル | `https://www.town.motobu.okinawa.jp/doc/2023103100062/file_contents/{ファイル名}.docx` or `.pdf` |

- 検索ページ・詳細ページの区別はない（単一ページに全リンクが掲載）
- ページネーションなし

## ページ構造

### 年度見出し

`<h2>` タグで年度が区切られている。

```
令和7年 / 令和6年 / 令和5年 / 令和4年 / 令和3年 / 令和2年 /
平成31年・令和元年 / 平成30年 / 平成29年 / 平成28年 / 平成27年 / 平成26年 / 平成25年
```

### 会議録リンク

各年度の `<h2>` 配下に、段落（`<p>`）内のリンクとして会議録ファイルが列挙される。

リンクテキストのパターン:

- 日付 + 号数: `3月6日（第1号）R7` / `6月18日（第1号）`
- 一般質問付き: `3月13日（第3号）一般質問R7`
- 臨時会: `2月19日（1回臨）R7` / `令和7年3月25日（3回臨）`
- 予算/決算委員会: `■令和7年3月6日（予委第1号）` / `■令和6年9月20日（決委第2号）`
- 会期日程: `第2回定例会（会期日程・議決の結果）R7`
- 目次・通告書: `目次・通告書（R7．3月合併号）`

## ファイル命名規則

### DOCX（令和6年以降）

命名パターンは不規則だが、以下の傾向がある:

- `{月日}{号数}R7.docx` — 例: `361R7.docx`（3月6日 第1号 R7年）
- `{月日}{号数}.docx` — 例: `371.docx`（3月7日 第1号）
- `{年下2桁}{月日}{号数}.docx` — 例: `76131.docx`（R7年6月13日 第1号）
- `{回数}.docx` — 例: `2R7.docx`（第2回定例会 会期日程）

### PDF（令和5年以前）

- `h{年号2桁}{月2桁}{日2桁}.pdf` — 例: `h300309.pdf`（平成30年3月9日）
- `r{年号1桁}{月}{日}{号数}.pdf` — 例: `r11212.pdf`（令和元年12月12日）

## スクレイピング戦略

### Step 1: 一覧ページからファイル URL を収集

会議録一覧ページ `https://www.town.motobu.okinawa.jp/doc/2023103100062/` から全リンクを取得する。

- 単一ページに全年度の会議録が掲載されているため、1 回の HTTP リクエストで全ファイル URL を取得可能
- `<h2>` タグで年度を識別し、各年度配下の `<a>` タグから `file_contents/` を含む href を抽出
- ページネーションなし

**収集方法:**

1. 一覧ページの HTML を取得
2. Cheerio で `<h2>` タグを走査し、年度情報を取得
3. 各 `<h2>` セクション内の `a[href*="file_contents/"]` からファイル URL とリンクテキストを抽出
4. リンクテキストから日付・会議種別（定例会/臨時会）・号数を正規表現でパース

### Step 2: ファイルのダウンロード

抽出した相対パスに一覧ページの URL をベースとして結合し、ファイルをダウンロードする。

- ベース URL: `https://www.town.motobu.okinawa.jp/doc/2023103100062/`
- ファイル URL: `https://www.town.motobu.okinawa.jp/doc/2023103100062/file_contents/{ファイル名}`

### Step 3: ファイルからテキスト抽出

#### DOCX ファイル

- `mammoth` や `docx` などの Node.js ライブラリで DOCX からテキストを抽出
- Word の書式情報（太字、下線等）は除去し、プレーンテキストとして処理

#### PDF ファイル

- `pdf-parse` 等のライブラリで PDF からテキストを抽出
- 段組み・ヘッダー/フッターの除去が必要

### Step 4: メタ情報の抽出

リンクテキストから以下のメタ情報を抽出する:

```typescript
// 日付の抽出（リンクテキストから）
const datePatternsFromLink = [
  /(\d+)月(\d+)日/,                         // 「3月6日」
  /(?:令和|平成)(\d+)年(\d+)月(\d+)日/,      // 「令和7年3月25日」
];

// 会議種別の判定
const sessionPatterns = {
  regular: /第(\d+)号/,                // 定例会
  extraordinary: /(\d+)回臨/,          // 臨時会
  budget: /予委第(\d+)号/,             // 予算審査委員会
  audit: /決委第(\d+)号/,              // 決算審査委員会
  schedule: /会期日程・議決の結果/,      // 会期日程
  toc: /目次・通告書/,                  // 目次・通告書
  generalQuestion: /一般質問/,          // 一般質問を含む
};

// 年度の判定（h2 テキストから）
const yearPattern = /(令和|平成)(\d+)年/;
```

### Step 5: 会議録本文のパース

DOCX/PDF から抽出したテキストの構造は、ファイルごとに異なる可能性がある。実際のファイルをダウンロードして構造を確認する必要がある。

想定される構造:

- 冒頭に会議名・日時・出席者等のメタ情報
- 発言者名 + 発言内容の繰り返し

---

## 注意事項

- 会議録が HTML ではなく DOCX/PDF で公開されているため、テキスト抽出にファイル解析ライブラリが必要
- DOCX と PDF が混在しているため、拡張子に応じたパーサーの切り替えが必要
- ファイル命名規則が不規則なため、ファイル名からのメタ情報抽出はリンクテキストに依存する
- 会期日程・目次・通告書のファイルは会議録本文ではないため、スクレイピング対象から除外する必要がある
- Joruri CMS のため、URL 構造が変更される可能性がある

---

## 推奨アプローチ

1. **一括取得が容易**: 単一ページに全年度の会議録リンクがあるため、リンク収集は 1 リクエストで完了する
2. **リンクテキストからメタ情報を抽出**: ファイル名は不規則だが、リンクテキストには日付・会議種別が記載されている
3. **ファイル種別で処理を分岐**: DOCX（令和6年〜）と PDF（〜令和5年）で異なるパーサーを使用
4. **会期日程・目次ファイルを除外**: リンクテキストに「会期日程」「目次・通告書」を含むものはスキップ
5. **レート制限**: 自治体サイトのため、ファイルダウンロード間に適切な待機時間（1〜2 秒）を設ける
6. **差分更新**: 一覧ページのリンク一覧を前回取得分と比較し、新規ファイルのみダウンロードする
