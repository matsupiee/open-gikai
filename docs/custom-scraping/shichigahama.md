## PDF ファイルの命名規則

### 命名パターン

ファイル名は日本語で URL エンコードされている。デコード後のパターンは以下の通り。

| パターン | ファイル名例 |
| --- | --- |
| 定例会 | `令和7年定例会6月会議.pdf` |
| 定例会（第N回） | `令和7年定例会1月第2回会議.pdf` |
| 予算審査特別委員会 | `令和7年予算審査特別委員会.pdf` |
| 決算審査特別委員会 | `令和6年定例会9月会議（決算審査特別委員会）.pdf` |
| 副本付き | `令和6年定例会3月会議（副本）.pdf` |

### 命名規則の揺れ

- 一部ファイルで末尾に `f` が付く場合がある（例: `令和6年定例会6月会議f.pdf`）
- 特別委員会の表記が括弧付きと独立ファイルの2パターンある

ファイル名を推測するのではなく、**一覧ページの `<a href>` から URL を直接収集する**ことを強く推奨する。

## PDF の内部構造

### 表紙

- 1ページ目: 年度、「七ヶ浜町議会会議録」、会議名と開会日・閉会日
- 例: `令和７年 七ヶ浜町議会会議録 6月会議 6月3日 開会 6月4日 閉会`

### 各日の構造

1つの PDF 内に複数日分の会議録が収録される場合がある。各日は以下の構成:

1. **日付・会議名ページ**: `令和７年６月３日（火曜日）` / `七ヶ浜町議会定例会６月会議会議録（第１日目）`
2. **会議録番号**: `令和7年七ヶ浜町議会定例会6月会議会議録第1号`
3. **開催日**: `令和6年6月3日（火曜日）`
4. **出席議員リスト**: `出席議員（14名）` + 番号・氏名の一覧
5. **欠席議員**: `欠席議員（なし）`
6. **説明のため出席した者**: 役職と氏名の一覧（町長、副町長、各課長等）
7. **事務局職員出席者**: 議会事務局長、書記
8. **議事日程**: `議事日程 第1号` + 日程項目一覧
9. **本日の会議に付した事件**
10. **本文**（発言内容）

### 発言者パターン

```
○議長（安倍敏彦君）　日程第１、会議録署名議員の指名を行います。
○副町長（平山良一君）　おはようございます。
○教育長（大槻泰弘君）　皆さん、おはようございます。
```

- `○` で始まり、役職 + 括弧内に氏名（敬称「君」付き）
- 役職例: 議長、副町長、教育長、総務課長兼デジタル推進室長 など

### 登壇表記

```
〔副町長　平山良一君　登壇〕
〔教育長　大槻泰弘君　降壇〕
```

- `〔` と `〕` で囲まれた登壇・降壇表記
- 形式: `〔役職　氏名君　登壇/降壇〕`

### ページ番号

`－ 1 －` 形式でページ下部に表示。

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（安倍敏彦君） → role="議長", name="安倍敏彦"

// 登壇・降壇の検出
const podiumPattern = /^〔(.+?)　(.+?)君　(登壇|降壇)〕/;
// 例: 〔副町長　平山良一君　登壇〕 → role="副町長", name="平山良一", action="登壇"

// 開催日の抽出（PDF本文内）
const datePattern = /令和(\d+)年(\d+)月(\d+)日/;

// 会議録番号の抽出
const recordPattern = /令和\d+年七ヶ浜町議会(.+?)会議録第(\d+)号/;
```

## スクレイピング戦略

### Step 1: 一覧ページから PDF URL を収集

`gikai4-010-10126.html` ページを取得し、PDF への `<a href>` をすべて抽出する。

- リンクは相対パス（`../../benricho/joho/assets/...`）のため、ベース URL を付与して絶対 URL に変換する
- PDF パスはすべて `/benricho/joho/assets/` 配下
- 同一ページで全年度・全会議が一覧できるため、ページネーション処理は不要

```typescript
// Cheerio での収集例
const links = $('a[href$=".pdf"]')
  .map((_, el) => {
    const href = $(el).attr('href')!;
    return new URL(href, 'https://www.shichigahama.com/benricho/joho/gikai4-010-10126.html').toString();
  })
  .toArray();
```

### Step 2: リンクテキストからメタ情報を取得

一覧ページのリンクテキストから年度・会議種別を取得する。

```typescript
// リンクテキストから年度と会議種別を抽出
const metaPattern = /令和(\d+)年(定例会\d+月(?:第\d+回)?会議|予算審査特別委員会|決算審査特別委員会)/;
// 例: "令和7年定例会6月会議.pdf" → year=7, session="定例会6月会議"
```

### Step 3: 各 PDF を取得・テキスト抽出

PDF ファイルを取得し、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

- 1つの PDF に1つの会議（複数日分）が収録されている
- 表紙、白紙ページ、出席者リスト等のメタ情報ページを読み飛ばし、本文部分を取得する
- テキスト抽出後、`○` で始まる発言者行で分割し、発言者と発言内容を構造化する

### Step 4: 発言の構造化

```typescript
// テキストを行ごとに処理
// 1. ○で始まる行を発言者行として検出
// 2. 発言者行から role と name を抽出
// 3. 次の発言者行までのテキストを発言内容として格納
// 4. 〔...登壇〕〔...降壇〕は発言内容から除外
```

## 注意事項

- **PDF テキスト抽出の品質**: PDF のテキストレイヤーが正しく埋め込まれている場合はテキスト抽出可能だが、スキャン PDF の場合は OCR が必要になる可能性がある
- **全角数字**: PDF 本文内の数字は全角で記載されている場合がある（例: `１４名`、`第１日目`）
- **複数日分の統合**: 1つの PDF に複数日分の会議録が含まれるため、日ごとの分割処理が必要
- **敬称「君」**: 発言者名に「君」が付いているため、パース時に除去する必要がある
- **ファイル名のエンコード**: URL が日本語ファイル名のパーセントエンコーディングのため、ダウンロード時にデコードが必要になる場合がある

## 推奨アプローチ

1. **一覧ページを単一取得**: `gikai4-010-10126.html` の 1 リクエストで全 PDF URL を収集できる
2. **リンクテキストを活用**: PDF ファイル名（リンクテキスト）から年度・会議種別を取得し、PDF 本文のパースの手がかりとする
3. **PDF テキスト抽出**: pdf-parse 等でテキスト化し、`○` 発言者パターンで発言を分割する
4. **レート制限**: 自治体サイトのため、PDF ダウンロード時もリクエスト間に 1〜2 秒の待機を設ける
5. **差分更新**: 既取得 URL のリストと比較し、新規 URL のみ PDF を取得する
