## ページ構造

### 会議録一覧ページ

- 一覧は `.loadbox` 内の `<ul><li>` リスト形式で表示される
- 各 `<li>` に更新日テキストと記事へのリンク（`<a href="kiji{記事ID}/index.html">`）が含まれる
- 「もっと見る」ボタンによる非同期読み込み（jQuery autopager プラグイン）を採用
- 初期表示: 30件、全件数: 74件以上（`全N件` 表示あり）
- 会議録以外にも議会広報紙「やまめ」等の記事も同一カテゴリに含まれる

#### ページネーション

`autopager` による動的読み込みのため、静的な `?pg=N` 形式では次ページを取得できない。
`rel="next"` 属性リンクを辿る、または Ajax エンドポイントを直接叩く方法を検討する。

### 会議録詳細ページ

- ページタイトル・H1 見出しに会議名が含まれる（例: 「五木村議会会議録（令和6年第3回定例会）」）
- 本文は会議録公開の簡潔な告知テキストのみ（議事内容は PDF 内）
- PDF ファイルのダウンロードリンクが 1 件掲載される

#### PDF ファイルの URL 構造

```
https://www.vill.itsuki.lg.jp/kiji{記事ID}/{連番}_{記事ID}_{ファイルID}_up_{ランダム文字列}.pdf
```

例:
```
https://www.vill.itsuki.lg.jp/kiji0032032/3_2032_2984_up_m4dlqqbi.pdf
https://www.vill.itsuki.lg.jp/kiji0031916/3_1916_2706_up_42x1c6e7.pdf
```

PDF URL は固定パターンではなくランダム文字列を含むため、詳細ページから動的に取得する必要がある。

## スクレイピング戦略

### Step 1: 会議録記事 URL の収集

一覧ページ `list00107.html` から会議録記事へのリンクを収集する。

- autopager による動的読み込みのため、Playwright 等のブラウザ自動化ツールで「もっと見る」ボタンを繰り返しクリックして全件を展開する
- `.loadbox a` を取得し、リンクテキストに「定例会」または「臨時会」を含む記事のみを会議録として絞り込む
- 記事 URL: `https://www.vill.itsuki.lg.jp/kiji{記事ID}/index.html`

### Step 2: 詳細ページから PDF URL を取得

各記事ページ `kiji{記事ID}/index.html` にアクセスし、PDF ダウンロードリンクを抽出する。

- PDF リンクは `<a href="...pdf">` 形式で掲載される
- ファイル名に年月が含まれる（例: `202503会議録`）ため、年月情報の補助として利用できる

### Step 3: メタ情報の抽出

各記事ページから以下を抽出:

- **会議名**: ページタイトルまたは H1 見出し（例: `五木村議会会議録（令和6年第3回定例会）`）
- **開催年度・回次**: 会議名文字列から正規表現で抽出
- **更新日**: ページ内の「最終更新日」テキスト

#### パース用正規表現（案）

```typescript
// 会議名からの年度・回次抽出
const sessionPattern = /（(令和|平成)(\d+)年第(\d+)回(定例会|臨時会)）/;
// 例: （令和6年第3回定例会） → era="令和", year="6", session="3", type="定例会"
```

### Step 4: PDF のダウンロード

取得した PDF URL から会議録 PDF をダウンロードし、テキスト抽出処理へ渡す。

---

## 注意事項

- PDF URL のランダム文字列部分は毎回異なるため、URL を直接推測することはできない。必ず詳細ページを経由して取得する
- 一覧ページに会議録以外の記事（議会広報紙「やまめ」等）が混在するため、タイトルフィルタリングが必須
- 一部の会議録は複数回分がまとめて公開される場合がある（令和6年 第1〜3回定例会が同日更新）
- レート制限: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **Playwright を使用**: autopager による動的ロードに対応するため、ブラウザ自動化が必要
2. **タイトルフィルタリング**: `定例会` または `臨時会` を含む記事のみを対象として絞り込む
3. **詳細ページ経由で PDF URL を取得**: URL のランダム文字列により直接推測不可
4. **差分更新**: 記事 ID（`kiji{ID}`）の数値部分は単調増加のため、前回取得済みの最大 ID 以降の新規記事を差分取得できる
