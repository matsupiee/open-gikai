各年度ページ（例: `/site/gikai/15030.html`）には、その年度内の全会議録が箇条書きリスト形式で掲載される。

- 会議は「第○回○月定例会」「第○回○月臨時会」の見出しでグループ化
- 各会議の下に「第◎日目（令和○年○月△日）」の形式で PDF リンクが列挙される
- 定例会は複数日（4〜5 日分）、臨時会は 1 日分の PDF が提供される

## PDF の提供形式

- 形式: PDF
- URL パターン: `https://www.town.heguri.nara.jp/uploaded/attachment/{attachment_id}.pdf`
- attachment_id は数値 ID（例: `10506`, `12473`, `12508`）
- ID は連番ではなく、ページ内の HTML から直接取得する必要がある

## スクレイピング戦略

### Step 1: 年度別・委員会別ページ URL の収集

#### 本会議録

1. `list47-37.html` を取得し、各年度の一覧ページへのリンク（`/site/gikai/{page_id}.html`）を抽出する
2. 平成23年〜令和8年（現在）の全年度が対象

#### 委員会会議録

1. `list47-82.html` を取得し、各委員会の一覧ページへのリンクを抽出する
2. 現在4委員会が存在（文教厚生・予算審査特別・決算審査特別・総務建設）

### Step 2: PDF リンクの収集

各年度ページ・委員会ページを取得し、PDF リンク（`/uploaded/attachment/{ID}.pdf`）を抽出する。

各 PDF に紐づくメタ情報（会議名・開催日）は、PDF リンク周辺の HTML テキストから取得する。

- 本会議録: 見出し（会議名）と箇条書き項目（「第◎日目（日付）」）を対応付けて抽出
- 委員会会議録: 年月グループと開催日テキストを対応付けて抽出

### Step 3: PDF の取得とテキスト抽出

収集した PDF URL から本文テキストを抽出する。

## メタ情報の抽出

### 本会議録のメタ情報

各 PDF リンク周辺から以下を抽出:

```
第2回3月定例会
  └ 第1日目（令和7年3月4日）  → /uploaded/attachment/XXXXX.pdf
  └ 第2日目（令和7年3月5日）  → /uploaded/attachment/XXXXX.pdf
  ...
```

- 会議種別: 「定例会」または「臨時会」
- 回次: 「第○回」
- 開催月: 「○月」
- 開催日: 「令和○年○月△日」（各 PDF ごと）

### 委員会会議録のメタ情報

```
令和7年12月
  └ 12月4日  → /uploaded/attachment/XXXXX.pdf
```

- 委員会名: ページタイトルまたは見出しから取得
- 開催日: 「令和○年○月△日」

## 注意事項

- PDF の attachment_id は連番ではないため、URL を推測する方法は使えない。必ず HTML ページから抽出すること
- 委員会ページ（`/site/gikai/1894.html` 等）の page_id は固定だが、将来的に委員会が追加・廃止される可能性がある。`list47-82.html` から動的に取得することが望ましい
- 本会議録のインデックスページ（`list47-37.html`）および委員会インデックスページ（`list47-82.html`）から再帰的に URL を収集することで、新年度の追加にも対応できる

---

## 推奨アプローチ

1. **インデックスページ起点**: `list47-37.html` と `list47-82.html` を起点に年度別・委員会別ページ URL を動的に収集する（ハードコードを避ける）
2. **2 段階クロール**: インデックス → 年度/委員会ページ → PDF URL の順に取得する
3. **メタ情報の構造的抽出**: 各ページ内の見出しと PDF リンクの親子関係を Cheerio で解析し、会議名・開催日を正確に紐付ける
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 各インデックスページを定期的に再取得し、新規 PDF リンクのみを対象とする差分取得を行う
