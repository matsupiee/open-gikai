# 千代田町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.chiyoda.gunma.jp/gikai/gikai15.html
- 分類: 静的 HTML ページに PDF リンクを掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は全て PDF 形式で公開。図書館等での閲覧も可能

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（単一ページ） | `https://www.town.chiyoda.gunma.jp/gikai/gikai15.html` |
| PDF（令和元年以降・ハッシュ型） | `https://www.town.chiyoda.gunma.jp/files/{32桁ハッシュ}.pdf` |
| PDF（令和6年・日本語ファイル名） | `https://www.town.chiyoda.gunma.jp/{URLエンコードされた日本語ファイル名}.pdf` |
| PDF（平成29年以前・定例会） | `https://www.town.chiyoda.gunma.jp/gikai/data/T_{YYYYMM}.pdf` |
| PDF（平成29年以前・臨時会） | `https://www.town.chiyoda.gunma.jp/gikai/data/R_{YYYYMMDD}.pdf` または `R_{YYYYMM}.pdf` |

- 検索ページ: なし（一覧ページのみ）
- ページネーション: なし（全年度が単一ページに掲載）

## PDF リンクのパターン

### パターン 1: ハッシュ型（令和2年〜令和7年の大部分）

```
../files/8f154240d77005b93bdec7f98f36f64c.pdf
../files/04122b2293b37965ae0eabeb7a4a8915.pdf
```

- 相対パス `../files/` で参照
- ファイル名は 32 桁の MD5 ハッシュ
- 会議名とファイル名に規則性なし（リンクテキストから会議情報を取得する必要あり）

### パターン 2: 日本語ファイル名（令和6年の一部）

```
../%E4%BB%A4%E5%92%8C%EF%BC%96%E5%B9%B4%E7%AC%AC%EF%BC%94%E5%9B%9E...pdf
```

- URL エンコードされた日本語ファイル名
- デコードすると「令和６年第４回千代田町議会定例会（12月開催）」のような名前

### パターン 3: 規則的ファイル名（平成29年以前）

```
data/T_201701.pdf   → 定例会（T = Teirei）
data/R_20161014.pdf → 臨時会（R = Rinji）
data/R_201601.pdf   → 臨時会（月のみ）
```

- `T_` プレフィックス: 定例会
- `R_` プレフィックス: 臨時会
- 日付は `YYYYMM` または `YYYYMMDD` 形式

| 種別 | 開催パターン | 備考 |
| --- | --- | --- |

| 臨時会 | 不定期 | 「第N回臨時会（M月D日）」と表記 |

## HTML 構造（一覧ページ）

```
<h1>議会会議録</h1>

<h2>令和７年</h2>
<ul>
  <li><a href="../files/{hash}.pdf">第4回定例会（12月）(PDF:XXX KB)</a></li>
  <li><a href="../files/{hash}.pdf">第2回臨時会（10月15日）(PDF:XXX KB)</a></li>
  ...
</ul>

<h2>令和６年</h2>
<ul>
  <li><a href="...">第4回定例会（12月）(PDF:XXX KB)</a></li>
  ...
</ul>

<!-- 平成18年まで同様の構造が続く -->
```

- 年度ごとに `<h2>` 見出しで区切られている
- 各会議録は `<ul>` / `<li>` 内の `<a>` タグで PDF へリンク
- リンクテキストに会議名・開催月・ファイルサイズが含まれる
- 掲載範囲: 平成18年〜令和7年（約20年分）

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `gikai15.html` から全 PDF リンクを抽出する。

1. `gikai15.html` を取得
2. `<h2>` タグから年度を抽出
3. 各年度セクション内の `<a>` タグから PDF URL とリンクテキスト（会議名）を抽出
4. 相対パスを絶対 URL に変換

**抽出すべき情報:**
- 年度（`<h2>` のテキスト → 「令和7年」「平成30年」等）
- 会議名（リンクテキスト → 「第4回定例会（12月）」等）
- PDF URL

### Step 2: PDF のダウンロード

収集した URL リストから PDF をダウンロードする。

- ファイルサイズは 195KB 〜 3.94MB 程度
- 全量で約 100 件以上の PDF

### Step 3: PDF からテキスト抽出

PDF からテキストを抽出し、構造化する。

- PDF パーサー（pdf-parse 等）でテキストを抽出
- 会議録の一般的な構造（開会宣告、出席議員、議事、閉会宣告）に基づいてパース

---

## 注意事項

- 全ての会議録が PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF の内部構造（レイアウト、フォント埋め込み等）は個別の PDF を確認する必要がある
- URL パターンが年代によって 3 種類に分かれるため、パターンごとの URL 解決処理が必要
- 令和6年の一部は日本語ファイル名のため、URL エンコーディングの考慮が必要
- 一覧ページが単一ページのため、クロール自体は軽量（1 リクエストで全リンク取得可能）

---

## 推奨アプローチ

1. **一覧ページからの全量取得**: `gikai15.html` の 1 リクエストで全 PDF リンクを収集できるため、シンプルなクローラーで対応可能
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使用して PDF からテキストを抽出
3. **メタ情報はリンクテキストから取得**: PDF ファイル名からは会議情報を特定できない（ハッシュ型）ため、`<a>` タグのリンクテキストと `<h2>` の年度情報を組み合わせてメタ情報を構築
4. **レート制限**: PDF ダウンロード時はリクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページの PDF リンク数を前回と比較し、新規追加分のみをダウンロードする
