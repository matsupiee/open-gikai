#### 旧形式（令和元年〜令和3年）

```

```

#### 新形式（令和4年〜）

ファイル名がハッシュ値に置き換わり、リンクテキストから会議情報を取得する必要がある。

```

```

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録一覧ページ `post_95.html` から全 PDF リンクとそのリンクテキスト（会議名）を抽出する。

- 一覧ページは単一ページ（ページネーションなし）
- 全年度の会議録が1ページに列挙されている
- リンクテキストから会議名・開催日を取得する

**収集方法:**

1. `post_95.html` を取得
2. ページ内の全 `<a>` タグから `.pdf` で終わるリンクを抽出
3. リンクテキストから会議情報（定例会/臨時会、回数、日目、開催日）をパース
4. 一般質問資料など会議録本文でないPDFは除外する

### Step 2: PDF のダウンロード

収集した PDF URL から各ファイルをダウンロードする。

- 旧形式 URL はファイル名に日本語（全角文字）を含むため、URL エンコードが必要
- 新形式 URL はハッシュ値のみなので特別な処理は不要

### Step 3: PDF のパース

ダウンロードした PDF からテキストを抽出し、会議録情報を構造化する。

#### メタ情報の抽出

リンクテキストから以下を抽出:

```
第１回定例会本会議 第１日(令和２年３月９日開催)
第２回臨時会本会議１日目（令和４年３月２５日開催）
```

- 会議種別: 定例会 / 臨時会
- 回数: 第N回
- 日目: 第N日 / N日目（表記揺れあり）
- 開催日: 令和X年X月X日

#### パース用正規表現（案）

```typescript
// リンクテキストから会議情報を抽出
const meetingPattern = /第([０-９\d]+)回(定例会|臨時会)本会議\s*[第]?([０-９\d]+)日[目]?\s*[(\(（]+(令和[０-９\d]+年[０-９\d]+月[０-９\d]+日)開催[)\)）]+/;

// 開催日の抽出（全角数字対応）
const datePattern = /令和([０-９\d]+)年([０-９\d]+)月([０-９\d]+)日/;
```

## 注意事項

- 一覧ページに会議録以外の PDF（一般質問資料等）も混在しているため、フィルタリングが必要
- 令和4年以降の PDF ファイル名はハッシュ値のため、メタ情報はリンクテキストからのみ取得可能
- 旧形式 URL のファイル名に全角スペースや括弧が含まれるため、URL エンコードに注意
- 「第N日」と「N日目」の表記揺れがある（令和元年〜3年は「第N日」、令和4年以降は「N日目」）
- PDF のテキスト抽出精度はファイルの作成方法に依存する（スキャン画像の場合は OCR が必要になる可能性あり）

---

## 推奨アプローチ

1. **一括取得が容易**: 単一ページに全リンクが掲載されているため、1回のリクエストで全 PDF URL を収集可能
2. **リンクテキスト活用**: 新形式 URL ではファイル名から会議情報を取得できないため、リンクテキストのパースが必須
3. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: 一覧ページの PDF リンク数を前回と比較し、新規追加分のみダウンロードする
