# 紋別市議会 カスタムスクレイピング方針

## 概要

- サイト: https://mombetsu.jp/gikai/minutes/
- 分類: 紋別市公式サイト独自 CMS による PDF 公開（既知の会議録検索システムは未使用）
- 文字コード: UTF-8
- 特記: Google Tag Manager（GTM-M5FPLL4）使用。会議録は全て PDF 形式で提供。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録トップ | `https://mombetsu.jp/gikai/minutes/` |
| 会議種別一覧 | `https://mombetsu.jp/gikai/minutes/?category={種別ID}` |
| 年度別一覧 | `https://mombetsu.jp/gikai/minutes/?category={年度ID}` |
| 会議録詳細 | `https://mombetsu.jp/gikai/minutes/?content={contentID}` |
| PDF ファイル | `https://mombetsu.jp/gikai/assets/images/content/content_{YYYYMMDD}_{HHMMSS}.pdf` |

---

## カテゴリ構造

### 会議種別カテゴリ（トップレベル）

| category | 会議種別 |
| --- | --- |
| `124` | 定例会 |
| `125` | 臨時会 |
| `126` | 委員会 |
| `127` | 特別委員会 |

### 年度カテゴリ（定例会の例）

会議種別カテゴリ配下に年度別のサブカテゴリが存在する。年度によってカテゴリ ID が異なる。

| category | 年度 |
| --- | --- |
| `233` | 令和7年 |
| `208` | 令和6年 |
| `196` | 令和5年 |
| ... | （平成22年まで遡る） |

※ 委員会・特別委員会も同様に年度別サブカテゴリを持つ（ID は異なる）。

---

## ページ構造

### 会議種別一覧ページ（`?category={種別ID}`）

- 年度別のサブカテゴリリンクを表示
- 各年度リンクは `?category={年度ID}` 形式

### 年度別一覧ページ（`?category={年度ID}`）

- 当該年度の定例会・臨時会・委員会等の content リンクを一覧表示
- 各リンクは `?content={contentID}` 形式
- 例: `?content=3509` → 令和6年第4回定例会

### 会議録詳細ページ（`?content={contentID}`）

- 会議名、開催期間が表示される
- 日程ごとに PDF リンクが掲載される
- 一般質問通告書、議決結果一覧等の付随資料も PDF で提供
- ファイルサイズが表示される

---

## PDF ファイルの構造

### URL パターン

```
https://mombetsu.jp/gikai/assets/images/content/content_{YYYYMMDD}_{HHMMSS}.pdf
```

- `YYYYMMDD`: ファイルアップロード日（会議開催日ではない）
- `HHMMSS`: アップロード時刻
- 例: `content_20250221_135713.pdf`

### PDF 内容

- 会議録本文（発言録）が PDF 形式で格納
- 発言者名、発言内容が含まれる
- PDF からのテキスト抽出が必要

---

## スクレイピング戦略

### Step 1: カテゴリツリーの収集

会議録トップページおよび各会議種別ページから、全カテゴリ ID を収集する。

1. トップページ `https://mombetsu.jp/gikai/minutes/` から会議種別カテゴリ（124〜127）を取得
2. 各会議種別カテゴリページから年度別サブカテゴリ ID を取得
3. 各年度カテゴリページから `content` ID を取得

**収集方法:**

```
トップ → ?category=124（定例会）→ ?category=208（令和6年）→ ?content=3509（第4回定例会）
                                → ?category=233（令和7年）→ ...
       → ?category=125（臨時会）→ ...
       → ?category=126（委員会）→ ...
       → ?category=127（特別委員会）→ ...
```

### Step 2: PDF URL の収集

各 `?content={ID}` ページにアクセスし、以下を抽出する:

- 会議名（例: 令和6年第4回紋別市議会定例会）
- 開催日（例: 12月3日〜12月12日）
- 各日程の PDF リンク（`/gikai/assets/images/content/content_*.pdf`）
- 付随資料の PDF リンク（一般質問通告書、議決結果一覧等）

**抽出対象の HTML 要素:**

- PDF リンクは `<a>` タグの `href` 属性で `assets/images/content/content_` を含む URL
- 相対パスで記載されているため、ベース URL からの解決が必要

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（例: `pdf-parse`, `pdfjs-dist`）でテキストを抽出
3. 抽出したテキストから発言者・発言内容を構造化

### Step 4: 発言録のパース

PDF から抽出したテキストの構造は PDF の作成方法に依存するため、実際の PDF を取得して分析する必要がある。

想定されるパースポイント:

- 発言者名のパターン（「○○番 △△議員」等）
- 議長発言のパターン
- 日付・議題の区切り

※ PDF のテキスト抽出結果を確認してから、具体的な正規表現を策定する。

---

## 注意事項

- **PDF 形式のみ**: HTML での会議録公開はなく、全て PDF からのテキスト抽出が必要
- **ファイル名に開催日が含まれない**: PDF の URL にはアップロード日時が使われており、会議の開催日とは一致しない。開催日は content ページのメタ情報から取得する必要がある
- **カテゴリ ID の規則性なし**: 年度別カテゴリ ID は連番だが、会議種別間で共有されていないため、全カテゴリを走査する必要がある
- **JavaScript による動的レンダリング**: 一部コンテンツが JavaScript で動的に読み込まれる可能性があるが、WebFetch で取得可能な範囲でリンク抽出は可能
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **段階的な URL 収集**: カテゴリツリーを上から順に辿り、全 content ID と PDF URL のマッピングを作成
2. **メタ情報の保持**: content ページから会議名・開催日を取得し、PDF と紐付けて保存
3. **PDF テキスト抽出の検証**: まず数件の PDF を手動でダウンロードし、テキスト抽出の品質を確認してからパース戦略を確定する
4. **差分更新**: content ID は数値連番のため、前回取得済みの最大 ID 以降のみを取得する差分更新が可能
