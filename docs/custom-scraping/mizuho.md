## 会議録の構造

### 年度別ページ

各年度ページには定例会・臨時会のリンクが列挙される。

例（令和6年 `/13295.htm`）:
- 第４回定例会（11月28日〜12月20日）→ `/13611.htm`
- 第3回定例会（9月2日〜27日）→ `/13520.htm`
- 第2回定例会（6月7日〜28日）→ `/13415.htm`
- 第2回臨時会（5月10日）→ `/13414.htm`
- 第1回定例会（2月22日〜3月15日）→ `/13347.htm`
- 第1回臨時会（2月2日）→ `/13296.htm`

会議種別は「定例会」「臨時会」のみ確認済み。委員会の会議録は本構造には含まれていない。

### 会議別ページ（PDF リスト）

各会議ページには日程ごとに分割された PDF リンクが列挙される。

例（令和6年第4回定例会 `/13611.htm`）:

| 日程 | ファイル名 | 容量 |
| --- | --- | --- |
| 目次 | `R6.4Tmokuji.pdf` | 53KB |
| 初日（11月28日） | `R6.4T.11.28.pdf` | 284KB |
| 総括質疑（12月6日） | `R6.4T.12.6.pdf` | 355KB |
| 一般質問（12月16日） | `R6.4T.12.16.pdf` | 581KB |
| 一般質問（12月17日） | `R6.4T.12.17.pdf` | 553KB |
| 最終日（12月20日） | `R6.4T.12.20.pdf` | 319KB |

PDF の格納パスは `/secure/{数字}/{ファイル名}.pdf` 形式。`{数字}` は会議ごとに異なり、ファイル名には年度・回次・日付が埋め込まれている。

古い年度（平成15年など）は `/secure/1593/3544_f_{番号}_o.pdf` のような別形式のファイル名になっている。

## スクレイピング戦略

### Step 1: 年度別 URL の収集

会議録トップ `https://www.city.mizuho.lg.jp/3412.htm` のサイドナビゲーションから年度別ページの URL を抽出する。

- リンクは `li` 要素内にあり、相対パス `/XXXXX.htm` 形式
- 上記「年度別ページ一覧」の URL は固定であるため、静的リストとして持つことも可能

### Step 2: 会議ページ URL の収集

各年度ページのメインコンテンツから会議（定例会・臨時会）ごとのリンクを抽出する。

- `<a href="/XXXXX.htm">` 形式のリンクを収集
- タイトルテキストから会議名と開催期間を取得

### Step 3: PDF リンクの収集

各会議ページから PDF のリンクを抽出する。

- `<a href="/secure/.../*.pdf">` 形式のリンクを収集
- 「目次」 PDF は本文でないため取得対象から除外してもよい
- タイトルテキストから会議日を取得

### Step 4: PDF の取得・解析

取得した PDF URL からファイルをダウンロードし、テキスト抽出を行う。

- PDF は日程ごとに分割されているため、1 会議につき複数ファイルを処理する
- テキスト抽出には `pdf-parse` 等のライブラリを使用する

## PDF 内の会議録構造（推定）

HTML 本文での確認はできないため、一般的な自治体議会会議録の書式を参考に推定する。

### メタ情報

ファイル名や会議ページのタイトルから取得:
- 年度・回次: 年度別ページおよび会議ページのタイトルから
- 開催日: ファイル名（例: `R6.4T.11.28.pdf` → 令和6年11月28日）または会議ページのリンクテキスト

### 発言の構造（推定）

一般的な議会会議録の形式:

```
○議長（氏名）
  発言内容テキスト

○XX番（氏名）
  発言内容テキスト
```

- `○` で始まる行が発言者を示す可能性が高い
- 実際のフォーマットは PDF を取得・確認して調整する

## キーワード検索

Google カスタムサーチ（CSE）を利用しており、スクレイピング用途には適さない。

```javascript
// サイト側の実装
function googleSearchInline() {
  var qs = $("#keywords").val();
  qs = encodeURI(qs);
  location.href = "http://www.city.mizuho.lg.jp/1249.htm?qs=" + qs;
}
```

- 検索結果は Google CSE ウィジェットで動的にレンダリングされるため、直接パース不可
- 全量収集は「年度 → 会議 → PDF」の階層クロールで行う

---

## 注意事項

- PDF は日程ごとに分割されており、1 会議あたり複数ファイルになる
- 古い年度（平成15年〜）は PDF ファイル名の命名規則が新しい年度と異なる
- `/secure/` 配下の PDF は認証不要で直接アクセス可能（確認済み）
- 委員会の会議録は年度別・会議録一覧ページには含まれていない可能性があり、別途調査が必要
- リクエスト間に適切な待機時間（1〜2 秒）を設けること

---

## 推奨アプローチ

1. **静的 URL リストを活用**: 年度別ページ URL は固定のため、静的テーブルとして管理し、毎回トップページをクロールしなくてよい
2. **階層クロール**: 年度ページ → 会議ページ → PDF リンクの 3 段階で収集する
3. **PDF テキスト抽出**: `pdf-parse` 等で PDF をテキスト化し、発言者パターンを正規表現で解析する
4. **差分更新**: 新しい年度ページ・会議ページは追加される都度取得し、取得済み PDF は再取得しない
5. **目次 PDF の除外**: 「目次」という名称の PDF は本文を含まないため、スキップする
