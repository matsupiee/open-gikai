## ページ構造

### 会議録一覧ページ（list7.html）

- 年度ごとのリンクが `<ul>` 内の `<li>` 要素としてリスト表示される
- 各 `<li>` に `<a href="/site/gikai/list7-{ID}.html">` 形式のリンクが含まれる
- ページネーションなし（全年度が1ページに表示）

### 年度別一覧ページ（list7-{ID}.html）

- 各年度に1件の記事リンクが含まれる
- `<li>` 内に更新日（`<span class="article_date">`）と記事タイトル（`<span class="article_title">`）が表示される
- 記事タイトルは「令和X年会議録について」形式
- リンク先が年度別会議録詳細ページ

### 年度別会議録詳細ページ（{記事ID}.html）

- ページ冒頭に「田布施町議会の会議録をPDFファイルでご覧いただけます。」という説明文
- 「ダウンロード」見出し（`<h2>`）の下に PDF リンクのリスト（`<ul class="file_list">`）
- 各 PDF リンクは `<a href="/uploaded/attachment/{ID}.pdf">` 形式
- リンクテキストに会議名とファイルサイズが含まれる

#### PDF リンクテキストの形式

```
令和6年第7回(12月)定例会会議録 [PDFファイル／2.21MB]
令和6年第6回(11月)臨時会会議録 [PDFファイル／319KB]
令和6年第5回(9月)定例会会議録 [PDFファイル／2.03MB]
令和6年第4回(6月)定例会会議録 [PDFファイル／2.27MB]
令和6年第3回(4月)臨時会会議録 [PDFファイル／330KB]
令和6年第2回(3月)定例会会議録 [PDFファイル／2.04MB]
令和6年第1回(1月)臨時会会議録 [PDFファイル／482KB]
```

## スクレイピング戦略

### Step 1: 年度別詳細ページ URL の収集

1. 会議録一覧ページ `list7.html` を取得
2. `<li>` 内の `<a href="/site/gikai/list7-{ID}.html">` を抽出して年度別一覧ページ URL を収集
3. 各年度別一覧ページにアクセスし、`<a href="/site/gikai/{記事ID}.html">` を抽出して詳細ページ URL を収集

### Step 2: PDF リンクの取得

各詳細ページにアクセスし、PDF ファイルへのリンクを抽出する。

```typescript
// PDF リンクの抽出パターン
const pdfLinks = $('ul.file_list a[href$=".pdf"]');
```

### Step 3: メタ情報の取得

PDF リンクテキストから以下を抽出:

```typescript
// リンクテキストからのメタ情報抽出
const metaPattern = /^(.+?)第(\d+)回\((\d+)月\)(定例会|臨時会)会議録/;
// 例: "令和6年第7回(12月)定例会会議録 [PDFファイル／2.21MB]"
// → 年度="令和6年", 回数="7", 月="12", 会議種別="定例会"
```

- 年度: `令和X年` / `平成XX年`
- 回数: `第X回`
- 月: `(X月)` 形式
- 会議種別: `定例会` / `臨時会`

### Step 4: PDF からのテキスト抽出

ダウンロードした PDF からテキストを抽出する。PDF 内の発言者・発言内容の構造は PDF のレイアウトに依存するため、テキスト抽出後に個別のパース処理が必要。

---

## 注意事項

- 年度別一覧ページの URL ID に規則性がないため、必ず一覧ページからリンクを収集する必要がある
- 会議録は PDF 形式のみで提供されており、HTML 形式の会議録本文は存在しない
- PDF ファイルの attachment ID も連番ではないため、リンクからの収集が必須
- PDF のファイルサイズは定例会で 2MB 前後、臨時会で 300KB〜500KB 程度
- リクエスト間には適切な待機時間（1〜2秒）を設けること
- 対象期間は平成23年〜現在（令和8年）

---

## 推奨アプローチ

1. **3階層のリンク収集**: 一覧ページ → 年度別一覧ページ → 詳細ページの順にリンクを辿り、全 PDF URL を収集する
2. **リンクテキストからのメタ情報取得**: PDF ファイル名ではなくリンクテキストから年度・回数・会議種別を抽出する
3. **PDF の直接ダウンロード**: 収集した URL から PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で本文を取得する
4. **差分更新**: 各年度詳細ページの更新日（`<span class="article_date">`）を監視し、更新があった年度のみ再取得する
