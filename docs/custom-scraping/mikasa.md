# 三笠市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.mikasa.hokkaido.jp/hotnews/category/307.html
- 分類: 公式サイト上で PDF 形式により会議録を公開（外部の会議録検索システムは未使用）
- 文字コード: UTF-8
- 特記: 会議録は全て PDF ファイルで提供されるため、PDF パーサーが必要

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録カテゴリトップ | `https://www.city.mikasa.hokkaido.jp/hotnews/category/307.html` |
| 年度別一覧（詳細ページ） | `https://www.city.mikasa.hokkaido.jp/assembly/detail/{ID}.html` |
| 会議録 PDF | `https://www.city.mikasa.hokkaido.jp/hotnews/files/{上位ID}/{詳細ページID}/{タイムスタンプ}.pdf` |

### 年度別一覧ページの ID 一覧（確認済み）

カテゴリトップページに全年度のリンクが一覧表示されている。ID は時系列で増加し、新しい年度ほど大きな番号となる。

例:
- 令和8年: `/assembly/detail/00016816.html`
- 令和7年: `/assembly/detail/00016223.html`
- 平成17年: `/assembly/detail/00001190.html`

---

## 検索パラメータ

**なし。** 検索機能やフィルタリング機能は提供されていない。全年度が静的なリストとして一覧表示される。

---

## ページネーション

**なし。** カテゴリトップページに全年度（平成17年〜令和8年、約21年分）のリンクが単一ページに列挙される。年度別詳細ページも同様にページネーションなし。

---

## サイト構造の詳細

### カテゴリトップページ (`/hotnews/category/307.html`)

- 年度ごとの詳細ページへのリンクを一覧表示
- リンクテキスト例: 「会議録 令和8年」「会議録 令和7年」...「会議録 平成17年」
- リンク先: `/assembly/detail/{ID}.html`

### 年度別詳細ページ (`/assembly/detail/{ID}.html`)

- 該当年度の全会議録 PDF へのリンクを表示
- 定例会・臨時会の区別あり
- 1つの定例会が複数日にわたる場合、日ごとに個別の PDF が提供される
- サイドバーに他年度への遷移リンクあり

### PDF リンクのテキストパターン

```
会議録 {和暦年} 第{N}回{定例会|臨時会} {M}月{D}日
```

例:
- `会議録 令和7年 第4回定例会 12月18日`
- `会議録 令和7年 第1回臨時会 1月24日`
- `会議録 令和7年 第1回定例会 3月3日`

### PDF の URL パターン

```
/hotnews/files/{上位ディレクトリID}/{詳細ページID}/{YYYYMMDDHHmmss}.pdf
```

例:
- `/hotnews/files/00016200/00016223/20260220172758.pdf`
- `/hotnews/files/00016800/00016816/20260220172920.pdf`

ファイル名はアップロード日時のタイムスタンプ（14桁）で、会議の開催日とは無関係。

## スクレイピング戦略

### Step 1: 年度別詳細ページ URL の収集

カテゴリトップページ `/hotnews/category/307.html` から全年度の詳細ページ URL を収集する。

- `/assembly/detail/{ID}.html` 形式のリンクを全て抽出
- ページネーションがないため、単一ページから全リンクを取得可能

### Step 2: PDF リンクの収集

各年度の詳細ページにアクセスし、PDF へのリンクを収集する。

- リンクテキストからメタ情報（和暦年、回次、会議種別、開催日）を抽出
- PDF URL を収集

**リンクテキストのパース用正規表現（案）:**

```typescript
// リンクテキストから会議情報を抽出
const meetingPattern = /会議録\s+((?:令和|平成)\d+年)\s+第(\d+)回(定例会|臨時会)\s+(\d+)月(\d+)日/;
// 例: "会議録 令和7年 第4回定例会 12月18日"
// → year="令和7年", session="4", type="定例会", month="12", day="18"
```

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、PDF パーサーでテキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキスト化
- PDF の内容構造は未調査のため、実際の PDF を取得して発言者パターン等を確認する必要あり

### Step 4: 会議録テキストのパース（要調査）

PDF からテキスト抽出後、以下の情報をパースする必要がある:

- 開催日
- 会議種別（定例会 / 臨時会）
- 発言者名・役職
- 発言内容

> **注意:** PDF 内部の構造（発言者のパターン、段落の区切り方等）は実際の PDF を確認してから決定する。リンクテキストから会議のメタ情報は取得可能なため、PDF からは主に発言者と発言内容の抽出が必要。

## 会議の構成

令和7年の実績:

| 会議種別 | 回数 | 日数（各回） |
| --- | --- | --- |
| 定例会 | 年4回（第1回〜第4回） | 2〜3日 |
| 臨時会 | 年1回 | 1日 |

- 第1回定例会: 3月（3日間）
- 第2回定例会: 6月（2日間）
- 第3回定例会: 9月（3日間）
- 第4回定例会: 12月（3日間）
- 臨時会: 1月（1日間）

---

## 注意事項

- 会議録は全て PDF 形式のため、HTML パーサーではなく PDF パーサーが必要
- PDF ファイル名はアップロード日時であり、会議の開催日とは関係がない。開催日はリンクテキストから取得する
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- 平成17年（2005年）から令和8年（2026年）まで約21年分のデータが存在

---

## 推奨アプローチ

1. **2段階クロール**: まずカテゴリトップ → 年度別詳細の順で全 PDF リンクを収集し、次に PDF をダウンロード・パースする
2. **メタ情報はリンクテキストから取得**: 和暦年、回次、会議種別、開催日はリンクテキストに含まれるため、PDF のパース前に取得可能
3. **PDF パーサーの選定**: pdf-parse や pdf.js 等を使用。テキスト抽出の品質を実際の PDF で検証する
4. **差分更新**: 年度別詳細ページの PDF リンク数を前回と比較し、新規追加分のみダウンロードする
5. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
