## HTML 構造

### 年度一覧ページ（list159.html）

```html
<li>
  <a href="/site/gikai/list159-604.html">会議録　令和7（2025）年</a>
</li>
<li>
  <a href="/site/gikai/list159-581.html">会議録　令和6（2024）年</a>
</li>
```

- 各年度へのリンクが `<li><a>` 形式で列挙されている
- リンクテキストは「会議録　令和X（XXXX）年」形式

### 年度別会議録一覧ページ（list159-{ID}.html）

```html
<li>
  <span>2025年2月21日更新</span>
  <a href="/site/gikai/31286.html">会議録（令和6（2024）年12月定例会）</a>
</li>
```

- 各会議録へのリンクが `<li>` 内の `<a>` タグで列挙されている
- 更新日付が `<span>` で付与されている
- リンクテキストは「会議録（令和X（XXXX）年X月定例会）」または「会議録（令和X（XXXX）年X月臨時会）」形式
- 年間 6 件程度（定例会 4 件 + 臨時会 2 件）

### 会議録詳細ページ（{ID}.html）

```html
<a href="/uploaded/attachment/24113.pdf">
  会議録（令和7（2025）年12月定例会） [PDFファイル／592KB]
</a>
```

- 1会議につき1つの PDF ファイルへのリンクが掲載される
- リンクテキストに会議名とファイルサイズが含まれる
- PDF の URL パターン: `/uploaded/attachment/{ID}.pdf`

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

トップページ `https://www.town.masaki.ehime.jp/site/gikai/list159.html` から年度別ページへのリンクを抽出する。

- `a[href*="list159-"]` セレクタで全年度ページへのリンクを抽出
- 年度ページの ID は非規則的なため、必ずトップページから動的に取得する

### Step 2: 会議録詳細ページ URL の収集

各年度別ページ `list159-{ID}.html` から会議録詳細ページへのリンクを抽出する。

- `a[href^="/site/gikai/"]` で会議録詳細ページへのリンクを抽出
- `list159-` を含むリンクは除外する（年度ページ間のナビゲーションリンク）
- リンクテキスト（会議名）と更新日も合わせて取得する

### Step 3: PDF リンクの収集

各会議録詳細ページ `{ID}.html` から PDF リンクを抽出する。

- `a[href$=".pdf"]` または `a[href*="/uploaded/attachment/"]` セレクタで PDF リンクを抽出
- 1ページにつき基本的に 1 件の PDF が掲載される
- リンクテキストから会議名・年度・会議種別を取得する

### Step 4: PDF のダウンロード

収集した URL から PDF をダウンロードする。

- 総 PDF 数は約 60〜80 件（12 年分 × 年 6 件程度）
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

### Step 5: PDF からのテキスト抽出・パース

PDF からテキストを抽出し、構造化データに変換する。

- PDF パーサー（pdf-parse 等）でテキストを抽出
- 1 つの PDF に 1 回の会議の全発言が含まれる

#### メタ情報の抽出

会議録詳細ページのリンクテキストから以下を抽出:

```
会議録（令和6（2024）年12月定例会）
```

- 開催年度: `令和X（XXXX）年` 形式
- 開催月: `X月` 形式
- 会議種別: `定例会` または `臨時会`

#### 発言の構造（PDF 内テキスト）

PDF 内のテキスト構造は実際のファイルをダウンロードして確認する必要がある。一般的な議会会議録 PDF では以下のパターンが想定される:

```
○議長（氏名）
  発言内容...

○番号番（氏名）
  発言内容...
```

※ 実際のフォーマットは PDF ダウンロード後に確認・調整が必要

## 注意事項

- **同名自治体との区別**: 北海道にも「松前町」が存在する。ファイル名は `masaki.md`（愛媛県松前町の読み「まさきちょう」）とし、北海道の `matsumae.md` と区別する。
- 年度別ページの URL（`list159-{ID}` の ID 部分）は非規則的で連番ではないため、URL を固定値でハードコードせずトップページから動的に取得する。
- PDF はテキスト PDF かスキャン PDF かを事前に数件ダウンロードして確認すること。スキャン PDF の場合は OCR が必要になる。

---

## 推奨アプローチ

1. **3段階のクロール**: トップページ → 年度別一覧ページ → 会議録詳細ページ → PDF という 3 段階の階層構造をたどる
2. **動的な URL 収集**: 年度ページの ID は非規則的なため、必ずトップページから URL を動的に取得する
3. **PDF テキスト品質の事前確認**: 数件の PDF をダウンロードしてテキスト抽出の品質を確認し、スキャン PDF かテキスト PDF かを判定する
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 会議録詳細ページの URL（`/site/gikai/{ID}.html` の ID）は数値のため、前回取得済みの最大 ID を記録しておくことで差分更新が可能
