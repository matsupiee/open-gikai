## 公開情報の種類

### 1. 会議の結果（HTML テキスト）

各年度ページに議決結果が HTML テキストとして掲載されている。

各議案の情報は以下の構成で記載される:

```
議案番号（例: 議第1号）
件名
議決日
議決の結果（可決 / 承認 / 同意 / 認定 など）
```

- ページネーションなし（1 ページに年度内全会期分を掲載）
- テーブル形式ではなく、見出し＋テキスト形式
- 会期の種別（定例会・臨時会）ごとに見出しで区切られている

**会期の種別と命名パターン:**

| 種別 | 例 |
| --- | --- |
| 定例会 | 第1回定例会、第2回定例会、第3回定例会、第4回定例会 |
| 臨時会 | 第1回臨時会、第2回臨時会、第3回臨時会 |

年度によって開催回数や臨時会の有無が異なる。令和元年ページは平成31年と令和元年の両方を掲載している。

### 2. 議会だより・一般質問（PDF のみ）

一般質問の内容は PDF 形式でのみ公開されており、HTML テキストとしての取得は不可。

- 各会期（定例会）ごとに 1 つの PDF ファイル
- PDF のファイルサイズは 1.5〜2.5 MB 程度
- Adobe Reader 必須の表記あり

## スクレイピング戦略

### 取得可能なデータ

**「会議の結果」ページから取得できる情報:**

- 議案番号
- 議案名（件名）
- 議決日
- 議決結果（可決・承認・同意・認定・否決など）
- 会期名（定例会・臨時会）
- 開催年度

**取得できない情報（専用システム非搭載のため）:**

- 発言録（誰が何を発言したか）
- 質疑応答の詳細内容
- 委員会審議の詳細
- 一般質問の本文（PDF のみ、テキスト抽出は別途対応が必要）

### Step 1: 年度別ページ URL の収集

議会トップページ `https://www.town-ono.jp/category/2-0-0-0-0-0-0-0-0-0.html` から「会議の結果」セクションのリンクを Cheerio で抽出する。

- 対象 URL パターン: `/0000XXXXXX.html` 形式の数値 ID
- 上記の年度別 URL 一覧をシードデータとして使用し、サイド変更に備えてトップページからの動的取得も実装する

### Step 2: 会議の結果ページの取得・パース

各年度ページを取得し、議決結果をパースする。

**パース対象の構造:**

```html
<!-- 会期の見出し -->
<h2>令和7年第1回臨時会</h2>

<!-- 各議案の記載（テキスト形式） -->
議第1号 ○○○○に関する件
議決日: 令和7年1月23日
議決の結果: 可決
```

実際の HTML 構造は CMS 生成のため確認が必要。Cheerio でテキストノードを抽出し、正規表現でパースする。

**パース用正規表現（案）:**

```typescript
// 議案番号の抽出
const billNumberPattern = /^(議第|発議第|諮第|承第|認第)(\d+)号/;

// 議決日の抽出
const decisionDatePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 議決結果の抽出
const outcomePattern = /(?:議決の結果|結果)[：:]\s*(可決|否決|承認|同意|認定|撤回|廃案)/;

// 会期の判定
const sessionPattern = /(?:令和|平成)(\d+)年第(\d+)回(定例会|臨時会)/;
```

### Step 3: PDF からのテキスト抽出（任意）

「議会だより・一般質問」ページの PDF から一般質問のテキストを抽出する場合は、PDF パースライブラリ（`pdf-parse` 等）を別途使用する。PDF の構造・テキスト品質は未検証のため、実運用前に手動での確認が必要。

## 注意事項

- **専用検索システムなし**: 発言者検索、キーワード検索などの機能は存在しない
- **会議録（発言録）の非公開**: 各発言の詳細テキストは公開されていない。詳細な会議録が必要な場合は議会事務局への問い合わせが必要
- **ページ ID は連番でない**: URL の数値 ID は年度順に連番ではない（例: 令和4年が `0000000592`、令和3年が `0000001590`）。トップページからのリンク取得が必要
- **令和元年ページの特殊性**: `0000001023.html` は平成31年と令和元年の両方の議決結果を掲載している
- **年度追加への対応**: 新年度のページ ID は事前に不明のため、トップページを定期的にクロールして新規リンクを検出する必要がある
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

## 推奨アプローチ

1. **トップページ起点**: `category/2-0-0-0-0-0-0-0-0-0.html` を定期取得し、「会議の結果」リンクを動的に収集する
2. **HTML テキスト取得に集中**: 発言録は存在しないため、議決結果の構造化データ取得に特化する
3. **議会だより PDF は別タスク**: PDF からのテキスト抽出は実装コストが高いため、フェーズ2以降で対応を検討する
4. **議会事務局への確認**: より詳細な会議録（発言録）の電子データが必要な場合は、大野町議会事務局に問い合わせる（TEL 等は公式サイト参照）
