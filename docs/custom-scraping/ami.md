## PDF ファイルの命名規則

ファイル名は統一された規則に従っており、以下のパターンが使用されている:

| パターン | 例 | 説明 |
| --- | --- | --- |
| 年号+回数+種別（通常） | `R0703_T1.pdf` | 令和7年3月第1回定例会 |
| 年号+回数+種別（臨時会） | `R0702_R1.pdf` | 令和7年2月第1回臨時会 |
| 特別委員会 | `[filename]_toku.pdf` | 予算決算特別委員会等 |

ファイル名の形式:
- `R[西暦下2桁][月（2桁）]_[T|R][回数].pdf`
- `T`: 定例会（定例会）
- `R`: 臨時会（臨時会）

## スクレイピング戦略

検索システムが存在しないため、インデックスページから PDF リンクを直接収集する。

### Step 1: インデックスページから PDF リンクを収集

1ページに全 PDF リンクが列挙されているため、インデックスページを1回取得するだけで全リンクが得られる。

```
GET https://www.town.ami.lg.jp/0000000309.html
```

- `cmsfiles/contents/0000000/309/` を含む `<a>` タグの `href` 属性を抽出する
- リンクテキスト（例: `R0703_T1.pdf`）も合わせて取得し、年号・回数・種別を記録する
- ファイル名から年号・月・回数・種別を正規表現で抽出する

### Step 2: 年号の標準化

ファイル名の `R[YY]` から年号を計算する:
- `R07` = 令和7年（2025年）
- `R06` = 令和6年（2024年）
- `R01` = 令和1年（2019年）
- `H31` 以前（ファイル確認不要） = 平成

月情報（`[MM]`）も同時に抽出し、会議の開催月を記録する。

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL から PDF をダウンロードし、テキストを抽出する。

- 1つの PDF ファイルに1回分の会議録が収録されている
- スキャン画像形式の PDF が含まれる可能性があるため、OCR 処理への対応を考慮すること

## 注意事項

- **差分更新**: インデックスページは会議録が追加されるたびに更新される。新しい会議録の追加を検知するには、インデックスページの PDF リンク一覧を前回取得時と比較する。
- **命名規則**: ファイル名は統一されているため、URL 単体での年号・回数・種別の解析が確実。
- **PDF のみ**: テキスト形式での提供はなく、全データが PDF 形式。
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける。
- **開催月の推定**: ファイル名の月情報は会議の開催月を示すため、会議の種別判定に活用できる（3月 = 第1回、6月 = 第2回、9月 = 第3回、12月 = 第4回）。
