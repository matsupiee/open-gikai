# 越知町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.ochi.kochi.jp/
- 分類: 公式サイトで PDF を静的公開（既存アダプターでは対応不可）
- 形式: PDF（テキスト埋め込み型）
- 特記: 議事録一覧ページから全 PDF リンクを取得可能

---

| ページ | URL パターン |

## PDF ファイル名パターン

### 基本構造

```
gijiroku{和暦年}.{月}-{会議日数}.pdf
```

- **和暦年**: 平成23〜令和7（数字のみ。平成31年は `31`、令和元年は `1`）
- **月**: 1〜12（数字）
- **会議日数**: 定例会は `-1`（第1日）、`-2`（第2日）、`-3`（第3日）の3日構成が基本
- **臨時会**: 月のみ or `r` / `rinji` / `R` サフィックス（例: `gijiroku2.5r.pdf`, `gijiroku3.3rinji.pdf`）

### 具体例

| 会議 | ファイル名 |
| --- | --- |
| 令和7年3月定例会 第1日 | `gijiroku7.3-1.pdf` |
| 令和7年3月定例会 第2日 | `gijiroku7.3-2.pdf` |
| 令和7年3月定例会 第3日 | `gijiroku7.3-3.pdf` |
| 令和7年1月臨時会 | `gijiroku7.1-1-1.pdf` |
| 令和2年5月臨時会 | `gijiroku2.5r.pdf` |
| 令和3年3月臨時会 | `gijiroku3.3rinji.pdf` |
| 平成29年3月臨時会 | `gijiroku29.3-R.pdf` |
| 令和4年7月臨時会 | `gijiroku4.7.pdf` |

### 注意: ファイル名の揺れ

ファイル名に一貫性がなく、以下のような揺れが存在する:

- 重複回避用の括弧付き番号: `gijiroku6.6-1(1).pdf`, `gijiroku4.9-3(1).pdf`
- 臨時会の表記揺れ: `r` / `rinji` / `R` / 月のみ
- 区切り文字の揺れ: `gijiroku25.3.1.pdf`（ドット区切り）、`gijiroku23.31.pdf`
- 誤字: `gojiroku4.1r.pdf`（`gijiroku` → `gojiroku`）
- ダブルドット: `gijiroku30.1-1..pdf`

**ファイル名のパターンマッチだけでは全件取得できないため、一覧ページからのリンク抽出が必須。**

## PDF 内部構造

### メタ情報（1ページ目）

```
3　令和７年第２回越知町議会定例会　会議録

令和７年３月１２日　越知町議会（定例会）を越知町役場議場に招集された。

１．開 議 日　令和７年３月１２日（水）　開議第３日
```

抽出可能な情報:
- 会議名: `令和X年第Y回越知町議会{定例会|臨時会}　会議録`
- 開催日: `令和X年X月X日`（全角数字）
- 会議日数: `開議第X日`

### 出席者情報（1ページ目）

```
２．出席議員　（１０人）
    １番　小田　壮一　　２番　上岡千世子　　３番　箭野　久美
    ４番　森下　安志　　５番　小田　範博

３．欠席議員　（なし）

４．事務局職員出席者
    事務局長　田村　幸三　　書　記　岩佐　由香

５．説明のため出席した者
    町　長　小田　保行　　副町長　國貞　誠志　　教育長　織田　誠
```

### 発言の構造

```
議　長（小 田 範 博 君）おはようございます。令和７年３月定例会開議３日目の応召御苦労さまです。
８　番（武 智　 龍 君）おはようございます。それでは、通告の順に従って一般質問をさせていただきます。
産業課長（武智 久幸 君）おはようございます。武智議員にお答えいたします。
```

発言者パターン:
- **議長**: `議　長（{氏名} 君）`
- **議員（番号付き）**: `{番号}　番（{氏名} 君）`
- **執行部（役職付き）**: `{役職}（{氏名} 君）`

特徴:
- 氏名の間にスペースが入る（例: `小 田 範 博`、`武 智　 龍`）
- 敬称は「君」で統一
- 全角スペースが多用される

## スクレイピング戦略

### Step 1: PDF リンクの収集

議事録閲覧ページ `https://www.town.ochi.kochi.jp/gikai/gijiroku` から全 PDF リンクを抽出する。

**収集方法:**

1. 一覧ページの HTML を取得
2. `/storage/files/gikai/gijiroku*.pdf` パターンに一致するリンクを Cheerio で抽出
3. 各リンクのテキスト（会議名・日付）も合わせて取得し、メタ情報として保存

**注意:**
- ファイル名に規則性がないため、一覧ページのスクレイピングが唯一の信頼できる取得方法
- 一覧ページは1ページに全件が表示されており、ページネーションなし

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF はテキスト埋め込み型のため、`pdf-parse` 等のライブラリでテキスト抽出が可能
- OCR は不要

### Step 3: 会議録のパース

#### メタ情報の抽出

```typescript
// 会議名の抽出
const titlePattern = /(?:令和|平成)\S+年第\S+回越知町議会(?:定例会|臨時会)\s*会議録/;

// 開催日の抽出
const datePattern = /(?:令和|平成)(\S+)年(\S+)月(\S+)日/;

// 開議日数の抽出
const dayPattern = /開議第(\S+)日/;
```

#### 発言者と発言内容の抽出

```typescript
// 議長の発言
const chairPattern = /^議\s*長（(.+?)君）/;

// 議員の発言（番号付き）
const memberPattern = /^(\S+)\s*番（(.+?)君）/;

// 執行部の発言（役職付き）
const officialPattern = /^(\S+?)（(.+?)君）/;
```

**注意:**
- 氏名中のスペース（全角・半角）を正規化する必要がある
- PDF のテキスト抽出時に改行位置がずれる可能性があるため、行結合処理が必要

---

## 注意事項

- PDF ファイル名に一貫性がなく、パターンマッチでの全件取得は不可能。一覧ページからの抽出が必須
- 議事日程（2ページ目〜3ページ目）に議案一覧が含まれる。議案情報の抽出も可能
- 出席議員・欠席議員・説明出席者の情報も構造的に記載されており、抽出可能
- 委員会の会議録は掲載されていない（本会議のみ）
- 議会だよりは別パス（`/storage/files/gikai/dayori/`）で公開されているが、会議録とは別コンテンツ

---

## 推奨アプローチ

1. **一覧ページからの全量取得を優先**: `https://www.town.ochi.kochi.jp/gikai/gijiroku` から全 PDF リンクとメタ情報を取得し、PDF URL のマスターリストを作成する
2. **PDF テキスト抽出**: `pdf-parse` 等でテキスト抽出。OCR は不要
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: 一覧ページのリンク数を前回取得時と比較し、新規追加分のみダウンロードする
5. **ファイル名の揺れへの対応**: ファイル名からのメタ情報抽出は補助的に使い、PDF 内部のテキストからメタ情報を正式に抽出する
