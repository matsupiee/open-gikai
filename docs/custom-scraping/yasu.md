# 野洲市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.yasu.lg.jp/gyoseijoho/gikai/teireikai-rinjikaikaigiroku/index.html
- 分類: 独自 CMS による PDF 公開（専用の会議録検索システムなし）
- 文字コード: UTF-8
- 特記: 年度別インデックスページから各年度ページへ遷移し、各開催日の PDF をダウンロードする構成

## PDF ファイルの命名規則

### 令和2年以降

```
r{年号数字}dai{回数}kai_{YYYYMMDD}.pdf   # 本文（開催日ごと）
r{年号数字}dai{回数}kai_mokuji.pdf       # 目次
```

例:
- `r7dai3kai_20250605.pdf` → 令和7年第3回定例会 6月5日分
- `r6dai6kai_mokuji.pdf` → 令和6年第6回定例会 目次
- `r7dai5kai_20251110.pdf` → 令和7年第5回臨時会 11月10日分

### 平成16〜23年（旧形式）

ファイル名が連番の数値 ID で管理されており、命名規則が統一されていない。

```
{数値ID}.pdf
```

例:
- `23754.pdf`〜`23787.pdf` → 平成23年の各会議録

---

## 会議録の構成

各年度に定例会・臨時会が複数回開催される。

- 定例会は年4〜6回（第2回、第3回、第4回、第6回が定例、第1回・第5回は臨時が多い）
- 臨時会は不定期開催
- 定例会は複数日にわたるため、開催日ごとに PDF が 1 ファイル存在する
- 各会期に目次 PDF が 1 ファイル付属する

---

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

インデックスページ `index.html` を取得し、各年度ページへのリンク（`/teireikai-rinjikaikaigiroku/{ID}.html`）を抽出する。

- ページネーションなし
- 令和7年（最新）から平成16年（最古）まで約23件のリンクが存在
- 「平成16〜23年（統合ページ）」はさらに各年度の旧形式ページへのリンクを持つ

### Step 2: 年度別ページから PDF リンクを収集

各年度ページを取得し、`/material/files/group/2/*.pdf` へのリンクを全て抽出する。

- ページネーションなし
- 1ページ内に当該年度の全会議録 PDF リンクが列挙される
- 本文 PDF と目次 PDF の両方が含まれる
- 旧形式（平成23年以前）は数値 ID のみのファイル名

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL から各ファイルをダウンロードし、テキストを抽出する。

- PDF から日付・会期情報・発言内容をパースする
- 目次 PDF（`_mokuji.pdf`）はメタ情報の補完に活用できる

---

## 注意事項

- 専用の全文検索システムは存在しない。PDF の直接ダウンロードのみで会議録を取得できる
- 旧形式（平成23年以前）は年度別ページが旧 CMS の URL（`/soshiki/gikai/teirei/`）に存在し、リンク先で連番 ID のファイル名が使われている
- 「平成16〜23年（統合ページ）」は各年の旧形式ページへのナビゲーションページであり、PDF リンクを直接持たない
- PDF ファイルは全て `https://www.city.yasu.lg.jp/material/files/group/2/` 配下に格納されている

---

## 推奨アプローチ

1. **インデックスページから年度 URL を収集**: `index.html` をパースして全年度ページ URL を取得する
2. **各年度ページから PDF URL を収集**: 各年度ページの `a[href$=".pdf"]` を抽出する
3. **目次 PDF を除外するオプション**: `_mokuji.pdf` は発言内容を含まないため、本文 PDF のみを処理対象とする選択肢もある
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 年度別ページを定期的に再取得し、新規 PDF リンクのみを追加処理する
