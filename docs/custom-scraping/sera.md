## PDF の内部構造

### メタ情報（冒頭ページ）

PDF の冒頭数ページに以下の情報が含まれる:

1. **表紙（1ページ目）**: 会議名称（例: `第３回世羅町議会定例会会議録`）、日付（例: `令和７年９月４日`）、日目（例: `第１日目`）
2. **議事日程（2ページ目）**: 会議の正式名称、開会時刻、場所（`於：世羅町役場議場`）、日程一覧
3. **出席議員一覧（3ページ目）**: 議員番号と氏名（12名）
4. **欠席議員一覧**
5. **会議録署名議員**
6. **説明のため出席した者の職氏名**: 町長、副町長、各課長等（19名）
7. **事務局職員**: 事務局長、書記等（3名）

### 発言の構造

発言者パターン:

```
○議長（髙橋公時）
○町長(奥田正和)
○６番（福永貴弘）
○教育長（早間貴之）
○産業振興課長（住田谷保）
```

- `○` で始まり、役職 or 議員番号 + 括弧内に氏名
- 括弧は全角 `（）` と半角 `()` が混在する
- 発言内容は発言者行の後に続くテキスト（同一行に続く場合と改行後に続く場合がある）

### パース用正規表現（案）

```typescript
// 発言者の抽出（全角・半角括弧両対応）
const speakerPattern = /^○(.+?)[（(](.+?)[）)]/;
// 例: ○６番（福永貴弘） → role="６番", name="福永貴弘"
// 例: ○町長(奥田正和) → role="町長", name="奥田正和"

// 開催日の抽出（表紙から）
const datePattern = /令和(\d+)年(\d+)月(\d+)日/;

// 会議名の抽出（表紙から）
const sessionPattern = /第(\d+)回世羅町議会(定例会|臨時会)会議録/;
```

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `https://www.town.sera.hiroshima.jp/soshiki/16/14729.html` から全 PDF リンクを抽出する。

- ページネーションなし。単一ページに全件掲載されているため、1回の取得で完了
- `<a href="/uploaded/attachment/{ID}.pdf">` パターンで PDF リンクを収集
- リンクテキストから会議名・日程のメタ情報を取得

**収集方法:**

```typescript
// PDF リンクの抽出
const pdfLinks = $('a[href$=".pdf"]');

// リンクテキストからメタ情報を抽出
const metaPattern = /令和(\d+)年第(\d+)回(定例会|臨時会)(?:第(\d+)日目)?[（(](\d+)月(\d+)日[）)]/;
```

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF は 50〜100 ページ程度（定例会）。臨時会は 10〜30 ページ程度
- テキスト抽出には `pdf-parse` や `pypdf` 等のライブラリを使用
- ページ番号がテキスト先頭に含まれる（例: `1 \n`）ため、除去が必要

### Step 3: 会議録のパース

抽出したテキストから発言データを構造化する。

1. **メタ情報の取得**: 表紙ページ・議事日程ページからの抽出（Step 1 のリンクテキストでも取得可能）
2. **発言者の分割**: `○` で始まる行を発言者の区切りとして分割
3. **出席者情報の取得**: 出席議員一覧・説明員一覧からの抽出

---

## 注意事項

- PDF の添付ファイル ID はサイト全体の共通連番であり、会議録専用の連番ではない。ID の連続性に依存した取得は不可
- 括弧の全角・半角が混在しているため、発言者パターンの正規表現は両方に対応する必要がある
- PDF からのテキスト抽出精度はライブラリに依存する。特に氏名の漢字（`髙`、`﨑` 等の異体字）が正しく抽出されるか要確認
- 出席議員一覧のレイアウトはスペース区切りのため、テキスト抽出時に整形が必要
- 一覧ページが将来的に年度別に分割される可能性があるため、ページ構造の変化に注意

---

## 推奨アプローチ

1. **一覧ページの一括取得**: 単一ページに全件掲載されているため、1回のリクエストで全 PDF リンクを収集する
2. **リンクテキストからのメタ情報抽出**: PDF を開く前にリンクテキストから会議名・日付等を取得し、DB 重複チェックに活用する
3. **PDF テキスト抽出**: ダウンロードした PDF からテキストを抽出し、発言者パターンで分割・構造化する
4. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 取得済み PDF の attachment ID を記録し、一覧ページの最新リンクと比較して未取得分のみダウンロードする
