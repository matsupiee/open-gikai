# 美祢市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.mine.lg.jp/soshiki/gikai/gijiroku/index.html
- 分類: 市公式サイト上で PDF 形式により直接公開（外部の会議録検索システムは使用していない）
- 文字コード: UTF-8（推定）
- 特記: 会議録は PDF 形式で提供されている。専用の全文検索システムは存在しない。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.city.mine.lg.jp/soshiki/gikai/gijiroku/index.html` |
| 各会議録 PDF | `https://www.city.mine.lg.jp/soshiki/gikai/gijiroku/{ファイル名}.pdf`（推定） |

---

## ページ構造（推定）

会議録一覧ページから各会議の PDF ファイルへのリンクが提供されている。

- 年度ごとに見出し（`<h2>` / `<h3>` 等）で区切られている可能性がある
- 各会議の PDF リンクが一覧で掲載されている
- 定例会・臨時会の区分がテキストに含まれると想定

---

## スクレイピング戦略

### Step 1: 一覧ページの取得・PDF URL 収集

`https://www.city.mine.lg.jp/soshiki/gikai/gijiroku/index.html` を取得し、PDF ファイルへのリンクをすべて抽出する。

```typescript
// Cheerio での収集例
const pdfLinks = $('a[href$=".pdf"]')
  .map((_, el) => {
    const href = $(el).attr('href')!;
    return new URL(href, 'https://www.city.mine.lg.jp/').toString();
  })
  .toArray();
```

- 一覧ページが単一ページか複数ページか（ページネーション有無）は実際のアクセスで確認が必要
- サブページ（年度別ページなど）が存在する場合は、一覧ページ内のリンクを再帰的に辿る

### Step 2: PDF の取得・テキスト抽出

各会議録 PDF を取得し、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

- PDF は会議録全文（発言者・発言内容を含む）
- ファイルサイズは会議によって異なる

### Step 3: メタ情報の取得

一覧ページのリンクテキストまたはページ見出しから以下を抽出する:

- 年度: `令和X年` / `平成XX年`
- 会議種別: `定例会` / `臨時会`
- 回次: `第X回`
- 開催日: ページ上に記載がある場合

**パース用正規表現（案）:**

```typescript
// 年度の抽出
const yearPattern = /(?:令和|平成)(\d+)年/;

// 会議種別と回次の抽出
const sessionPattern = /第(\d+)回(定例会|臨時会)/;
```

---

## 注意事項

- サイトへのアクセスが不安定な場合がある（調査時に DNS 解決不可を確認）。リトライ処理を組み込むこと
- 会議録の全文テキストは HTML ページには含まれず、すべて PDF 内にある
- PDF ファイルの命名規則は実際のリンクから確認が必要。ファイル名を推測・構築せず、一覧ページの `<a href>` から URL を直接収集すること
- 一覧ページの構造（年度別の見出し、リンクの配置パターン等）は実際のアクセスで確認が必要
- リクエスト間には適切な待機時間（1〜2秒）を設けること

---

## 推奨アプローチ

1. **一覧ページからリンク収集**: 会議録一覧ページの全 PDF リンクを起点とし、URL のハードコードを避ける
2. **サブページの探索**: 一覧ページから年度別・会議種別のサブページへのリンクがある場合は再帰的に辿る
3. **PDF テキスト抽出**: pdfjs や pdfminer 相当のライブラリで PDF を解析する
4. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
5. **差分更新**: 既取得 PDF URL のリストと比較し、新規 URL のみ PDF を取得する
