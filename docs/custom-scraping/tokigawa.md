# ときがわ町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.tokigawa.lg.jp/div/203010/htm/gijiroku/index.html
- 分類: 町独自の静的 HTML 公開（既存アダプターでは対応不可）
- 文字コード: Shift_JIS（CP932）
- 特記: 平成18年（2006年）以降の会議録を年度別・回次別の静的 HTML ファイルで公開

### 会議録ファイル名の規則

```
{MMDD}{4桁連番}.htm
```

- `MMDD`: 開催月日（例: `0304` → 3月4日）
- 連番: `0000`（目次）、`0001` 以降（本会議各日）
- 特別委員会は連番の上位桁が異なる（例: `2000`〜 → 予算審査特別委員会）

例:
- `r07/03040001.htm` → 令和7年3月4日本会議
- `r07/03062001.htm` → 令和7年3月6日予算審査特別委員会
- `h18/03070101.htm` → 平成18年3月7日第1回臨時会

## スクレイピング戦略

### Step 1: 年度別インデックスページの収集

平成18年（`h18.html`）から令和7年（`index.html`）まで、全年度のインデックスページを取得する。

- 年度ごとに1ページ、合計約20ページ
- ページネーションなし

**URL 生成方法:**

```typescript
// 平成18〜31年
const heiseiYears = Array.from({ length: 14 }, (_, i) => `h${String(18 + i).padStart(2, "0")}.html`);
// h18.html〜h31.html

// 令和2年〜（現在）
const reiwaYears = Array.from({ length: 6 }, (_, i) => `r${String(2 + i).padStart(2, "0")}.html`);
// r02.html〜r07.html（最新年は index.html としても参照可能）
```

### Step 2: 会議録ファイルリストの抽出

各年度インデックスページの HTML を Shift_JIS でデコードし、会議録リンクを抽出する。

```typescript
// インデックスページ内のリンク形式
// <a href="./r07/03040001.htm">３月４日（開会、一般質問）</a>
// <a href="./h18/03070101.htm">３月７日（正副議長選挙、議案上程...）</a>

// 抽出パターン
const linkPattern = /<a href="\.\/([^/]+)\/(\d{8}\.htm)"[^>]*>([^<]+)<\/a>/g;
```

目次ページ（連番 `0000`）はスキップし、本文ファイルのみ取得する。

### Step 3: 会議録本文の取得

各 `.htm` ファイルを Shift_JIS でデコードして取得する。

- HTML は `<body><TT><font size=3>` で始まるシンプルな構造
- CSS やナビゲーション要素はほぼ存在しない
- `<A NAME="PP{番号}">` アンカーでページ内の節が区切られる

### Step 4: 会議録のパース

#### メタ情報

ファイル先頭付近に以下の形式で記載される:

```
令和７年第１回ときがわ町議会定例会
```

- 開催日: インデックスページのリンクテキストから取得（例: `３月４日（開会、一般質問）`）
- 会議名: インデックスページの見出し（例: `令和７年第１回定例会`）

#### 発言の構造

発言者パターン:

```
○神山　俊議長　皆さん、おはようございます。
○渡邉一美町長　田中議員ご質問の…
○６番　田中紀吉議員　議長の発言許可をいただきましたので…
◇　田　中　紀　吉　議員
```

- `○` で始まり、役職または番号 + 氏名 + 発言内容が続く（同一行）
- `◇` で始まる行は一般質問の発言議員の見出し（発言内容は直後の `○` 行から始まる）
- 議長の発言: `○{氏名}議長`
- 町長の発言: `○{氏名}町長`
- 議員の発言: `○{番号}番　{氏名}議員`

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)(?:\t|　)(.+)$/;
// 例: ○神山　俊議長　本日の議事日程を... → role_name="神山　俊議長", speech="本日の..."
// 例: ○６番　田中紀吉議員　議長の発言許可を... → role_name="６番　田中紀吉議員", speech="議長の..."

// 役職と氏名の分離
const rolePattern = /^(\d+番)\s+(.+?)議員$/;       // 議員
const chairPattern = /^(.+?)議長$/;                // 議長
const mayorPattern = /^(.+?)町長$/;                // 町長
const vicePattern = /^(.+?)副町長$/;               // 副町長
const officerPattern = /^(.+?課長|.+?局長)$/;      // 行政職員

// 質問者の見出し
const questionerPattern = /^◇\s*(.+?)\s*議員$/;

// 開催日の抽出（インデックスページのリンクテキストより）
const datePattern = /^(\d+)月(\d+)日/;

// 年号の抽出（本文冒頭より）
const eraPattern = /(?:令和|平成)(\d+)年第(\d+)回/;
```

---

## 注意事項

- 文字コードが Shift_JIS（CP932）のため、`iconv` 等でデコードが必要
- `<A NAME="PP{番号}">` アンカーはページネーションではなく印刷用の節区切りであり、1ファイルが1日分の会議録
- 目次ファイル（連番 `0000`）はリンク一覧のみで発言内容を含まないため、スクレイピング対象外
- 氏名は全角スペース区切りで1文字ずつ離されている場合があるため（例: `渡　邉　一　美`）、名寄せ時は正規化が必要
- `◇` 行（質問者見出し）は発言ではないため、発言としてパースしない

---

## 推奨アプローチ

1. **インデックス起点**: 全年度インデックスページ（約20ページ）を起点として全会議録ファイルのURLリストを作成する
2. **目次をスキップ**: 連番 `0000` のファイルは目次のため除外し、`0001` 以降のみ取得する
3. **Shift_JIS デコード**: `iconv-lite` 等で CP932 としてデコードする（`shift_jis` ではなく `cp932` を指定）
4. **発言単位の抽出**: `○` で始まる行を発言の区切りとして処理する
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
6. **差分更新**: 最新年度のインデックスページを定期的に確認し、新規ファイルのみ追加取得する
