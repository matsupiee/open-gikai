## 会議の種類

- **定例会**: 年 4 回（3 月・6 月・9 月・12 月が基本）
- **臨時会**: 必要に応じて随時開催

---

## PDF ファイル命名規則

### 近年（令和4年以降）の統一パターン

| 会議種別 | ファイル名パターン | 例 |
| --- | --- | --- |
| 定例会 | `teireikai{YY}{MM}.pdf` | `teireikai0703.pdf`（令和7年3月） |
| 臨時会 | `rinnjikai{YY}{MM}.pdf` | `rinnjikai0701.pdf`（令和7年1月） |

- `{YY}` は和暦の下2桁（令和7年 → `07`）
- `{MM}` は開催月の2桁（3月 → `03`）

### 古い年度（平成22年以前）の命名

ファイル名の命名規則が**統一されていない**。以下のような多様なパターンが混在する:

- `rinji20100118.pdf`（日付ベース）
- `H220204RINJI.pdf`（和暦+日付+大文字）
- `h2209teireikai.pdf`（和暦+小文字）
- `dai4kaiteireikai.pdf`（日本語ローマ字）
- `teirei190903_01.pdf`（連番分割、平成19年）

古い年度では 1 つの定例会が日程ごとに複数 PDF に分割されているケースもある（例: 平成19年9月定例会は目次 + 4 日分の計 5 ファイル）。

## スクレイピング戦略

### Step 1: 年度一覧ページから年度別ページの URL を収集

会議録トップページ `/www/gikai/genre/1204443645837/index.html` にアクセスし、年度別ページへのリンク一覧を取得する。

- リンクは `<ul><li>` 構造で列挙されている
- `/www/gikai/contents/{年度ID}/index.html` パターンのリンクを抽出
- ページネーションなし（全年度が単一ページに掲載）

### Step 2: 各年度ページから PDF リンクを収集

各年度ページにアクセスし、PDF ファイルへのリンクを収集する。

- PDF リンクは `<ul><li><a>` 構造で列挙
- リンクテキストに会議名・開催月・ファイルサイズが含まれる
  - 例: `令和７年第１回（３月）定例会会議録(1954KB)(PDF)`
- ページネーションなし（1 年度分が単一ページに掲載）

**リンクテキストからのメタ情報抽出:**

```typescript
// リンクテキストから会議情報を抽出
const meetingPattern = /(?:令和|平成)(\S+?)年第(\d+)回(?:（(\S+?)月）)?(定例会|臨時会)会議録/;
// 例: "令和７年第１回（３月）定例会会議録(1954KB)(PDF)"
//   → era="令和", year="７", session="1", month="３", type="定例会"
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容をパース

**PDF テキストのパース方針:**

- PDF 内のテキスト構造は年度・会議によって異なる可能性があるため、実際の PDF を複数ダウンロードして構造を確認する必要がある
- 一般的な議会会議録 PDF では、発言者名が行頭に記載され、続くテキストが発言内容となるパターンが多い

---

## 注意事項

- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- PDF ファイルサイズが大きい場合がある（最大約 9MB）。ダウンロード時のタイムアウトに注意
- 古い年度（平成22年以前）はファイル命名が不統一なため、年度ページから動的にリンクを収集する方式が必須
- 平成19年9月定例会のように 1 会議が複数 PDF に分割されているケースでは、目次ファイルと日程別ファイルを結合して扱う必要がある
- 年度一覧ページの URL リスト自体が更新される可能性があるため、年度一覧からの動的取得を基本とする

---

## 推奨アプローチ

1. **2 段階クロール**: 年度一覧 → 年度別ページ → PDF リンク収集の 2 段階で URL を収集
2. **PDF テキスト抽出**: HTML ではなく PDF からのテキスト抽出が必要。pdf-parse や pdf.js 等のライブラリを使用
3. **レート制限**: リクエスト間に 1〜2 秒の待機時間を設ける
4. **差分更新**: 年度 ID をキーにして、新しい年度ページの追加を検知。各年度ページ内の PDF リンク数の変化で新規会議録の追加を検知
5. **リンクテキストからメタ情報を取得**: PDF ファイル名は命名規則が不統一なため、リンクテキスト（会議名・回次・月）をメタ情報の主要ソースとする
