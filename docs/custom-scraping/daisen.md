## サイト構造

### 3 階層のナビゲーション

1. **会議録トップ** (`/gikai/9/`) — 年度別リンクの一覧
2. **年度別ページ** (`/gikai/9/{年度ID}/`) — その年の定例会・臨時会の一覧
3. **定例会サブページ** (`/gikai/9/{年度ID}/{定例会ID}/`) — 定例会の各日程の PDF リンク（定例会のみ）

### 会議種別によるリンク形式の違い

- **臨時会**: 年度別ページから PDF への直接リンク（1 回 = 1 PDF）
- **定例会（近年）**: 年度別ページからサブページへのリンク → サブページ内で日程ごとの PDF リンク
- **定例会（平成25年以前）**: 年度別ページから日程ごとの PDF への直接リンク（サブページなし）

### リンクテキストのパターン

```
# 臨時会
第1回大山町議会臨時会（1月20日）

# 定例会（年度ページ上）
第2回大山町議会定例会（2月28日～3月21日）

# 定例会（サブページ上、各日程）
第1日（2月28日）
第2日（3月1日）

# 定例会（旧形式、日程ごとに直接リンク）
平成25年第2回定例会第1日（3月4日）
```

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録トップページ (`/gikai/9/`) から全年度のリンクを取得する。

- 年度リンクは `/gikai/9/{年度ID}/` 形式
- 年度 ID は規則性がないため、トップページから動的に取得する必要がある

### Step 2: 各年度ページから会議録リンクの収集

各年度ページをクロールし、以下の 2 種類のリンクを収集する。

#### a) PDF への直接リンク（臨時会 + 旧形式の定例会）

- リンク先が `.pdf` で終わるものを収集
- リンクテキストから会議名・日付を抽出

#### b) 定例会サブページへのリンク

- リンク先が `/gikai/9/{年度ID}/{定例会ID}/` 形式のものを収集
- サブページをさらにクロールし、日程ごとの PDF リンクを取得

### Step 3: PDF のダウンロードとテキスト抽出

全会議録が PDF 形式のため、以下の手順で処理する。

1. PDF ファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストからメタ情報と発言内容をパース

### Step 4: メタ情報の抽出

リンクテキストから以下を抽出する。

```typescript
// 臨時会のリンクテキストから
const rinjiPattern = /第(\d+)回大山町議会臨時会（(\d+)月(\d+)日）/;
// 例: "第1回大山町議会臨時会（1月20日）"

// 定例会のリンクテキストから（年度ページ上）
const teireiPattern = /第(\d+)回大山町議会定例会（(\d+)月(\d+)日～(\d+)月(\d+)日）/;
// 例: "第2回大山町議会定例会（2月28日～3月21日）"

// 定例会の各日程リンクテキストから（サブページ上）
const dayPattern = /第(\d+)日（(\d+)月(\d+)日）/;
// 例: "第1日（2月28日）"

// 旧形式の定例会リンクテキストから
const oldTeireiPattern = /平成(\d+)年第(\d+)回定例会第(\d+)日（(\d+)月(\d+)日）/;
// 例: "平成25年第2回定例会第1日（3月4日）"
```

## PDF の URL パターン

### 新形式（令和2年以降）

```
/user/filer_public/{hash1}/{hash2}/{UUID}/{ファイル名}.pdf
```

- `{hash1}/{hash2}`: UUID の先頭から派生したディレクトリ構造
- `{UUID}`: ファイルの一意識別子
- `{ファイル名}`: 日本語をローマ字に変換した名称（例: `ling-he-7nian-di-3hui-lin-shi-hui-51.pdf`）

### 旧形式（平成25年以前）

```
/system/site/upload/live/{ID}/atc_{タイムスタンプ}.pdf
```

- `{ID}`: 数値 ID（年度ごとに同じ値の場合あり）
- `{タイムスタンプ}`: UNIX タイムスタンプ

---

## 注意事項

- 検索機能は存在しないため、全ページのクロールが必須
- 会議録はすべて PDF 形式（HTML 本文なし）であり、PDF パーサーが必要
- PDF 内の発言構造はファイルごとに異なる可能性があるため、パース精度の検証が必要
- 年度 ID に規則性がないため、ハードコーディングせずトップページから動的に取得する
- 旧形式と新形式で PDF の URL パターンが異なる
- ページネーションは存在しない（各ページに全件表示）

---

## 推奨アプローチ

1. **トップダウンでクロール**: トップページ → 年度ページ → 定例会サブページの順にリンクを収集
2. **PDF テキスト抽出**: `pdf-parse` 等のライブラリで PDF からテキストを抽出し、発言情報をパース
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 最新の年度ページのみを再クロールし、新規 PDF のみをダウンロードする
5. **リンクテキストの活用**: PDF 内からのメタ情報抽出が困難な場合、リンクテキストから会議名・日付を取得する
