# 湧別町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.yubetsu.lg.jp/administration/town/detail.html?content=516
- 分類: 自治体公式サイト上の単一ページに PDF リンクを掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8（HTML）/ PDF 形式で会議録を公開
- 特記: Google Tag Manager（GTM-NDD6GW5）使用

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議結果・会議録一覧 | `https://www.town.yubetsu.lg.jp/administration/town/detail.html?content=516` |
| PDF ファイル | `https://www.town.yubetsu.lg.jp/common/img/content/content_{YYYYMMDD}_{HHMMSS}.pdf` |

### PDF URL のパターン

PDF のファイル名は `content_{アップロード日}_{アップロード時刻}.pdf` 形式で、会議の開催日とは一致しない。

例:
- `content_20250318_141316.pdf` → 令和7年第1回臨時会の会議録（開催日: 令和7年1月10日）
- `content_20240815_133505.pdf` → 令和6年第1回定例会の会議録（開催日: 令和6年3月6日）

HTML 上では相対パス `../../common/img/content/content_{YYYYMMDD}_{HHMMSS}.pdf` で記載されている。

## ページ構造

### 一覧ページの構成

- 単一ページに全ての会議録・会議結果のリンクが掲載される（ページネーションなし）
- 年度ごとにセクションが分かれている（令和8年、令和7年、令和6年の3年分を確認）
- 各年度内は会議ごとにグループ化されている
- 過去年度（令和5年以前）へのリンクは現時点では存在しない

### 会議の種類

| 種別 | 例 |
| --- | --- |
| 定例会 | 第1回定例会〜第4回定例会 |
| 臨時会 | 第1回臨時会〜第6回臨時会 |

### 各会議に含まれるドキュメント

| ドキュメント種別 | 説明 |
| --- | --- |
| 会議結果 | 議案の採決結果等をまとめた PDF |
| 会議録 | 会議の発言内容を記録した PDF |

- 定例会は複数日にわたることがあり、日ごとに「会議結果」「会議録」が分かれる（例: 1日目、2日目...）
- 臨時会は通常1日で終了し、「会議結果」と「会議録」が各1ファイル
- 最新の会議は「会議結果」のみで「会議録」が未掲載の場合がある

## PDF の構造

### 表紙（1ページ目）

```
令和７年第１回

湧別町議会臨時会会議録

湧別町議会
```

### メタ情報（2ページ目〜）

冒頭に以下の情報が記載される:

```
令和７年第１回湧別町議会臨時会会議

令和７年１月１０日湧別町議会議場に招集された。
```

- 開催日: `令和X年X月X日` 形式（全角数字）
- 会議名: `第X回湧別町議会{定例会|臨時会}会議`

続いて以下のセクションがある:
1. 応招議員（議員番号 + 氏名）
2. 不応招議員
3. 出席議員
4. 欠席議員
5. 説明のため出席する者（町長、副町長、各課長等）
6. 本会議の書記

### 議事日程

```
議事日程（第１日）
                            令和７年１月１０日
日程第　１　　　　会議録署名議員の指名
日程第　２　　　　会期の決定
日程第　３　　　　諸般の報告
...
```

### 発言の構造

発言者パターン:

```
○議　　長　ただいまの出席議員は９名でございます。
○議会事務局長　議案第１号、令和６年度湧別町一般会計補正予算。
○町　　長　前回の議会以降における行政上の諸課題について...
○４　　番　今説明がありました説明資料の９ページの...
○農政課長　村川議員の質問にお答えします。
○全　　員　（異　議　な　し）
```

- `○` で始まり、役職名（議長、町長、議会事務局長、農政課長 等）または議員番号が続く
- 役職名や番号の間にスペースが挿入されている場合がある（例: `○議　　長`、`○４　　番`）
- 発言内容は発言者行の後に続くテキスト
- 議員の指名は `４番、村川君。` のように番号 + 氏名で行われる

#### パース用正規表現（案）

```typescript
// 発言者の抽出（スペースを含む役職名に対応）
const speakerPattern = /^○(.+?)$/;
// 例: ○議　　長 → "議　　長"
// 例: ○４　　番 → "４　　番"
// 例: ○農政課長 → "農政課長"

// 役職名の正規化（スペース除去）
const normalizeRole = (role: string) => role.replace(/[\s　]+/g, "");
// "議　　長" → "議長"
// "４　　番" → "４番"

// 開催日の抽出
const datePattern = /令和(\d+)年(\d+)月(\d+)日/;

// 全員の応答
const allMembersPattern = /^○全\s*員/;
```

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `detail.html?content=516` から会議録 PDF のリンクを抽出する。

**収集方法:**

1. 一覧ページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` を抽出
3. リンクテキストから「会議録」を含むものだけをフィルタリング（「会議結果」は除外）
4. 相対パスを絶対 URL に変換（`../../common/img/content/` → `https://www.town.yubetsu.lg.jp/common/img/content/`）
5. リンクテキストからメタ情報（開催日、会議種別、日数目）を抽出

**リンクテキストのパターン:**

```
会議録　PDF(740KB)                    → 臨時会等（1日のみ）
9月16日　会議録（1日目）　PDF(1294KB)  → 定例会（複数日）
```

### Step 2: PDF のダウンロード

抽出した URL から PDF ファイルをダウンロードする。

- PDF のファイルサイズは 150KB〜5MB 程度
- レート制限を考慮し、リクエスト間に適切な待機時間（1〜2秒）を設ける

### Step 3: PDF からのテキスト抽出

PDF からテキストを抽出し、会議録の内容をパースする。

**推奨ライブラリ:** `pdf-parse` または `pdfjs-dist`

1. PDF をバッファとして読み込み
2. ページごとにテキストを抽出
3. 表紙ページ（1ページ目）と議員一覧ページ（2〜3ページ目）はメタ情報として処理
4. 議事日程ページ以降を発言録として処理

### Step 4: 発言のパース

テキストから発言者と発言内容を構造化する。

1. `○` で始まる行を発言者の区切りとして検出
2. 発言者の役職名を正規化（スペース除去）
3. 次の `○` 行までのテキストを発言内容として抽出
4. 「開会宣告」「散会宣告」等の見出し行は別途処理

---

## 注意事項

- PDF 形式のため、テキスト抽出時にレイアウト由来の不要な改行やスペースが混入する可能性がある
- 議員番号と氏名の間にスペースが挿入されている場合がある（`○４　　番` 等）
- 現時点では令和6年〜令和8年の3年分のみ掲載されており、過去の年度は確認できない
- 「会議結果」と「会議録」は別ファイルであり、スクレイピング対象は「会議録」のみ
- 定例会の複数日にわたる会議は日ごとに個別の PDF として公開される
- 最新の会議は会議録が未掲載の場合がある（会議結果のみ先行公開）

---

## 推奨アプローチ

1. **会議録のみを対象**: リンクテキストに「会議録」を含むものだけを取得し、「会議結果」は除外する
2. **PDF テキスト抽出**: `pdf-parse` 等で PDF からテキストを抽出し、`○` 記号を発言者の区切りとして構造化する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: PDF ファイル名のタイムスタンプ（`content_{YYYYMMDD}_{HHMMSS}`）を利用して、前回取得済みのファイル以降のみを取得する差分更新が可能
5. **ページ構成の変化に注意**: 過去年度が追加される可能性があるため、定期的に一覧ページを確認する
