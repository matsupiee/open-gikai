# 東彼杵町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.higashisonogi.lg.jp/soshiki/gikai/795.html
- 分類: 自治体公式サイトに PDF を直接掲載（検索システムなし）
- 公開形式: PDF
- 特記: 全会議録が単一ページに年度別で一覧掲載されている

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.town.higashisonogi.lg.jp/soshiki/gikai/795.html` |
| 会議録 PDF | `https://www.town.higashisonogi.lg.jp/material/files/group/44/{ファイル名}.pdf` |

### PDF ファイル名の命名規則

ファイル名はローマ字で構成され、以下のパターンに従う:

```
{年号}{年}nenndai{回数}kai{会議種別}{月}gatu{日}niti.pdf
```

例:
- `reiwa7nenndai4kaiteireikai12gatu11niti.pdf` → 令和7年第4回定例会 12月11日
- `reiwa6nenndai1kairinnjikai1gatu25niti.pdf` → 令和6年第1回臨時会 1月25日
- `dai1kaiteireikai-reiwa3nenn3gatu8niti.pdf` → 令和3年第1回定例会 3月8日（古い形式）

**注意**: ファイル名の命名規則は完全に統一されておらず、年度によって微妙な差異がある。URL をファイル名から生成するのではなく、一覧ページから直接取得する方式が安全。

```
    </a>
    ...
  <h3>臨時会</h3>
    <a href="//www.town.higashisonogi.lg.jp/material/files/group/44/xxx.pdf">
      第1回臨時会（令和7年1月29日開催） (PDFファイル: 100.5KB)
    </a>
    ...
<h2>令和6年議会会議録</h2>
  ...
```

- 年度見出し: `<h2>` タグ（例: `令和7年議会会議録`）
- 会議種別見出し: `<h3>` タグ（`定例会` / `臨時会`）
- PDF リンク: `<a>` タグ（`<h3>` の後に直接配置、`<ul>/<li>` なし）
- ページネーション: なし（単一ページに全件掲載）

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録一覧ページ `https://www.town.higashisonogi.lg.jp/soshiki/gikai/795.html` から全 PDF リンクを抽出する。

**収集方法:**

1. 一覧ページの HTML を取得
2. `<h2>` タグで年度を判定
3. `<h3>` タグで会議種別（定例会 / 臨時会）を判定
4. `<a>` タグから PDF の URL とリンクテキストを抽出
5. リンクテキストから開催日を正規表現で抽出

**パース用正規表現（案）:**

```typescript
// 年度の抽出（h2 テキスト）
const fiscalYearPattern = /(?:令和|平成)(\d+)年議会会議録/;

// 会議種別の判定（h3 テキスト）
const sessionTypePattern = /^(定例会|臨時会)$/;

// リンクテキストから会議情報を抽出
const meetingPattern = /第(\d+)回(定例会|臨時会)（(?:令和|平成)(\d+)年(\d+)月(\d+)日開催）/;
// 例: "第4回定例会（令和7年12月11日開催）" → 回数=4, 種別=定例会, 年=7, 月=12, 日=11
```

### Step 2: PDF のダウンロード

収集した URL から PDF をダウンロードする。

- href 属性が `//` で始まるプロトコル相対 URL のため、`https:` を先頭に付与する
- ファイルサイズはリンクテキストに記載あり（数百 KB〜数 MB 程度）

### Step 3: PDF のテキスト抽出

ダウンロードした PDF からテキストを抽出する。

- PDF パーサー（`pdf-parse` 等）を使用してテキストを抽出
- 議事録の構造（発言者・発言内容）は PDF 内のレイアウトに依存するため、実際の PDF を確認して抽出ロジックを調整する必要がある

---

## 注意事項

- 検索システムが存在しないため、全量取得が前提となる
- PDF 形式のため、テキスト抽出の精度は PDF の作成方法に依存する（画像 PDF の場合は OCR が必要になる可能性あり）
- ファイル名の命名規則が年度によって異なるため、URL の推測ではなく一覧ページからのリンク収集が必須
- href 属性がプロトコル相対 URL（`//` 始まり）であることに注意

---

## 推奨アプローチ

1. **一覧ページからの全量収集**: 単一ページに全件掲載されているため、1回の HTML 取得で全 PDF リンクを収集可能
2. **メタ情報はリンクテキストから抽出**: 開催日・会議種別・回数はリンクテキストに含まれるため、PDF を開かずにメタ情報を取得できる
3. **年度・会議種別は HTML 構造から判定**: `<h2>` / `<h3>` の階層構造を利用して、各リンクがどの年度・種別に属するかを判定する
4. **レート制限**: PDF ダウンロード時はリクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 一覧ページを定期的に取得し、既知の URL と比較して新規追加分のみダウンロードする
