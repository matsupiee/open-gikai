# 坂城町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.sakaki.nagano.jp/site/gikai/
- 分類: 自治体 CMS による PDF 公開（会議ごとの PDF が一覧形式で掲載）
- 文字コード: UTF-8
- 特記: Google Tag Manager（GTM-MTLSMDXX）使用、アクセシビリティツール（_tsutaeruMO）搭載

---

## 会議の種類

- **定例会**: 年4回開催（第1回〜第4回）
- **臨時会**: 不定期開催（年によって0〜複数回）

会議録詳細ページのタイトル例:
- `令和7年第1回坂城町議会定例会会議録`
- `令和7年第1回坂城町議会臨時会会議録`

---

## PDF の構成

各会議録詳細ページには以下の PDF が掲載される:

| 種別 | 説明 |
| --- | --- |
| 全ページ一括ダウンロード | 会議録全体を1ファイルにまとめたもの |
| 日程 | 会期日程表 |
| 付議事件及び審議結果 | 議案と審議結果の一覧 |
| 目次 | 会議録の目次 |
| 分割 会議録（1日目〜N日目） | 各日の会議録（日付入り） |
| 一般質問通告一覧表 | 一般質問の通告内容（定例会のみ） |
| 意見書 | 意見書（ある場合のみ） |

PDF の URL パターン: `/uploaded/attachment/{連番ID}.pdf`

---

## ページネーション

- 年度別一覧ページ（`list10-{ID}.html`）にはページネーションなし
- 各年度の会議録は1ページにすべて表示される

---

## スクレイピング戦略

### Step 1: 年度別一覧ページの URL 収集

議会トップページ `/site/gikai/` または会議録一覧 `/site/gikai/list10.html` から、年度別一覧ページの URL（`list10-{ID}.html`）を収集する。

### Step 2: 会議録詳細ページの URL 収集

各年度別一覧ページ（`list10-{ID}.html`）から、個別の会議録詳細ページの URL（`/site/gikai/{ページID}.html`）を抽出する。

- 各会議のリンクテキストには会議名が含まれる（例: `令和7年第3回坂城町議会定例会会議録`）
- 更新日も取得可能

### Step 3: PDF URL の収集

各会議録詳細ページから PDF リンク（`/uploaded/attachment/{ファイルID}.pdf`）を抽出する。

- リンクテキストから PDF の種別（全ページ一括、日程、分割会議録など）を判別可能
- スクレイピング対象は「全ページ一括ダウンロード」PDF を優先（1ファイルで全内容を取得できるため）

### Step 4: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキスト抽出ツール（pdf-parse 等）でテキスト化する
- 分割会議録の場合は日付情報がリンクテキストに含まれる（例: `分割 会議録1日目（9月1日）`）

---

## メタ情報の抽出

### 会議名・開催回

詳細ページのタイトルから抽出:

```
令和{N}年第{M}回坂城町議会{定例会|臨時会}会議録
```

### 開催日

分割会議録のリンクテキストから各日の日付を取得:

```
分割 会議録{N}日目（{M}月{D}日）
```

### パース用正規表現（案）

```typescript
// 会議名の抽出
const meetingPattern = /(令和|平成)\d+年第\d+回坂城町議会(定例会|臨時会)会議録/;

// 分割会議録の日付抽出
const dayPattern = /(\d+)日目（(\d+)月(\d+)日）/;

// PDF URL の抽出
const pdfPattern = /\/uploaded\/attachment\/(\d+)\.pdf/;
```

---

## 注意事項

- PDF はスキャン画像ベースの可能性があり、OCR が必要な場合がある（特に古い年度）
- 「全ページ一括ダウンロード」PDF は大容量（数十 MB）になることがある
- 平成20年以前の会議録は掲載されていない可能性がある
- Adobe Reader が必要な旨のガイダンスがページに記載されている

---

## 推奨アプローチ

1. **3段階クロール**: 年度一覧 → 会議詳細 → PDF URL の順に段階的に収集する
2. **全ページ一括 PDF を優先**: 分割 PDF よりも全ページ一括 PDF を取得することでリクエスト数を削減する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度別一覧ページの更新日を確認し、新規追加分のみを取得する
