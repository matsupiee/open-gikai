## PDF の公開形式

各年度ページには定例会・臨時会ごとにPDFリンクが掲載される。

### 定例会の構成パターン

- 複数日程の定例会は日ごとに分割されたPDFで公開（例: 1日目、4日目、8日目）
- 一般質問は議員名ごとに個別PDFで公開される場合がある（令和7年で確認）
- 単一PDFのみの定例会もある

### 臨時会の構成パターン

- 通常は単一PDFで公開

### PDF URL の特徴

- パスは `/village/wp-content/uploads/{YYYY}/{MM}/{ハッシュ}.pdf` 形式
- ファイル名はランダムなハッシュ値（32文字の16進数）で、会議名や日付を含まない
- メタ情報（会議名、開催日）はリンクテキストから取得する必要がある

## スクレイピング戦略

### Step 1: 年度別記事ページの URL を収集

カテゴリ一覧ページ `category/c-00-admininfo/c-03/c-03-02` から各年度の記事ページへのリンクを収集する。

- リンクは `c-03-02/{記事ID}` 形式
- WordPress のカテゴリアーカイブ構造

**収集方法:**

1. カテゴリページの HTML を取得
2. `c-00-admininfo/c-03/c-03-02/{記事ID}` パターンのリンクを抽出
3. 記事 ID の一覧を構築

### Step 2: 各年度ページから PDF リンクとメタ情報を収集

各年度ページの HTML から PDF リンクとその周辺テキスト（会議名・日程情報）を抽出する。

**収集方法:**

1. 各年度ページの HTML を取得
2. `wp-content/uploads/.../*.pdf` パターンのリンクを抽出
3. リンクテキストまたは周辺の見出し・テキストから以下を抽出:
   - 定例会/臨時会の区分
   - 第N回（回次）
   - 日目（複数日開催の場合）
   - 議員名（一般質問の場合）

### Step 3: PDF のダウンロードとテキスト抽出

PDFファイルをダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には pdf-parse 等のライブラリを使用
- 抽出したテキストから発言者・発言内容を構造化

---

## 注意事項

- PDF ファイル名がハッシュ値のため、ファイル名から会議情報を推定できない。リンクテキストの解析が必須
- 令和4年の議事録が欠落している可能性がある（カテゴリページに該当リンクなし）
- 年度によって公開形式が異なる（日ごと分割、単一PDF、一般質問の分離など）
- WordPress サイトのため、記事の追加・更新でページ構造が変わる可能性がある

---

## 推奨アプローチ

1. **2段階クロール**: カテゴリページ → 年度ページ → PDF の順で段階的に取得
2. **メタ情報はリンク周辺から抽出**: PDF ファイル名に情報がないため、HTML 上のリンクテキスト・見出しからメタ情報を組み立てる
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
4. **差分更新**: カテゴリページの記事 ID 一覧を比較し、新規追加分のみを取得する
5. **PDF テキスト抽出**: 会議録本文は PDF 内にあるため、PDF パーサーによるテキスト抽出が必要
