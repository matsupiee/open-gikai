# 苓北町議会 カスタムスクレイピング方針

## 概要

- サイト: https://reihoku-kumamoto.jp/
- 分類: 町公式サイトで PDF を直接公開（専用の会議録検索システムは存在しない）
- 文字コード: UTF-8
- 特記: 自治体向け独自 CMS を使用。jQuery ベース

---

| ページ | URL |
| --- | --- |
| 会議録一覧 | `https://reihoku-kumamoto.jp/kiji003676/index.html` |
| PDF ファイル | `https://reihoku-kumamoto.jp/kiji003676/{ファイル名}.pdf` |

## ページ構造

会議録一覧ページは単一の HTML ページ。ページネーションなし。全会議録が時系列降順で一覧表示されている。

各 PDF リンクは `<a target="_blank">` タグで提供されており、リンク内に PDF アイコン画像と会議名テキストが含まれる。年月のグループ分け見出しは構造化タグ（`<h3>` 等）ではなく、プレーンテキストとして配置されている。

```html
<!-- PDFリンクの基本パターン -->
<a href="3_676_3438_up_h6vpiiko.pdf" target="_blank">
  <img src="[PDF アイコン]" />
  令和8年第1回臨時会（PDF：432.2キロバイト）
  <img src="[別ウィンドウアイコン]" />
</a>
```

- リンクテキスト形式: `{元号}{年}年第{回数}回{定例会|臨時会}（PDF：{サイズ}キロバイト）`
- PDF URL はすべて相対パス（ベース URL: `https://reihoku-kumamoto.jp/kiji003676/`）

## PDF ファイルの命名規則

ファイル名の規則は時期によって大きく異なり、一貫したパターンが存在しない。主な形式は以下のとおり。

### 新形式（令和6年以降）

`3_676_{連番}_up_{ランダム文字列}.pdf`

例:
- `3_676_3438_up_h6vpiiko.pdf` → 令和8年第1回臨時会
- `3_676_3437_up_gusv3uc3.pdf` → 令和7年第5回定例会

### 中間形式（令和4年〜令和5年）

`3_676_{連番}_{元号略称}{年}_{回数}_{種別}.pdf`

例:
- `3_676_1_R5_1_rinjikai.pdf` → 令和5年第1回臨時会
- `3_676_6_R4.6_teirei.pdf` → 令和4年第3回定例会

### 旧形式（令和3年以前）

`3_676_{連番}_{MD5ハッシュ}.pdf`

例:
- `3_676_32_7b7cd6248fe70ef38ba59ee98e3a7fb4.pdf` → 平成30年第22回定例会

ファイル名からメタ情報を推測することは困難なため、**一覧ページの `<a href>` から URL を直接収集する**ことを強く推奨する。

## スクレイピング戦略

### Step 1: 一覧ページから PDF URL を収集

`https://reihoku-kumamoto.jp/kiji003676/index.html` を取得し、`kiji003676/` を含む `.pdf` へのすべての `<a href>` を抽出する。

- PDF URL はすべて相対パスのため、ベース URL `https://reihoku-kumamoto.jp/kiji003676/` を付与して絶対 URL に変換する
- 同一ページで全年度・全会議が一覧できるため、ページネーションは不要

```typescript
// Cheerio での収集例
const links = $('a[href$=".pdf"]')
  .map((_, el) => {
    const href = $(el).attr('href')!;
    return new URL(href, 'https://reihoku-kumamoto.jp/kiji003676/').toString();
  })
  .toArray();
```

### Step 2: リンクテキストからメタ情報を取得

各 `<a>` タグのテキストノードから会議名・年度・会議種別を抽出する。

**パース用正規表現（案）:**

```typescript
// リンクテキストから会議名を抽出
const titlePattern = /(?:令和|平成|平成\d+年)\d+年第\d+回(?:定例会|臨時会)/;
// 例: "令和8年第1回臨時会（PDF：432.2キロバイト）" → "令和8年第1回臨時会"

// 元号・年・回次・種別を個別に抽出
const sessionPattern = /(令和|平成)(\d+)年第(\d+)回(定例会|臨時会)/;
// 例: "令和8年第1回臨時会" → era="令和", year=8, count=1, type="臨時会"
```

### Step 3: 各 PDF を取得・テキスト抽出

PDF ファイルを取得し、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

- PDF は会議録全文（発言者・発言内容を含む）
- ファイルサイズは会議によって異なる（数百 KB 程度）

---

## 注意事項

- **ファイル名の規則が不統一**: 時期によって命名規則が大きく異なるため、URL は常に HTML から直接収集する
- **リンクテキストにファイルサイズが含まれる**: 正規表現でサイズ情報（`（PDF：N.Nキロバイト）`）を除去してから会議名を取得する
- **最古の会議録は平成25年（2013年）**: ただし連続して掲載されていない年度が存在する可能性がある

---

## 推奨アプローチ

1. **一覧ページを単一取得**: `index.html` の 1 リクエストで全 PDF URL を収集できる
2. **リンクテキストからメタ情報を取得**: `<a>` タグのテキストから年度・会議種別・回次を正規表現で抽出する
3. **レート制限**: PDF ダウンロード時もリクエスト間に 1〜2 秒の待機を設ける
4. **差分更新**: 既取得 PDF URL のリストと比較し、新規 URL のみ PDF を取得する
