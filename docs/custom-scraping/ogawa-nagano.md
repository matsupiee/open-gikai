## 概要

- サイト: https://www.vill.ogawa.nagano.jp/docs/64436.html
- 分類: 自治体公式サイト上で PDF を公開（既存アダプターでは対応不可）

### PDF ファイル名の命名規則

```
{和暦年}.{月}.{日}{会議種別}.pdf
```

- 和暦年: `7` = 令和7年、`R8` = 令和8年（表記ゆれあり）
- 月: 1〜2桁（`6`, `09`, `12` など。ゼロ埋めに統一性なし）
- 日: 1〜2桁
- 会議種別: 開会議案説明 / 一般質問 / 質疑 / 討論採決 / 説明討論採決 / 発議議会構成 / 臨時会 など

### PDF パスの `/fs/` ディレクトリ構造

`/fs/{a}/{b}/{c}/{d}/{e}/_/` の数字部分は連番 ID と推測される。規則性がないため、必ず HTML ページからリンクを収集する必要がある。

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録ページ `https://www.vill.ogawa.nagano.jp/docs/64436.html` から全 PDF リンクを収集する。

- ページは単一ページで全件掲載のため、ページネーション処理は不要
- `<h2>` 見出しから月別セクション（6月 / 9月 / 12月 / 臨時会）を判定
- `<h3>` 見出しから会議種別（説明 / 一般質問 / 質疑 / 討論・採決）を判定
- `<a>` タグの `href` が `/fs/` を含み `.pdf` で終わるリンクを抽出

**収集するメタ情報:**

```typescript
interface OgawaNaganoMinutes {
  month: string;         // "6月" | "9月" | "12月" | "臨時会"
  type: string;          // "議案説明" | "一般質問" | "質疑" | "討論採決" など
  date: string;          // "令和7年6月5日" など（ファイル名から抽出）
  pdfUrl: string;        // PDF の絶対 URL
}
```

### Step 2: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容をパース

### Step 3: テキストのパース（推定）

PDF 内のテキスト構造は一般的な議会会議録形式と推定される。

- 開催日・会議名はヘッダー部分に記載
- 発言者は「○議長」「○N番」等の形式で記載される可能性がある
- PDF のレイアウトに依存するため、実際のファイルを確認して正規表現を調整する必要あり

---

## 注意事項

- 単一 HTML ページに全 PDF リンクが掲載されているが、掲載範囲は現年度分のみ。過去年度の会議録が別ページに存在するかは不明
- 3月定例会は見出しのみで PDF が未掲載。今後追加される可能性あり
- PDF ファイル名の和暦年表記にゆれがある（`7.` と `R8.`）
- PDF パスの数字部分に規則性がないため、必ず HTML ページからリンクを辿る必要がある
- PDF 形式のため、HTML スクレイピングとは異なり PDF パーサーが必要
- PDF のレイアウト（段組み・ヘッダー/フッター等）によってテキスト抽出の精度が変わるため、実際の PDF を確認してパーサーの調整が必要

---

## 推奨アプローチ

1. **単一ページクロール**: 会議録ページ 1 ページのみを取得し、全 PDF リンクを収集する。ページネーションや年度別ページの巡回は不要
2. **PDF テキスト抽出の事前検証**: 数件の PDF を手動でダウンロードし、テキスト抽出の品質を確認する。段組み・文字化け等の問題がないか事前に検証
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: PDF リンクの URL をキーに管理し、前回取得済みの URL はスキップする。定期的にページを再取得して新規追加分のみダウンロードする
5. **ページ URL の変更監視**: 過去年度の会議録が別ページに移動・追加される可能性があるため、定期的にページ構成を確認する
