## 定例会ページの構造

各定例会ページはテーブル形式で質問者一覧を表示し、各質問者に対応する PDF リンクを持つ。

### テーブル列構成

| No. | 質問者 | 質問事項 | 質問要旨 | 答弁者 | 質問時間 |
| --- | --- | --- | --- | --- | --- |
| 連番 | 姓　名（全角スペース区切り） | 質問の大項目 | 詳細内容 | 町長等 | 30〜60分 |

- 質問者名が PDF へのリンクになっている
- 1 人の質問者が複数の質問事項を持つ場合がある

## PDF 会議録の構造

### ファイル形式

- Microsoft Word 2016 で作成された PDF
- 1 ファイル = 1 議員の質問・答弁の全文
- 平均 5〜11 ページ程度

### 発言者パターン

```
○３番（枝　史子君）
○議長（倉持　功君）
○町長（橋本正裕君）
○町長（橋本正裕君）
```

- `○` + 役職/番号 + `（` + 氏名 + `君）` の形式
- 議員は `○{番号}番（{氏名}君）` 形式
- 議長は `○議長（{氏名}君）` 形式
- 町長・副町長等は `○{役職}（{氏名}君）` 形式

### 議場注記パターン

```
［「全体ということですか」と言う者あり］
［町長　橋本正裕君登壇］
［何事か言う者あり］
```

- 角括弧 `［ ］` で囲まれた議場の状況説明

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○３番（枝　史子君） → role="３番", name="枝　史子"
// 例: ○町長（橋本正裕君） → role="町長", name="橋本正裕"
// 例: ○議長（倉持　功君） → role="議長", name="倉持　功"

// 議場注記の検出
const notePattern = /^［(.+?)］/;

// 登壇の検出
const podiumPattern = /［(.+?)登壇］/;
```

## スクレイピング戦略

### Step 1: 定例会ページ URL の収集

一般質問会議録トップページ `dir000145.html` から全定例会ページの URL を収集する。

1. トップページの HTML を取得
2. `page/page{XXXXXX}.html` 形式のリンクをすべて抽出
3. 年別ディレクトリページも巡回し、トップに表示されていない過去の定例会ページも収集

### Step 2: PDF URL の収集

各定例会ページから PDF リンクを収集する。

1. 定例会ページの HTML を取得
2. `/data/doc/{timestamp}_doc_88_0.pdf` 形式のリンクをすべて抽出
3. リンクテキスト（質問者名）も同時に取得

**収集データ:**

```typescript
type MeetingRecord = {
  year: string;          // "令和7年" など
  session: string;       // "第4回定例会" など
  questioner: string;    // "枝　史子" など
  pdfUrl: string;        // PDF の完全 URL
  pageUrl: string;       // 定例会ページの URL
};
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 各 PDF をダウンロード
2. PDF からテキストを抽出（`pdf-parse` 等のライブラリを使用）
3. テキストを発言単位でパースする

### Step 4: 発言のパース

PDF から抽出したテキストを発言単位に分割する。

1. `○` で始まる行を発言者の区切りとして認識
2. 発言者パターン正規表現で role と name を抽出
3. 次の `○` 行までを発言内容として取得
4. `［ ］` 内の議場注記は発言から除外するか、メタ情報として保持

## 注意事項

- 一般質問のみの公開であり、本会議全体や委員会の会議録は含まれない
- PDF は Word から生成されているため、テキスト抽出時にレイアウト由来の改行やスペースが混入する可能性がある
- 質問者名は「姓　名」（全角スペース区切り）で統一されている
- 敬称は「君」で統一（「○３番（枝　史子君）」）
- ページネーションはなく、トップページにすべての定例会が一覧表示される
- PDF の URL はタイムスタンプベース（例: `1773276029`）で、ID の規則性はない
- 平成18年から現在まで約20年分のデータが存在する

---

## 推奨アプローチ

1. **トップダウンでクロール**: トップページ → 年別ディレクトリ → 定例会ページ → PDF の順に階層的に URL を収集
2. **PDF テキスト抽出**: `pdf-parse` または `pdfjs-dist` で PDF からテキストを抽出し、Word 由来のフォーマットノイズを除去
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 定例会ページの URL（`page{XXXXXX}.html`）のページ番号が単調増加するため、前回取得済みの最大ページ番号以降のみを取得する差分更新が可能
5. **メタ情報はページから取得**: 開催年・定例会回次・質問者名は定例会ページのテーブルから取得し、PDF 内のテキストパースに依存しすぎない
