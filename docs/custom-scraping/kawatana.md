# 川棚町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.kawatana.jp/cat05/c5-11/post_267/
- 分類: 公式サイト内に年度別で PDF 会議録を掲載（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録はすべて PDF 形式で公開。Google Analytics（UA-143178231-1）使用

## ページ構造

### 会議録トップページ

年度別の見出し（`<h1>`）の下に、定例会・臨時会へのリンクが `<ul><li><a>` 形式で列挙されている。

```html
<h1>令和6年</h1>
<div class="entry">
  <ul>
    <li><a href="../../../parliament/record/r-teirei/post-954.html">令和６年定例会</a></li>
    <li><a href="../../../parliament/record/r-rinji/post-939.html">令和６年臨時会</a></li>
  </ul>
</div>
```

### 定例会・臨時会ページ

各ページには月別の会議録 PDF リンクが掲載されている。

- 定例会: 「令和６年12月　定例会（3日目）」のように月 + 日目で表記
- 臨時会: 「令和６年11月臨時会」のように月のみで表記
- リンク先はすべて PDF ファイル

```
令和６年12月　定例会（3日目）  351.6 KB → item/2d7f70b24c95d3f48df40a87a9479235.pdf
令和６年12月　定例会（2日目）  383.0 KB → item/86ddfb0c73fdc47c1c18df09f1d92b6f_1.pdf
令和６年12月　定例会（1日目）  849.6 KB → item/1009e22b6da9561d6b482798631da9b8_4.pdf
```

### PDF ファイル名

- ハッシュ値ベースのファイル名（例: `d618919dbb5788c5919eef193c0c5e9d_3.pdf`）
- ファイル名から会議の内容は推測不可
- サフィックス `_N` が付く場合があるが、規則性は不明

## スクレイピング戦略

### Step 1: 年度一覧ページから定例会・臨時会リンクの収集

会議録トップページ `https://www.kawatana.jp/cat05/c5-11/post_267/` をパースし、各年度の定例会・臨時会ページの URL を収集する。

**収集方法:**

1. `<section>` 内の `<h1>` から年度を取得
2. 各 `<div class="entry">` 内の `<a>` タグから定例会・臨時会ページの URL を抽出
3. リンクテキストから「定例会」「臨時会」を判別

### Step 2: 各年度ページから PDF リンクの収集

定例会・臨時会ページをそれぞれパースし、PDF ファイルの URL とメタ情報を収集する。

**収集方法:**

1. 各ページ内の `<a>` タグから `item/*.pdf` パターンの URL を抽出
2. リンクテキストから以下のメタ情報をパース:
   - 年号（令和/平成）
   - 月
   - 会議種別（定例会/臨時会）
   - 日目（定例会のみ）

**パース用正規表現（案）:**

```typescript
// 定例会リンクテキストのパース
const teireiPattern = /(?:令和|平成)(\S+?)年(\d+)月\s*定例会（(\d+)日目）/;
// 例: "令和６年12月　定例会（3日目）" → era="令和", year="６", month="12", day="3"

// 臨時会リンクテキストのパース
const rinjiPattern = /(?:令和|平成)(\S+?)年(\d+)月\s*臨時会/;
// 例: "令和６年11月臨時会" → era="令和", year="６", month="11"
```

### Step 3: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキストを抽出
- PDF 内から開催日、出席議員、発言内容等を抽出

## 注意事項

- 会議録はすべて PDF 形式のため、HTML パースだけでは本文取得不可。PDF パーサーが必須
- PDF ファイル名がハッシュ値のため、ファイル名からの内容推測は不可能。リンクテキストからメタ情報を取得する必要がある
- slug に統一規則がないため、トップページからのリンク収集が必須（URL を直接生成するアプローチは不可）
- 年度ページの HTML 構造は比較的シンプルだが、古い年度で構造が異なる可能性がある
- PDF のファイルサイズは 300KB〜850KB 程度

---

## 推奨アプローチ

1. **トップダウンでクロール**: トップページ → 年度ページ → PDF の3段階でリンクを辿る
2. **メタ情報はリンクテキストから取得**: PDF ファイル名には情報がないため、リンクテキストのパースが重要
3. **PDF テキスト抽出**: pdf-parse 等のライブラリで PDF からテキストを抽出
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: トップページの `最終更新日` を確認し、更新があった場合のみ再クロールする
