## ページ構造

### 一覧ページの HTML 構造

各一覧ページは以下の構造で構成される:

- `<h2>` タグで年度の見出し（例: 「令和7年」）
- `<h3>` タグで会議種別（例: 「定例会」「予算特別委員会会議録」）
- `<ul>` / `<li>` タグで PDF リンクのリスト
- `<a>` タグの `href` 属性に `/uploaded/attachment/{ID}.pdf` 形式の URL

### PDF リンクのパターン

```
令和7年第4回定例会 → /uploaded/attachment/27759.pdf
令和7年第3回定例会 → /uploaded/attachment/27235.pdf
令和7年12月12日総務教育常任委員会 → /uploaded/attachment/27760.pdf
```

- PDF の ID は数値連番（5桁）
- ファイルサイズが括弧内に表記される場合あり（例: `(542KB)`）

### ページネーション

なし。各一覧ページに全年度分が1ページに掲載されている。

## スクレイピング戦略

### Step 1: PDF リンクの収集

4つの一覧ページから全 PDF リンクを収集する。

**対象ページ:**

1. `https://www.city.shiogama.miyagi.jp/soshiki/2/2578.html`（定例会・特別委員会）
2. `https://www.city.shiogama.miyagi.jp/soshiki/2/6884.html`（常任委員会）
3. `https://www.city.shiogama.miyagi.jp/soshiki/2/47219.html`（全員協議会）
4. `https://www.city.shiogama.miyagi.jp/soshiki/2/30644.html`（長期総合計画特別委員会）

**収集方法:**

1. 各ページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` セレクタにより PDF リンクを抽出
3. リンクテキストから会議名・開催日を抽出
4. `<h2>` / `<h3>` の見出し階層から年度・会議種別を特定

### Step 2: メタ情報の抽出

リンクテキストと見出し階層から以下のメタ情報を抽出する:

```
リンクテキスト例:
- "令和7年第4回定例会"
- "令和7年12月12日総務教育常任委員会"
- "令和6年度決算特別委員会"
```

**抽出用正規表現（案）:**

```typescript
// 定例会・臨時会のパターン
const sessionPattern = /^(令和|平成)(\d+)年第?(\d+)回?(定例会|臨時会)/;
// 例: "令和7年第4回定例会" → era="令和", year=7, session=4, type="定例会"

// 委員会（日付付き）のパターン
const committeeDatePattern = /^(令和|平成)(\d+)年(\d+)月(\d+)日(.+)/;
// 例: "令和7年12月12日総務教育常任委員会" → date=12/12, name="総務教育常任委員会"

// 特別委員会（年度付き）のパターン
const specialCommitteePattern = /^(令和|平成)(\d+)年度(.+)/;
// 例: "令和6年度決算特別委員会" → era="令和", year=6, name="決算特別委員会"
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容を分離

### Step 4: 発言のパース

PDF から抽出したテキストの構造は PDF の作成方法に依存するため、実際の PDF を取得して構造を確認する必要がある。以下は想定されるパターン:

```typescript
// 発言者パターン（一般的な議会会議録の形式）
const speakerPattern = /^○(.+?)(?:（(.+?)）)?/;
// 例: ○議長（氏名） → role="議長", name="氏名"

// 番号付き発言者
const numberedSpeakerPattern = /^○(\d+番)\s*（(.+?)）/;
// 例: ○１番（氏名） → number="１番", name="氏名"
```

※ 実際の PDF テキスト構造に応じて正規表現を調整する必要がある。

---

## 注意事項

- 全ての会議録が PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF のテキスト抽出精度は PDF の作成方法（スキャン画像 vs テキスト埋め込み）に依存する
- スキャン画像ベースの PDF の場合は OCR が必要になる可能性がある
- PDF の attachment ID は数値連番だが、欠番がある可能性があるため、一覧ページからリンクを収集する方式が確実
- ページネーションがないため、一覧ページの取得は各カテゴリ1回で完了する

---

## 推奨アプローチ

1. **一覧ページからの PDF リンク収集を優先**: 4ページの HTML をパースするだけで全 PDF の URL とメタ情報が得られる
2. **PDF のサンプル確認を先行**: 実装前に数件の PDF をダウンロードし、テキスト抽出の可否と発言構造を確認する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの HTML を前回取得時と比較し、新規 PDF リンクのみを処理する
