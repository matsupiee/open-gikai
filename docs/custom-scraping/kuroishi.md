# 黒石市議会 カスタムスクレイピング方針

## 概要

- サイト: http://www.city.kuroishi.aomori.jp/shisei/gikai/gikai_kaigiroku.html
- 分類: 年別 PDF ファイル提供型（検索システムなし、単一ページに全年度を掲載）
- 文字コード: UTF-8
- 特記: 会議録は全て PDF で提供。HTML 上の本文テキストはなく、PDF をダウンロードして内容を抽出する必要がある

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（全年度） | `http://www.city.kuroishi.aomori.jp/shisei/gikai/gikai_kaigiroku.html` |
| 会議録 PDF | `http://www.city.kuroishi.aomori.jp/shisei/gikai/files/{ファイル名}.pdf` |

### PDF ファイル名パターン

ファイル名の命名規則は平成期と令和期で異なる。

#### 平成期（H19〜H31）

| 種別 | パターン | 例 |

## 検索パラメータ

検索システムは存在しない。単一の HTML ページに平成19年〜令和7年の全会議録 PDF リンクが掲載されている。

### 会議種別

- **定例会**: 年4回（第1回〜第4回）
- **臨時会**: 不定期開催（第1回〜複数回）

### 年号範囲

- 平成19年（2007年）〜 令和7年（2025年）

---

## HTML 構造

### ページ構成

- 全年度が1ページ内に `<h4>` タグで年号別にセクション分けされている
- 各年号セクション内に定例会・臨時会ごとの PDF リンクがリスト形式で配置
- テーブルは使用されていない
- アンカーリンク（`#H19`〜`#H07`）で各年度に直接ジャンプ可能

### リンク構造

- PDF リンクは `files/` ディレクトリからの相対パスで記載
- 各会議ごとに会期日程、一般質問通告表、審議議案、目次、各号（第1〜4号等）の PDF がリストされる

---

## ページネーション

なし。全年度の会議録が単一ページに掲載されている。

## スクレイピング方針

### Step 1: PDF リンクの収集

会議録一覧ページ（`gikai_kaigiroku.html`）を取得し、全ての PDF リンクを抽出する。

**収集方法:**

1. `gikai_kaigiroku.html` を取得
2. Cheerio 等で `a[href$=".pdf"]` のリンクを全て抽出
3. ファイル名から年号・回数・種別（定例会/臨時会）・号数をパースして構造化

**ファイル名パース用正規表現（案）:**

```typescript
// 平成期の会議録本文
const heiseiPattern = /^H(\d{2})_(\d+)(?:R)?_(\d+)\.pdf$/;
// 例: H19_1_1.pdf → 年=19, 回=1, 号=1
// 例: H19_1R_1.pdf → 年=19, 回=1(臨時会), 号=1

// 令和期の会議録本文
const reiwaPattern = /^R(\d{2})_(\d+)(T|R)_(\d{2})\.pdf$/;
// 例: R07_1T_01.pdf → 年=07, 回=1, 定例会(T), 号=01
// 例: R07_1R_01.pdf → 年=07, 回=1, 臨時会(R), 号=01

// 補助資料の除外パターン
const auxiliaryPattern = /^(Schedule|Tuukoku|Shingigian)_/;
const contentsPattern = /(contents|_C|_S\.pdf|_T\.pdf|_G_)$/;
```

### Step 2: PDF のダウンロード

抽出したリンクから会議録本文の PDF のみをダウンロードする。

- 補助資料（会期日程、一般質問通告表、審議議案、目次）はスキップ
- `files/` ディレクトリからの相対パスをベース URL と結合して絶対 URL を構築

### Step 3: PDF からのテキスト抽出

ダウンロードした PDF から `pdf-parse` 等のライブラリでテキストを抽出する。

- PDF 内の発言者パターンや会議メタ情報の構造は PDF の内容に依存するため、サンプル PDF を実際にダウンロードして確認が必要

---

## 注意事項

- 会議録は全て PDF 形式のため、HTML パースでは本文テキストを取得できない
- PDF のテキスト抽出精度はファイルの作成方法に依存する（スキャン画像の場合は OCR が必要になる可能性あり）
- 平成期と令和期でファイル名の命名規則が異なるため、両方のパターンに対応する必要がある
- 令和期の審議議案 PDF には日付サフィックス（`_0314` 等）が付くパターンがある

---

## 推奨アプローチ

1. **一覧ページから全 PDF リンクを一括取得**: 単一ページに全年度が掲載されているため、1回のリクエストで全リンクを収集できる
2. **会議録本文 PDF のみをフィルタリング**: 補助資料（目次・日程・通告表・議案）を除外し、会議録本文のみを対象にする
3. **サンプル PDF の事前確認**: テキスト抽出可能か（テキスト PDF かスキャン画像か）をサンプルで確認する
4. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2 秒）を設ける
5. **年号変換テーブル**: 平成→西暦、令和→西暦の変換テーブルを用意し、統一的な日付管理を行う
