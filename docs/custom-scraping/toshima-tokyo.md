# 利島村議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.toshimamura.org/about/assembly.html
- 分類: 自治体公式サイト上で年度別に PDF を公開（既存アダプターでは対応不可）
- 形式: PDF ファイル
- 文字コード: UTF-8
- 特記: 会議録検索システムなし。議会情報ページに議案書・議事録の PDF リンクを掲載。2021年第4回定例会以降 YouTube 配信あり

---

## サイト構造

### 議会情報ページ

`https://www.toshimamura.org/about/assembly.html` が議会に関する総合ページ。以下の情報を掲載。

- 議員情報（定数6名、議長・副議長・議員の氏名）
- 議会の仕事説明
- 年度別の議案書・議事録 PDF リンク
- 議会だより PDF リンク
- YouTube ライブ配信チャネルへのリンク

### お知らせ一覧（アーカイブ）

`https://www.toshimamura.org/archives/` は村全体のお知らせ一覧ページ。議会関連の記事も含まれるが、議事録 PDF は議会情報ページに直接掲載されている。

## URL 構造

### 議事録 PDF

PDF ファイルは以下のパターンで配置される。

```
https://www.toshimamura.org/fs/{d}/{d}/{d}/{d}/{d}/{d}/_/{ファイル名}.pdf
```

`/fs/` 以下は1桁の数字がスラッシュ区切りで6桁続き、`/_/` の後にファイル名が続く。数字部分はファイル ID のような連番で、規則性は不明。

## ページ構造

### HTML 構造

議会情報ページ（`assembly.html`）の議事録セクションは以下の構造を持つ。

- `<h3>` 見出しで年度別セクションに分割（例: 「2025年（令和7年）」「2024年（令和6年）」）
- 各セクション内に `<p>` タグで囲まれた `<a>` タグで PDF リンクが並ぶ
- `<a>` タグには特別な class/id 属性なし。`href` 属性のみ
- リンクテキストで「議案書」と「議事録」を区別（テキスト内容による判別が必要）

### 会議の種類

| 種類 | 説明 |
| --- | --- |
| 定例会 | 年4回（第1回〜第4回）開催 |
| 臨時会 | 不定期に開催 |

### 議会だより

議会広報紙「議会だより」も PDF で公開されている（第89号〜第101号を確認）。議事録とは別コンテンツ。

## スクレイピング戦略

### Step 1: 議事録 PDF リンクの収集

議会情報ページ `https://www.toshimamura.org/about/assembly.html` にアクセスし、議事録 PDF へのリンクを収集する。

- `<h3>` 見出しから年度情報を抽出
- `<a>` タグで `href` が `/fs/` を含み `.pdf` で終わるリンクを抽出
- リンクテキストに「議事録」を含むものをフィルタリング（「議案書」「施政方針」「決算書」等を除外）
- リンクテキストから会議種別（定例会/臨時会）と回次を抽出

**収集するメタ情報:**

```typescript
interface ToshimaTokyoMinutes {
  year: string;        // "2025年（令和7年）" など
  session: string;     // "第1回定例会" | "第2回臨時会" など
  type: "定例会" | "臨時会";
  pdfUrl: string;      // PDF の絶対 URL
}
```

### Step 2: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL からファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストから発言者・発言内容をパース

### Step 3: テキストのパース（推定）

PDF 内のテキスト構造は一般的な議会会議録形式と推定される。

- 開催日・会議名はヘッダー部分に記載
- 発言者は「○議長」「○N番」等の形式で記載される可能性がある
- PDF のレイアウトに依存するため、実際のファイルを確認して正規表現を調整する必要あり

## 注意事項

- PDF 形式のため、HTML スクレイピングとは異なり PDF パーサーが必要
- PDF のレイアウト（段組み・ヘッダー/フッター等）によってテキスト抽出の精度が変わるため、実際の PDF を確認してパーサーの調整が必要
- PDF ファイル名に日本語が含まれており、URL エンコードに注意が必要
- `/fs/` 配下の数字パスに規則性がないため、必ず議会情報ページからリンクを辿る必要がある
- 全ページが単一の `assembly.html` に掲載されているため、ページネーション処理は不要
- 議案書・施政方針・決算書など議事録以外の PDF も同一ページに混在しているため、リンクテキストによるフィルタリングが必須

---

## 推奨アプローチ

1. **単一ページクロール**: `assembly.html` のみを取得すれば全議事録 PDF リンクを収集可能。複数ページの巡回は不要
2. **リンクテキストによるフィルタリング**: 「議事録」を含むリンクのみを抽出し、議案書等を除外する
3. **PDF テキスト抽出の事前検証**: 数件の PDF を手動でダウンロードし、テキスト抽出の品質を確認する。段組み・文字化け等の問題がないか事前に検証
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 年度単位で管理し、過去の年度は再取得しない。最新年度のみ定期的に確認する
6. **定例会 → 臨時会の順**: まず定例会の議事録を優先して対応し、臨時会は段階的に拡張する
