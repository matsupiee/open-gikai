## PDF の URL パターン

PDF ファイルは `/fs/` 配下に格納されており、パスは以下の形式:

```
/fs/{d1}/{d2}/{d3}/{d4}/{d5}/_/{ファイル名}.pdf
```

- `{d1}〜{d5}`: 数字1桁ずつ（ファイル管理用 ID）
- `{ファイル名}`: 日本語ファイル名（日付＋会議種別を含む）

例:
- `/fs/5/8/3/7/6/_/2_20_______.pdf`（議案説明会）
- `/fs/5/3/0/5/2/_/2024.06.14_____.pdf`（全員協議会）
- `/fs/5/9/0/7/2/_/2025.04.24_________.pdf`（総務文教常任委員会）
- `/fs/5/8/7/7/7/_/2025.03.17_____________.pdf`（予算等審査特別委員会）

## スクレイピング戦略

### Step 1: 年度別ページの URL 収集

会議録トップページ `page2014063000011.html` から各年度ページへのリンクを収集する。

- トップページには全年度（平成27年〜令和7年）へのリンクが一覧表示されている
- ページネーションなし（単一ページに全年度が掲載）
- `/docs/{ページID}.html` 形式のリンクを抽出

### Step 2: 各年度ページから PDF リンクの収集

各年度ページをクロールし、PDF ファイルへのリンクを全て抽出する。

- 各年度ページにはテーブル形式で会議録が一覧表示されている
- PDF リンクは `/fs/` で始まるパスで提供される
- リンクテキストから会議種別・開催日を推定可能

**収集方法:**

1. 各年度ページの HTML を取得
2. `a[href*="/fs/"]` または `a[href$=".pdf"]` で PDF リンクを抽出
3. リンクのテキストやセクション見出しから会議種別・開催日のメタ情報を取得

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF リンクからファイルをダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用
- ファイル名に日本語が含まれるため、URL エンコードに注意

### Step 4: メタ情報の付与

PDF のテキストとリンク元の情報を組み合わせてメタ情報を構成する。

- 開催日: 年度ページのセクション見出し・テーブル内容から抽出
- 会議種別: セクション見出しから判定（定例会、常任委員会等）
- 号数: 定例会の場合「第N号」の表記から抽出

---

## 注意事項

- 会議録は全て PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF ファイル名に日本語（全角文字）が含まれるため、URL エンコーディングの処理が必要
- 年度ページの URL パターンが統一されていない（`6603.html` と `page2022031400015.html` が混在）
- 年度別ページの URL はトップページからの動的取得が必要（ハードコード不可）
- Adobe Reader 形式と記載されており、標準的な PDF フォーマット

---

## 推奨アプローチ

1. **トップダウンでクロール**: トップページ → 年度ページ → PDF リンクの順に辿る
2. **PDF テキスト抽出**: `pdf-parse` や `pdfjs-dist` を使って PDF からテキストを抽出
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度ページ単位でクロール済みかどうかを管理し、新しい年度・更新された年度のみ再取得する
5. **ファイル名からのメタ情報推定**: PDF ファイル名に日付情報が含まれるため、補助的なメタ情報源として活用可能
