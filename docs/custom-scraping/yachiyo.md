## サイト構造

### 3 階層構造

```
審議結果・会議録トップ (dir000425.html)
  └── 年度別一覧 (dir{XXXXXX}.html)
        └── 会議詳細ページ (page{XXXXXX}.html)
              ├── 第1号 PDF (会議録)
              ├── 第2号 PDF (会議録)
              ├── ...
              └── 審議結果 PDF
```

### 会議種別

各年度に以下の会議が含まれる:

- **定例会**: 年4回（3月・6月・9月・12月）
- **臨時会**: 不定期

### 会議詳細ページの構造

各会議詳細ページには以下が含まれる:

- 会期情報（開始日〜終了日）
- 議事日程テーブル（号数・開催日・主な内容）
- 各号の会議録 PDF へのリンク
- 審議結果 PDF へのリンク

例（令和6年第4回定例会）:

| 号数 | 開催日 | 主な内容 | PDF |

## PDF ファイルの特徴

- ファイル名パターン: `{タイムスタンプ}_doc_{XX}_0.pdf`
  - タイムスタンプ: UNIX タイムスタンプ（アップロード日時と推定）
  - `_doc_{XX}_0`: ドキュメント種別を示す数値（`61`, `62` など）
- サイズ: 概ね 150KB〜800KB 程度
- PDF からのテキスト抽出が必要

---

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

トップページ `dir000425.html` から全年度ページへのリンクを取得する。

- `dir{XXXXXX}.html` 形式のリンクをすべて抽出
- 19年度分（平成20年〜令和8年）が存在

### Step 2: 会議詳細ページ URL の収集

各年度ページから会議詳細ページへのリンクを取得する。

- `page{XXXXXX}.html` 形式のリンクをすべて抽出
- 定例会・臨時会の区別はリンクテキストから判定（例: 「第4回（12月）定例会」「第1回（1月）臨時会」）

### Step 3: PDF リンクの収集

各会議詳細ページから PDF ファイルへのリンクを取得する。

- `../data/doc/{タイムスタンプ}_doc_{XX}_0.pdf` 形式のリンクを抽出
- 審議結果 PDF は除外し、会議録 PDF のみを対象とする
- テーブルから号数・開催日・主な内容のメタ情報も取得

### Step 4: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- `pdf-parse` や `pdfjs-dist` などのライブラリでテキスト抽出
- PDF の品質により OCR が必要な可能性あり

### Step 5: テキストのパース

抽出したテキストから発言者と発言内容を分離する。

- PDF のテキスト構造は実際にダウンロードして確認が必要
- 一般的な議会会議録 PDF の場合、発言者名が行頭に記載されるパターンが多い

---

## 注意事項

- 検索機能がないため、全量クロールが必要
- PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF の品質（テキスト埋め込み or スキャン画像）は実際にダウンロードして確認する必要がある
- 審議結果 PDF は会議録ではないため、スクレイピング対象から除外する
- 年度ページの URL パターン（`dir{XXXXXX}.html`）に規則性がないため、トップページからの動的取得が必須

---

## 推奨アプローチ

1. **3 段階クロール**: トップ → 年度一覧 → 会議詳細の順に URL を収集し、最後に PDF をダウンロード
2. **PDF テキスト抽出の事前検証**: 数件の PDF をダウンロードし、テキスト抽出の品質を確認してからパーサーを実装
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度ページ単位での差分チェックが可能（新しい年度ページや会議詳細ページの追加を検知）
5. **メタ情報の保持**: リンクテキストから取得できる会議種別（定例会/臨時会）・回数・月をメタ情報として保存
