- サイト: https://www.town.hashikami.lg.jp/index.cfm/9,0,46,438,html
- 分類: 町公式サイト（ColdFusion CMS）内の単一ページに全会議録の PDF リンクを一覧掲載
- 文字コード: UTF-8

## HTML 構造

### 会議録一覧ページ

全会議録が単一の `<table>` に行として列挙される。

```
div#Main > div.content > section
  h2.titleOfContent: "議会会議録"
  div.contentBodyBox > div.contentBody
    table[border="1"]
      tbody
        tr (ヘッダー行)
          td: "名称"
          td: "ファイル"
        tr (データ行、繰り返し)
          td: "令和7年第7回階上町議会臨時会"  ← 会議名
          td
            a[href="/index.cfm/9,11528,c,html/11528/{filename}.pdf"]
              img[src="/images/icons/pdf.gif"]
              "令和7年第7回臨時会　階上町議会会議録"  ← リンクテキスト
```

### テーブル構造の特徴

- 1行目はヘッダー行（「名称」「ファイル」）
- 2行目以降がデータ行で、各行が1つの会議録に対応
- 1列目（`td:first-child`）: 会議名（例: 「令和7年第7回階上町議会臨時会」）
- 2列目（`td:nth-child(2)`）: PDF リンク（`<a>` タグ）
- 一部の行はリンクテキストがファイル名のまま（例: `20230904-140449.pdf`）で、発言訂正申出書などの補足資料

### 会議名のパターン

- `{和暦}年第{N}回階上町議会定例会`
- `{和暦}年第{N}回階上町議会臨時会`
- `{和暦}年第{N}回定例会における発言訂正申出書`（補足資料）

## スクレイピング方針

### 1. 一覧ページの取得

`https://www.town.hashikami.lg.jp/index.cfm/9,0,46,438,html` にアクセスし、`div.contentBody` 内の `<table>` を取得する。

### 2. テーブル行の解析

`<table>` 内の `<tr>` を走査し、各行から以下を抽出:

- 1列目のテキスト: 会議名
- 2列目の `<a>` タグの `href` 属性: PDF の URL

ヘッダー行（1行目）はスキップする。

### 3. フィルタリング

- 会議名に「定例会」または「臨時会」を含む行のみを対象とする
- 「発言訂正申出書」などの補足資料はスキップする

### 4. PDF のダウンロードとテキスト抽出

- PDF の URL は相対パス `/index.cfm/9,11528,c,html/11528/{filename}.pdf` 形式
- ベース URL: `https://www.town.hashikami.lg.jp`
- PDF をダウンロードし、テキスト抽出を行う

### 5. メタデータの構成

1列目の会議名テキストから以下を解析:

- 会議種別: 「定例会」または「臨時会」
- 和暦年: 「令和{N}年」「平成{N}年」
- 回次: 「第{N}回」

### 注意事項

- ページ全体で約228件の PDF リンクがあるため、一括取得時のリクエスト間隔に配慮する
- 一部の PDF リンクの `href` に全角スペース等が含まれるケースがある（例: `20241004-114718.pdf　`）。URL のトリム処理が必要
- 平成8年以前は回次の番号体系が異なる（年間の通し番号で、第8回、第13回など大きい番号になる）
- 1つの会議に対して1つの PDF ファイルが対応する（日ごと・議題ごとの分割はなし）
