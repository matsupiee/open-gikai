# 御船町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.mifune.kumamoto.jp/gikai/
- 分類: 独自 CMS（hpkiji）による HTML 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 会議録形式: PDF ファイル
- 特記: ASP.NET 環境。一覧ページは autopager プラグインによる動的読み込み方式

---

| 一覧の追加読み込み | `https://www.town.mifune.kumamoto.jp/gikai/hpkiji/pub/List.aspx?c_id=3&class_set_id=6&class_id=6006&pg={ページ番号}` |

## 一覧ページの構造

- 全 120 件（2026年3月時点）
- 令和2年度（2020年）〜令和6年度（2025年）の定例会会議録を掲載
- 1ページあたり 20 件表示、「もっと見る」ボタンで追加読み込み（autopager）
- 各エントリには以下が含まれる:
  - 会議録タイトル（例: `令和4年度第5回定例会（9月会議）会議録`）
  - 詳細ページ URL（`/gikai/page{ID}.html` 形式）
  - 最終更新日

### 年度別の詳細ページの分類

| 年度 | エントリ形式 | 例 |
| --- | --- | --- |

| 令和4年度以前 | 1回の定例会ごとに個別の詳細ページ | `令和4年度第5回定例会（9月会議）会議録` → `page7510.html` |

## 詳細ページの構造

### 令和5年度以降（年度まとめ型）

`https://www.town.mifune.kumamoto.jp/gikai/page8722.html` などの年度まとめページ。

- 1ページ内に年度内の全定例会（6〜9回）の PDF リンクを列挙
- 大規模な定例会は PDF が複数分割されている場合あり（例: 令和5年度第5回は4分割）
- ページネーションなし

### 令和4年度以前（定例会単位型）

`https://www.town.mifune.kumamoto.jp/gikai/page7510.html` などの各回ページ。

- 1ページに1回の定例会分の PDF リンク
- ページネーションなし

### 各エントリの含有情報

- 会議名（例: `令和6年度 第1回御船町議会定例会（6月会議）`）
- PDF ファイルへのリンク
- ファイルサイズ表示（例: `2.2メガバイト`）
- 「別ウィンドウで開きます」アイコン

## PDF URL の構造

```
https://www.town.mifune.kumamoto.jp/common/UploadFileOutput.ashx?c_id=3&id={ページID}&sub_id=1&flid={ファイルID}
```

| パラメータ | 意味 |
| --- | --- |
| `c_id=3` | カテゴリID（議会関連固定） |
| `id` | 詳細ページの ID（`page{ID}.html` の数値部分と対応） |
| `sub_id` | サブID（通常 `1` または `2`） |
| `flid` | ファイル固有 ID（連番、詳細ページごとに異なる） |

## スクレイピング戦略

### Step 1: 一覧ページから詳細ページ URL を収集

一覧ページ `List.aspx?c_id=3&class_set_id=6&class_id=6006` から詳細ページ URL を全件取得する。

1. `pg=1` から順にリクエストし、各ページの `<a>` タグから `/gikai/page{ID}.html` 形式のリンクを抽出
2. 「もっと見る」ボタン相当の次ページ URL（`pg=N`）が存在する限り繰り返す
3. 全件取得したら重複を排除してリスト化

**留意点:**

- autopager による動的読み込みのため、`pg` パラメータを手動でインクリメントして全ページ取得する
- `List.aspx` のレスポンスには先頭から累積でエントリが含まれるため、前ページのエントリとの重複排除が必要

### Step 2: 詳細ページから PDF URL を収集

各詳細ページにアクセスし、PDF ダウンロードリンク（`UploadFileOutput.ashx`）を抽出する。

- `id="mainBlock"` 内の `<a href>` タグで `UploadFileOutput.ashx` を含むものを対象とする
- 分割 PDF がある場合は複数リンクが存在するため、すべてを収集する
- 会議名は `<a>` タグのテキストまたは周辺テキストから抽出

### Step 3: PDF のダウンロードと処理

収集した PDF URL からファイルをダウンロードしてテキストを抽出する。

- PDF ファイルは `UploadFileOutput.ashx` エンドポイント経由で取得
- ファイルサイズは最大 2.4MB 程度
- 分割 PDF は複数ファイルを結合してから処理する

## メタ情報の取得

詳細ページのタイトルおよびリンクテキストから以下を抽出:

```
令和6年度 第1回御船町議会定例会（6月会議）
令和5年度 第7回御船町議会定例会（12月会議）
```

**抽出用正規表現（案）:**

```typescript
// 年度の抽出
const eraYearPattern = /(?:令和|平成)(\d+)年度/;

// 回数・開催月の抽出
const sessionPattern = /第(\d+)回御船町議会定例会（(.+?)会議）/;

// 開催年・月の変換（令和X年度 第N回 → 開催年月の推定）
// 例: 令和6年度 第1回（6月会議）→ 2024年6月
```

---

## 注意事項

- 令和5年度・6年度は年度まとめ型（1詳細ページ内に複数回分）、令和4年度以前は定例会単位型と形式が異なる
- 一部の定例会では PDF が複数ファイルに分割されている
- 会議録に開催日の明示がなく、会議名（月）から推定が必要な場合がある
- 委員会の会議録は掲載されておらず、定例会のみが対象

---

## 推奨アプローチ

1. **一覧ページの全量取得**: `pg` パラメータをインクリメントして全エントリの詳細ページ URL を収集
2. **詳細ページのクロール**: 各詳細ページから `UploadFileOutput.ashx` リンクをすべて抽出
3. **年度まとめ型の対応**: 令和5年度以降は1ページに複数回分が含まれるため、リンクテキストから各回を識別
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 詳細ページの最終更新日（`最終更新日：YYYY年MM月DD日`）を記録し、前回取得以降に更新されたページのみ再取得する
