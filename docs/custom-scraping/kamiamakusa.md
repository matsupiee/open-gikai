# 上天草市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.kamiamakusa.kumamoto.jp/
- 分類: 市公式サイト内 PDF 公開（独自 CMS）
- 文字コード: UTF-8
- 特記: 会議録本文の HTML 提供なし。PDF のみ提供。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.city.kamiamakusa.kumamoto.jp/q/list/389.html` |
| 会議録一覧（2ページ目以降） | `https://www.city.kamiamakusa.kumamoto.jp/q/list/389.html?pg={ページ番号}` |
| 会議録詳細 | `https://www.city.kamiamakusa.kumamoto.jp/q/aview/389/{ID}.html` |
| PDF ダウンロード | `https://www.city.kamiamakusa.kumamoto.jp/dl?q={ファイルID}_filelib_{ハッシュ}.pdf` |

---

## ページネーション構造

- 一覧ページは1ページあたり最大16件表示
- `?pg=1` パラメータで2ページ目へ遷移（0始まりではなく1始まり）
- 総ページ数: 2ページ（2016年12月時点、計22件）

---

## 会議録一覧ページの構造

各エントリは以下の要素で構成される:

- 掲載日（例: `2016年12月8日`）
- タイトル（例: `平成28年度第4回定例会(10月)議事録`）
- 詳細ページへのリンク: `/q/aview/389/{ID}.html`

掲載されている会議録の期間: 平成25年（2013年）〜平成28年（2016年）

---

## 会議録詳細ページの構造

詳細ページは会議ごとに複数の PDF ファイルへのリンクを含む。各 PDF は章番号付きで分割されている場合がある。

### PDF リンク形式

```
https://www.city.kamiamakusa.kumamoto.jp/dl?q={ファイルID}_filelib_{MD5ハッシュ}.pdf
```

例:
```
https://www.city.kamiamakusa.kumamoto.jp/dl?q=15437_filelib_bafa9783a44999599f954d22bc797058.pdf
```

### PDF の分割構成例（平成28年第3回定例会）

| 章番号 | 内容 | ファイルサイズ |
| --- | --- | --- |
| 1章 | 議事日程〜請願書 | 409KB |
| 7章 | 質疑・討論〜請願書 | 388KB |
| 8章 | 一般質問 | 596KB |
| 9章 | 一般質問 | 552KB |
| 18章 | 委員会報告 | 49KB |
| 22章 | 採決結果・議決 | 277KB |

※ 会議ごとに PDF 分割数・章番号は異なる。

## スクレイピング戦略

### Step 1: 詳細ページ URL の収集

一覧ページ `q/list/389.html?pg={N}` を全ページ分クロールし、`/q/aview/389/{ID}.html` 形式のリンクを全件収集する。

1. `pg=` パラメータを1から順に増やしてページネーションを処理
2. 各ページの各エントリから詳細ページ URL（`/q/aview/389/{ID}.html`）を抽出
3. エントリがゼロになったらクロール終了

### Step 2: PDF リンクの収集

各詳細ページ `/q/aview/389/{ID}.html` にアクセスし、`/dl?q=` 形式の PDF ダウンロードリンクを全件抽出する。

- Cheerio で `a[href*="/dl?q="]` を抽出
- 同一会議録が複数 PDF に分割されている場合は全件取得

### Step 3: PDF のダウンロードとテキスト抽出

取得した PDF ダウンロード URL から PDF を取得し、テキストを抽出する。

- PDF はスキャン画像でなく電子テキスト埋め込みの場合はそのまま抽出可能
- OCR が必要かどうかは個別に確認が必要

---

## 注意事項

- 会議録はすべて PDF 形式であり、HTML テキストでの本文提供はない
- PDF は章番号で複数ファイルに分割されているため、同一会議の全 PDF を紐付けて管理する必要がある
- `dl?q=` の URL に含まれるハッシュ値（MD5）はファイル固有の識別子として利用可能
- リクエスト間に適切な待機時間（1〜2秒）を設けること

---

## 推奨アプローチ

1. **全件収集優先**: 一覧ページ全ページを走査し、詳細ページ URL の全量リストを作成
2. **PDF 紐付け管理**: 同一詳細ページ URL に属する PDF を会議単位でグループ化して保存
3. **差分更新**: 詳細ページの `{ID}` は数値のため、前回取得済みの最大 ID 以降のみ取得する差分更新が可能
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
