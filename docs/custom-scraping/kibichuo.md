## PDF の構造

### 各会議の PDF 構成

1会議あたり以下の PDF が存在する:

- **一般質問概要書**: 一般質問の一覧（定例会のみ）
- **1日目〜N日目**: 各日の会議録本文（通常4〜6日分）

臨時会は1日分の PDF のみ（日付がファイル名代わりのリンクテキスト）。

### 発言の構造（4ページ目以降）

発言者パターン:

```
○議長（西山宗弘君）
○町長（山本雅則君）
○教育長（石井孝典君）
○総務課長（山本敦志君）
○３番（我妻瑛子君）
```

- `○` で始まり、役職または議席番号 + 括弧内に氏名 + `君` の敬称
- 発言内容は発言者行の次の行から続くテキスト
- 議事の区切りは `～～～～～～～～～～～～` で表現
- 傍聴席の反応は `［「異議なし」の声］` の形式で記載

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)君）/;
// 例: ○議長（西山宗弘君） → role="議長", name="西山宗弘"
// 例: ○３番（我妻瑛子君） → role="３番", name="我妻瑛子"

// 議事区切りの検出
const separatorPattern = /^～{5,}/;

// 傍聴席の反応
const audiencePattern = /^［(.+?)］/;

// 開催日の抽出（メタ情報から）
const datePattern = /令和(\d+)年(\d+)月(\d+)日/;

// 会議タイトルの抽出
const titlePattern = /^(令和|平成)\d+年第\d+回（.+?）吉備中央町議会会議録（(\d+)日目）/;
```

## スクレイピング戦略

### Step 1: PDF リンクの収集

会議録ページ `https://www.town.kibichuo.lg.jp/site/gikai/7847.html` から全 PDF リンクを抽出する。

- 単一ページに全年度分が掲載されているため、1回のリクエストで全リンクを取得可能
- `/uploaded/attachment/{ID}.pdf` パターンのリンクを Cheerio で抽出
- リンクテキストと前後の見出し要素から年度・会議名・日数を紐付ける

**収集方法:**

1. `7847.html` を取得し、HTML をパース
2. 年度見出し（例: `2024年(令和6年)`）と会議見出し（例: `第6回12月定例会`）をセクション区切りとして認識
3. 各セクション内の PDF リンクを `{ year, session, dayOrType, url }` として構造化

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF からのテキスト抽出には `pdf-parse` や `pdfjs-dist` 等のライブラリを使用
- 「一般質問概要書」は発言録ではないため、スキップまたは別扱いとする

### Step 3: 会議録のパース

抽出したテキストから発言データを構造化する。

1. **メタ情報の抽出**: 1ページ目から開催日、会議区別、出席議員等を抽出
2. **発言の分割**: `○` で始まる行を発言者の区切りとして分割
3. **発言者情報の抽出**: 役職/議席番号と氏名を正規表現で抽出
4. **議事区切りの処理**: `～～～` の区切り線で議事日程の切り替わりを検出

## 注意事項

- 会議録は PDF 形式のみで提供されており、HTML 版は存在しない
- PDF のテキスト抽出精度はフォント埋め込み状況に依存する（テスト時に要確認）
- 全角数字が多用されている（議席番号、日付など）
- 敬称「君」が全議員・全役職者に付与される
- 2020年は12月定例会のみ掲載（それ以前の会議録は現在のページには掲載されていない）
- ユーザーの備考には「2011年（平成23年）から」と記載があるが、現在の公式ページで確認できるのは2020年以降のみ

## 推奨アプローチ

1. **単一ページからの一括取得**: 会議録一覧が1ページにまとまっているため、クローリングは単純
2. **PDF テキスト抽出の事前検証**: 1〜2件の PDF で `pdf-parse` によるテキスト抽出精度を検証する
3. **一般質問概要書の除外**: 発言録ではないため、リンクテキストが「一般質問概要書」のものはスキップする
4. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: PDF の attachment ID は数値連番のため、前回取得済みの最大 ID 以降のみを取得する差分更新が可能
