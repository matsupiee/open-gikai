## HTML 構造

### ページ全体

単一ページ内に全年度の会議録 PDF リンクが掲載されている。年度ごとに `<h2>` 見出しで区切られ、その下に `<ul><li>` で PDF リンクが列挙される。同一年度内の定例会間は `<hr>` で区切られている。

```html
<h2>令和７年</h2>
<ul>
  <li><a href="files/r07-04t-01-1203.pdf" target="_blank">
    令和７年第４回蓬田村議会定例会会議録（第１号）
    <img alt="PDFファイル" class="wcv_ww_fileicon" src="../../_wcv/images/icon/pdf.gif">
    <span class="wcv_ww_filesize">(338KB)</span>
  </a></li>
  <li>...</li>
</ul>
<hr>
<ul>
  <li><a href="files/r07-03t-01-0909.pdf" target="_blank">
    令和７年第３回蓬田村議会定例会会議録（第１号）...
  </a></li>
</ul>
```

### PDF リンクの構造

- `<a>` タグの `href` 属性は相対パス `files/{ファイル名}.pdf`
- リンクテキストに会議名・号数が含まれる（例: `令和７年第４回蓬田村議会定例会会議録（第１号）`）
- 一部リンクは `<span style="font-size:100%;">` や `<p>` でラップされている（年度により HTML 構造が微妙に異なる）

### PDF ファイル名の命名規則

ファイル名の命名規則は年度により統一されていない。主なパターン:

| 年度 | パターン | 例 |
| --- | --- | --- |
| 令和7年 | `r07-{回}t-{号}-{MMDD}.pdf` | `r07-04t-01-1203.pdf` |
| 令和6年（第4回） | `r06-{回}t-{号}-{MMDD}.pdf` | `r06-04t-01-1204.pdf` |
| 令和6年（第1〜3回） | `R6-{回}-teirei-gijiroku{号}.pdf` | `R6-3-teirei-gijiroku1.pdf` |
| 令和5年 | `giji_r05-{回}-{号}.pdf` or `giji_r05_{回}_{号}.pdf` | `giji_r05-04-01.pdf` |
| 令和4年 | `giji_r04_{回}_{号}.pdf` or `giji_r04-{回}-{号}.pdf` | `giji_r04_04_01.pdf` |
| 特別委員会（令和7年） | `r07-k-{号}-{MMDD}.pdf`（決算）, `r07-y-{号}-{MMDD}.pdf`（予算） | `r07-k-01-0909.pdf` |
| 特別委員会（令和6年） | `R6-1-teirei-gijiroku-yosantokubetuiinkai{号}.pdf` | 長い日本語ローマ字名 |
| 特別委員会（令和5年） | `giji_kessan{号}_{YYYYMMDD}.pdf`, `giji_yosan{号}_{YYYYMMDD}.pdf` | `giji_kessan1_20230905.pdf` |

ファイル名に規則性がないため、URL をファイル名から推測することはできない。HTML ページからリンクを収集する方式が必須。

## スクレイピング戦略

### Step 1: 一覧ページの取得

`gijiroku.html` を1回取得するだけで全 PDF リンクを収集できる。

### Step 2: PDF リンクの収集

ページ内の `<a>` タグから `href` が `.pdf` で終わるリンクをすべて収集する。

```typescript
const pdfLinks = $('a[href$=".pdf"]');
```

- `href` は相対パス（`files/xxx.pdf`）なので、ベース URL `https://www.vill.yomogita.lg.jp/sonsei/gikai/` を付与して絶対 URL を構築する
- リンクテキストから会議名・号数を抽出する

### Step 3: 年度・会議種別の判定

直前の `<h2>` 見出しから年度を判定する。

```typescript
// h2 の内容から年度を取得
const yearHeadings = $('h2');
// 例: "令和７年", "令和６年", ...
```

リンクテキストから以下を正規表現で抽出:

```typescript
// 定例会
const teireiPattern = /令和(\d+)年第(\d+)回蓬田村議会定例会会議録（第(\d+)号）/;

// 特別委員会
const tokubetsuPattern = /(予算|決算)特別委員会会議録（第(\d+)号）/;
```

### Step 4: PDF のダウンロードとテキスト抽出

収集した URL から PDF をダウンロードし、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

## 注意事項

- **ファイル名の不統一**: 年度ごとにファイル名の命名規則が異なるため、ファイル名からの URL 推測は不可能。必ず HTML ページからリンクを収集すること
- **HTML 構造のばらつき**: 年度によりリンクの囲み方が異なる（`<span>` ラップあり/なし、`<p>` ラップあり/なし）。PDF リンクの抽出は `a[href$=".pdf"]` で統一的に取得可能
- **全角数字**: リンクテキスト内の数字は全角（例: `第１号`、`第４回`）。パース時に全角→半角変換が必要
- **特別委員会の年度帰属**: 特別委員会のリンクテキストには年度が含まれない場合がある。直前の `<h2>` 見出しから年度を判定する必要がある
- **PDF 総数**: 現時点で約112件の PDF リンクが存在

---

## 推奨アプローチ

1. **単一ページ起点**: `gijiroku.html` の1リクエストで全 PDF URL を収集できるため、クローリングは非常にシンプル
2. **`a[href$=".pdf"]` で一括取得**: HTML 構造のばらつきを無視し、PDF リンクのみを抽出する
3. **リンクテキストからメタデータ抽出**: ファイル名ではなくリンクテキストから会議名・号数を取得する
4. **レート制限**: PDF ダウンロード時にリクエスト間に1〜2秒の待機時間を設ける
5. **差分更新**: 既取得 URL のリストと比較し、新規 URL のみをダウンロードする
