# 斜里町議会 カスタムスクレイピング方針

## 概要

- サイト: http://gikai-sharitown.net/
- 分類: 独自ドメインの静的 HTML サイト（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 会議録形式: PDF ファイル（`img/` ディレクトリ配下に格納）
- 特記: ホームページ・ビルダー（`hpb-sp-20-1011-01`）で構築された静的サイト。検索機能なし

## 会議録の形式

### 本会議の会議録

- **形式**: PDF ファイル
- **配置場所**: `img/` ディレクトリ
- **ファイル名パターン**: `kaigiroku_{年度}_{月}_{日}.pdf`
  - 例: `kaigiroku_r3_5_6.pdf`（令和3年5月6日）
  - 例: `kaigiroku_r3_6_23.pdf`（令和3年6月23日）
  - 例: `kaigiroku_r2_3_4.pdf`（令和2年3月4日）
- **リンク元**: 各会議の議事日程テーブル内の「会議録」列

### 委員会の会議記録

- **形式**: PDF ファイル
- **配置場所**: `img/` ディレクトリ
- **ファイル名パターン**: `{YYMMDD}_kiroku.pdf`
  - 例: `230301_kiroku.pdf`（令和5年3月1日）
  - 例: `220916_kiroku2.pdf`（令和4年9月16日）

### 議事日程 PDF

- **ファイル名パターン**: `{年度}.{月}.{日}_nittei.pdf`
  - 例: `r7.6.18_nittei.pdf`

## スクレイピング戦略

### Step 1: 年度別一覧ページの収集

年度ページのURL一覧をクロールして全年度のページを収集する。

**起点ページ:**

1. `r7giji.html`（最新）から開始
2. 各ページの前年リンクを辿り、`h30giji.html`（最古）まで遡る

**既知の年度ページ一覧:**

- `r7giji.html`, `r6giji.html`, `r5giji.html`, `r4giji.html`, `r3giji.html`
- `giji.html`（令和2年）
- `r1giji.html`（令和元年）, `h31giji.html`（平成31年）, `h30giji.html`（平成30年）

### Step 2: 会議録 PDF リンクの抽出

2段階の構造に対応する必要がある:

#### パターン A: 令和6年〜令和7年（テーブル内に直接情報）

- 年度ページのテーブルから議事日程 PDF リンクと会議録リンクを直接取得
- ただし「会議録」列が空の場合が多い（まだアップロードされていない可能性）

#### パターン B: 令和5年以前（個別会議ページ経由）

1. 年度ページのテーブルから個別会議ページへのリンクを収集
   - 例: `r4_3kaigi.html`, `r3-5-6.html`, `0623-1.html`
2. 各個別会議ページのテーブルから `img/kaigiroku_*.pdf` へのリンクを抽出

**PDF リンク抽出の正規表現:**

```typescript
// 本会議の会議録 PDF
const kaigirokuPattern = /href="(img\/kaigiroku_[^"]+\.pdf)"/g;

// 委員会の会議記録 PDF
const kirokuPattern = /href="(img\/\d{6}_kiroku\d?\.pdf)"/g;

// 全 PDF リンク（汎用）
const allPdfPattern = /href="(img\/[^"]+\.pdf)"/g;
```

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）でテキスト化する
- PDF 内のメタ情報（開催日、会議名）はファイル名またはリンク元ページから取得する

### Step 4: メタ情報の抽出

#### 開催日

- ファイル名から抽出: `kaigiroku_r3_5_6.pdf` → 令和3年5月6日
- リンクテキストから抽出: `令和3年5月6日（木）`

#### 会議名

- リンク元テーブルの `th.col-title` から取得（例: `3月定例会議`, `1月臨時会議`, `招集会議`）
- ページタイトルから取得（例: `令和3年斜里町議会定例会　招集会議`）

## 注意事項

- HTTP 接続のみ（HTTPS は証明書エラーで接続不可: `ERR_TLS_CERT_ALTNAME_INVALID`）
- 会議録 PDF は令和3年以前の個別会議ページにのみリンクがある。令和6年・7年のページでは「会議録」列がほぼ空欄（PDF 未掲載 or 掲載遅延）
- 個別会議ページの URL に統一的な命名規則がない（`r3-5-6.html`, `0623-1.html`, `r4_3kaigi.html`, `2021-0302.html` など多様）
- 委員会の会議記録は委員会ページ（`soumu.html` 等）に直接 PDF リンクがある
- YouTube の録画リンクも多数含まれるが、会議録（テキスト）とは別物
- ホームページ・ビルダーで構築されているため HTML が正規の構造に沿わない箇所がある（`<li>` の閉じタグ欠落等）→ パーサーの寛容モードが必要

---

## 推奨アプローチ

1. **2段階クロール**: まず年度一覧ページ → 個別会議ページ → PDF リンクの順で全 PDF URL を収集
2. **PDF テキスト抽出**: ダウンロードした PDF から pdf-parse 等でテキストを抽出し、発言内容をパース
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **HTTP のみ使用**: HTTPS ではなく `http://` で全リクエストを行う
5. **委員会は別系統**: 本会議と委員会で PDF の配置構造が異なるため、別々のクローラーを用意する
6. **ファイル名ベースの差分更新**: PDF ファイル名に日付が含まれるため、既取得ファイル名のリストと照合して差分更新が可能
