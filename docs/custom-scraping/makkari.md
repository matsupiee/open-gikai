# 真狩村議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.vill.makkari.lg.jp/songikai/kaigiroku/
- 分類: PDF ファイルによる会議録公開（HTML 会議録検索システムなし）
- 文字コード: UTF-8（一覧ページ）/ PDF（会議録本文）
- 特記: 会議録は全て PDF 形式で公開。検索機能なし。単一ページに全年度の会議録リンクを掲載

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧 | `https://www.vill.makkari.lg.jp/songikai/kaigiroku/` |
| 会議録 PDF | `https://www.vill.makkari.lg.jp/resources/output/contents/file/release/1506/{ID}/{ファイル名}.pdf` |

### PDF ファイル名の命名規則

ファイル名は `{年号}-{月}-{日}-{回次}{種別}.pdf` のパターンに従う。

- 令和7年以降: `R{年号}-{月}-{日}-{回次}{種別}.pdf`
  - 例: `R7-12-11-4tei.pdf`（令和7年第4回定例会 12月11日）
  - 例: `R7-1-28-1rin.pdf`（令和7年第1回臨時会 1月28日）
- 令和6年以前: `{年号}-{月}-{日}-{回次}{種別}.pdf`
  - 例: `6-3-8-1tei.pdf`（令和6年第1回定例会 3月8日）
  - 例: `5-2-15-1rin.pdf`（令和5年第1回臨時会 2月15日）

種別コード:
- `tei` = 定例会
- `rin` = 臨時会

## 会議の種類

| 種別 | 説明 |
| --- | --- |
| 定例会 | 年4回開催（3月・6月・9月・12月）。定例会は複数日にわたる場合あり（1日目、2日目、3日目） |
| 臨時会 | 不定期開催。年に数回 |

## HTML 構造（一覧ページ）

年度ごとに `<h2 class="mk-title large">` 見出しがあり、その下に `<ul><li>` リストで会議録 PDF へのリンクが並ぶ。

```html
<h2 class="mk-title large">2025年（令和7年）会議録</h3>
<ul>
  <li>
    <a href="/resources/output/contents/file/release/1506/11537/R7-1-28-1rin.pdf"
       title="令和7年第1回臨時会(1月28日）" target="_blank">
      令和7年第1回臨時会(1月28日）
    </a>
    <!--(PDF形式:184KB)-->
  </li>
  ...
</ul>
```

- リンクテキストに会議名・開催日が含まれる（例: `令和7年第1回定例会　1日目(3月10日）`）
- PDF サイズはコメントアウトされた形式で記載（例: `<!--(PDF形式:184KB)-->`）
- `target="_blank"` で新しいタブに開く設定

### 議事経過のテーブル構造

議事本文は表形式で記載される:

```
                議       事       の   経   過
日   程 発   言   者              発               言
10：15   議       長    ただいまの出席議員数は、8 人です。
開会      (佐伯秀範)       定足数に達しておりますので、...
```

カラム構成:
1. **日程**: 時刻・日程番号（例: `10：15`、`日程 1`）
2. **発言者**: 役職と氏名（例: `議長 (佐伯秀範)`、`村長 (岩原清一)`）
3. **発言**: 発言内容

### 発言者パターン

```
議       長
(佐伯秀範)

村        長
( 岩 原 清 一 )

5 番
(向井忠幸)
```

- 役職（議長、村長、副村長、教育長、○番）の後に括弧内の氏名
- PDF のレイアウト上、文字間にスペースが入ることがある（例: `岩 原 清 一`）
- 一般議員は `{番号} 番` + `({氏名})` の形式

## スクレイピング戦略

### Step 1: PDF URL の収集

一覧ページ `https://www.vill.makkari.lg.jp/songikai/kaigiroku/` から全 PDF リンクを抽出する。

**収集方法:**

1. 一覧ページの HTML を取得
2. `<a href="...pdf">` のリンクを Cheerio で抽出
3. リンクテキスト（`title` 属性）から会議名・開催日を取得

```typescript
// リンクテキストからメタ情報を抽出
const metaPattern = /令和(\d+)年第(\d+)回(定例会|臨時会)[\s　]*((\d+)日目)?[（(](\d+)月(\d+)日[)）]/;
// 例: "令和7年第1回定例会　1日目(3月10日）"
//   → 年=7, 回次=1, 種別=定例会, 日目=1, 月=3, 日=10
```

### Step 2: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、`pdftotext` 等のツールでテキストを抽出する。

- PDF は比較的小さいサイズ（100KB〜600KB 程度）
- テキスト抽出可能な PDF（画像ベースではない）

### Step 3: テキストのパース

#### メタ情報の抽出

```typescript
// 会議名の抽出
const titlePattern = /令和\s*(\d+)\s*年\s*第(\d+)回真狩村議会(定例会|臨時会)会議録/;

// 開会日時の抽出
const datePattern = /開会\s+令和\s*(\d+)\s*年\s*(\d+)\s*月\s*(\d+)\s*日\s*午[前後]\s*(\d+)\s*時\s*(\d+)\s*分/;
```

#### 発言者と発言内容の抽出

PDF テキストはテーブルレイアウトのため、列位置に基づくパースが必要。`pdftotext -layout` で抽出した場合、スペースの位置で列を判別する。

```typescript
// 発言者パターン（スペース入りの可能性あり）
const speakerPattern = /^\s*(議\s*長|副\s*議\s*長|村\s*長|副\s*村\s*長|教\s*育\s*長|\d+\s*番)\s*$/;
const namePattern = /^\s*[（(]\s*(.+?)\s*[)）]\s*$/;
```

---

## 注意事項

- PDF 形式のため、HTML スクレイピングとは異なり `pdftotext` 等のテキスト抽出ツールが必要
- PDF のテーブルレイアウトにより、文字間にスペースが挿入されることがある（氏名・役職名など）
- 一覧ページに掲載されている年度が限定的（令和5年〜現在）。過去の会議録が追加される可能性あり
- 定例会は複数日にわたる場合があり、日ごとに別 PDF として公開される
- リンクテキストの括弧が全角・半角混在している（`（` と `(` の両方が使用される）

---

## 推奨アプローチ

1. **一覧ページの定期チェック**: ページネーションがなく単一ページのため、一覧ページを定期的にフェッチして新しい PDF リンクの追加を検出する
2. **PDF テキスト抽出**: `pdftotext -layout` を使用してレイアウトを保持したテキストを取得し、列位置ベースでパースする
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 一覧ページの PDF リンク一覧をキャッシュし、新規追加分のみダウンロード・パースする
5. **文字間スペースの正規化**: PDF 抽出時に入る不要なスペースを除去する前処理を実装する
