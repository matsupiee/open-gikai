# 釧路町議会 カスタムスクレイピング方針

## 概要

- サイト: http://www.town.kushiro.lg.jp/gikai/
- 分類: 自治体独自 CMS による HTML + PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 会議録形式: PDF（1 会議日ごとに 1 PDF ファイル）
- 特記: Google Analytics（UA-61798839-1）使用

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 議会トップ | `http://www.town.kushiro.lg.jp/gikai/` |
| 年度別一覧 | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{年}.html` |
| 会議録詳細（定例会） | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{回数}/1/{年}.html` |
| 会議録詳細（臨時会） | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{回数}/2/{年}.html` |
| 会議録 PDF | `http://www.town.kushiro.lg.jp/gikai/kaigiroku/{年}/teirei_{回数}/kaigi{連番}.pdf` |
| 会議録 PDF（臨時会） | `http://www.town.kushiro.lg.jp/gikai/kaigiroku/{年}/rinji_{回数}/kaigi{連番}.pdf` |
| 日程 PDF | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{年}/teirei_{回数}/nittei.pdf` |
| 審議結果 PDF | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{年}/teirei_{回数}/kekka.pdf` |
| 一般質問 PDF | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{年}/teirei_{回数}/shitsumon.pdf` |
| 代表質問 PDF | `http://www.town.kushiro.lg.jp/gikai/gijiroku/{年}/teirei_{回数}/shitsumon_dai.pdf` |

### URL パターンの補足

- `{年}` は西暦（例: `2024`）
- `{回数}` は年内の開催回数（例: `1`, `2`, `3`, `4`）
- 定例会は URL 第2セグメントが `1`、臨時会は `2`（例: `/gijiroku/1/1/2024.html` = 第1回定例会、`/gijiroku/1/2/2024.html` = 第1回臨時会）
- 会議録 PDF のファイル名 `kaigi{連番}` は通し番号（例: `kaigi376.pdf`）

## 会議種別

### 定例会

年4回開催（第1回〜第4回）。各定例会ページには以下の資料がある:

- 日程内容（PDF）
- 審議結果（PDF）
- 代表質問（PDF、第1回定例会のみ）
- 一般質問（PDF）
- 会議録（HTML ページ → 各日の PDF リンク）

### 臨時会

年に複数回開催（回数は年による）。各臨時会ページには以下がある:

- 審議結果（PDF）
- 会議録（HTML ページ → 各日の PDF リンク）

## ページ構造

### 年度別一覧ページ（`gijiroku/{年}.html`）

- `<h2>` に「令和X年に開催した定例会・臨時会」
- `<h3>` で「定例会」「臨時会」を分割
- 各会議はテーブル形式で表示
- テーブルのヘッダ: 「会期」「日程内容」「審議結果」「代表質問」「一般質問」「会議録」
- 臨時会のテーブルヘッダ: 「会期」「審議結果」「会議録」
- 会議録列に詳細ページへのリンク（`<a href="../../gikai/gijiroku/{回数}/{種別}/{年}.html">`）
- ページネーションなし（1ページに全会議を表示）

### 会議録詳細ページ（`gijiroku/{回数}/{種別}/{年}.html`）

- `<h2>` に「令和X年第Y回定例会 会議録」または「令和X年第Y回臨時会 会議録」
- テーブル形式で各日の会議録を一覧表示
- ヘッダ: 「掲載内容」（幅80%）、「会議録閲覧」（幅20%）
- 掲載内容の例: `06.03.04（月）町政執行方針、教育行政執行方針、代表質問（3名）ほか`
- 会議録閲覧列に PDF リンク（`<a href="../../../../gikai/kaigiroku/{年}/teirei_{回数}/kaigi{連番}.pdf">`）
- ページネーションなし

## 会議録 PDF の構造

### メタ情報（1ページ目冒頭）

```
令和６年第１回釧路町議会定例会会議録（第１号）

                令和６年３月４日（月曜日）　午前１０時００分開議
```

- タイトル: `令和X年第Y回釧路町議会{定例会/臨時会}会議録（第Z号）`
- 開催日: `令和X年M月D日（曜日）　午前/午後HH時MM分開議`
- 全角数字使用

### 目次セクション

```
○議事日程
○会議に付した事件
○出席議員（16名）
○欠席議員（0名）
○地方自治法第121条第1項により説明のため出席した者の職氏名
○職務のため議場に出席した事務局職員の職氏名
```

### 発言の構造

発言者パターン:

```
○議長（橋口春樹）
○議会運営委員長（中田磨）
○町長（小松茂）
○副町長（佐々木喜代孝）
○教育長（辻川尚志）
○6番（中田磨）
```

- `○` で始まり、役職または議席番号 + 括弧内に氏名
- 議事日程の見出しは `◎日程第N　{議題名}` の形式
- 開会・閉会宣告は `◎開会・開議宣告` `◎散会宣告` 等

### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（橋口春樹） → role="議長", name="橋口春樹"
// 例: ○6番（中田磨） → role="6番", name="中田磨"

// 議事日程の見出し
const agendaPattern = /^◎日程第(\d+)\s+(.+)/;
// 例: ◎日程第1　会議録署名議員指名

// タイトルの抽出
const titlePattern = /(?:令和|平成)([\d０-９]+)年第([\d０-９]+)回釧路町議会(.+?)会議録/;

// 開催日の抽出
const datePattern = /(?:令和|平成)([\d０-９]+)年([\d０-９]+)月([\d０-９]+)日/;
```

## スクレイピング戦略

### Step 1: 年度別一覧ページから会議録詳細ページ URL を収集

年度別一覧ページ `gijiroku/{年}.html`（2013〜現在）をクロールし、会議録詳細ページへのリンクを抽出する。

**収集方法:**

1. `gijiroku/{年}.html`（年 = 2013〜現在）を順次取得
2. 各ページから `gijiroku/{回数}/{種別}/{年}.html` へのリンクを Cheerio で抽出
3. `<h3>` タグで定例会（種別 = `1`）と臨時会（種別 = `2`）を判別

### Step 2: 会議録詳細ページから PDF URL を収集

各詳細ページから PDF リンクを抽出する。

**収集方法:**

1. 各詳細ページの `<table>` から PDF リンクを抽出
2. `<td>` の掲載内容テキストから日付・議題を取得
3. PDF URL パターン: `kaigiroku/{年}/teirei_{回数}/kaigi{連番}.pdf`

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出してパースする。

**抽出方法:**

1. PDF をダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 2段組レイアウトのため、テキスト抽出時にカラム順序の考慮が必要

### Step 4: 会議録テキストのパース

抽出したテキストから発言者・発言内容を構造化する。

**パース手順:**

1. タイトル行から会議名・号数を抽出
2. 日付行から開催日を抽出
3. `○` で始まる行を発言者として検出
4. 発言者行から役職と氏名を正規表現で抽出
5. 次の発言者行までのテキストを発言内容として取得

---

## 注意事項

- 会議録は HTML ではなく PDF で公開されている。PDF のテキスト抽出が必要
- PDF は2段組レイアウト（左右カラム）のため、テキスト抽出時にカラム順序が崩れる可能性がある
- `http://www.town.kushiro.lg.jp/gikai/gijiroku/` への直接アクセスは 403 エラーを返す（ディレクトリ一覧は非公開）
- User-Agent を設定しないとアクセスが拒否される場合がある
- 全角数字が多用されている（議席番号、日付等）
- 会議録 PDF の連番（`kaigi{連番}`）は年をまたいで通し番号になっている

---

## 推奨アプローチ

1. **3段階クロール**: 年度一覧 → 会議録詳細 → PDF の順で段階的にクロール
2. **PDF テキスト抽出**: `pdf-parse` または `pdfjs-dist` を使用。2段組レイアウトの処理が課題
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: PDF ファイル名の連番（`kaigi{N}`）を利用し、前回取得済みの最大連番以降のみを取得する差分更新が可能
5. **掲載内容テキストの活用**: 詳細ページの掲載内容テキスト（例: `06.03.04（月）町政執行方針...`）から日付と概要を事前取得し、PDF パースの補助情報として利用
