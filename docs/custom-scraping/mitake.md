# 御嵩町議会 カスタムスクレイピング方針

## 概要

- サイト: https://mitake-gikai.com/side/minutes
- 分類: 独自ドメインの公式 HP による PDF 公開（年度・会議種別ごとのリスト表示、専用検索システムなし）
- 文字コード: UTF-8
- 会議録形式: **すべて PDF ファイル**（HTML テキスト形式の会議録なし）
- 対象期間: 令和5年（2023年）〜 現在（令和5年2月9日が最古）

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧ページ（単一） | `https://mitake-gikai.com/side/minutes` |
| PDF ダウンロード | `https://mitake-gikai.com/download_file/view/{ファイルID}/296` |

### 特記

- 会議録一覧は **単一ページ** に全年度・全会議種別がまとめて掲載される（ページネーションなし）
- PDF へのリンクは CMS の `download_file/view/{ID}/296` 形式で統一されており、直接 PDF バイナリへリダイレクト（HTTP 303）される
- `/296` はページ ID（固定値）であり、ファイルの識別子は `{ファイルID}` 部分のみ

---

## 一覧ページの HTML 構造

会議録一覧ページは以下の階層構造で構成される。

```html
<h3>令和6年（2024年）</h3>

<!-- 会議種別（strong または太字テキストで区別） -->
◆第2回 定例会

<strong>●令和6年6月12日</strong>
<a href="https://mitake-gikai.com/download_file/view/445/296">→ダウンロード</a>

<strong>●令和6年6月19日</strong>
<a href="https://mitake-gikai.com/download_file/view/446/296">→ダウンロード</a>

<h3>令和5年（2023年）</h3>
...
```

- 年度は `<h3>` タグで区切られる
- 会議種別（第N回定例会・臨時会・常任委員会）は `◆` 記号付きのテキストで示される（セマンティックな見出しタグではなく、段落内テキスト）
- 各会議録は `<strong>●日付</strong>` と `<a>→ダウンロード</a>` の組み合わせで表示される

## スクレイピング戦略

### Step 1: 一覧ページから PDF リンクを収集

`https://mitake-gikai.com/side/minutes` を取得し、`download_file/view/` を含むすべてのリンクを Cheerio で抽出する。

```typescript
// PDF ダウンロードリンクの抽出
const pdfLinks = $('a[href*="download_file/view/"]').map((_, el) => ({
  url: $(el).attr("href"),
  // 周辺のstrongタグから日付を取得
  dateText: $(el).prev("strong").text().trim(),
}));
```

各リンクの前後の DOM（`<h3>` 見出し、`◆` テキスト、`<strong>` の日付）を解析して以下のメタ情報を構築する。

- 年度: 直近の `<h3>` テキストから抽出（例: `令和6年（2024年）`）
- 会議種別: `◆` で始まるテキストノードから抽出（例: `◆第2回 定例会`）
- 開催日: リンク直前の `<strong>` テキストから抽出（例: `●令和6年6月12日`）

### Step 2: PDF をダウンロード

`download_file/view/{ID}/296` へ GET リクエストを送信すると HTTP 303 リダイレクトで実際の PDF URL へ転送される。リダイレクト先の URL まで追従して PDF バイナリを取得する。

```typescript
// リダイレクトを追従して PDF を取得
const response = await fetch(url, { redirect: "follow" });
const pdfBuffer = await response.arrayBuffer();
```

### Step 3: PDF からテキストを抽出

**この自治体の会議録はすべて PDF 形式**のため、PDF からのテキスト抽出が必要。

- PDF テキスト抽出ライブラリ（例: `pdf-parse`, `pdfjs-dist`）を使用する
- スキャン PDF（画像 PDF）の場合は OCR が必要になる可能性がある

### Step 4: PDF テキストのパース

PDF から抽出したテキストの構造は一般的な議会会議録の形式に準じていると想定される。発言者パターンの例：

```
○議長（氏名）
○町長（氏名）
○N番（氏名）
```

---

## 注意事項

- **PDF のみ**: HTML テキスト形式の会議録は存在しない。テキスト検索・発言者抽出には PDF パースが必須
- **単一ページ構成**: 全年度・全会議録が1ページにまとめられているため、ページネーション処理は不要
- **ダウンロードリダイレクト**: `download_file/view/{ID}/296` は HTTP 303 で実際の PDF へリダイレクトする。リダイレクト追従を有効にすること
- **新規追加の検出**: 一覧ページを定期クロールし、既知のファイルID 以外の新規リンクを差分として処理する
- **「準備中」セクションあり**: ページ内に一部「以下準備中」の記載がある。ダウンロードリンクが存在するもののみを処理対象とする
- **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **一覧ページから動的に PDF リンクを収集**: 固定リストではなく、毎回一覧ページをクロールして新規リンクを検出する
2. **ファイルID を識別子として活用**: `download_file/view/{ID}/296` の `{ID}` を一意識別子として使い、取得済み PDF の差分更新を行う
3. **DOM の前後関係で会議種別を特定**: `<h3>`（年度）→ `◆` テキスト（会議種別）→ `<strong>`（日付）の順序を Cheerio で走査してメタ情報を構築する
4. **PDF テキスト抽出**: `pdf-parse` 等のライブラリで本文テキストを取得し、議会会議録の標準的な発言者パターンでパースする
