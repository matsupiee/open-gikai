# むかわ町議会 カスタムスクレイピング方針

## 概要

- サイト: http://www.town.mukawa.lg.jp/2872.htm
- 分類: 自治体 CMS（i-SITE PORTAL）による単一ページ + PDF 公開
- 文字コード: UTF-8
- 特記: 会議録は全て PDF 形式で公開。1ページに全年度分のテーブルが掲載されており、各セルから PDF ファイルへ直接リンクされている。ページネーションなし。

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録一覧（単一ページ） | `http://www.town.mukawa.lg.jp/2872.htm` |
| 会議録 PDF | `http://www.town.mukawa.lg.jp/secure/8403/{ファイル名}.pdf` |

### PDF ファイル名の命名規則

ファイル名には複数の命名パターンが混在している:

| パターン | 例 | 対象年度 |
| --- | --- | --- |
| `{YYYYMMDD}-{元号略}{年}-{回}{種別}.pdf` | `20250310-R07-1teirei.pdf` | 令和5年〜令和7年 |
| `{YYYYMMDD}-{DD}-{元号略}{年}-{回}{種別}.pdf` | `20230309-14-R05-1teirei.pdf` | 令和5年（複数日開催） |
| `{元号略}{YYMMDD}-{元号略}{年}.{回}{種別略}.pdf` | `R040307-R04.1tei.pdf` | 令和3年後半〜令和4年前半 |
| `{YYYYMMDD}-{元号略}{年}-{回}{種別}.pdf` | `20210308-R03-1teirei.pdf` | 令和2年〜令和3年前半 |
| `{YYMMDD}-{元号略}{年}-{回}{種別}.pdf` | `200327-R02-2rinnji.pdf` | 令和2年（一部） |
| `{元号略}{YYMMDD}-{元号略}{年}-{回}{種別}.pdf` | `H300306-H30-1teirei.pdf` | 平成28年後半〜平成30年 |
| `{元号略}{年}-{回}{種別}.pdf` | `H27-1teirei.pdf` | 平成27年〜平成28年前半 |

種別略称:
- `teirei` / `tei` = 定例会
- `rinji` / `rinnji` / `rin` = 臨時会（表記揺れあり）

元号略称:
- `H` = 平成
- `R` = 令和

## ページ構造

### HTML 構造

- 単一の `<table>` 要素で全年度の会議録を一覧表示
- テーブルヘッダ: `開催年` + 会議名リンク列（7列）
- 各行は1年度分で、年度ラベルのセル（`rowspan="2"` の場合もあり）+ 会議リンクセルで構成
- リンクがあるセルは `background-color: #fdeada`（オレンジ系背景色）
- 年度ラベルセルは `background-color: #c6d9f0`（水色背景）
- PDF リンクは `<a href="/secure/8403/{ファイル名}.pdf">` 形式
- 同一セル内に同じ PDF への複数リンクが含まれる場合がある（リンクテキストが分割されている）

### 掲載範囲

| 年度 | 和暦 |
| --- | --- |
| 最古 | 平成27年（2015年） |
| 最新 | 令和7年（2025年） |

### 会議の種類

- **定例会**: 年4回（第1回〜第4回）
- **臨時会**: 年によって回数が異なる（0〜8回程度）

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `http://www.town.mukawa.lg.jp/2872.htm` から全 PDF リンクを抽出する。

- ページネーションなし（単一ページに全件掲載）
- `<a href="/secure/8403/...pdf">` を全て取得
- 同一 PDF への重複リンクを除去（`Set` で一意化）

**収集方法:**

1. 一覧ページの HTML を取得
2. Cheerio で `a[href$=".pdf"]` を全て抽出
3. `href` 属性を一意化してリスト化
4. 各リンクの周辺テキストから会議名・開催日を抽出

### Step 2: メタ情報の抽出

PDF リンクの周辺テキストとファイル名から以下を抽出:

- **年度**: テーブル行の年度ラベルセルのテキスト（例: `令和7年`）
- **会議名**: リンクテキスト（例: `第1回定例会`、`第2回臨時会`）
- **開催日**: リンクテキストまたは隣接テキストの括弧内（例: `(3/10-14)`、`(1月28日)`）
- **ファイル名からのフォールバック**: リンクテキストが不完全な場合、ファイル名の日付部分（`YYYYMMDD`）から開催日を推定

```typescript
// ファイル名から日付を抽出する正規表現
const dateFromFilename = /^(\d{8})-/; // 20250310-R07-1teirei.pdf → 20250310
const dateFromFilenameShort = /^(\d{6})-/; // 200327-R02-2rinnji.pdf → 200327
const dateFromFilenameEra = /^[HR](\d{6})-/; // R040307-R04.1tei.pdf → 040307

// 会議種別の判定
const isTeirei = /teirei|tei/i; // 定例会
const isRinji = /rinji|rinnji|rin/i; // 臨時会
```

### Step 3: PDF のダウンロードとテキスト抽出

1. 収集した PDF URL から PDF ファイルをダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 抽出したテキストを会議録本文として保存

---

## 注意事項

- PDF ファイル名の命名規則が年度によって異なるため、ファイル名パースに複数パターンの対応が必要
- 臨時会の表記揺れ（`rinji` / `rinnji` / `rin`）に注意
- 同一セル内に同じ PDF への `<a>` タグが複数存在する場合がある（テキスト分割のため）。重複除去が必須
- 一部リンクに誤ったファイル名が含まれる可能性がある（例: 令和6年のセルに `R02` のファイル名）
- `個人情報保護の観点から一部原本と違うものがある` との記載あり
- 掲載年以前の議事録は議会事務局への問い合わせが必要（サイト上では取得不可）

---

## 推奨アプローチ

1. **単一ページ取得**: ページネーションがないため、1回の HTTP リクエストで全リンクを収集可能
2. **PDF 直接ダウンロード**: HTML の会議録本文は存在せず、PDF からのテキスト抽出が必須
3. **ファイル名ベースの重複検知**: PDF ファイル名をキーとして差分更新を行う
4. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2秒）を設ける
5. **テーブル構造の行単位パース**: 年度ラベルと PDF リンクを行単位で対応付けることで、正確なメタ情報を取得する
