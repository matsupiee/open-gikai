## ページ構造

### 会議録トップページ

年度別ページへのリンクが `<ul>` リスト形式で並んでいる。各リンクの `href` にドキュメント ID が含まれる。

```html
<ul>
  <li><a href="/docs/2025052200015/">令和7年議会の会議録</a></li>
  <li><a href="/docs/2024060500025/">令和6年議会の会議録</a></li>
  ...
</ul>
```

### 年度別ページ

テーブル形式で会議ごとに PDF リンクが掲載されている。

```html
<!-- 定例会: <th> に会議名 -->
<th colspan="3" scope="col" style="text-align: left;">令和7年第4回定例会(12月議会)</th>

<!-- 臨時会: <p> に会議名（一部年度では <th> の場合もあり） -->
<p>令和7年第5回臨時会(12月22日)</p>

<!-- PDF リンク -->
<td><a class="iconFile iconPdf" href="file_contents/teirei7_4mokuji.pdf">会議録目次[PDF：468KB]</a></td>
<td><a class="iconFile iconPdf" href="file_contents/teirei7_1201.pdf">第1号【12月1日】[PDF：879KB]</a></td>
```

## 会議の種類

本会議のみ公開（委員会の会議録は非公開）。

- **定例会**: 年4回（3月、6月、9月、12月）。複数日にわたり第1号〜第4号程度
- **臨時会**: 随時開催。通常1日（第1号のみ）

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録トップページ `https://www.town.tamamura.lg.jp/docs/2019052900044/` から年度別ページへのリンクを抽出する。

- `<a href="/docs/{ドキュメントID}/">` を Cheerio で取得
- ドキュメント ID は固定リストとしてハードコードも可（上記一覧参照）

### Step 2: 各年度ページから PDF URL の収集

各年度ページのテーブルから PDF リンクを抽出する。

**収集方法:**

1. `a.iconFile.iconPdf` セレクタで全 PDF リンクを取得
2. リンクテキストから会議情報をパース:
   - 「会議録目次」→ 目次 PDF（スキップ可）
   - 「第N号【M月D日】」→ 会議録本文 PDF
3. 直前の `<th>` または `<p>` から会議名を取得:
   - 「令和7年第4回定例会(12月議会)」
   - 「令和7年第5回臨時会(12月22日)」

**パース用正規表現（案）:**

```typescript
// 会議名の抽出
const meetingPattern = /(?:令和|平成)\d+年第\d+回(定例会|臨時会)[(\[（【](.+?)[)\]）】]/;

// PDF リンクテキストから日付を抽出
const datePattern = /第(\d+)号【(\d+)月(\d+)日】/;

// PDF の href から相対パスを取得
const pdfHrefPattern = /file_contents\/(.+\.pdf)/;
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF を取得後、テキストを抽出して会議録データを構築する。

- PDF からのテキスト抽出には `pdf-parse` 等のライブラリを使用
- PDF 内の構造は議会事務局のフォーマットに依存するため、テキスト抽出後の整形が必要

---

## 注意事項

- **検索機能なし**: 全量取得はページを巡回して PDF を収集する方式のみ
- **PDF 形式のみ**: HTML での会議録公開はなく、全て PDF ダウンロードが必要
- **ファイル名パターンの変遷**: 旧年度と近年でファイル名規則が異なるため、href 属性をそのまま使用するのが安全
- **会議種別の表記揺れ**: 定例会は `(12月議会)` 形式、臨時会は `(12月22日)` 形式と括弧の中身が異なる。また古い年度では角括弧 `[12月議会]` が使われている
- **本会議のみ**: 委員会の会議録は公開されていない

---

## 推奨アプローチ

1. **固定リスト + 巡回**: 年度別ページの URL はトップページから動的に取得するか、上記の固定リストを使用
2. **PDF リンクの一括収集**: 各年度ページから `a.iconPdf` で PDF URL を全て収集し、会議メタ情報と紐づける
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度別ページの最終更新日（`updatedAt`）を確認し、更新があった年度のみ再取得する
5. **PDF テキスト抽出**: PDF の構造が不定形のため、テキスト抽出後に発言者パターン等の正規表現で構造化する
