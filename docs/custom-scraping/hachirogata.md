## 概要

- サイト: https://www.town.hachirogata.akita.jp/gikai/1001560/index.html
- 分類: 自治体 CMS による PDF 公開（既存アダプターでは対応不可）
- 文字コード: UTF-8

### PDF URL のパターン

PDF ファイルの URL は会議 ID をもとに構成される。会議 ID を3桁ずつ区切ったパスが使われる。

例:
- 会議 ID `1004139` → `/_res/projects/default_project/_page_/001/004/139/{ファイル名}.pdf`
- 会議 ID `1001968` → `/_res/projects/default_project/_page_/001/001/968/{ファイル名}.pdf`

PDF ファイル名は統一されておらず、会議ごとに異なる（例: `giji12.pdf`, `r603giji.pdf`, `1001968_001.pdf`）。

## スクレイピング戦略

### Step 1: 年別一覧ページから会議ページ ID を収集

議事録トップページ `/gikai/1001560/index.html` から各年の一覧ページ URL を取得し、年別一覧ページから各会議の個別ページ URL（会議 ID）を抽出する。

**収集方法:**

1. トップページのリンクから年 ID の一覧を取得
2. 各年別一覧ページ `/gikai/1001560/{年ID}/index.html` にアクセス
3. ページ内のリンクから会議 ID を正規表現 or Cheerio で抽出

```typescript
// 会議ページリンクの抽出
// リンクパターン: /gikai/1001560/{年ID}/{会議ID}.html
const meetingLinkPattern = /\/gikai\/1001560\/\d+\/(\d+)\.html/;
```

### Step 2: 会議ページから PDF URL を取得

各会議ページ `/gikai/1001560/{年ID}/{会議ID}.html` にアクセスし、添付ファイルセクションから PDF のリンクを抽出する。

- 各会議ページには 1 つの PDF ファイルが添付されている
- PDF リンクは `_res/projects/default_project/_page_/` 以下の相対パスで記述

```typescript
// PDF リンクの抽出
const pdfLinkPattern = /\/_res\/projects\/default_project\/_page_\/[\d/]+\/[^"]+\.pdf/;
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、PDF パーサー（pdf-parse 等）でテキストを抽出する。

- PDF 内のテキストから開催日、会議名、発言者、発言内容をパースする
- PDF のレイアウトは議事録形式（発言者名と発言内容の交互記述）が想定される

### Step 4: 会議録のパース

#### メタ情報

会議ページのタイトルおよび PDF 内のヘッダーから以下を抽出:

- 開催年: 年 ID とページタイトルから特定（例: 「令和6年12月定例会」）
- 会議種別: 定例会 or 臨時会（ページタイトルから判別）
- 開催月: タイトルから抽出

#### 発言の構造

PDF 内部の発言パターンは PDF テキスト抽出後に確認が必要。一般的な町議会議事録では以下のようなパターンが想定される:

```
○議長（氏名）
○○番（氏名）
○町長（氏名）
```

---

## 注意事項

- 会議録は PDF 形式のため、HTML パースではなく PDF テキスト抽出が必要
- PDF ファイル名に統一規則がないため、会議ページから都度 PDF URL を取得する必要がある
- 平成22年にサイトリニューアルが行われており、それ以前のデータ構造が異なる可能性がある（ただし公開範囲は平成24年以降）
- PDF 内のテキスト抽出品質はファイルの作成方法（スキャン画像 vs テキスト埋め込み）に依存する

---

## 推奨アプローチ

1. **2段階クロール**: まず年別一覧 → 会議ページ → PDF URL の順に全 PDF の URL リストを作成し、その後 PDF をダウンロード
2. **PDF テキスト抽出**: `pdf-parse` または同等のライブラリで PDF からテキストを抽出
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年 ID・会議 ID の組み合わせで管理し、新規追加分のみをダウンロードする
5. **PDF 品質チェック**: テキスト抽出が不可能な PDF（画像スキャン等）がないか事前に確認する
