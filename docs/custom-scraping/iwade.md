# 岩出市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.iwade.lg.jp/gikai/kaigiroku/
- 分類: 独自ウェブサイト上で PDF 形式の会議録を公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 会議録は PDF ファイルで提供。全体版と日程別・議員別の分割版がある

---

## URL 構造

| ページ | URL パターン |
| --- | --- |
| 会議録トップ | `https://www.city.iwade.lg.jp/gikai/kaigiroku/` |
| 定例会ページ | `https://www.city.iwade.lg.jp/gikai/kaigiroku/{ページID}.html` |
| PDF ファイル | `https://www.city.iwade.lg.jp/gikai/kaigiroku/files/{ファイル名}.pdf` |

### ページ ID のパターン

ページ ID（各定例会 HTML のファイル名）は年度によって命名規則が異なる:

| 時期 | パターン例 | 備考 |
| --- | --- | --- |
| 令和7年〜 | `R7-1.html`, `R7-1-rinji.html` | `R{年}-{回}` 形式 |
| 令和4〜6年 | `2024-0612-R6-1.html`, `2022-0916-r4-02.html` | 日付プレフィックス付き |
| 令和2〜3年 | `R3-4.html`, `R2-rinji2.html` | `R{年}-{回}` 形式 |
| 平成26〜31年 | 同様の不統一パターン | 年度ごとに異なる |

**注意:** ページ ID に一貫した命名規則がないため、トップページからリンクを辿って各定例会ページを発見する必要がある。

---

## トップページの構造

トップページ（`/gikai/kaigiroku/`）は全年度分の会議録リンクを1ページに一覧表示している。

- ページネーション: なし（全件を1ページに表示）
- 検索機能: なし
- 年号ごとにセクション分けされたリスト構造
- 各リンクのテキストに「第N回定例会（M月議会）」「第N回臨時会」等の会議種別が含まれる
- 対象範囲: 平成26年（2014年）〜令和7年（2025年）

---

## 定例会ページの構造

各定例会ページは以下のセクションで構成される:

### 1. 会議録（全体分）

定例会全体を1つにまとめた PDF ファイル。

### 2. 会議録（日程別）

各開催日ごとの PDF ファイル。通常 3〜4 日分。

| 例（令和7年第1回定例会） | 日付 | ファイル名 |
| --- | --- | --- |
| 第1日 | 2月28日 | `R7-1-1R070228.pdf` |
| 第2日 | 3月6日 | `R7-1-2R070306.2.pdf` |
| 第3日 | 3月18日 | `R7-1-3R070318.pdf` |
| 第4日 | 3月21日 | `R7-1-4R070321.2.pdf` |

### 3. 一般質問通告一覧

一般質問の通告内容をまとめた PDF。

### 4. 会議録（議員別一般質問）

議員ごとの一般質問を個別 PDF で提供。

---

## PDF ファイル名パターン

PDF ファイル名にも一貫した命名規則がなく、年度によって異なる:

- `R7-1zzentai.2.pdf`（全体版）
- `R7-1-1R070228.pdf`（日程別: 年号 + 日付）
- `R7-1-1fukouka.pdf`（議員別: 議員名ローマ字）
- `R6-4zentai.pdf`（全体版、別パターン）
- `1R6-4fukuoka.pdf`（議員別、番号プレフィックス）

---

## スクレイピング戦略

### Step 1: 定例会ページの URL 収集

トップページ（`/gikai/kaigiroku/`）から全定例会ページへのリンクを収集する。

**収集方法:**

1. トップページの HTML を取得
2. `<a>` タグから `.html` で終わるリンクを抽出（相対パス）
3. ベース URL `https://www.city.iwade.lg.jp/gikai/kaigiroku/` と結合して絶対 URL を生成
4. リンクテキストから会議名（「第N回定例会」「第N回臨時会」）と年度を抽出

### Step 2: 各定例会ページから PDF リンクの収集

各定例会ページの HTML を取得し、PDF ファイルへのリンクを抽出する。

**収集方法:**

1. 各定例会ページの HTML を取得
2. `<a>` タグから `.pdf` で終わるリンクを抽出
3. リンクテキストから PDF の種別（全体版 / 日程別 / 議員別）を判定
4. ベース URL と結合して絶対 URL を生成

**種別判定のヒューリスティック:**

- 「全体分」を含む → 全体版
- 「第N日」「N月N日」を含む → 日程別
- 「一般質問通告」を含む → 一般質問通告一覧
- 議員名を含む → 議員別一般質問

### Step 3: PDF のダウンロードとテキスト抽出

PDF ファイルをダウンロードし、テキストを抽出する。

**推奨ライブラリ:** `pdf-parse` または `pdfjs-dist`

**抽出対象:** 全体版 PDF を優先的に取得する（日程別・議員別は全体版の分割であるため）。

### Step 4: テキストのパース

PDF から抽出したテキストを構造化データに変換する。

#### メタ情報の抽出

PDF の冒頭やファイル名から以下を推定:

- 開催日: 日程別 PDF のファイル名に日付が含まれる（例: `R070228` → 令和7年2月28日）
- 会議名: 定例会ページのタイトルから取得（例: 「令和7年第1回定例会（3月議会）」）

#### 発言の構造

PDF 内のテキスト構造は PDF のレイアウトに依存するため、実際の PDF を取得して分析する必要がある。一般的な議会会議録 PDF では以下のパターンが多い:

- 発言者名が行頭に記載され、発言内容が続く
- 議長の発言で議事進行が区切られる

---

## 注意事項

- PDF ファイルのため、テキスト抽出の精度は PDF の作成方法に依存する（画像 PDF の場合は OCR が必要）
- ファイル名・ページ ID に一貫した命名規則がないため、トップページからのリンク辿りが必須
- 臨時会は不定期に開催されるため、トップページのリンクを定期的にチェックする必要がある
- PDF のファイルサイズは 150KB〜1MB 程度

---

## 推奨アプローチ

1. **トップページ起点の 2 段階クロール**: トップページ → 定例会ページ → PDF リンクの順に辿る。URL パターンの不統一があるため、直接 URL を推測するのではなくリンク辿りで対応する
2. **全体版 PDF を優先取得**: 日程別・議員別は全体版の分割であるため、全体版のみ取得すれば重複を避けられる
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: トップページの定例会リンク一覧と取得済みリストを比較し、新規分のみ取得する
5. **PDF テキスト抽出の検証**: 初回は実際の PDF を数件ダウンロードして、テキスト抽出が正常に行えるか（画像 PDF でないか）を確認する
