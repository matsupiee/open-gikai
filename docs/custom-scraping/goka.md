## 会議録 PDF の構造

### PDF ファイルの構成

各会議ページには以下の PDF が添付されている:

1. **審議結果** PDF（1ファイル）
2. **会議録** PDF（目次 + 日程ごとに分割）
   - 目次（`_doc_88_0.pdf`）
   - 第1号（`_doc_88_1.pdf`）
   - 第2号（`_doc_88_2.pdf`）
   - 第3号（`_doc_88_3.pdf`）※ 会議日程数により増減

### PDF URL パターン

```
https://www.town.goka.lg.jp/data/doc/{タイムスタンプ}_doc_88_{連番}.pdf
```

- `{タイムスタンプ}`: Unix タイムスタンプ（10桁）
- `doc_88`: 会議録カテゴリの固定 ID
- `{連番}`: 0 から始まる連番（0=目次 or 審議結果、1〜=第N号）

#### 発言の構造

発言者パターン:

```
○議長（鈴木喜一郎君）
○町長（染谷森雄君）
○6番（新井　庫君）
○都市建設課長（大橋　勝君）
○教育長（森田恵美子君）
```

- `○` で始まり、役職または議席番号 + 括弧内に氏名（「君」付き）
- 議事進行の区切り: `◎` で始まるセクション見出し（例: `◎開会の宣告及び議長挨拶`、`◎一般質問`）
- 登壇表記: `〔6番　新井　庫君　発言席へ移動〕`
- 発言者の紹介: `◇ 新 井　庫 君`（質問者の見出し）
- 発言内容は発言者行の後に続くテキスト

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（鈴木喜一郎君） → role="議長", name="鈴木喜一郎君"
// 例: ○6番（新井　庫君） → role="6番", name="新井　庫君"

// セクション見出しの検出
const sectionPattern = /^[◎◉](.+)/;
// 例: ◎一般質問 → section="一般質問"

// 質問者見出しの検出
const questionerPattern = /^[◇◆]\s*(.+)/;
// 例: ◇ 新 井　庫 君 → questioner="新井庫君"

// 登壇・移動の検出
const movementPattern = /^〔(.+?)〕/;
// 例: 〔6番　新井　庫君　発言席へ移動〕

// 開催日の抽出（タイトルから）
const datePattern = /(?:令和|平成)\s*(\S+?)年\s*(\S+?)月\s*(\S+?)日/;

// 会議名の抽出（タイトルから）
const meetingPattern = /((?:令和|平成)\S+年第\S+回五霞町議会(?:定例会|臨時会))会議録/;
```

## スクレイピング戦略

### Step 1: 会議ページ URL の収集

審議結果・会議録一覧ページから各会議ページへのリンクを収集する。

1. 一覧ページ `shingikeeka-kaigiroku/` を取得
2. 年度別リンク（`reiwa8nen/`、`gikaijimukyoku/` など）を抽出
3. 各年度ページから個別会議ページ（`page{ID}.html`）のリンクを抽出
4. ページネーションなし（全件が1ページに表示される）

**注意:** 一覧ページ自体に全年度・全会議のリンクが含まれているため、年度別ページを個別にクロールする必要はない可能性がある。

### Step 2: 会議録 PDF URL の収集

各会議ページから PDF ダウンロードリンクを収集する。

1. 各 `page{ID}.html` を取得
2. `data/doc/{タイムスタンプ}_doc_88_{連番}.pdf` パターンのリンクを抽出
3. 審議結果 PDF と会議録 PDF を区別する（リンクテキストから判定）

### Step 3: PDF のダウンロードとテキスト抽出

1. 会議録 PDF をダウンロード（目次 PDF は不要）
2. `pdftotext` 等のツールで PDF からテキストを抽出
3. Word 由来の PDF のため、テキスト抽出の精度は比較的良好

### Step 4: 会議録のパース

抽出したテキストから以下を構造化する:

1. **メタ情報の抽出**
   - 会議名: PDF 冒頭のタイトル行から
   - 開催日: タイトル直下の日付行から
   - 議事日程番号: タイトルの「（第N号）」から

2. **出席情報の抽出**
   - 出席議員リスト
   - 欠席議員リスト
   - 説明員リスト
   - 事務局職員リスト

3. **発言の抽出**
   - `○` で始まる行を発言者の区切りとして分割
   - 発言者の役職と氏名を正規表現で抽出
   - 発言者行以降、次の発言者行までを発言内容として取得

## 注意事項

- 会議録は「個人・団体等の権利及び利益並びに町の公益等を保護する観点」から一部発言が「○○○」と表記されている場合がある
- 会議録の無断転載は禁止されている旨の記載あり
- PDF は Word から直接生成されているため、テキストレイヤーが含まれており OCR 不要
- 議員数は10名（小規模町議会）
- 定例会は年4回、臨時会は不定期に開催
- データ範囲は平成31年（2019年）以降が確認できる

---

## 推奨アプローチ

1. **一覧ページから全量収集**: 審議結果・会議録一覧ページを起点に、全年度・全会議のページ URL を収集
2. **PDF ダウンロード**: 各会議ページから会議録 PDF のリンクを抽出してダウンロード
3. **pdftotext でテキスト変換**: PDF からプレーンテキストを抽出（Word 由来のため高精度）
4. **正規表現でパース**: `○` パターンで発言者を分割し、メタ情報と合わせて構造化
5. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
6. **差分更新**: ページ ID（`page{ID}.html`）の数値が単調増加のため、前回取得済みの最大 ID 以降のみを取得する差分更新が可能
