## PDF ファイル名パターン

### 近年（令和以降）の命名規則

`r{和暦年2桁}-{月2桁}{会議種別}-{連番2桁}-{月日4桁}.pdf`

| 会議種別コード | 意味 |
| --- | --- |
| `t` | 定例会議 |
| `r` | 随時会議 |
| `y` | 予算特別委員会 |
| `k` | 決算特別委員会 |

**例:**

- `r06-12t-01-1202.pdf` → 令和6年12月定例会議 第1回（12月2日）
- `r06-12r-1223.pdf` → 令和6年12月随時会議（12月23日）
- `r06-y-02-0305.pdf` → 令和6年予算特別委員会 第2回（3月5日）
- `r06-k-03-0910.pdf` → 令和6年決算特別委員会 第3回（9月10日）

### 古い年度（平成20年頃）

数値連番のファイル名（例: `709.pdf`, `710.pdf`）。規則性が異なるため、リンク先から直接取得する必要がある。

## HTML 構造

### トップページ（年度一覧）

```html
<article id="contents">
  <!-- 年度ごとのリンクがリスト表示 -->
  <a href="https://www.town.taiwa.miyagi.jp/choseijoho/gikai/kaigiroku/{ページID}.html">
    {年度名}大和町議会会議録
  </a>
</article>
```

### 年度別ページ

```html
<article id="contents">
  <h1 class="title">令和6年大和町議会会議録</h1>

  <!-- 会議種別ごとの h2 セクション -->
  <h2><span class="bg"><span class="bg2"><span class="bg3">令和6年12月定例会議</span></span></span></h2>

  <!-- PDF リンク -->
  <p class="file-link-item">
    <a class="pdf" href="//www.town.taiwa.miyagi.jp/material/files/group/21/r06-12t-01-1202.pdf">
      12月2日分（再開、諸般の報告、行政報告、一般質問） (PDFファイル: 1.2MB)
    </a>
  </p>
</article>
```

**ポイント:**

- 会議種別は `<h2>` の中の `<span class="bg3">` テキストから抽出可能
- PDF リンクは `<p class="file-link-item">` 内の `<a class="pdf">` 要素
- リンクテキストに開催日と議事内容が含まれる
- ファイルサイズもリンクテキストに含まれる

## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

トップページ `kaigiroku/index.html` から全年度ページへのリンクを取得する。

- `<a>` 要素の `href` 属性から `/choseijoho/gikai/kaigiroku/{ID}.html` パターンを抽出
- 平成20年〜現在までの18年度分

### Step 2: PDF URL とメタ情報の収集

各年度ページから PDF リンクとメタ情報を抽出する。

**収集方法:**

1. `<h2>` 要素から会議種別を抽出（`令和X年{月}{会議種別}` パターン）
2. `<p class="file-link-item">` 内の `<a class="pdf">` から PDF URL を取得
3. リンクテキストから開催日と議事内容を抽出

**抽出用正規表現（案）:**

```typescript
// h2 から会議種別を抽出
const meetingTypePattern = /(令和|平成)\d+年(.+)$/;
// 例: "令和6年12月定例会議" → era="令和", rest="12月定例会議"

// リンクテキストから日付と内容を抽出
const linkTextPattern = /^(\d+)月(\d+)日分[【（]?(.+?)[）】]?\s*\(PDFファイル/;
// 例: "12月2日分（再開、諸般の報告、行政報告、一般質問） (PDFファイル: 1.2MB)"
//   → month=12, day=2, content="再開、諸般の報告、行政報告、一般質問"
```

### Step 3: PDF のダウンロードとテキスト抽出

PDF からテキストを抽出する。

- PDF パーサー（pdf-parse 等）を使用してテキスト化
- 発言者・発言内容の構造化は PDF 内のテキスト構造に依存するため、サンプル PDF を確認して個別対応が必要

---

## ページネーション

なし。トップページに全年度が一覧表示され、各年度ページにも全会議録が一覧表示される。

---

## 注意事項

- 会議録は HTML テキストではなく PDF で公開されているため、テキスト抽出に PDF パーサーが必要
- PDF のリンク URL はプロトコル相対（`//www.town.taiwa.miyagi.jp/...`）で記載されている。`https:` を付加して使用する
- 古い年度の PDF ファイル名は命名規則が異なる（数値連番）ため、ファイル名からメタ情報を推測できない。リンクテキストと `<h2>` 見出しからメタ情報を取得する
- 一部の PDF ファイル名が規則に従わないケースがある（例: `0405.pdf`）。リンク先から直接取得する方針が安全

---

## 推奨アプローチ

1. **2段階クロール**: トップページ → 年度ページ → PDF URL の順で収集
2. **メタ情報は HTML から取得**: PDF ファイル名ではなく、`<h2>` 見出しとリンクテキストから会議種別・開催日・議事内容を取得する
3. **PDF テキスト抽出**: pdf-parse 等のライブラリで PDF からテキストを抽出し、発言者・発言内容を構造化
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
5. **差分更新**: 年度ページの更新日（`<p class="update">` 要素）を確認し、変更があった年度のみ再取得する
