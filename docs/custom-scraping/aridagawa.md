## PDF ファイル命名規則

ファイルは `material/files/group/12/` 配下に格納されており、命名規則が統一されている。

形式: `R{年度}-{回数}-{日数}{会議種別}R{年月日}.pdf`

| 要素 | 説明 | 例 |
| --- | --- | --- |
| `R{年度}` | 令和年度 | `R7`（令和7年）、`R6`（令和6年） |
| `{回数}` | 第N回 | `1`〜`4`（定例会は年4回） |
| `{日数}` | 開催N日目 | `1`〜（複数日にわたる場合） |
| `{会議種別}` | 定例会 or 臨時会 | `teirei`（定例会）、`rinji`（臨時会） |
| `R{年月日}` | 実際の開催日（令和表記） | `R070603`（令和7年6月3日） |

令和7年の例:
- `R7-2-1teireiR070603.pdf` → 令和7年第2回定例会 第1日（6月3日）
- `R7-2-2teireiR070612.pdf` → 令和7年第2回定例会 第2日（6月12日）
- `R7-2-3teireiR070617.pdf` → 令和7年第2回定例会 第3日（6月17日）
- `R07-3-1rinjiR070624.pdf` → 令和7年第3回臨時会 第1日（6月24日）
- `R07-2-1rinjiR070218.pdf` → 令和7年第2回臨時会 第1日（2月18日）

令和6年の例:
- `R6-4-1teireiR061203.pdf` → 令和6年第4回定例会 第1日（12月3日）
- `R6-4-2teireiR061212.pdf` → 令和6年第4回定例会 第2日（12月12日）
- `R6-4-3teireiR061217.pdf` → 令和6年第4回定例会 第3日（12月17日）

## スクレイピング戦略

### Step 1: 会議録一覧トップから年度別ページ ID の収集

会議録一覧トップ (`kaigiroku/index.html`) から各年度ページへのリンクを収集する。

- 令和2年〜令和7年の6年分が主要掲載範囲
- 過去年度（平成18年以降）も掲載

### Step 2: 年度別ページから PDF リンクの抽出

各年度ページ（`kaigiroku/{ID}.html`）には当該年度の全会議 PDF が一覧表示されている。

年度別ページから直接 `material/files/group/12/*.pdf` へのリンクを全件抽出する。

### Step 3: PDF のダウンロード

抽出したリンクから PDF をダウンロードする。1 年度につき通常 10〜15 件の PDF が存在する（定例会4回×複数日 + 臨時会）。

---

## 注意事項

- 年度フォルダ名（`R7` vs `R07`）が一貫していない場合がある（ファイル名の先頭に `R7` と `R07` が混在）
- 議会映像インターネット配信（kensakusystem）はテキスト検索非対応のため対象外
- 一覧ページに「全ての項目を見る」リンクが存在する場合があり、ページネーションまたは展開が必要
- テキスト検索システムは存在しないため、全 PDF のダウンロードが必要

---

## 推奨アプローチ

1. **2段階クロール**: 年度別一覧 → PDF リンク抽出（会議詳細ページを経由せず直接 PDF を収集できる）
2. **ファイル名からメタ情報を抽出**: 命名規則が統一されているため、ファイル名から年度・回数・開催日・会議種別を正規表現で解析可能
3. **「全ての項目を見る」の対応**: ページに展開リンクがある場合は展開後の全リンクを取得する
4. **レート制限**: リクエスト間に 1〜2 秒の待機時間を設ける

### ファイル名解析の正規表現（案）

```typescript
// R7-2-1teireiR070603.pdf または R07-3-1rinjiR070624.pdf
const pattern = /^R0?(\d+)-(\d+)-(\d+)(teirei|rinji)R(\d{2})(\d{2})(\d{2})\.pdf$/;
// groups: [年度, 回数, 日数, 種別, 年(2桁), 月, 日]
```
