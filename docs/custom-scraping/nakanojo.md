## 発言者パターン

### 議長

```
○議長（安原賢一）みなさん、おはようございます。
○議長（安原賢一）３番、山本さん
○議長（安原賢一）町長
○議長（安原賢一）教育長
```

### 町長・副町長・教育長等

```
○町長（外丸茂樹）みなさん、おはようございます。
○町長（外丸茂樹）それでは、日程に従いまして...
○教育長（山口暁夫）それでは、山本修議員の質問にお答えいたします。
```

### 議員（番号付き）

```
○３番（山本修）詳しいご説明いただきまして、大変ありがとうございます。
```

### パターンまとめ

- `○{役職}（{氏名}）` — 議長、町長、副町長、教育長、各課長など
- `○{番号}番（{氏名}）` — 一般議員
- 役職と氏名は全角括弧 `（）` で囲まれる
- 議長が次の発言者を指名する行が挟まる: `○議長（安原賢一）３番、山本さん` / `○議長（安原賢一）町長`

### その他の表記

- 議場の声: `（「異議なし」の声）`
- 休憩・再開: `◎　休　憩（午前１０時５０分）` / `◎　再　開（午前１１時０５分）`
- ページ番号: `－ 1 －`（各ページ下部）

## パース用正規表現（案）

```typescript
// 発言者の抽出（役職付き）
const speakerWithRolePattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（安原賢一） → role="議長", name="安原賢一"
// 例: ○町長（外丸茂樹） → role="町長", name="外丸茂樹"

// 発言者の抽出（番号付き議員）
const speakerWithNumberPattern = /^○(\d+)番（(.+?)）/;
// 例: ○３番（山本修） → number="３", name="山本修"

// セクション見出しの検出
const sectionPattern = /^◎\s*(.+)/;
// 例: ◎　再開前のあいさつ → section="再開前のあいさつ"

// 休憩・再開の検出
const breakPattern = /^◎\s*(休\s*憩|再\s*開)（(.+?)）/;
// 例: ◎　休　憩（午前１０時５０分） → type="休憩", time="午前１０時５０分"

// セクション区切り線の検出
const separatorPattern = /^─+\s*○\s*─+$/;

// 議場の声
const voicePattern = /^（「(.+?)」の声）$/;

// 表紙からの開催日抽出
const datePattern = /(?:令和|平成)[０-９]+年[０-９]+月[０-９]+日/;

// PDF リンクテキストからの情報抽出
const linkTextPattern = /(.+?)(?:定例会議|臨時会議|招集会議)\((\d+)月(\d+)日(?:～(\d+)日)?\)/;
```

## スクレイピング戦略

### Step 1: PDF リンクの収集

一覧ページ `https://www.town.nakanojo.gunma.jp/site/nakanojo-gikai/1097.html` から全 PDF リンクを抽出する。

- HTML は単一ページ、ページネーションなし
- `<a href="/uploaded/attachment/{ID}.pdf">` のパターンで PDF リンクを収集
- リンクテキストから会議名・開催日を抽出（例: `第1回定例会12月定例会議(12月3日～16日)`）
- 年度は HTML の見出し（`<h2>` 等）から取得

**収集方法:**

1. 一覧ページを Cheerio でパース
2. 年度見出し（h2）配下の PDF リンクを抽出
3. 各リンクの `href`（attachment ID）とテキスト（会議名・日付）を記録

### Step 2: PDF のダウンロードとテキスト抽出

各 PDF を `https://www.town.nakanojo.gunma.jp/uploaded/attachment/{ID}.pdf` からダウンロードし、テキストを抽出する。

- PDF はテキスト埋め込み型（画像ではない）のため、`pdf-parse` や `pdfjs-dist` でテキスト抽出が可能
- フォントは MS-Mincho / MS-Gothic / BIZ-UDMincho-Medium（日本語テキスト埋め込み済み）

### Step 3: テキストのパース

抽出したテキストから以下を構造化する:

1. **メタ情報**: 表紙・2ページ目のテーブルから開催日、会議名、出席議員を抽出
2. **議事日程**: 議案番号と件名の一覧
3. **発言記録**: `○` で始まる発言者行を境界として発言ブロックに分割

## 会議の分類

一覧ページのリンクテキストから以下の会議種別が確認できる:

| 会議種別 | 例 |
| --- | --- |
| 招集会議 | 第1回定例会招集会議(1月16日) |
| 定例会議 | 第1回定例会12月定例会議(12月3日～16日) |
| 臨時会議 | 第1回定例会第3回臨時会議(10月21日) |

- 定例会議は年4回（3月、6月、9月、12月）
- 臨時会議は不定期（年1〜5回程度）
- 招集会議は各定例会の冒頭（1月）

---

## 注意事項

- PDF 直接掲載のため、HTML ベースの会議録検索システムとは異なるアプローチが必要
- PDF のテキスト抽出品質に依存するため、抽出結果の検証が重要
- 全角数字（１２３）と半角数字（123）が混在する可能性がある
- PDF 内のテーブル（出席者一覧等）はテキスト抽出時にレイアウトが崩れる可能性がある
- attachment ID は連番ではなく、他ページの添付ファイルと共有の通し番号

---

## 推奨アプローチ

1. **一括取得**: 一覧ページが単一のため、1回の HTML 取得で全 PDF リンクを収集可能
2. **PDF テキスト抽出**: `pdfjs-dist` または `pdf-parse` を使用してテキスト層を抽出
3. **発言者ベースの分割**: `○` で始まる行を発言者の境界としてパース
4. **レート制限**: 自治体サイトのため、PDF ダウンロード間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 一覧ページの PDF リンク数を前回と比較し、新規追加分のみをダウンロード・処理する
