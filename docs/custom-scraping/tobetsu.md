## スクレイピング戦略

### Step 1: 年度別ページ URL の収集

会議録一覧ページ（18717.html）から年度別ページへのリンクを抽出する。

- リンクは `/site/gikai/{ページID}.html` 形式
- 全18年度分が単一ページに掲載（ページネーションなし）
- HTML のリスト構造から Cheerio で抽出可能

### Step 2: PDF リンクの収集

各年度ページから PDF ファイルへのリンクを抽出する。

- リンクは `/uploaded/attachment/{ファイルID}.pdf` 形式
- 各リンクのテキストに会議名が含まれる（例: 「第1回定例会（3月）」）
- ファイルサイズ情報も併記されている

**収集方法:**

1. Step 1 で取得した18個の年度別ページ URL を順次アクセス
2. 各ページ内の PDF リンク（`/uploaded/attachment/*.pdf`）を Cheerio で抽出
3. リンクテキストから会議名（定例会/臨時会、回数、開催月）をパース

### Step 3: PDF のダウンロードとテキスト抽出

- PDF をダウンロードし、テキスト抽出ライブラリ（pdf-parse 等）で本文を取得
- PDF 内部の構造は会議ごとに異なる可能性があるため、テキスト抽出後にパースする

### Step 4: 会議録のパース

PDF から抽出したテキストを解析し、以下の情報を構造化する:

- 開催日
- 会議種別（定例会/臨時会）
- 回次
- 発言者と発言内容

※ PDF の内部構造は実際にダウンロードして確認する必要がある。

---

## 注意事項

- 会議録は PDF 形式のため、HTML スクレイピングとは異なるパイプラインが必要
- PDF のテキスト抽出精度はファイルの作成方法に依存する（スキャン画像の場合は OCR が必要）
- 平成20年〜令和7年の18年分、年あたり4〜6ファイル（合計約80〜100ファイル程度）
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける

---

## 推奨アプローチ

1. **2段階クロール**: 一覧ページ → 年度別ページ → PDF リンク収集 → PDF ダウンロードの順で段階的に処理
2. **PDF テキスト抽出の事前検証**: 数ファイルをサンプルでダウンロードし、テキスト抽出の品質を確認してからパーサーを実装する
3. **メタ情報はリンクテキストから取得**: PDF 内部のパースに依存せず、リンクテキスト（「第1回定例会（3月）」等）から会議種別・回次・開催月を抽出する
4. **差分更新**: 年度別ページの PDF リンク数を前回取得時と比較し、新規追加分のみをダウンロードする
