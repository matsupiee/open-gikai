# 日野町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.hino.tottori.jp/1660.htm
- 分類: 静的リスト型（年度別・回数別のリンク一覧から個別ページへ遷移し、PDF を取得）
- 文字コード: UTF-8
- 特記: 会議録本文は PDF ファイルで提供（HTML 上には本文なし）

## サイト構造

### 一覧ページ（/1660.htm）

- 令和元年〜令和8年の会議録リンクを単一ページに全件掲載
- ページネーションなし
- 各リンクのテキスト形式: `{年号}年第{N}回日野町議会{定例会|臨時会}会議録`
- 会議種別は「定例会」と「臨時会」の2種類

### 詳細ページ（/{ページID}.htm）

- 会議名をタイトルとして表示
- 1回の会議が複数日にわたる場合、日ごとに個別の PDF リンクを掲載
  - 定例会: 通常2〜3日分の PDF
  - 臨時会: 通常1日分の PDF
- PDF リンクのテキスト: `第1日`、`第2日`、`第3日` など

### PDF ファイル名パターン

| 年代 | パターン | 例 |
| --- | --- | --- |
| 令和2年以降 | `{回次:2桁}R{年:2桁}{月:2桁}{日:2桁}.pdf` | `06R071205.pdf`（令和7年第6回、12月5日） |
| 令和元年 | `R{年}-{回次:2桁}.pdf` または `{回次:2桁}{月:2桁}{日:2桁}.pdf` | `R1-03.pdf`、`081205.pdf` |

※ ファイル名パターンは年代により不統一のため、詳細ページから直接リンクを取得する方が確実。

## スクレイピング戦略

### Step 1: 会議録一覧の取得

一覧ページ `/1660.htm` から全会議録の詳細ページ URL を収集する。

- `<a>` タグの `href` 属性から `/{数字}.htm` 形式のリンクを抽出
- リンクテキストから年号・回次・会議種別を正規表現で抽出

**収集方法:**

1. `/1660.htm` を取得
2. Cheerio で `a[href]` を選択し、`/\d+\.htm` にマッチするリンクを抽出
3. リンクテキストから会議メタ情報をパース

### Step 2: PDF リンクの収集

各詳細ページから PDF ファイルの URL を収集する。

- 詳細ページ内の `<a>` タグから `/secure/` で始まる `.pdf` リンクを抽出
- 1つの会議に対して複数の PDF が存在する場合がある（日ごと）

**収集方法:**

1. 各詳細ページ `/{ページID}.htm` を取得
2. Cheerio で `a[href$=".pdf"]` を選択
3. PDF の URL とリンクテキスト（第1日、第2日など）を記録

### Step 3: PDF のダウンロードとテキスト抽出

PDF をダウンロードし、テキストを抽出する。

- PDF は Microsoft Word 2016 で作成されている
- フォント: MS-Mincho、MS-Gothic
- `pdftotext` や `pdf-parse` 等のライブラリでテキスト抽出可能

### Step 4: 会議録のパース

#### メタ情報

PDF 冒頭から以下を抽出:

```
令和８年第１回日野町議会臨時会を次のとおり招集する。
令和８年１月16日
```

```
第１回 日野町議会臨時会会議録 （第１日）
令和８年１月20日（火曜日）
```

- 年号・回次・会議種別: タイトル行から抽出
- 開催日: `令和X年XX月XX日（X曜日）` 形式
- 出席議員: `出席議員（N名）` セクションから抽出

#### 発言者の構造

発言者パターン:

```
○議長（中原 信男君）
○町長（﨏田 淳一君）
○議員（８番 安達 幸博君）
○議員（３番 坪倉 敏君）
○議員（４番 中山 法貴君）
○総務課長（景山 政之君）
○企画政策課長（神崎 猛君）
○健康福祉課長（住田 秀樹君）
○産業振興課長（杉原 昭二君）
○教育課長（三好 ...）
```

- `○` で始まり、役職名 + 括弧内に氏名（`君` 付き）
- 議員の場合: `○議員（{番号}番 {氏名}君）`
- 議長の場合: `○議長（{氏名}君）`
- 執行部の場合: `○{役職名}（{氏名}君）`
- 発言内容は発言者行の後に続くテキスト

#### 議事進行の表記

```
〔質疑なし〕
〔討論なし〕
〔「異議なし」と呼ぶ者あり〕
〔賛成者起立〕
```

#### パース用正規表現（案）

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（中原 信男君） → role="議長", name="中原 信男君"

// 議員番号付き発言者
const memberPattern = /^○議員（(\d+)番\s+(.+?)君）/;
// 例: ○議員（８番 安達 幸博君） → number="８", name="安達 幸博"

// 開催日の抽出
const datePattern = /(?:令和|平成)(\d+)年(\d+)月(\d+)日/;

// 議事進行の検出
const proceedingPattern = /^〔(.+?)〕/;
```

## 注意事項

- PDF のテキスト抽出時、改行位置が元のレイアウトに依存するため、発言者パターンが行をまたぐ場合がある（例: `○総務課長（景山` + 改行 + `政之君）`）
- 氏名の間にスペースが入る場合がある（例: `中原 信男君`、`安達 幸博君`）
- 全角数字と半角数字が混在する（PDF 内は主に全角、ファイル名は半角）
- ページ番号（`－1－`、`－2－` 等）がテキストに含まれるため、除去が必要
- 罫線文字（`───`）が区切りとして使われている

## 推奨アプローチ

1. **2段階クロール**: 一覧ページ → 詳細ページ → PDF の順に URL を収集し、PDF をダウンロード
2. **PDF テキスト抽出**: `pdf-parse` や `pdftotext` を用いて PDF からテキストを抽出
3. **改行の正規化**: PDF 由来の不規則な改行を正規化してからパースする
4. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2秒）を設ける
5. **差分更新**: 一覧ページのリンク数を前回と比較し、新規追加分のみを取得する
