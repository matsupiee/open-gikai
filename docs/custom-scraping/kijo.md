# 木城町議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.town.kijo.lg.jp/gyouseizyouhou/gikai/kaigiroku/index.html
- 分類: 自治体サイトで年度別ページに PDF を直接公開（会議録検索システムなし）
- 公開範囲: 平成23年〜令和7年
- 形式: PDF ファイル（目次 + 日別の会議録）

## 会議種別

各年度で「定例会」と「臨時会」が交互に開催される。年間おおむね以下の構成:

- **定例会**: 年4回程度（3月・6月・9月・12月）、複数日開催（1〜3日間）
- **臨時会**: 年2〜4回程度、単日開催が多い

各回は「第N回」として通し番号が振られる（例: 第1回臨時会、第2回定例会、...）。

## PDF ファイル命名規則

ファイル名の命名規則は年度によって統一されておらず、複数のパターンが混在する。

### パターン 1: ローマ字日付形式（平成23〜25年頃）

```
1h23-7.pdf        → 平成23年 第7回 目次（1=目次）
2h23-7.pdf        → 平成23年 第7回 第1日
3h23-7.pdf        → 平成23年 第7回 第2日
4h23-7.pdf        → 平成23年 第7回 第3日
```

### パターン 2: 英語表記形式（平成23〜25年の一部）

```
H23-5th-minutes-contents.pdf  → 平成23年 第5回 目次
H23-5th-minutes-1st.pdf       → 平成23年 第5回 第1日
H23-5th-minutes-2nd.pdf       → 平成23年 第5回 第2日
```

### パターン 3: 年号プレフィックス形式（令和6年頃）

```
R6-mokuji.pdf           → 令和6年 目次
R6-1.pdf                → 令和6年 第1回 第1日
R602mokuji.pdf          → 令和6年 第2回 目次
R602ichinichime.pdf     → 令和6年 第2回 第1日目
```

### パターン 4: dai + 回数形式（令和6〜7年）

```
dai3kaiteireikai-mokuji.pdf    → 第3回定例会 目次
dai3kaiteireikai-1nichime.pdf  → 第3回定例会 第1日目
```

### パターン 5: 月日形式（令和7年の一部）

```

```

## スクレイピング戦略

### Step 1: 年度別ページの巡回

会議録トップページから年度別ページへのリンク一覧を取得する。

- トップページ `index.html` をパースし、`/gyouseizyouhou/gikai/kaigiroku/{ID}.html` 形式のリンクを全て抽出
- 13 ページ（平成23年〜令和7年分）を巡回対象とする

### Step 2: PDF リンクの収集

各年度別ページから PDF ファイルへのリンクを抽出する。

- `//www.town.kijo.lg.jp/material/files/group/5/` 配下の `.pdf` リンクを Cheerio で収集
- 相対パス・プロトコル省略（`//`）のリンクも考慮する
- 各 PDF リンクに対し、ページ内のテキストから会議種別（定例会/臨時会）・回数・日付のメタ情報を関連付ける

**メタ情報の抽出:**

PDF リンクの周辺テキスト（見出しやリスト項目）から以下を抽出:

```
第3回定例会
  目次 (3月7日)
  第1日目 (3月7日)
  第2日目 (3月10日)
  第3日目 (3月18日)
```

- 会議回数: `第(\d+)回`
- 会議種別: `定例会` or `臨時会`
- 開催日: `(\d+)月(\d+)日`

### Step 3: PDF のダウンロードとテキスト抽出

収集した PDF URL からファイルをダウンロードし、テキストを抽出する。

- PDF パーサー（pdf-parse 等）でテキストを抽出
- 目次 PDF（ファイル名に `mokuji` / `contents` / 先頭番号 `1` を含む）はメタ情報の補完に使用
- 日別 PDF から会議録本文を取得

### Step 4: テキストのパース

PDF から抽出したテキストを発言単位に分割する。

- PDF テキストは HTML と異なり構造化されていないため、正規表現ベースでパースする
- 発言者パターンは PDF の内容を実際に確認して決定する必要がある

---

## 注意事項

- ファイル名の命名規則が年度ごとに異なるため、PDF リンクの収集はファイル名パターンではなく HTML のリンク要素から行う
- 平成23〜25年は1ページにまとまっているため、年度の判別にはページ内の見出しを使用する
- PDF のテキスト抽出品質はファイルの作成方法（スキャン画像 vs テキスト埋め込み）に依存する。スキャン画像の場合は OCR が必要になる可能性がある
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
- PDF の総数は年間 15〜25 ファイル程度 x 13 年度分 = 推定 200〜300 ファイル

---

## 推奨アプローチ

1. **HTML リンク収集を優先**: ファイル名パターンが不統一なため、各年度ページの HTML から PDF リンクを直接収集する
2. **メタ情報は周辺テキストから取得**: PDF リンクの前後にある見出し・テキストから会議種別・日付を関連付ける
3. **目次 PDF を活用**: 目次 PDF には会議の構成情報が含まれており、メタ情報の補完に利用できる
4. **段階的な実装**: まず PDF リンク一覧の収集 → PDF ダウンロード → テキスト抽出 → パースの順で段階的に実装する
5. **レート制限**: 自治体サイトのため、リクエスト間に 1〜2 秒の待機時間を設ける
