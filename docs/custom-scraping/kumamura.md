# 球磨村議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.kumamura.com/
- 分類: 独自 CMS による PDF 直接公開（既存アダプターでは対応不可）
- 文字コード: UTF-8
- 特記: 外部の会議録検索システムは使用していない。公式サイト上で PDF ファイルとして会議録を直接公開している。

## 会議録一覧ページの構造

- URL: `https://www.kumamura.com/list00249.html`
- 全件数: 53件（令和3年〜令和7年）
- 一覧は `<ul>/<li>` 形式で表示され、各 `<li>` に更新日と会議録詳細ページへのリンクが含まれる
- 「もっと見る」ボタンによる AJAX 読み込みあり

### AJAX エンドポイント

追加の会議録は以下のエンドポイントで取得する（`pg` パラメータでページ番号を指定）:

```
https://www.kumamura.com/dynamic/hpkiji/pub/hpkijilistpagerhandler.ashx?c_id=3&class_id=249&class_set_id=1&pg={ページ番号}&kbn=kijilist&top_id=0
```

## 会議録詳細ページの構造

各定例会・臨時会の詳細ページ（`kiji{記事ID}/index.html`）に以下の情報が含まれる:

- **タイトル**: 会議名（例: 「令和６年第８回定例会　会議録」）
- **会期**: 開始日〜終了日（例: 「令和6年9月9日～17日」）
- **審議内容**: 報告・認定・議案・同意の件数
- **PDF リンク**: 審議結果・各日程の会議録

### 掲載 PDF の種類

| ファイル | 内容 |
| --- | --- |
| `3_{記事ID}_1_{ハッシュ}.pdf` | 審議結果 |
| `3_{記事ID}_2_{ハッシュ}.pdf` | 初日（開会・上程） |
| `3_{記事ID}_3_{ハッシュ}.pdf` | 2日目以降（一般質問等） |
| `3_{記事ID}_N_{ハッシュ}.pdf` | 最終日（審議採決・閉会） |

## スクレイピング戦略

### Step 1: 会議録詳細ページ URL の収集

1. `https://www.kumamura.com/list00249.html` にアクセスし、一覧内の全リンクを取得する
2. AJAX エンドポイント（`pg=2`, `pg=3`, `pg=4`, ...）を順次リクエストし、追加の会議録リンクを取得する
3. 全リンクが `https://www.kumamura.com/kiji{記事ID}/index.html` 形式であることを確認する

### Step 2: 各会議録詳細ページの取得

各 `kiji{記事ID}/index.html` にアクセスし、以下を抽出する:

- 会議名（`<h2>` タグ）
- 会期（開始日・終了日）
- 審議内容の概要
- 掲載されている全 PDF リンク

### Step 3: PDF の取得

各 PDF を以下のパターンで取得する:

```
https://www.kumamura.com/kiji{記事ID}/3_{記事ID}_{連番}_{MD5ハッシュ}.pdf
```

PDF はテキスト抽出可能な形式（スキャン PDF でなく電子的に生成された PDF）と想定されるが、OCR が必要な場合もある。

## 注意事項

- PDF のファイル名に含まれる MD5 ハッシュ値は固定であり、URL が変更されることはない
- 各詳細ページの HTML 構造はシンプルで、テーブルではなく `<p>` タグや改行によるレイアウトを使用している
- 会議録 PDF は複数日程に分割されており、1 つの会議につき 2〜5 件程度の PDF が存在する
- レート制限: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **一覧ページの全量取得**: 一覧ページ + AJAX エンドポイントで全会議録の `kiji{記事ID}` を収集する
2. **詳細ページのクロール**: 各詳細ページから PDF リンクを抽出する
3. **PDF のダウンロード・テキスト抽出**: 各 PDF をダウンロードし、テキストを抽出する
4. **差分更新**: 取得済みの `kiji{記事ID}` リストと照合し、新規追加分のみを処理する
