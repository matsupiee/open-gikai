## 年度ページ構造

### 公開範囲

平成23年（2011年）〜令和7年（2025年）現在まで。トップページから各年度ページへリンクされている。

### 年度ページ内の構造

- 月別に「○月会議」としてグループ化
- 1月のみ「定例会」と表記される場合がある
- 各会議は「会議録（X月Y日）」のリンクテキストで列挙
- 「一般質問」セクションが別途存在する会議がある
- ページネーションなし（1ページにすべての会議を表示）

### リンク形式の混在

| 時期 | 形式 | 備考 |
| --- | --- | --- |
| 令和7年（2025年） | PDF のみ | `gijiroku_R7.files/r{YYMMDD}.pdf` |
| 令和6年（2024年） | HTML + PDF 混在 | 12月・11月は PDF、1月〜9月は HTML |
| 平成23年〜令和5年 | HTML のみ | フレームセット構造 |

※ PDF への移行が進行中と推測される。

## PDF 会議録の構造

### ファイル名の命名規則

`r{YY}{MM}{DD}.pdf` 形式。例: `r070702.pdf` → 令和7年7月2日

### PDF の内容構成

1. **タイトル**: `令和X年蔵王町議会定例会Y月会議`
2. **開催日**: `令和X年Y月Z日（曜日）`
3. **出席議員一覧**: `出席議員（N名）` + 番号・氏名のリスト
4. **欠席議員**: `欠席議員（なし）` 等
5. **説明のため出席した者**: 町長、副町長、各課長の氏名一覧
6. **事務局職員出席者**: 事務局長等
7. **議事日程**: `日程第N　議案第XX号　...` の形式
8. **本文**: 発言の記録

### 発言者パターン

```
○議長（佐藤長成君）
○町長（村上英人君）
○副町長（平間喜久夫君）
○総務課長（鈴木　賢君）
○教育長（文谷政義君）
○１番（平間徹也君）
```

- `○` で始まり、役職 or 番号 + 括弧内に氏名（敬称「君」付き）
- 発言内容は発言者行の後に続くテキスト

### 登壇表記

```
〔町長　村上英人君　登壇〕
```

- `〔` で始まり `登壇〕` で終わる
- 役職 + 氏名 + 「君」 + 「登壇」

### その他の記録

```
〔「異議なし」と呼ぶ者あり〕
午前１０時００分　開議
午前１０時０４分　休憩
午前１０時４５分　再開
```

## HTML 会議録の構造

### フレームセット

`*main.html` がフレームセットのエントリポイント。2カラム構成:

- 左フレーム（15%）: `*menu.html` — 日程一覧のナビゲーション
- 右フレーム（85%）: `*.htm` — 会議録本体

### HTML 本体

- 文字コード: Shift_JIS（`<META http-equiv="Content-Type" content="text/html; charset=Shift_JIS">`）
- 生成ツール: IBM WebSphere Studio Homepage Builder Version 13
- MS Word 由来のスタイル（`MsoNormal` 等）が含まれる
- 発言者パターンは PDF と同一（`○議長（...）` 形式）

### ディレクトリ命名規則

| パターン | 例 | 意味 |
| --- | --- | --- |
| `R{YY}{MM}-{N}` | `R0606-2` | 令和6年6月会議の2日目 |
| `R{YY}{MM}{DD}` | `R060201` | 令和6年2月の1日目（旧形式） |
| `{YY}{MM}` | `2301` | 平成23年1月 |
| `{YY}{MM}-{N}-{M}` | `2306-2-1` | 平成23年6月第2回の1日目 |

## スクレイピング戦略

### Step 1: 年度ページの URL 一覧を生成

トップページ `index.html` から全年度ページへのリンクを取得する。

- 令和: `gijiroku_R{1〜7}.html`
- 平成: `gijiroku_h{23〜31}.html`（h31 は令和元年と兼用）

### Step 2: 各年度ページから会議録リンクを収集

各年度ページの HTML をパースし、PDF リンク（`.pdf`）と HTML リンク（`main.html`）を収集する。

**収集方法:**

1. 年度ページの `<a>` タグから `href` を抽出
2. `.pdf` で終わるリンク → PDF 会議録として記録
3. `main.html` で終わるリンク → HTML 会議録として記録
4. リンクテキストから開催日を抽出（`会議録（X月Y日）` 形式）
5. 直前のセクション見出しから会議種別（○月会議 / 定例会）と一般質問の区分を判定

### Step 3: 会議録本文の取得

#### PDF の場合

1. PDF をダウンロード
2. PDF パーサー（pdf-parse 等）でテキストを抽出
3. 発言者パターンでテキストを分割

#### HTML の場合

1. `*main.html` からフレームセットを解析し、本体 `.htm` の URL を特定
2. Shift_JIS でデコードして HTML を取得
3. Cheerio でテキストを抽出（`MsoNormal` 等のスタイル要素を無視）

### Step 4: 会議録のパース

#### メタ情報の抽出

```
令和６年蔵王町議会第２回定例会６月会議
令和６年６月１１日（火曜日）
```

- 会議名: タイトル行から抽出
- 開催日: 全角数字で `令和X年XX月XX日（曜日）` 形式

#### 発言の分割

```typescript
// 発言者の抽出
const speakerPattern = /^○(.+?)（(.+?)）/;
// 例: ○議長（佐藤長成君） → role="議長", name="佐藤長成君"
// 例: ○１番（平間徹也君） → role="１番", name="平間徹也君"

// 登壇の検出
const podiumPattern = /^〔(.+?)登壇〕/;

// 開催日の抽出
const datePattern = /(?:令和|平成)([\d０-９]+)年([\d０-９]+)月([\d０-９]+)日/;

// 休憩・再開の検出
const breakPattern = /^(午前|午後)([\d０-９]+)時([\d０-９]+)分\s*(開議|休憩|再開|散会)/;
```

## 注意事項

- PDF と HTML の2形式が混在しており、両方のパーサーが必要
- HTML 版は Shift_JIS エンコーディングのため、デコード処理が必須
- HTML 版はフレームセット構造のため、`*main.html` → `*.htm` の2段階アクセスが必要
- HTML 版は MS Word 由来の冗長なスタイル定義が含まれるため、テキスト抽出時に無視する必要がある
- 氏名に「君」の敬称が付いている（`佐藤長成君`）ため、氏名抽出時に除去が必要
- 議員番号は全角数字（`１番`、`１０番` 等）
- 検索機能は存在しないため、全ページを順次クロールする必要がある

---

## 推奨アプローチ

1. **PDF 優先**: 令和7年以降は PDF のみで提供されており、今後 PDF に統一される可能性が高い。PDF パーサーを優先的に実装する
2. **段階的対応**: まず PDF 会議録のパースを実装し、次に HTML 版に対応する
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度ページの会議録リンク数を前回取得時と比較し、新規追加分のみを取得する
5. **文字コード変換**: HTML 版は Shift_JIS → UTF-8 の変換を行う
