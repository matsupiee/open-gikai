## PDF ファイルの構成

各会議詳細ページには以下の種類の PDF が掲載される。

| ファイル種別 | 備考 |
| --- | --- |
| 会期日程 | 会期のスケジュール（会議録本文ではない） |
| 一般質問事項 | 質問項目の一覧（会議録本文ではない） |
| 採決結果 | 議案の採決結果（ページによっては掲載なし） |
| 会議録（日ごと） | 開催日ごとに個別 PDF。これが会議録本文 |

### 会議録 PDF のファイル名パターン

ファイル名の形式は年代によって異なる。

| 年代 | ファイル名形式 | 例 |
| --- | --- | --- |
| 平成26年〜平成28年頃 | `{月日4桁}kaigiroku.pdf` | `1208kaigiroku.pdf`（12月8日） |
| 令和2年以降 | `{YYYYMMDD}.pdf` | `20200305.pdf`（2020年3月5日） |

いずれもリンクテキストに「〇月〇日　会議録」または「〇月〇日会議録」と記載されており、会議録本文の PDF を識別できる。

## スクレイピング戦略

### Step 1: 会議詳細ページ URL の収集

会議録トップページ（`category/32-5-0-0-0-0-0-0-0-0.html`）から、各会議詳細ページへのリンクを抽出する。

- リンク先は `https://www.town.toyosato.shiga.jp/{10桁ID}.html` 形式
- トップページに全年度の会議リンクがまとめて掲載されている
- ページネーションなし（1ページに全件表示）
- 年度別カテゴリページ（`category/32-5-{年度ID}-...`）からも同様に収集可能

**収集方法:**

1. トップページの `<a>` タグを Cheerio で解析し、`/\d{10}\.html` にマッチするリンクを抽出
2. リンクテキスト（会議名）と URL をペアで記録する

### Step 2: 各会議詳細ページから PDF リンクの収集

各会議詳細ページ（例: `https://www.town.toyosato.shiga.jp/0000003781.html`）から PDF リンクを抽出する。

- `<a href="...pdf">` のうち、リンクテキストに「会議録」を含むものが対象
- 「会期日程」「一般質問事項」「採決結果」は会議録本文ではないため除外する
- 1 つの会議に複数日分の PDF が存在する（開催日ごと）

**会議録 PDF の識別条件:**

```
リンクテキストに「会議録」が含まれる
かつ
href が .pdf で終わる
```

### Step 3: PDF の取得とテキスト抽出

各会議録 PDF を取得し、テキストを抽出する。

- PDF は日ごとに分割されているため、同一定例会の複数 PDF を結合して扱う
- テキスト抽出は既存の PDF パーサーを使用

## 会議の種類

定例会と臨時会が存在する。臨時会は同一年度内に複数回開催されることがある。

| 会議種別 | 備考 |
| --- | --- |
| 定例会 | 年4回程度（3月・6月・9月・12月） |
| 臨時会 | 随時（同一年度に複数回の場合あり） |

---

## 注意事項

- 会議録 PDF のファイル名形式が年代によって異なるため、ファイル名ではなくリンクテキストで会議録 PDF を識別する
- 一部の定例会は「2月定例会」「11月定例会」等、通常と異なる月に開催される場合がある
- 令和8年3月定例会（最新）は会期日程と一般質問事項のみ掲載で、会議録 PDF はまだ公開されていない場合がある（会議録の公開は会議終了後）
- 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける

---

## 推奨アプローチ

1. **トップページから全件収集**: ページネーションがないため、トップページ 1 件のアクセスで全会議の URL を取得できる
2. **リンクテキストで PDF を識別**: ファイル名パターンが年代によって異なるため、「会議録」を含むリンクテキストを基準に会議録 PDF を選別する
3. **差分更新**: 取得済みの会議詳細ページ URL を記録しておき、新規追加分のみを取得することで効率的な差分更新が可能
4. **日付の抽出**: リンクテキスト（「〇月〇日　会議録」）または PDF ファイル名（`YYYYMMDD.pdf`）から開催日を取得する
