# 由布市議会 カスタムスクレイピング方針

## 概要

- サイト: https://www.city.yufu.oita.jp/
- 分類: 市公式サイトで PDF を直接公開（外部の会議録検索システムは未導入）
- 文字コード: UTF-8
- 特記: WordPress ベースのサイト。旧ファイルは `/wp-content/uploads/` 配下、新ファイルは `/uploads/files/` 配下に格納

| ページ | URL |
| --- | --- |
| 会議録一覧 | `https://www.city.yufu.oita.jp/city_council/shigikai/shigikai_cate5/gijirokukensaku` |

## ページ構造

会議録一覧ページは単一の HTML ページ。年度ごとに `<h3>` 見出しで区切られ、各定例会・臨時会の PDF リンクが `<p>` タグ内に列挙されている。

```html
<h3>令和７年</h3>
<p>◆第1回定例会</p>
<p><a href="/uploads/files/2025/07/R7-1-2.25.pdf">初日（R7.2.25）</a></p>
<p><a href="/uploads/files/2025/07/R7-2-2.28.pdf">２日目（R7.2.28）</a></p>
<p><a href="/uploads/files/2025/07/R7-3-3.3.pdf">３日目（R7.3.3）</a></p>
```

- 年度セクションはアンカー ID `#member01` 〜 `#member21` でジャンプ可能
- ページネーションなし（全年度分が単一ページに集約）

## PDF ファイルの命名規則

### 命名パターン

ファイル名は年度・時期によって命名規則が異なり、統一されていない。

| 時期 | パスパターン | ファイル名例 |
| --- | --- | --- |
| 令和5年〜 | `/uploads/files/{年}/{月}/` | `R7-1-2.25.pdf`（`{元号略称}{年}-{日数目}-{月}.{日}.pdf`） |
| 令和5年 | `/uploads/files/{年}/{月}/` | `R5-yufushiteireikai-1.pdf` |
| 令和4年 | `/uploads/files/{年}/{月}/` | `1r4teireikai1.pdf` |
| 令和3年以前 | `/wp-content/uploads/{年}/{月}/` | `r3-2-25.pdf`、`r20227.pdf`、`h31.2.27.pdf` |

### 命名規則の揺れ

- 元号略称の大文字・小文字混在: `R7` / `r3` / `h31`
- 区切り文字の不統一: ハイフン（`-`）、ドット（`.`）
- ファイル名構造自体が年度ごとに異なる

ファイル名を推測するのではなく、**一覧ページの `<a href>` から URL を直接収集する**ことを強く推奨する。

## 会議種別

| 種別 | 一覧ページでの表記 | 備考 |
| --- | --- | --- |
| 定例会 | `◆第N回定例会` | 年4回が標準 |
| 臨時会 | `◆第N回臨時会` | 年1〜複数回（年によって異なる） |

各定例会は複数日にわたり、日ごとに個別の PDF ファイルが用意されている（「初日」「2日目」「3日目」...「最終日」）。

委員会別の会議録は掲載されていない。すべて本会議（定例会・臨時会）のみ。

## スクレイピング戦略

### Step 1: 一覧ページから PDF URL を収集

`gijirokukensaku` ページを取得し、PDF への `<a href>` をすべて抽出する。

- リンクは相対パスのため、ベース URL `https://www.city.yufu.oita.jp` を付与して絶対 URL に変換する
- PDF のパスには `/uploads/files/` と `/wp-content/uploads/` の2パターンが存在する
- 同一ページで全年度・全会議が一覧できるため、ページネーション処理は不要

```typescript
// Cheerio での収集例
const links = $('a[href$=".pdf"]')
  .map((_, el) => {
    const href = $(el).attr('href')!;
    return new URL(href, 'https://www.city.yufu.oita.jp').toString();
  })
  .toArray();
```

### Step 2: 各 PDF を取得・テキスト抽出

PDF ファイルを取得し、テキスト抽出ツール（pdf-parse 等）でテキスト化する。

- PDF は会議録全文（発言者・発言内容を含む）
- 各定例会は日ごとに分割されたPDFとなっている

### Step 3: メタ情報の取得

PDF URL と一覧ページのテキストからメタ情報を取得する。

**一覧ページから取得できる情報:**

- 年度: `<h3>` テキスト（例: `令和７年`）
- 会議種別と回次: `◆` で始まる `<p>` テキスト（例: `◆第1回定例会`、`◆第1回臨時会`）
- 開催日・日数目: リンクテキスト（例: `初日（R7.2.25）`、`２日目（R7.2.28）`）

**パース用正規表現（案）:**

```typescript
// <h3> から年度を抽出
const yearPattern = /(?:令和|平成)(\d+)年/;
// 例: "令和７年" → era="令和", year=7（全角数字の場合あり）

// <p> テキストから会議種別と回次を抽出
const sessionPattern = /◆第(\d+)回(定例会|臨時会)/;
// 例: "◆第1回定例会" → count=1, type="定例会"

// リンクテキストから開催日を抽出
const datePattern = /[（(](?:R|H)(\d+)\.(\d+)\.(\d+)[)）]/;
// 例: "初日（R7.2.25）" → year=7, month=2, day=25
```

---

## 注意事項

- **PDF パスの変遷**: 令和4年頃を境に `/wp-content/uploads/` から `/uploads/files/` にパスが変更されている。両方のパターンに対応する必要がある
- **ファイル名の不統一**: 年度ごとにファイル命名規則が異なるため、URL は必ず HTML から直接収集する
- **全角数字**: 年度見出し（`<h3>`）やリンクテキストに全角数字が含まれる場合がある（例: `令和７年`、`２日目`）
- **日ごとの分割**: 1つの定例会が複数の PDF に分割されているため、同一会議の紐付けが必要

---

## 推奨アプローチ

1. **一覧ページを単一取得**: `gijirokukensaku` の 1 リクエストで全 PDF URL を収集できる
2. **リンクあり判定**: `$('a[href$=".pdf"]')` で PDF リンクが存在する会議のみを対象にする
3. **年度・会議種別はページから取得**: `<h3>` と `◆` 付き `<p>` テキストを紐付けてメタ情報とする
4. **レート制限**: PDF ダウンロード時もリクエスト間に 1〜2 秒の待機を設ける
5. **差分更新**: 既取得 URL のリストと比較し、新規 URL のみ PDF を取得する
