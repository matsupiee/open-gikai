## HTML 構造

### 一覧ページ（index.html）

年度ごとに `<h2>` 見出しで区切られ、各定例会・臨時会が会期付きのリンクとして列挙される。

- リンク形式: `honkaigi_R{N}.html#{アンカーID}`（相対パス）
- アンカーID パターン: `r{N}dai{X}kaiteirei`（定例会）、`r{N}dai{X}kairinji`（臨時会）
- ページネーションなし（全年度が単一ページに掲載）

### 年度別本会議ページ（honkaigi_R{N}.html）

各定例会・臨時会は `<h2>` 見出しで区切られ、以下の情報が含まれる。

#### 会期日程テーブル

4列構成のテーブル:

| 日程 | 月日（曜日） | 開会予定時間 | 内容 |
| --- | --- | --- | --- |

#### 議決結果テーブル

4列構成のテーブル:

| 議案番号 | 提出年月日 | 件名 | 議決結果 |
| --- | --- | --- | --- |

#### 一般質問セクション

階層的な見出し構造:

```
h3: 一般質問の主な項目
  h4: 議員名（会派名）
    h5: 質問テーマ
      h6: 詳細項目
        箇条書き: 具体的な質問内容
```

## ページネーション

なし。一覧ページ（index.html）に全年度のリンクが掲載されており、年度別ページにも当該年度のすべての定例会・臨時会情報が1ページにまとめられている。

---

## 掲載内容の特徴

- **発言録の全文は未掲載**: 一般質問については「主な項目」として質問テーマと質問内容の要約のみが掲載される。議員の発言や答弁の全文テキストは含まれない
- **議案一覧**: 議案番号・件名・議決結果が掲載される
- **会期日程**: 各定例会・臨時会の日程表が掲載される

---

## 会議種別

定例会と臨時会の両方が掲載されている。

- 定例会: 年4回（3月・6月・9月・12月）が主体
- 臨時会: 不定期に開催

## スクレイピング方針

### Step 1: 一覧ページから年度別ページ URL を収集

一覧ページ `index.html` から、年度別本会議ページへのリンクをすべて収集する。

- リンクは相対パスで記載されているため、ベース URL `https://www.town.itayanagi.aomori.jp/gikai/teirei/` を付与して絶対 URL に変換する
- ファイル名部分のみを抽出し、重複を排除する（同一ファイルに複数アンカーでリンクされているため）

```typescript
const baseUrl = "https://www.town.itayanagi.aomori.jp/gikai/teirei/";
const links = $('a[href*="honkaigi_"], a[href*="2020-0312"]');
const uniqueFiles = new Set<string>();
links.each((_, el) => {
  const href = $(el).attr("href");
  if (href) {
    const filename = href.split("#")[0];
    uniqueFiles.add(filename);
  }
});
```

### Step 2: 各年度別ページから会議情報を抽出

各年度別ページにアクセスし、`<h2>` 見出しごとに定例会・臨時会を分割する。

各セクションから以下の情報を抽出:

1. **会議名**: h2 見出しテキスト（例: 「第8回定例会（令和7年12月5日~11日）」）
2. **会期**: h2 見出しテキスト内の括弧から日付を抽出
3. **議案一覧**: 議決結果テーブルの各行
4. **一般質問**: h3 以下の階層構造から議員名・質問テーマ・質問内容を抽出

### Step 3: データの構造化

```typescript
// 会議名からの情報抽出
const meetingPattern = /第(\d+)回(定例会|臨時会)（(.+?)）/;
// 例: 第8回定例会（令和7年12月5日～11日）

// 議員名の抽出（h4テキストから）
const memberPattern = /(.+?)（(.+?)）/;
// 例: 山田太郎（無所属） → name="山田太郎", faction="無所属"
```

## 注意事項

- **令和2年のファイル名が例外**: 令和2年のみ `2020-0312-1647-18.html` という CMS 生成風のファイル名で、`honkaigi_R2.html` パターンに従わない
- **発言全文なし**: このサイトは議案一覧と一般質問の主な項目のみを公開しており、会議録の発言全文は掲載されていない。発言録が必要な場合は別の情報源（町議会だよりなど）を検討する必要がある
- **アンカー ID の不確実性**: index.html のリンクにはアンカー ID が含まれるが、年度別ページ側の h2 要素に対応する id 属性が設定されていない可能性がある
- **全角数字**: 見出しや本文中の数字に全角が使われている場合がある
- **年度の起算**: 3月起算のため、例えば「令和7年」のセクションには令和7年3月から令和8年2月までの会議が含まれる（3月定例会が年度の最初の会議）

---

## 推奨アプローチ

1. **一覧ページを起点にする**: `index.html` の1リクエストで全年度ページの URL を収集できる
2. **ファイル名の重複排除**: 同一ファイルへの複数リンク（アンカー違い）があるため、ファイル名単位で重複排除する
3. **h2 見出しベースで分割**: 各年度ページは h2 見出しで会議が区切られているため、h2 をセパレータとして処理する
4. **レート制限**: 自治体サイトのため、リクエスト間に1~2秒の待機時間を設ける
5. **差分更新**: 既取得 URL のリストと比較し、新規・更新分のみを取得する
