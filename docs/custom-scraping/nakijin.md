## PDF ファイルの命名規則

年度によって命名規則が異なる。

### 令和5年〜令和6年（統一形式）

```
reiwa{N}nendai{回数}kai{teireikai|rinjikai}{MMDD}.pdf
```

例:
- `reiwa6nendai1kaiteireikai0305.pdf` → 令和6年第1回定例会 3月5日
- `reiwa6nendai2kairinjikai0801.pdf` → 令和6年第2回臨時会 8月1日
- `reiwa6nendai1kaiteireikaimokujitou.pdf` → 令和6年第1回定例会 目次・通告書
- `reiwa6nendai1kaiteireikaikaikinitteitou.pdf` → 令和6年第1回定例会 会期日程・議決結果

### 令和2年〜令和4年

```
R{N}-{回数}kaiteireikai{MMDD}.pdf
R{N}-{回数}kairinjikai{MMDD}.pdf
R{N}{回数}mokujituukokusyo.pdf
```

例:
- `R2-1kaiteireikai309.pdf` → 令和2年第1回定例会 3月9日

### 平成30年〜平成31年

一部は数値 ID ベースのファイル名（`95291794.pdf` 等）、一部は HTML 中間ページ経由で PDF リンクが提供される混在形式。

1. **定例会**（第1回〜第4回）

## PDF の構造

### ページ構成

1. **表紙・メタ情報**（1ページ目）
   - タイトル: `令和X年第Y回今帰仁村議会定例会会議録`
   - 招集年月日
   - 招集場所: 今帰仁村議会議場
   - 開会・散会時刻
   - 出席（応招）議員一覧（議席番号・氏名）
   - 欠席（不応招）議員
   - 会議録署名議員
   - 職務のため出席したもの（事務局長・議事係長）
   - 地方自治法第121条により説明のため出席した者の職氏名（村長・副村長・教育長・各課長等）

2. **議事日程**（2〜3ページ目）
   - 日程番号・議案番号・事件名・摘要の表形式

3. **会議録本文**（4ページ目以降）
   - 発言者ごとの逐語録

### 発言者パターン

```
○　上原祐希　議長　ただいまの出席議員は11名です。...
○　久田浩也　村長　皆さんおはようございます。...
```

- `○` + 全角スペース + **氏名**（太字） + 全角スペース + **役職**（太字） + 全角スペース + 発言内容
- 氏名と役職は太字（PDF 内で Bold フォント）
- 発言内容は同行から開始し、次の発言者行まで続く

#### 役職パターン

| 表記 | 説明 |
| --- | --- |
| `議長` | 議長 |
| `村長` | 村長 |
| `副村長` | 副村長 |
| `教育長` | 教育長 |
| `○番（氏名）` | 議員（議席番号付き） |
| `{課名}課長` | 各課長（総務課長、企画財政課長等） |

### パース用正規表現（案）

```typescript
// 発言者の抽出（PDF テキスト抽出後）
const speakerPattern = /^○\s+(.+?)\s+(議長|副議長|村長|副村長|教育長|.+?課長|事務局長)\s/;
// 例: ○　上原祐希　議長　→ name="上原祐希", role="議長"

// 議員番号付きの発言者
const memberPattern = /^○\s+(\d+)番\s+(.+?)\s/;
// 例: ○　5番　石嶺美奈実　→ number="5", name="石嶺美奈実"

// 開催日の抽出
const datePattern = /(令和|平成)(\d+)年(\d+)月(\d+)日/;

// 開会時刻
const openingPattern = /開会時刻\s+(午前|午後)(\d+)時(\d+)分/;
```

## スクレイピング戦略

### Step 1: 年度別一覧ページの URL 収集

議事録トップページ `https://www.nakijin.jp/pagtop/gyosei/songikai/1281.html` から年度別一覧ページへのリンクを収集する。

- リンクは `/pagtop/kakuka/gikai/2/2/2/gijiroku/` 配下の HTML ページを指す
- 年度ごとに固有 ID のため、トップページからの動的取得が必要

### Step 2: PDF リンクの収集

各年度別一覧ページから PDF ファイルへのリンクを収集する。

- リンクは `//www.nakijin.jp/material/files/group/1/{ファイル名}.pdf` 形式
- 目次・通告書、会期日程・議決結果、各日の議事録を区別して収集
- H30 は HTML 中間ページ（`/pagtop/kakuka/gikai/2/2/2/gijiroku/heisei30nen/{ID}.html`）を経由する場合がある

**収集方法:**

1. 年度一覧ページの HTML を取得
2. `<a href="...pdf">` リンクを Cheerio で抽出
3. H30 の中間ページがある場合は、そのページも再帰的に取得して PDF リンクを収集
4. ファイル名から会議種別（定例会/臨時会）、回数、日付を推定

### Step 3: PDF のダウンロードとテキスト抽出

各 PDF をダウンロードし、テキストを抽出する。

- PDF はテキスト埋め込み型（画像 PDF ではない）のため、`pdf-parse` 等のライブラリでテキスト抽出が可能
- 1ファイルあたり約 30〜40 ページ程度

### Step 4: テキストのパース

抽出したテキストから会議録情報を構造化する。

1. **メタ情報の抽出**: タイトル行から会議名・開催日を取得
2. **出席者情報の抽出**: 表紙ページの出席議員一覧を取得
3. **発言の分割**: `○` で始まる行を発言者行として認識し、発言ブロックを分割
4. **発言者情報の抽出**: 氏名・役職を正規表現で抽出

---

## 注意事項

- PDF ファイルは FlateDecode 圧縮されており、直接テキスト読み取りはできない。PDF パーサーライブラリが必須
- 年度によって PDF のファイル名命名規則が異なるため、ファイル名からのメタ情報推定は補助的に使い、PDF 本文内のタイトルを正とする
- H30 年度は PDF ファイル名が数値 ID ベース（`95291794.pdf` 等）のものがあり、ファイル名からの情報抽出が困難
- 会議録 PDF には議事日程（表形式）が含まれるが、PDF からの表抽出は精度が低いため、発言録部分のみをパース対象とするのが現実的
- 発言者パターンの `○` 記号は PDF テキスト抽出時に正しく取得できることを事前に確認する必要がある

---

## 推奨アプローチ

1. **トップダウンでリンク収集**: トップページ → 年度一覧 → PDF リンクの順に辿る
2. **PDF テキスト抽出には `pdf-parse` を使用**: テキスト埋め込み型 PDF のため、OCR は不要
3. **レート制限**: 自治体サイトのため、リクエスト間に適切な待機時間（1〜2 秒）を設ける
4. **差分更新**: 年度一覧ページの PDF リンク一覧と既存取得済みリストを比較し、新規分のみをダウンロード
5. **ファイル名ではなく PDF 本文からメタ情報を取得**: 命名規則の不統一に対応するため
