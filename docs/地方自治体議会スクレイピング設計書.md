# 全自治体議会データ収集プロジェクト
**スクレイピング設計書 v1.1　　2026年3月**

---

## 1. プロジェクト概要

日本全国約1,700自治体の議会データ（議事録テキスト・日程・開催情報・議員情報）を自動収集し、統合検索サービスを構築することを目的とする。

**収集対象データ（3種）**

- 議事録テキスト — 本会議・委員会の発言録（発言者・内容・日付）
- 日程・開催情報 — 定例会・臨時会の日時・会議種別
- 議員情報 — 氏名・会派・期数など（将来対応）

---

## 2. 現状分析：自治体議会システムの生態系

最大の課題は、1,700自治体が複数の異なるシステム・形式でデータを公開していることである。スクレイピングはシステムごとに専用アダプターを作る「マルチアダプター方式」を採用している。

### 2.1 主要会議録システムの分布

| 会議録システム | ベンダー | 採用割合（推定） | 難易度 | 実装状況 |
|---|---|---|---|---|
| **DiscussNet** | NTT-AT / 会議録研究所 | ★★★★★（最多） | 中（HTML規則的） | ✅ 実装済 |
| **NDL（国会図書館）** | 国立国会図書館 | 国会のみ | 低（API提供） | ✅ 実装済 |
| ssp.kaigiroku.net | 会議録研究所 | ★★☆☆☆（鹿児島市等） | 中（APIあり） | ✅ 実装済 |
| 独自HTML | 各自治体 | ★★★☆☆ | 高 | ✅ 実装済（Local） |
| DB-Search | 大和速記情報センター | ★★★☆☆ | 中 | 未対応 |
| Sophia | 神戸綜合速記 | ★★☆☆☆ | 中 | 未対応 |
| VOICES | フューチャーイン | ★★☆☆☆ | 中 | 未対応 |
| PDF/HTMLファイル | （システムなし） | ★★☆☆☆ | 高 | 未対応 |

### 2.2 公開形式の3分類

**① 会議録検索システム経由（DiscussNet 等）**
- HTML構造が各ベンダーで規則的 → アダプター1本で数百自治体をカバー可能
- DiscussNetは全国最多。URLパターンも統一されており最も取り組みやすい

**② 独自CMSによるHTML公開**
- Local アダプターで CSS セレクタを設定して対応
- HTMLの構造がバラバラ → 個別対応が必要（優先度は下げて対応）

**③ PDFファイル直接公開**
- PDF→テキスト変換（pdfplumber / pdfminer）が必要
- 将来フェーズで対応予定

## 3. システムアーキテクチャ

### 3.1 技術スタック（実装）

| カテゴリ | 技術 |
|---|---|
| スクレイピング | TypeScript + Bun + fetch |
| HTML解析 | Cheerio |
| データ保存 | PostgreSQL + Drizzle ORM |
| 全文検索 | PostgreSQL 標準 tsvector |
| 意味検索 | pgvector（OpenAI 埋め込み） |
| ジョブキュー | Cloudflare Queue |
| スケジューラー | Cloudflare Cron（1分ごと） |
| API | oRPC |
| フロントエンド | TanStack Start + React |
| インフラ | Cloudflare Workers（wrangler deploy） |

### 3.2 処理フロー

```
管理画面でジョブ作成 → scraper_jobs に INSERT (status: pending)
  ↓
Cron（1分ごと）が pending を queued に更新し、Queue に start-job 投入
  ↓
Queue コンシューマーが start-job を処理
  ↓
ソース別にサブメッセージ投入（discussnet-list, ndl-page, kagoshima-council, local-target）
  ↓
各ハンドラーが 1 自治体/1 ページ単位で取得 → meetings / statements に保存
  ↓
process-meetings が pending の meetings を statements に変換（embedding 生成）
```

### 3.3 実装済みアダプター

| ソース | 対象 | 備考 |
|--------|------|------|
| **ndl** | 国会会議録 | 国会図書館 API、ページング対応 |
| **kagoshima** | 鹿児島市 | ssp.kaigiroku.net API |
| **local** | 任意の自治体 | baseUrl + CSS セレクタ設定 |
| **discussnet** | DiscussNet 導入自治体 | municipalities マスタから対象取得 |

## 4. データモデル設計

### 4.1 DBテーブル構成（実装）

| テーブル名 | 主なカラム | 説明 |
|---|---|---|
| `municipalities` | id, code, name, prefecture, system_type, base_url, enabled | 自治体マスタ（約1,700件） |
| `meetings` | id, title, meeting_type, held_on, raw_text, assembly_level, prefecture, municipality, external_id, status | 会議単位 |
| `statements` | id, meeting_id, kind, speaker_name, speaker_role, content, content_hash, embedding | 発言単位（全文・ベクトル検索対象） |
| `scraper_jobs` | id, source, status, config, total_inserted, total_skipped | ジョブ管理 |
| `scraper_job_logs` | id, job_id, level, message | ジョブログ |
| `policy_tags` / `statement_policy_tags` | 政策タグ（将来利用） | |

### 4.2 全文検索・意味検索

- **全文検索**: PostgreSQL 標準 `tsvector` + `to_tsquery`
- **意味検索**: pgvector + OpenAI text-embedding-3-small

```sql
-- 全文検索
WHERE statements.content_tsv @@ to_tsquery('simple', '検索語')

-- 意味検索（コサイン類似度）
ORDER BY embedding <=> '[...]'::vector
```

## 5. アダプター設計

### 5.1 DiscussNet アダプター（実装済）

`apps/scraper-worker/src/scrapers/discussnet.ts`

- `municipalities` テーブルから `system_type = 'discussnet'` かつ `enabled = true` の自治体を取得
- 議事録一覧ページ（discussnet-list）→ 個別議事録ページ（discussnet-meeting）の 2 段階キュー
- セルフチェーンで全ページを巡回

### 5.2 NDL アダプター（実装済）

`apps/scraper-worker/src/handlers/ndl.ts`

- 国会図書館 API（https://kokkai.ndl.go.jp/api/meeting）
- 日付範囲（from / until）指定、10件/ページでページング

### 5.3 Local アダプター（実装済）

`apps/scraper-worker/src/scrapers/local.ts`

- baseUrl + listSelector / contentSelector / dateSelector を設定して任意の HTML サイトに対応

### 5.4 将来対応：PDF アダプター

- pdfplumber でテキスト抽出後、正規表現で発言者・発言内容を分離
- Cloudflare Workers では Node.js ネイティブモジュールが使えないため、別 Worker や VPS での実行を検討

## 6. 主な課題と対策

### 6.1 アクセス制限・ブロック対策

- **対策①** リクエスト間隔：各ハンドラーで 500ms〜1500ms のウェイトを設定
- **対策②** User-Agent にプロジェクト名と連絡先を明記（検討中）
- **対策③** 同時実行数を制限（Cloudflare Queue: maxConcurrency=2）

### 6.2 robots.txt・利用規約

- `municipalities.enabled = false` で対象外とする
- 収集前に robots.txt 確認スクリプトでフラグを設定（Phase 1）

### 6.3 1,700自治体 URL リスト

- `packages/db/src/seeds/municipalities.csv` を基準に `municipalities` テーブルを投入
- DiscussNet 導入自治体は chiholog 等から補完（`municipalities-discussnet.ts`）

## 8. 法的・倫理的留意事項

> ⚠️ **必ず守ること**
> - `robots.txt` を確認し、Disallow されているパスはクロールしない
> - 利用規約でスクレイピング禁止の記載がある場合は対象外とする
> - データは公開情報のみ収集（ログイン不要のページのみ）
> - サーバーに過大な負荷をかけない（クロール間隔・並列数を制限）
> - データの帰属を明示し、出典として各自治体の URL をリンク表示する
